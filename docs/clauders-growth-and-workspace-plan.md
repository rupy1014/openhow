# Clauders 운영 구조 + 모집 성장 계획

작성일: 2026-02-28

## 1) 운영 목표
- 멤버 UX 단순화 (자기 기수 중심)
- 기수 간 지식 누적 (아카이브/위키)
- 운영진 검수로 품질 유지

## 2) 워크스페이스 구조
1. `clauders-core`
   - 운영진 정본: 강의/운영 매뉴얼/정책/공식 위키
2. `clauders-{cohort}` (예: `clauders-3rd`)
   - 기수별 활동: 공지/자기소개/질문/실습/회고
3. `clauders-archive`
   - 기수 종료 후 공개 가능한 문서의 읽기 전용 아카이브

## 3) UX 원칙
- 멤버: 자기 기수 워크스페이스 중심 사용
- 타 기수 열람: 아카이브에서 검색/필터로 해결
- 운영진: core + cohort + archive 횡단 운영

## 4) 기수 템플릿 구조
- `00-notice/`
- `01-introductions/`
- `02-qna/`
- `03-build-log/`
- `04-retros/`
- `05-best-practices/`

## 5) 위키 승격 파이프라인
1. 기수 워크스페이스에서 문서 생성
2. 주간 운영진 검수 (품질/중복/보안)
3. 통과 문서만
   - `clauders-core/wiki` (공식)
   - `clauders-archive` (사례 공유)

## 6) 문서 메타데이터 규칙 (frontmatter)
- `cohort: 3rd`
- `topic: openclaw | ai-agent | automation | ops`
- `status: draft | reviewed | canonical`
- `visibility: team | public`
- `owner: @username`
- `updatedAt: YYYY-MM-DD`

## 7) 운영 리듬
- 주 1회: 승격 회의 30분
- 월 1회: 중복 정리/병합
- 기수 종료 주: 아카이브 이관 + 다음 기수 스타터팩 배포

## 8) 2주 도입 일정
### Week 1
- 워크스페이스 3종 생성
- 기수 템플릿/문서 초기화
- 자기소개 게시판 오픈

### Week 2
- 검수 루프 시작
- 첫 승격 문서 5~10개 생성
- 아카이브 태그/검색 정리

## 9) 매 기수 모집 성장 전략

### 핵심 프레임
- 퍼널: 노출 → 관심 → 신청 → 참석 → 완주 → 추천
- 기수마다 같은 홍보 반복 대신, 시스템으로 자동화

### A. 모집 전(상시) 자산화
1. 공개 랜딩 1개 고정
   - 기수별로 페이지 새로 만들지 말고, 같은 URL에서 기수만 업데이트
2. 증거 자산 누적
   - 수강생 결과물, 전/후 비교, 후기, 데모 영상 30~90초
3. FAQ 위키화
   - 매번 받는 질문(난이도/시간/비용/혜택) 문서화

### B. 모집 기간(2주) 운영
1. 3단 메시지 시퀀스
   - D-14: 문제 제기 + 가치
   - D-7: 커리큘럼/후기 + 실전 결과
   - D-2: 좌석/마감 리마인드
2. 채널 믹스
   - 카톡 커뮤니티, 텔레그램, 기존 수강생 리퍼럴, 개인 SNS
3. 신청 장벽 최소화
   - 신청 폼 1분 이내(필수 항목 최소화)

### C. 모집 후(기수 진행 중)
1. 진행 중 하이라이트 공개
   - 주 1회 결과물 스냅샷 공개 (다음 기수 유입용)
2. 리퍼럴 트리거
   - 2주차/마지막주에 추천 요청 + 템플릿 제공
3. 완주자 앰배서더
   - 다음 기수 Q&A 세션 초대, 사례 발표 기회 제공

### D. 최소 자동화 세트
- Notion/Openhow: 모집 상태 보드
- 템플릿: 모집 공지/리마인드/FAQ/후기 요청
- CRM 간단 태그: `lead`, `applied`, `enrolled`, `alumni`, `referrer`
