# 블로그 가이드 발행

사용자 요청: $ARGUMENTS

## 개요

`/publish`는 검수 완료된 가이드를 openhow 워크스페이스에 발행합니다.

```
/publish subscription-planning     # 특정 가이드 발행
/publish                           # 가장 최근 검수된 가이드 발행
```

---

## 실행 파이프라인

```
┌──────────────────────────────────────────────┐
│ 1. 발행 대상 확인                               │
│   - docs/guides/{slug}.md 존재 확인             │
│   - 검수 완료 여부 확인 (미검수 시 /review 먼저)  │
└──────────────────────────────────────────────┘
                      ↓
┌──────────────────────────────────────────────┐
│ 2. openhow 형식 변환                            │
│   - 마크다운 → openhow 문서 형식                │
│   - 메타데이터 추가 (제목, 설명, 태그)           │
│   - 이미지 경로 확인/조정                        │
└──────────────────────────────────────────────┘
                      ↓
┌──────────────────────────────────────────────┐
│ 3. openhow publish 실행                        │
│   - cd examples/blog.bootpay.ai               │
│   - openhow publish                           │
│   - 발행 URL 확인                              │
└──────────────────────────────────────────────┘
                      ↓
┌──────────────────────────────────────────────┐
│ 4. 발행 확인                                    │
│   - 발행된 페이지 접근 확인                      │
│   - 렌더링 정상 여부 (비교표, 코드블록 등)        │
└──────────────────────────────────────────────┘
                      ↓
┌──────────────────────────────────────────────┐
│ 5. 완료 보고                                    │
│   - 발행 URL                                   │
│   - 커뮤니티 공유용 한 줄 소개 문구               │
└──────────────────────────────────────────────┘
```

---

## 커뮤니티 공유용 문구 생성

발행 완료 시, 각 가이드에 맞는 공유용 문구를 함께 생성합니다.

```yaml
결제 도입:
  질문: "사이드 프로젝트에 결제 넣으려는데 뭐부터 해요?"
  답변: "이거 한 번 보세요. PG사 선택부터 라이브까지 전체 과정 정리되어 있어요."

구독 기획:
  질문: "SaaS 구독 모델 만드려는데 정기결제 어떻게 해요?"
  답변: "기획 단계에서 정해야 할 것들 정리된 글이에요. 빌링키부터 이탈 방지까지."

마켓플레이스 정산:
  질문: "중개 플랫폼인데 판매자 정산 어떻게 하죠?"
  답변: "정산 모델 3가지 비교 + 세무 이슈까지 정리된 가이드에요."

결제 운영:
  질문: "결제 장애 나면 어떡해요?"
  답변: "결제 운영 체크리스트에요. 모니터링부터 장애 대응까지."
```
