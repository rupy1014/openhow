# 용어 규칙: 취소 vs 환불

> 모든 가이드, 대본, 리서치 노트에서 이 규칙을 따른다.

## 정의

| 용어 | 의미 | 처리 주체 |
|------|------|-----------|
| **취소** | PG사/카드사 전산에서 원래 결제를 되돌리는 것 | 시스템 (PG 전산) |
| **환불** | 고객 계좌로 직접 돈을 입금하는 것 | 가맹점 (직접 송금) |

## 결제수단별 분류

| 결제수단 | 기본 처리 | 비고 |
|----------|----------|------|
| 신용카드/체크카드 | **취소** | 승인취소 또는 매입취소 (PG 전산) |
| 간편결제 (카카오페이, 네이버페이 등) | **취소** | 원래 결제수단으로 자동 되돌림 |
| 계좌이체 | **취소** | PG 전산에서 처리 |
| 가상계좌 (입금 전) | **취소** | 가상계좌 자체를 폐기 |
| 가상계좌 (입금 후) | **환불** | 고객 실제 계좌로 직접 입금 |
| 휴대폰 결제 (당월) | **취소** | 통신사 청구에서 차감 |
| 휴대폰 결제 (익월) | **환불** | 고객 계좌로 직접 입금 |

## 예외: 취소 기간 경과

취소 가능 기간이 지난 건은 결제수단과 무관하게 **환불**로 처리한다.

- 카드 매입취소 기한 초과 → 환불 (고객 계좌로 입금)
- 간편결제 취소 기한 초과 → 환불

## 표기 규칙

```yaml
올바른 표기:
  ✅ "카드 결제 취소" (PG 전산 처리)
  ✅ "가상계좌 환불" (고객 계좌로 입금)
  ✅ "취소 및 환불 정책" (둘 다 포함할 때)
  ✅ "취소 기간이 지나면 환불로 처리"

잘못된 표기:
  ❌ "카드 환불" (카드는 취소임)
  ❌ "가상계좌 취소" (입금 후는 환불임)
  ❌ "환불"로 모든 되돌림을 통칭
  ❌ "취소"로 가상계좌 입금 후 처리를 지칭
```

## 업계 근거

| PG사 | API 경로 | 카드 되돌리기 | 가상계좌 (입금 후) |
|------|---------|-------------|-----------------|
| 토스페이먼츠 | `/cancel` | 취소 | 취소 + `refundReceiveAccount` |
| 나이스페이먼츠 | `/cancel` | 취소 | 환불 (refund 파라미터) |
| 이니시스 | `/refund` | 취소/환불 혼용 | 환불 |
| KCP | `/cancel` | 거래취소 | 자동 환불 미지원 (가맹점 직접) |

대부분의 PG사가 전산 처리는 `cancel`, 돈 돌려주기는 `refund`로 구분한다.
우리도 이 관행에 맞춰 취소/환불을 명확히 구분한다.
