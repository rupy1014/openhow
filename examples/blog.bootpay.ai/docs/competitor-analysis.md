# 경쟁사 블로그 분석

## 1. 포트원 (PortOne)

### 현황
- **글 수**: 200+개
- **포지셔닝**: "PG 연동의 교과서" — 중립적 PG사 비교 + 무료 가입 강조
- **타겟**: 개발자 + 기획자
- **핵심 전략**: SEO 물량전. "PG", "결제연동", "수수료 비교" 등 핵심 키워드를 광범위하게 선점

### 주제별 콘텐츠 현황

**결제 도입 (가장 강한 영역)**
- PG사 완전정복 시리즈, 국내 주요 PG사 비교(매년 업데이트), 가맹점 가입 가이드, 결제수단 선택 가이드 등 10편+
- PG 비교표와 수수료 정보가 상세함
- AI CodeMate(MCP 서버)로 개발자 연동 경험 차별화 시도 중
- ❌ 없는 것: 결제 시스템 설계론(멱등성, 상태 머신), 테스트 환경 구축, 법적 고려사항

**구독/정기결제 (약한 영역, 3편)**
- 빌링키 개념, 슈퍼빌링키(PG 교체 시 빌링키 유지), SaaS 창업자 가이드
- ❌ 없는 것: 가격 모델 설계, 무료체험→유료 전환, 이탈 방지(dunning), 프로레이션, 환불 정책

**마켓플레이스 정산 (법률 대응은 강함, 8편)**
- 파트너 정산 자동화, 전금법 개정 대응, 정산 한도 관리, 하위셀러 정산 사례 등
- ❌ 없는 것: 정산 모델 비교(수취 후 지급 vs 분리정산 vs 에스크로), 유형별 설계론, 분쟁 처리

**결제 운영 (멀티PG 반복, 8편)**
- 멀티PG 도입 이유(반복적), 애널리틱스 베타, 트래픽 급증 대응, 스마트 라우팅
- ❌ 없는 것: 장애 대응 플레이북, 정산 대사, fraud 탐지, CS 대응 가이드, 운영 체크리스트

### 전체 톤
"포트원 쓰면 해결됩니다" — 리드젠 마케팅 콘텐츠. **도메인 자체에 대한 교육은 약함.**
커뮤니티에서 공유하기엔 광고 느낌이 강함.

## 2. 토스페이먼츠 (Toss Payments)

### 현황
- **2개 채널 운영**: 기술 블로그(tosspayments.com/blog) + 사장님 매거진(blog.tosspayments.com, 토스페이먼츠피드)
- **포지셔닝**: "사장님이 궁금할 때 바로 찾는 매거진" + 개발자 기술 콘텐츠
- **타겟**: 쇼핑몰 운영자(매거진) + 개발자(기술 블로그)
- **핵심 전략**: 비즈니스 단계별 콘텐츠 + 리드 수집(리포트/가이드 PDF)

### 주제별 콘텐츠 현황

**결제 도입 (기획자 시리즈가 좋음, 8~10편)**
- "이커머스 기획자가 알아두면 좋을 결제 이야기" Part 1, 2 — 비개발자도 이해하는 설명
- 결제창 호출 방식 총정리, 주문서에서 시작하기, 결제 플로우 설명
- 인프랩 CTO 인터뷰 등 고객 사례
- ❌ 없는 것: 업종별 가이드, PG 선택 비교(1편만 존재), 비즈니스 의사결정 관점

**구독/정기결제 (개발 연동만 존재, 4~5편)**
- 빌링키 발급·스케줄링 API 연동 시리즈, 구독경제 소비자보호 규제 정리
- ❌ 없는 것: 구독 모델 설계, 플랜 구조, 이탈 방지, 프로레이션, 환불 정책 — **기획 관점 전무**

**마켓플레이스 정산 (제품 소개 수준, 3~4편)**
- 지급대행 서비스 정책, 관리자 가이드, KYC 프로세스
- ❌ 없는 것: 정산 모델 비교, 아키텍처 설계, 세무 이슈, 다자간 정산 — **설계 전략 콘텐츠 전무**

**결제 운영 (단편적 트러블슈팅, 5~6편)**
- FDS 안내, API 응답 처리, 에러 가이드, 전환율 개선
- ❌ 없는 것: 체계적 운영 가이드, 모니터링 체계, 장애 대응, 정산 대사

### 전체 톤
양쪽 채널 모두 자사 제품(결제위젯, 상점관리자, 지급대행)으로의 전환 유도가 목적.
매거진 채널은 친근하지만, 결제 도메인 자체에 대한 깊은 교육은 부족.

## 3. Bootpay가 노릴 수 있는 빈 자리

양쪽 경쟁사의 공통 약점은 명확하다:
1. **모든 콘텐츠가 자사 제품 세일즈로 귀결** → 커뮤니티 공유 시 "광고잖아" 반응
2. **기획자를 위한 의사결정 콘텐츠 부재** → "어떤 걸 선택해야 하나"를 도와주지 않음
3. **라이브 이후(운영) 콘텐츠 거의 없음** → 결제 연동까지만 다루고 끝

| 빈 영역 | 포트원 상태 | 토스페이먼츠 상태 | Bootpay 기회 |
|---------|-----------|---------------|-------------|
| 구독 비즈니스 기획 (플랜 설계, 이탈 방지, 환불 정책) | 빌링키 설명 3편 | API 연동 4편 | **한국어로 이 주제를 제대로 다루는 글 자체가 없음** |
| 마켓플레이스 정산 설계 (모델 비교, 세무, 분쟁) | 전금법 대응 중심 | 제품 소개 수준 | **설계론을 다루는 중립적 교육 콘텐츠 없음** |
| 결제 운영 실무 (장애 대응, 정산 대사, fraud) | 멀티PG 반복 | 단편적 FAQ | **체계적 운영 가이드 전무** |
| 결제 도입 의사결정 ("내 상황에 맞는 선택") | PG 비교표 (정보 나열) | 기획자 시리즈 (결제 개념 설명) | **판단 기준 제공** 관점 부족 |
