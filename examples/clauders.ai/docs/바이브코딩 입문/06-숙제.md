**지금 배운 프로세스를 써먹어봐.**

읽기만 하면 3일 안에 잊어. 직접 만들어봐야 해.

---

## 뭘 만들까?

뭐든 돼. 조건은 하나 — **너의 아이디어.**

예시:

- 나만의 포트폴리오 사이트
- 할 일 관리 앱
- 레시피 모음 페이지
- 동네 카페 추천 사이트
- 읽은 책 기록 페이지

규모는 작을수록 좋아. "쇼핑몰 전체" 말고 "상품 카드 3개 보여주는 페이지"처럼.

---

## 프로세스를 따라가봐

**Step 1.** 바이브 기획

```
[나의 아이디어]를 만들고 싶어.

- [핵심 기능 1]
- [핵심 기능 2]
- [핵심 기능 3]

안 만들 것:
- 로그인
- [안 할 것 1]
- [안 할 것 2]

가장 단순한 방법으로 PRD.md 정리해줘.
코드는 아직 짜지 마.
```

**Step 2.** 레퍼런스 주기

비슷한 서비스를 찾아서 스크린샷을 찍어. Claude한테 "이거랑 비슷하게"라고 보여줘.

CLAUDE.md에 기본 규칙도 적어두면 좋아.

**Step 3.** 한 입씩 만들기

PRD에서 기능 목록을 뽑고, 1번부터 하나씩.

기능 하나가 끝날 때마다 → 저장 → 정리 → 다음.

```
1번 기능부터 만들어줘. 나머지는 아직 하지 마.
```

끝나면:

```
지금까지 만든 거 git으로 저장해줘.
그리고 지금까지 만든 기능을 정리해줘.
```

**Step 4.** 고치기

에러가 나면 복붙. 3번 안 되면 되돌려.

```
이런 에러가 나:
[에러 메시지]

[뭘 했는데] [뭐가 됐어] [뭐가 돼야 해]
```

잘 먹힌 프롬프트는 메모해둬. 다음 프로젝트에서 재사용할 수 있어.

**Step 5.** 배포하기

다 만들었으면 세상에 공개해.

```
이 프로젝트를 GitHub에 올려줘.
API 키는 환경변수로 관리하고, .env.local은 .gitignore에 추가해줘.
```

GitHub에 올렸으면 Vercel에 연결하고 환경변수 넣으면 끝.

## 체크리스트

| 항목 | 했어? |
|------|------|
| PRD.md를 먼저 만들었어 | |
| "코드는 아직 짜지 마"를 말했어 | |
| 레퍼런스(스크린샷, 비슷한 서비스)를 줬어 | |
| 한 기능씩 만들고 확인했어 | |
| 기능마다 저장(git commit)했어 | |
| 기능마다 정리(문서화)를 시켰어 | |
| 에러는 복붙으로 고쳤어 | |
| Vercel(또는 다른 곳)에 배포했어 | |
| 잘 먹힌 프롬프트를 메모했어 | |

전부 체크하면 숙제 완료야.

---

## 왜 직접 해봐야 해?

강의 보는 것과 손으로 만드는 건 달라.

에러가 나고, 예상대로 안 되고, 다시 고치는 과정에서 프로세스가 몸에 배.

중요한 건 "뭘 만들었냐"가 아니야. **"어떻게 만들었냐"**야.

PRD부터 시작해서, 레퍼런스를 주고, 한 입씩 만들고, 에러를 고치는 이 흐름.

이 흐름이 익숙해지면, 다음 프로젝트는 2배 빨라져.
