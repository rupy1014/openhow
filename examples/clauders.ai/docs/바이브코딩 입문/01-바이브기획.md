**코드부터 짜면 안 돼. 기획부터야.**

"AI 캐릭터 채팅 만들어줘"라고 하면 뭐가 나올까?

AI가 알아서 기술을 골라. 화면 구조도 정해. 기능도 넣어.

근데 그게 네가 원한 거야? 아마 아닐 거야.

---

## 왜 기획 없이 시작하면 망해?

실제로 해보면 이래:

```
나: "AI 캐릭터 채팅 서비스 만들어줘"
```

AI가 뭘 하냐:

- Next.js + TypeScript + Prisma + PostgreSQL을 깔아
- 로그인 시스템을 만들어
- 관리자 페이지를 만들어
- 다크모드를 넣어
- 1,500줄짜리 코드가 나와

너는 그냥 캐릭터 5개랑 대화하고 싶었을 뿐인데.

**AI가 네 대신 기획을 해버린 거야.** 네 의도랑 전혀 다른 방향으로.

---

## 제1원칙: "코드는 짜지 마"

이 한 줄이 전부야:

```
코드는 아직 짜지 마. 먼저 계획을 보여줘.
```

이 말을 먼저 하면:

- AI가 복잡한 기술을 안 골라
- 화면 구조를 네가 정할 수 있어
- 한 번에 수백 줄이 안 쏟아져

Peter Yang(바이브코딩 12원칙 저자)이 이렇게 말했어:

> "이 한마디가 90%의 과잉 설계를 막아준다."

계획을 보고, 마음에 드는 부분만 "이거 실행해줘"라고 하면 돼.

---

## 기획 = PRD 한 장

기획이라고 거창한 게 아니야.

**PRD**(Product Requirements Document) = "내가 뭘 만들고 싶은지 정리한 문서".

A4 한 장이면 돼. 10장짜리 기획서가 아니야.

이걸 AI한테 먼저 만들어달라고 해.

```
나 AI 캐릭터 채팅 서비스 만들고 싶어.
PRD.md로 정리해줘. 코드는 아직 짜지 마.
```

AI가 코드 대신 문서를 내놓아.

그 문서를 읽고, 마음에 안 드는 건 바꾸라고 하면 돼.

**코드는 PRD가 확정된 다음에.**

---

## 같은 프로젝트, 다른 결과

| | 기획 없이 | 기획하고 |
|---|---|---|
| 첫 프롬프트 | "채팅 서비스 만들어줘" | "PRD 먼저 만들어줘" |
| AI가 하는 일 | 기술 선택 + 코드 생성 | 문서 작성 |
| 결과물 | 1,500줄 코드 | 1장짜리 기획서 |
| 내 통제력 | 없음 | 있음 |
| 수정 비용 | 처음부터 다시 | 문서 한 줄 고치기 |

같은 프로젝트인데 시작이 다르면 결과가 완전히 달라.

---

## 한 줄 정리

"코드는 짜지 마"를 먼저 말해. 그 한마디가 기획의 시작이야.
