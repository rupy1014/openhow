**에러는 무조건 나. 당연한 거야. 고치는 법만 알면 돼.**

바이브코딩에서 에러는 버그가 아니야. 과정이야.

프로 개발자도 에러 없이 코드를 짜지 못해. 차이는 고치는 속도야.

---

## 에러가 나면 어떻게 해?

**Step 1.** 에러 메시지를 통째로 복사해

브라우저에서 빨간 글씨가 보이면, 그대로 복사해서 Claude한테 붙여넣어.

```
이런 에러가 나:
TypeError: Cannot read properties of undefined (reading 'map')
    at CharacterList (CharacterList.jsx:12:23)
```

읽을 줄 몰라도 돼. Claude가 읽으니까.

**Step 2.** 뭘 하려고 했는지 같이 말해

```
이런 에러가 나:
[에러 메시지]

캐릭터 선택 화면에서 카드를 클릭했을 때 발생해.
카드 클릭하면 채팅 화면으로 넘어가야 하는데, 대신 빈 화면이 떠.
```

에러 메시지 + 상황 설명. 이 두 개가 있으면 Claude가 빠르게 고쳐.

**Step 3.** 고친 뒤에 다시 확인해

브라우저 새로고침하고 같은 동작을 해봐.

됐으면 다음으로. 또 에러가 나면 반복해.

---

## 3번 시도해도 안 되면?

같은 에러를 3번 고쳤는데 계속 나와?

**되돌려.**

```
이 에러 3번째 시도인데 계속 나.
마지막으로 잘 되던 시점으로 되돌려줘.
```

왜? AI가 고치면서 다른 곳을 망가뜨리는 경우가 있어. 이걸 **고치다가 꼬이는 루프**라고 해.

A를 고쳤더니 B가 터져. B를 고쳤더니 A가 다시 터져.

이때는 멈추고, 잘 되던 시점으로 돌아가서 다시 시작하는 게 빨라.

그래서 기능마다 저장(git commit)이 중요한 거야.

---

## 말을 다듬으면 결과가 달라져

같은 요청인데 표현을 바꾸면 결과가 달라져.

```
❌ "이거 고쳐줘"
✅ "캐릭터 카드 클릭했을 때 채팅 화면으로 안 넘어가.
    콘솔에 이런 에러가 나: [에러 메시지]
    카드 클릭 이벤트를 확인해줘."
```

**3가지를 같이 말해:**

| 뭘 말해 | 예시 |
|---------|------|
| 뭘 했는데 | "카드를 클릭했는데" |
| 뭐가 됐어 | "빈 화면이 떠" |
| 뭐가 돼야 해 | "채팅 화면으로 넘어가야 해" |

이 세 가지가 있으면 Claude가 문제를 정확히 찾아.

---

## 브라우저 콘솔 열기

에러는 화면에 안 보일 때도 있어. 그때 **브라우저 콘솔**을 열어봐.

- **Windows**: `F12` → Console 탭
- **Mac**: `Cmd + Option + J`

빨간 글씨가 있으면 그게 에러야. 복사해서 Claude한테 보내.

화면에서는 잘 되는 것 같은데 뭔가 이상하면, 콘솔에 빨간 글씨가 있을 수 있어.

---

## 안 할 것도 말해

고칠 때 AI가 관련 없는 걸 같이 바꿔버리는 경우가 있어.

```
❌ "이 에러 고쳐줘"
✅ "이 에러 고쳐줘. 다른 파일은 건드리지 마."
```

```
❌ "채팅 화면 수정해줘"
✅ "채팅 화면에서 말풍선 위치만 수정해줘. 색상이나 폰트는 바꾸지 마."
```

범위를 정해주면 AI가 딴짓을 안 해.

## 잘 된 프롬프트를 모아둬

에러를 고치다 보면 "이 표현이 잘 먹히네" 싶은 게 생겨.

그걸 메모해둬.

```
# 잘 먹히는 프롬프트

## 에러 고칠 때
- "이 에러 고쳐줘. 다른 파일은 건드리지 마."
- "콘솔 에러 [붙여넣기]. 이 파일만 수정해."

## 기능 추가할 때
- "1번 기능부터 만들어줘. 나머지는 아직 하지 마."
- "PRD.md 기반으로 구현 순서 정해줘. 코드는 짜지 마."

## 디자인 고칠 때
- "이 스크린샷이랑 비슷하게 만들어줘."
- "카드 간격만 넓혀줘. 다른 건 바꾸지 마."
```

이걸 왜 해? 다음 프로젝트에서 똑같이 쓸 수 있어.

매번 처음부터 표현을 고민할 필요 없어. 검증된 프롬프트를 꺼내 쓰면 돼.

메모장이든, 노션이든, 텍스트 파일이든 어디에 적어도 돼. 모아두는 게 중요해.

---

## 한 줄 정리

에러는 복붙. 3번 안 되면 되돌려. 잘 먹힌 프롬프트는 모아둬.
