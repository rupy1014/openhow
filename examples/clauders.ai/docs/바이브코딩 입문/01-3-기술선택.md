**가장 단순한 걸 골라.**

AI한테 물어보면 기술 옵션을 3~4가지 줘.

근데 비개발자가 어떻게 골라? 기준이 있으면 쉬워.

---

## 왜 기술 선택이 중요해?

같은 서비스라도 기술에 따라 난이도가 완전 달라.

| 기술 | 난이도 | 배포 | 유지보수 |
|------|--------|------|---------|
| HTML + JS | 쉬움 | 어디서든 | 그냥 파일 수정 |
| Next.js | 중간 | Vercel 원클릭 | 프레임워크 이해 필요 |
| Next.js + DB | 어려움 | 설정 많음 | DB 관리까지 |

바이브코딩에서 제일 잘 통하는 건 **가장 단순한 기술**이야.

복잡한 기술 = 에러 나면 고치기 어려워 = 바이브가 안 돼.

---

## AI한테 옵션 물어보는 법

PRD를 만든 다음에 이렇게 물어봐:

```
이 PRD를 구현하는 방법이 여러 가지일 텐데,
가장 단순한 것부터 3가지 옵션을 보여줘.

각 옵션마다 이걸 알려줘:
- 기술 스택
- 배포 방법
- 장점과 단점
- 난이도 (쉬움/중간/어려움)

아직 코드는 짜지 마.
```

AI가 이런 식으로 보여줘:

- **옵션 A: HTML + 바닐라 JS** — 가장 단순, 파일 몇 개. API 키 노출 위험.
- **옵션 B: Next.js + API Route** — API 키 안전, Vercel 배포. 프레임워크 필요.
- **옵션 C: Next.js + DB + 인증** — 데이터 저장 가능. 설정 복잡.

---

## 어떻게 골라?

3가지 기준만 봐.

**1. 배포가 쉬워?**

바이브코딩의 목표는 "세상에 공개하기"야.

배포가 어려우면 결국 내 컴퓨터에서만 돌아가.

→ Vercel, GitHub Pages처럼 원클릭 배포가 되는 걸 골라.

**2. 에러를 고칠 수 있어?**

복잡한 기술은 에러 메시지도 복잡해.

바이브코딩에서 에러를 고치는 방법은 "에러 메시지를 AI한테 복붙"이야.

기술이 단순하면 AI가 에러를 더 잘 고쳐.

→ 의심되면 더 단순한 옵션을 골라.

**3. 나중에 키울 수 있어?**

단순하게 시작하고, 필요하면 나중에 업그레이드해.

HTML로 시작 → 잘 되면 Next.js로 전환.

처음부터 풀스택으로 시작하면 → 안 되면 다 버려야 해.

## 프로젝트별 추천

| 프로젝트 | 추천 기술 | 이유 |
|---------|-----------|------|
| 포트폴리오 | HTML + CSS | 파일 1~2개로 끝남 |
| AI 채팅 | Next.js + API Route | API 키 보호 필요 |
| 블로그 | HTML 또는 Next.js | 정적이면 HTML, 동적이면 Next.js |
| 대시보드 | Next.js | 데이터 표시 기능 필요 |
| 쇼핑몰 | Next.js + DB | 상품/주문 데이터 관리 |

**90%의 MVP는 HTML이나 Next.js로 충분해.**

프레임워크를 3개 이상 조합하는 옵션이 나오면 → 그건 MVP가 아니야.

---

## 한 줄 정리

기술 옵션 중 제일 단순한 걸 골라. 복잡하면 바이브가 안 돼.
