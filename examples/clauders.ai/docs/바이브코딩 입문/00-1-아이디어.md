**"뭘 만들지 모르겠어"가 제일 많은 질문이야.**

대단한 아이디어 필요 없어. 네 귀찮음에서 시작하면 돼.

---

## 아이디어 = 가설이야

아이디어는 정답이 아니야. 가설이야.

"이런 문제가 있고, 이렇게 풀면 사람들이 쓸 거야."

맞을 수도, 틀릴 수도 있어. 빨리 만들어서 확인하는 게 목표야.

---

## 어디서 찾아?

**1. 내 불편함에서 시작**

이게 제일 좋아.

"매번 귀찮은데...", "이런 거 있으면 편할텐데..."

이런 생각이 아이디어야.

| 내 불편함 | AI 능력 | 만들 것 |
|----------|---------|--------|
| "매일 영어 문장 알림받고 싶어" | 텍스트 생성 + 자동화 | 영어 학습 알림 봇 |
| "상품 설명 쓰기 귀찮아" | 텍스트 생성 | 상품 설명 자동 생성기 |
| "썸네일 만들기 귀찮아" | 이미지 생성 | 썸네일 자동 생성기 |

**2. 지금 하는 일에서 시작**

꼭 새 서비스 안 만들어도 돼.

- 매주 반복하는 보고서 → 자동화
- 엑셀로 하던 정리 → 자동화
- 고객 문의 답변 → 템플릿 + AI 보조

"매번 손으로 하는데 귀찮아" = 자동화 기회야.

**3. 내 전문 분야에서 시작**

네가 잘 아는 분야가 뭐야?

회계사라면 회계 업무 자동화. 마케터라면 콘텐츠 제작 도구. 쇼핑몰 운영자라면 상품 설명 자동화.

10년 경험이 있으면, 뭐가 불편한지 네가 제일 잘 알아.

그 분야 모르는 사람은 절대 생각 못 할 아이디어가 나와.

**4. 꼭 "서비스"가 아니어도 돼**

웹사이트나 앱만 생각하지 마.

- 자동화 스크립트 — 내 업무용 도구
- 슬랙봇 / 디스코드봇 — 팀 내부용
- 나만 쓰는 도구

나만 쓰는 도구로 시작해도 돼. 잘 되면 다른 사람한테도 줄 수 있어.

---

## "비슷한 게 이미 있는데?"

검색해보니 비슷한 게 있어. 포기해야 하나?

**아니. 오히려 좋은 신호야.**

비슷한 게 있다 = 사람들이 원한다는 증거야.

성공한 서비스 대부분이 "이미 있는 시장"에서 시작했어:

- Google — Yahoo, AltaVista 이미 있었어
- Slack — HipChat, IRC 있었어
- Notion — Evernote, OneNote 있었어

차별점은 이 3가지 중 하나로 잡아:

| 방향 | 예시 |
|------|------|
| 타겟 좁히기 | "모든 영어 학습자" → "아침 출근하는 직장인만" |
| 기능 줄이기 | 기능 20개 → 기능 1개 (근데 10배 잘함) |
| 가격 다르게 | 월 $20 구독 → 1회 $2 |

## 가설 3줄로 정리해

아이디어가 잡혔으면, 이 3줄만 써.

```
문제: [누가] [뭐가] 불편하다
해결책: [어떻게] 해서 [뭘] 해결한다
핵심 가정: [이것]이 맞다면, 사람들이 [이렇게] 할 것이다
```

예시:

```
문제: 영어 공부하고 싶은데 시간이 없어
해결책: 하루에 문장 1개만 푸시 알림으로 보내주기
핵심 가정: "짧으면 매일 볼 것이다"
```

**핵심 가정이 가장 중요해.** 이게 틀리면 전체가 틀린 거야.

Claude한테 정리시켜도 돼:

```
나 이런 거 만들고 싶어.
[네 아이디어]

가설을 정리해줘.
- 문제: 뭐야?
- 해결책: 뭐야?
- 핵심 가정: 뭐야?
코드는 아직 짜지 마.
```

---

## 한 줄 정리

거창한 아이디어 필요 없어. 네 귀찮음에서 시작하고, 가설 3줄로 정리하면 끝이야.
