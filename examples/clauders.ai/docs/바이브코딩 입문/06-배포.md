**내 컴퓨터에서만 돌아가면 아무도 못 써. 배포하면 URL이 나와.**

그 URL을 카톡에 보내면 누구나 내 서비스를 써.

---

## 어떻게 배포해?

**Step 1.** Vercel 계정을 만들어

[vercel.com](https://vercel.com)에서 GitHub 계정으로 가입하면 돼.

**Step 2.** Claude한테 배포해달라고 해

```
이 프로젝트 Vercel로 배포해줘.
OpenRouter API 키는 환경변수로 설정해줘.
```

Claude가 배포 설정을 잡아줘.
환경변수로 API 키를 옮기는 것도 알아서 해.

**Step 3.** 배포된 URL을 확인해

배포가 끝나면 `https://내프로젝트.vercel.app` 같은 주소가 나와.
모바일에서도 열어봐. 친구한테 보내봐.

---

## API 키를 왜 환경변수로?

코드에 API 키를 직접 적으면, 코드를 보는 사람 누구나 내 키를 가져갈 수 있어.
내 돈으로 남이 API를 쓰는 거야.

환경변수는 서버에만 저장돼. 코드에는 안 보여.
Claude한테 "환경변수로 옮겨줘"라고 하면 알아서 처리해.

---

## 여기서 더 하려면

배포까지 했으면 서비스 하나를 끝까지 만든 거야.

더 해보고 싶은 것들:

- **캐릭터 추가** — 이미지 더 넣고 "새 캐릭터 만들어줘"
- **TTS** — 캐릭터가 목소리로 대답하게
- **이미지 생성** — 대화 중에 캐릭터가 그림을 그려주게
- **로그인** — 사용자별로 대화 기록 저장

전부 같은 방식이야. 말로 설명하면 Claude가 짜.

---

## 한 줄 정리

배포하면 URL이 나오고, 누구나 쓸 수 있어. 시스템 프롬프트를 잘 쓰는 게 곧 서비스 기획이야.
