**PRD가 다르면 결과가 완전히 달라.**

같은 "채팅 서비스"라도 PRD를 어떻게 쓰느냐에 따라 나오는 게 달라.

실제 PRD 3개를 보여줄게. 복사해서 네 프로젝트에 맞게 바꿔봐.

---

## 예시 1: AI 캐릭터 채팅

이 시리즈에서 계속 쓰는 예시야.

```markdown
# PRD: AI 캐릭터 채팅

## 한 줄 설명
AI 캐릭터 5명 중 하나를 골라 한국어로 대화하는 웹앱.

## 대상 사용자
AI 캐릭터 대화에 관심 있는 일반 사용자.

## 핵심 기능
1. 캐릭터 선택 화면 (카드 5개)
2. 채팅 화면 (말풍선 UI)
3. OpenRouter API로 AI 응답
4. 한국어 대화

## 안 만들 것
- 로그인/회원가입
- 대화 기록 저장 (새로고침 시 초기화)
- 캐릭터 생성/편집
- 음성 대화
- 관리자 페이지

## 사용자 흐름
캐릭터 선택 → 대화 시작 → 메시지 입력 → AI 응답 확인

## 성공 기준
캐릭터를 골라서 10번 이상 대화할 수 있으면 완성.

## 기술 스택
Next.js + API Route + Vercel 배포
```

**포인트:** "안 만들 것"이 5개. AI가 딴짓할 여지를 막아놨어.

## 예시 2: 포트폴리오 사이트

```markdown
# PRD: 개인 포트폴리오

## 한 줄 설명
내 프로젝트 3개를 보여주는 1페이지 포트폴리오.

## 대상 사용자
채용 담당자, 협업 파트너.

## 핵심 기능
1. 프로필 섹션 (사진 + 한 줄 소개)
2. 프로젝트 카드 3개 (이미지 + 설명 + 링크)
3. 연락처 섹션 (이메일 + GitHub)

## 안 만들 것
- 블로그
- CMS (내용은 HTML에 직접 작성)
- 방문자 통계
- 댓글
- 다국어
- 다크모드

## 사용자 흐름
페이지 접속 → 프로필 확인 → 프로젝트 클릭 → 연락처 확인

## 성공 기준
URL로 접속해서 프로젝트 3개가 보이면 완성.

## 기술 스택
HTML + CSS (파일 2개) + GitHub Pages 배포
```

**포인트:** 기술이 HTML+CSS뿐이야. AI가 React를 꺼내들 여지가 없어.

## 예시 3: 팀 TODO 리스트

```markdown
# PRD: 팀 TODO 리스트

## 한 줄 설명
팀원들이 할 일을 공유하는 간단한 웹앱.

## 대상 사용자
5명 이하 소규모 팀.

## 핵심 기능
1. 할 일 추가/완료/삭제
2. 팀원 이름 태그
3. 실시간 동기화 (새로고침 없이)

## 안 만들 것
- 로그인 (URL 공유로 접근)
- 마감일/캘린더
- 파일 첨부
- 알림
- 히스토리/되돌리기

## 사용자 흐름
URL 접속 → 할 일 추가 → 이름 태그 → 완료 체크

## 성공 기준
두 명이 동시에 접속해서 할 일을 추가/완료할 수 있으면 완성.

## 기술 스택
Next.js + Supabase (실시간 DB) + Vercel 배포
```

**포인트:** "실시간 동기화"가 필요해서 DB가 들어가. 근데 그 외엔 다 뺐어.

---

## PRD 쓸 때 흔한 실수

| 실수 | 왜 문제야 | 어떻게 고쳐 |
|------|---------|-----------|
| 핵심 기능이 10개 | MVP가 아니야 | 5개 이하로 줄여 |
| "안 만들 것"이 없어 | AI가 알아서 추가해 | 최소 3개 적어 |
| 성공 기준이 모호해 | 언제 끝인지 몰라 | 숫자로 적어 ("10번 대화 가능") |
| 기술을 미리 정해 | AI가 더 잘 아는 영역 | 옵션을 물어봐 |
| 디자인까지 적어 | PRD는 기능 문서야 | 디자인은 레퍼런스로 |

---

## 한 줄 정리

PRD 예시를 참고해서 네 프로젝트에 맞게 바꿔봐. "안 만들 것"을 먼저 채워.
