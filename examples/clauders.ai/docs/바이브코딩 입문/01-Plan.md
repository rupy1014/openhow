**코드부터 짜는 게 아니야. Plan부터 세워.**

"이런 거 만들고 싶어" → Claude가 계획 잡아줘 → 확인하고 → 실행.
이게 바이브코딩의 시작이야.

---

## 왜 Plan부터야?

바로 "만들어줘"라고 하면, Claude가 알아서 구조를 정해.
그러면 나중에 "이거 왜 이렇게 됐지?" 싶은 순간이 와.

Plan을 먼저 세우면 내가 방향을 잡아.
Claude는 그 안에서 일해.

---

## 어떻게 해?

**Step 1.** 바탕화면에 `my-chat` 폴더를 만들어

- **Windows**: 바탕화면 우클릭 → 새로 만들기 → 폴더 → `my-chat`
- **Mac**: 바탕화면 우클릭 → 새로운 폴더 → `my-chat`

**Step 2.** VS Code로 열고 터미널에서 `dsclaude` 실행

```bash
dsclaude
```

**Step 3.** 만들고 싶은 걸 말해

```
AI 캐릭터 채팅 서비스 만들고 싶어.

- 캐릭터 이미지 5장을 넣으면 AI가 각 캐릭터의 성격과 말투를 잡아
- 캐릭터를 골라서 대화할 수 있어
- AI 대화는 OpenRouter API로 연결해
- 한국어로 대화해

먼저 PRD.md로 계획을 정리해줘. 코드는 아직 짜지 마.
```

마지막 줄이 핵심이야. **"코드는 아직 짜지 마."** 이걸 안 하면 바로 코드부터 짜.

**Step 4.** PRD.md 확인하고 수정해

Claude가 PRD.md를 만들어줘. 기술 스택, 화면 구성, 기능 목록이 정리돼 있어.

```
PRD.md 보여줘
```

읽어보고 마음에 안 드는 게 있으면 말해.

```
캐릭터 선택 화면에서 카드 형태로 보여줬으면 좋겠어.
그리고 대화 화면은 카카오톡처럼 말풍선으로.
```

Claude가 PRD.md를 수정해. 이 과정을 원하는 만큼 반복해.

---

## PRD.md가 뭐야?

Product Requirements Document. 제품 요구사항 문서야.

쉽게 말하면 **"내가 만들고 싶은 거 정리한 문서"**야.
Claude는 이 문서를 기준으로 코드를 짜.

PRD.md가 구체적일수록 결과물이 내 의도에 가까워.
"대충 만들어줘"보다 "이런 화면에, 이런 기능으로"가 훨씬 나아.
