**"안 만들 것"이 기획의 절반이야.**

AI한테 "쇼핑몰 만들어줘"라고 하면 뭘 만드냐:

- 회원가입 + 로그인
- 상품 관리 CMS
- 장바구니 + 결제
- 리뷰 시스템
- 포인트 적립
- 관리자 대시보드
- 다크모드

너는 상품 3개 보여주는 페이지가 필요했을 뿐인데.

---

## AI는 왜 기능을 잔뜩 붙여?

AI는 "좋은 서비스"를 만들려고 해.

근데 AI가 생각하는 "좋은 서비스" = 기능이 많은 서비스야.

네가 원하는 건 "작동하는 서비스"인데.

**시키지 않으면 AI는 항상 과잉 설계해.** 이게 기본값이야.

"하지 마"를 안 말하면 = "해도 돼"로 받아들여.

---

## AI가 자주 추가하는 것들

| AI가 넣는 것 | 너한테 필요해? | 결과 |
|-------------|-------------|------|
| 로그인/회원가입 | MVP에선 거의 불필요 | 복잡도 3배 |
| 다크모드 | 있으면 좋지만 지금은 아냐 | CSS 코드 2배 |
| 관리자 페이지 | 너 혼자 쓰는 서비스인데? | 화면 2배 |
| 다국어 지원 (i18n) | 한국어만 하면 되는데 | 코드 복잡도 ↑ |
| 에러 처리 (ErrorBoundary) | MVP에서 과한 에러 처리 | 코드량 ↑ |
| 반응형 전체 지원 | 데스크톱에서만 쓸 건데 | CSS 3배 |
| 테스트 코드 | 바이브코딩에서 테스트? | 코드 2배 |

이 중 하나라도 들어가면 프로젝트 복잡도가 2~3배 뛰어.

## "안 만들 것" 리스트 쓰는 법

**Step 1.** AI가 PRD를 주면 기능 목록을 봐

**Step 2.** 각 기능마다 이 질문을 해:

```
"이 기능이 없으면 서비스가 안 돌아가?"
```

"없어도 돌아가" → 안 만들 것 목록에 넣어.

**Step 3.** 명시적으로 적어

```
안 만들 것:
- 로그인/회원가입 (인증 없이 바로 사용)
- 대화 기록 저장 (새로고침하면 초기화)
- 관리자 페이지 (캐릭터는 코드에 하드코딩)
- 다크모드
- 모바일 최적화 (데스크톱 우선)
```

괄호 안에 **왜 안 만드는지** 이유를 쓰면 AI가 더 잘 이해해.

## 프로젝트별 "안 만들 것" 예시

### AI 채팅 서비스

```
안 만들 것:
- 로그인 (인증 없이 바로 사용)
- 대화 저장 (DB 없이, 세션에서만)
- 캐릭터 생성 기능 (미리 정의된 5명만)
- 음성 대화
- 이미지 생성
```

### 포트폴리오 사이트

```
안 만들 것:
- 블로그
- CMS (내용은 HTML에 직접)
- 방문자 통계
- 댓글
- 다국어
```

### 커뮤니티 게시판

```
안 만들 것:
- 소셜 로그인 (이메일/비밀번호만)
- 파일 첨부
- 실시간 알림
- 채팅/DM
- 관리자 페이지 (DB 직접 수정)
```

이 목록을 복사해서 네 프로젝트에 맞게 수정하면 돼.

"안 만들 것"을 명확히 쓰면 AI가 딴짓을 안 해. 기획의 절반은 빼는 거야.
