**대화가 되긴 하는데, 뭔가 아쉬워. 여기서부터가 진짜야.**

---

## 대화가 끊기는 문제

캐릭터한테 "아까 뭐라고 했어?"라고 물어봐.
기억 못하면 **대화 히스토리**가 안 되는 거야.

```
대화 히스토리를 유지해줘. 이전 메시지를 API에 같이 보내서
캐릭터가 맥락을 기억하게.
```

이 한마디면 Claude가 대화 기록을 누적해서 보내는 구조로 바꿔줘.

---

## 캐릭터가 밋밋할 때

시스템 프롬프트를 수정하면 돼.

캐릭터가 너무 착해:
```
1번 캐릭터 시스템 프롬프트에 "호의적이지만 가끔 독설을 섞어" 추가해줘
```

대답이 너무 길어:
```
모든 캐릭터 시스템 프롬프트에 "대답은 3문장 이내로" 추가해줘
```

이모지가 과해:
```
3번 캐릭터는 이모지 안 쓰게 해줘
```

프롬프트 한 줄 바꿨을 뿐인데, 캐릭터 느낌이 확 달라져.
이게 프롬프트 엔지니어링이야. 거창한 게 아니야.

---

## 에러가 나면?

바이브코딩에서 에러는 자주 나. 당연한 거야.

브라우저에서 에러가 보이면, 그냥 그대로 복사해서 Claude한테 붙여넣어.

```
이런 에러가 나:
[에러 메시지 붙여넣기]
```

Claude가 고쳐줘. 에러를 읽을 줄 몰라도 돼. Claude가 읽으니까.

브라우저 콘솔(`F12` → Console)에 빨간 글씨가 보이면 그것도 복사해서 보내.
정보가 많을수록 빨리 고쳐져.

---

## 한 줄 정리

히스토리로 맥락을 살리고, 프롬프트로 개성을 잡아. 에러는 복붙하면 돼.
