**대부분은 맞아. 근데 몇 군데는 문서와 구현의 강도가 달라.**

길게 늘이지 않고 필요한 것만 적을게.

---

## 1. 승인 타입은 코드가 하나 더 보여

문서에서 주로 보이는 건:

- `hire_agent`
- `approve_ceo_strategy`

근데 코드 상수엔 이것도 있어.

- `budget_override_required`

즉,
예산 초과 승인 타입은 구현 쪽이 더 또렷해.

---

## 2. CEO 전략 승인은 문서가 더 분명해

문서와 API 예시엔 `approve_ceo_strategy`가 분명히 있어.

근데 구현에서 `hire_agent`만큼 전용 후처리가 눈에 띄는 건 덜 보여.

즉,
이건 문서/API 흐름으로 읽는 게 제일 안전해.

---

## 3. 80%와 100%는 무게가 다르게 보이더라

문서 기준으론:

- 80%: warning
- 100%: hard stop / auto-pause

구현과 테스트까지 보면,
100% 쪽 차단은 확실해.

반대로 80%는 경고와 운영 가이드 성격이 더 강해 보여.

즉,
80%는 "주의",
100%는 "실제 차단"에 더 가까워.

---

## 4. checkout / 409 / blocked는 잘 맞아

이건 비교적 자신 있게 봐도 돼.

- checkout 먼저
- 409면 다른 일로 이동
- blocked면 댓글 남기고 올리기

문서와 구현이 꽤 잘 맞는 축이야.

---

## 한 줄 정리

문서와 구현이 완전히 어긋나는 건 아니야. 다만 승인 타입, 80% warning, CEO 전략 후처리 쪽은 조금 더 조심해서 읽는 게 좋아.
