**이번 정리는 한 기준으로 다시 했어.**

> "이 말이 Paperclip 공식 문서나 구현 가이드에 실제로 있나?"

그래서 추론이 강한 부분은 많이 걷어냈어.

---

## 뭘 걷어냈어?

처음 버전엔 이런 추천이 있었어.

- 처음엔 몇 명이 최고다
- 어떤 팀부터 만들어라
- 팀별 운영법은 이게 제일 좋다

이건 실전 팁으론 쓸 수 있어.
근데 Paperclip 공식 설명 그 자체는 아니야.

그래서 이번엔:

- 공식 문서에 직접 있는 흐름
- API 예시에 보이는 구조
- 코드와 테스트로 확인되는 규칙

위주로 남겼어.

---

## 지금 남긴 축은 뭐야?

- 회사 → 목표 → CEO → 조직도 → heartbeat 흐름
- 승인 구조
- 비용 / 예산 / hard stop
- issue checkout / 409 / blocked
- board 개입 위치
- 막히면 어디부터 보는지

즉,
"어떻게 쓰면 좋을까"보다
**"Paperclip은 실제로 이렇게 설명되고 이렇게 움직인다"** 쪽으로 맞췄어.

---

## 어디는 조심해서 읽어야 해?

이건 91편에 따로 정리했어.

예:

- 문서에 보이지만 구현 후처리가 덜 또렷한 것
- 구현엔 있는데 문서에서 덜 강조하는 것
- 경고와 강제 차단이 같은 강도가 아닌 것

즉,
91편은 "다 맞다"가 아니라
"여긴 톤 차이가 있다"를 적어둔 문서야.
