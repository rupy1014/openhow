**말이 아무리 빨라도 마구(harness) 없이는 마차를 못 끌어. AI도 마찬가지야.**

[revfactory/harness](https://github.com/revfactory/harness) — Claude Code 플러그인이야. '하네스 구성해줘'라고 말하면 에이전트 팀이랑 스킬(작업 도구)을 자동으로 만들어줘.

---

## 왜 이걸 써?

이 플러그인 만든 팀이 A/B 테스트를 했어. 15개 코딩 과제로.

| | 하네스 없이 | 하네스 있으면 |
|--|-----------|------------|
| 평균 품질 점수 | 49.5 | 79.3 (**+60%**) |
| 이긴 횟수 | — | **15전 15승** |

과제가 어려울수록 차이가 커.

쉬운 과제는 +24점, 어려운 과제는 +36점.

그리고 [Harness 100](https://github.com/revfactory/harness-100)이라고, 10개 도메인에 100개 팀 세트가 이미 만들어져 있어.

콘텐츠, 개발, 데이터, 마케팅 다 있어.

---

## 이게 뭐야?

하네스는 **에이전트 팀 설계기**야.

여기서 에이전트 팀은 역할을 나눈 AI 묶음이고, 스킬은 각 에이전트가 꺼내 쓰는 작은 작업 도구야.

이 플러그인을 쓰면 이런 파일이 생겨:

| 뭐가 생겨? | 무슨 뜻이야? |
|------------|-------------|
| `.claude/agents/*.md` | 역할별 에이전트 설명서가 생겨 |
| `.claude/skills/*/skill.md` | 에이전트가 쓰는 작업 도구가 생겨 |
| 팀 모드 / 서브 에이전트 모드 | 팀으로 굴릴지, 하위 작업자로 쪼갤지 고를 수 있어 |

하네스가 자주 쓰는 패턴도 미리 잡아줘:

| 패턴 | 쉬운 말 | 언제 써? |
|------|---------|---------|
| 파이프라인 | 순서대로 넘겨 | 기획 → 디자인 → 개발 → 테스트 |
| 팬아웃/팬인 | 나눠서 하고 다시 모아 | 파일 10개를 동시에 리뷰 |
| 전문가 풀 | 역할별로 골라 써 | 질문 유형에 따라 다른 전문가 호출 |
| 생성-검증 | 만들고 바로 검사해 | 코드 짜고 바로 테스트 돌려 |
| 감독자 | 총괄 하나가 흐름 잡아 | 복잡한 프로젝트, 동적 작업 분배 |
| 계층적 위임 | 큰 일부터 작은 일로 쪼개 | 대규모 리팩터링 |

## 어떻게 깔아?

마켓플레이스로 까는 쪽이 제일 빨라.

```bash
/plugin marketplace add revfactory/harness
/plugin install harness@harness
```

전역으로 직접 넣고 싶으면 이것도 돼:

```bash
cp -r skills/harness ~/.claude/skills/harness
```

에이전트 팀 기능은 실험 기능이라 이 변수도 켜야 해:

```bash
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
```

설치 끝나면 이렇게 말하면 돼:

```text
하네스 구성해줘
하네스 설계해줘
```

이 프롬프트(AI한테 하는 말)도 자주 먹혀:

| 프롬프트 예시 |
|-------------|
| SaaS 대시보드 리뉴얼용 하네스 구성해줘. 기획, 디자인, 프론트, 검수 팀으로 나눠줘. |
| 기술 문서 리팩터링 하네스 설계해줘. 조사, 집필, 팩트체크, 편집 흐름으로 짜줘. |
| 코드 리뷰 전용 하네스 구성해줘. 버그, 회귀 검사, 문서 확인 역할로 나눠줘. |
