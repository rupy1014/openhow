**긁어온 뉴스가 50개야. 다 쓸 거야? 아니지.**

쓸만한 것만 골라내고, AI가 한국어 리뷰를 자동으로 만드는 단계야.

이전 글에서 `content/2025-03-08/` 폴더에 원문이 쌓였잖아. 이제 여기서 시작해.

---

## 필터링은 어떻게 하냐?

수집한 원문 중에는 쓸 수 없는 것도 있어. 에러 페이지, 너무 짧은 글, 광고성 글...

MAX5에서는 이런 기준으로 걸러:

- 본문 500자 미만 → 내용이 없으니까 버림
- 에러 페이지(403, 404, Access Denied) → 긁기 실패한 거니까 버림
- 이미 리뷰 만든 파일 → 중복이니까 스킵

이 필터링이 별도 스크립트가 아니야. **리뷰 생성 스크립트 안에 이미 들어가 있어.** 실행하면 알아서 걸러내고, 통과한 것만 리뷰를 만들어.

---

## 카테고리도 자동으로 매겨

뉴스 종류에 따라 다른 스타일로 리뷰해야 해. GitHub 프로젝트 소개랑 창업 성공 사례는 리뷰 방향이 다르잖아.

스크립트가 키워드를 보고 자동으로 분류해:

```
제목이나 본문에 이런 단어가 나오면:
- GitHub URL, 설치 명령어, 오픈소스 → dev_reviewer
- Product Hunt, 출시, 가격 → product_reviewer
- MRR, 매출, 인터뷰, 교훈 → story_reviewer
```

데이터 정리 파트에서 배운 "키워드 매칭"이야. 점수를 매겨서 가장 높은 유형으로 분류해. 예를 들어 GitHub URL이 있으면 개발 도구 리뷰(dev_reviewer)로 보내.

---

## 종류별 리뷰어가 뭐냐?

뉴스 종류마다 다른 톤으로 리뷰를 만들어. 총 3종류:

| 종류 | 리뷰어 | 리뷰 방향 |
|------|----------|-----------|
| GitHub/오픈소스 | `dev_reviewer` | 설치법, 활용 프롬프트 |
| 서비스 출시 | `product_reviewer` | 사용 팁, 비즈니스 기회 |
| 뉴스/사례 | `story_reviewer` | 핵심 교훈, 적용법 |

리뷰어 프롬프트는 `.agent/skills/` 폴더에 써둬:

```markdown
<!-- .agent/skills/dev_reviewer/SKILL.md -->
# Dev Reviewer

오픈소스 프로젝트, 개발 도구를 리뷰해.
핵심: "이걸 내 작업에 어떻게 쓸 수 있냐?"

톤: 개발자 친구에게 슬랙으로 공유하듯 반말체
포함: 설치 명령어, 활용 프롬프트 예시
금지: 마케팅 문구, 과장 표현
```

이건 처음에 AI한테 만들어달라고 하면 돼:

```
> .agent/skills/ 폴더에 dev_reviewer, product_reviewer, story_reviewer
> 3개의 리뷰어 프롬프트를 만들어줘.
> 각각 어떤 톤으로, 뭘 포함하고, 뭘 금지할지 정의해줘.
```

## 리뷰 생성 스크립트를 돌려

예전에는 원문을 복사해서 ChatGPT에 붙여넣고, 결과를 다시 파일로 저장했어. 뉴스 50개면 50번 반복. 미친 짓이지.

이제는 스크립트 하나로 전부 자동이야:

```bash
# 미리보기 (API 호출 없이 유형 판별만)
npm run generate:dry

# 전체 자동 생성
npm run generate

# 5개만 테스트 (처음엔 이걸로 확인)
node scripts/generate-reviews.js --limit 5
```

:::tip 처음엔 --limit 5로 시작해
한꺼번에 50개 돌리면 시간도 오래 걸리고, 프롬프트가 맘에 안 들 수도 있어. 5개 먼저 돌려보고, 결과 확인한 다음에 전체를 돌려.
:::

스크립트가 하는 일을 순서대로 보면:

```
1. content/2025-03-08/ 폴더에서 원문 파일 읽기
2. 500자 미만이면 버림 (필터링)
3. 이미 content-workflow/에 있으면 스킵 (중복 방지)
4. 키워드로 유형 판별 (GitHub URL → dev, MRR → story...)
5. 유형에 맞는 프롬프트로 AI에게 리뷰 요청
6. 결과를 content-workflow/ 폴더에 저장
```

실행하면 이런 로그가 나와:

```
🤖 MAX5 AI 리뷰 생성기
━━━━━━━━━━━━━━━━━━━━━━
📅 날짜: 2025-03-08
📄 원문 47개 발견

  🔍 hacker-news-claude-code-multi-agent.md
     → dev_reviewer (high)
     ✅ 생성 완료 (3241자)

  🔍 product-hunt-new-ai-tool.md
     → product_reviewer (high)
     ✅ 생성 완료 (2847자)

  ⏭️  hacker-news-old-article.md (이미 있음)
  ⏭️  marktechpost-short-post.md (312자 — 너무 짧음)
━━━━━━━━━━━━━━━━━━━━━━
✨ 생성: 12개
⏭️  스킵: 35개
✅ 완료!
```

47개 중 12개만 새로 만들어졌어. 나머지 35개는? 이전에 이미 만들었거나(중복 스킵), 너무 짧아서 버린 거야. 자동으로 알아서 처리해.

## 리뷰 결과는 이렇게 생겨

`content-workflow/` 폴더에 저장된 파일을 열어보면:

```markdown
# 🔧 MCP Inspector — AI 에이전트 디버깅이 이렇게 쉬웠어?

> 💡 한줄 요약: MCP 서버를 브라우저에서 바로 테스트하고
> 디버깅할 수 있는 오픈소스 도구가 나왔어.

## 이게 뭔데?
MCP 서버를 만들어봤으면 알 거야.
디버깅이 진짜 고통이거든...

## 핵심 정리
- 브라우저에서 바로 테스트
- 요청/응답 실시간 확인
...

## 바이브 코딩 레시피
추천 프롬프트:
> 이 MCP 서버 코드를 분석하고 테스트 시나리오 3개 만들어줘.

---
**원문**: [MCP Inspector](https://github.com/...)
**카테고리**: vibe_coding
**태그**: #MCP #AI에이전트 #디버깅
```

원문 하나가 이런 한국어 리뷰로 변환되는 거야. 파일 이름은 원문과 같아서 어떤 원문에서 나온 건지 바로 알 수 있어.

## 환경 설정

리뷰 생성 스크립트는 AI 모델한테 요청을 보내야 하잖아. 그 설정을 `.env` 파일에 넣어야 해.

:::info .env 파일이 뭐야?
비밀번호, API 주소 같은 민감한 설정을 저장하는 파일이야. 코드에 직접 안 쓰고 따로 관리해. 이 파일은 절대 GitHub에 올리면 안 돼.
:::

```bash
# .env 파일에 추가
LLM_MUX_ENDPOINT=https://openrouter.ai/api/v1   # AI 서비스 주소
LLM_MUX_MODEL=google/gemini-2.0-flash-exp        # 사용할 모델
LLM_MUX_API_KEY=sk-or-v1-xxx                     # API 키
```

위 예시는 OpenRouter라는 서비스를 쓰는 경우야. 여러 AI 모델을 하나의 API로 쓸 수 있어.

:::info 엔드포인트가 뭐야?
프로그램이 요청을 보내는 주소야. 식당 주소 같은 거야 — 주소를 알아야 주문할 수 있잖아. `LLM_MUX_ENDPOINT`는 "AI한테 보낼 주소"야.
:::

MAX5에서는 자체 중계 서버(llm-mux)를 써. 하지만 처음 만들 때는 OpenRouter나 OpenAI API를 직접 쓰는 게 더 간단해.

| 방법 | 장점 | 설정 |
|------|------|------|
| OpenRouter | 무료 모델 있음, 여러 모델 선택 | API 키만 발급 |
| OpenAI API | GPT 모델 직접 사용 | API 키 + 결제 등록 |
| llm-mux (자체 서버) | 비용 절감, 모델 자유롭게 교체 | 서버 세팅 필요 |

## 폴더 흐름 정리

지금까지의 데이터 흐름을 정리하면:

```
content/2025-03-08/                    ← 1단계: 수집된 원문 (영어)
  ├── hacker-news-claude-code.md
  ├── product-hunt-new-tool.md
  └── ...

     ↓ generate-reviews.js (필터 + 분류 + 리뷰)

content-workflow/                      ← 2단계: AI가 만든 리뷰 (한국어)
  ├── hacker-news-claude-code.md       ← 같은 파일명, 내용은 리뷰
  ├── product-hunt-new-tool.md
  └── ...
```

파일 이름이 같으니까, 원문과 리뷰가 1:1로 매칭돼.
