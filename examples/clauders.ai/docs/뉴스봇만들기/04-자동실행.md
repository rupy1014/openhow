**매일 아침 자동으로 돌아가게 만들어.**

수집 → AI 리뷰 → 뉴스레터 → 발송. 이걸 하나로 묶고, 매일 실행.

---

## 전체를 하나로 묶어

지금까지 만든 것들:

```
1. fetch-all.js           — RSS에서 뉴스 긁기
2. generate-reviews.js    — 필터링 + 분류 + AI 리뷰
3. generate-newsletter.js — 뉴스레터 텍스트 만들기
4. send-kakao.js          — 카톡/슬랙으로 보내기
```

지금은 이걸 하나씩 따로 실행하잖아. 귀찮아. 하나로 묶자.

**Step 1.** 전체 파이프라인 스크립트를 만들어

파이프라인은 공장 조립 라인이야. 앞 단계가 끝나면 다음 단계로 자동으로 넘어가.

```
> 전체를 순서대로 실행하는 셸 스크립트 만들어줘.
> 1. fetch-all.js 실행 (수집)
> 2. generate-reviews.js 실행 (AI 리뷰)
> 3. generate-newsletter.js 실행 (뉴스레터)
> 4. send-kakao.js 실행 (발송)
> 하나라도 실패하면 로그 남기고 중단.
> --dry-run 옵션 넣어줘 (실제로 안 하고 미리보기만).
> scripts/run-workflow.sh로 저장.
```

:::info 셸 스크립트가 뭐야?
터미널 명령어를 파일에 모아둔 거야. `node xxx.js`를 4번 치는 대신, 셸 스크립트 하나 실행하면 4개가 순서대로 돌아가. `.sh` 확장자를 써.
:::

MAX5의 `run-workflow.sh`가 이거야:

```bash
./scripts/run-workflow.sh              # 전체 실행
./scripts/run-workflow.sh --dry-run    # 미리보기
./scripts/run-workflow.sh --fetch-only # 수집만
```

실행하면:

```
🚀 콘텐츠 워크플로우
━━━━━━━━━━━━━━━━━━
📡 1단계: RSS 수집
✅ 수집 완료 (47개)

🤖 2단계: AI 리뷰 생성
  ✨ 새로 생성: 12개
  ⏭️ 스킵: 35개
✅ 생성 완료

📰 3단계: 뉴스레터 생성
✅ newsletter.txt 저장 완료

📦 4단계: 발송
✅ 카카오톡 전송 완료

⏱️ 소요 시간: 127초
```

4개 스크립트가 순서대로 실행되고, 전체 결과를 한눈에 볼 수 있어.

## 매일 자동으로 돌려

이제 이 스크립트를 매일 아침에 자동 실행해.

**Step 2.** cron으로 예약해

:::info cron이 뭐야?
알람 시계야. 근데 알람 대신 프로그램을 실행해. "매일 아침 7시에 이 스크립트 돌려줘"를 설정하는 도구야. 맥이나 리눅스에 기본으로 들어있어.
:::

```
> 매일 아침 7시에 run-workflow.sh를 자동 실행하는
> cron 설정을 알려줘.
> 로그를 날짜별로 저장하도록.
```

이런 설정이 들어가:

```bash
0 7 * * * cd /path/to/news-bot && ./scripts/run-workflow.sh >> logs/$(date +\%Y-\%m-\%d).log 2>&1
```

뜻: "매일(`* * *`) 7시(`0 7`)에 이 스크립트를 실행하고, 결과를 로그 파일에 저장해."

:::info 로그가 뭐야?
프로그램이 실행되면서 남기는 기록이야. "몇 시에 시작했고, 뭘 했고, 성공/실패했는지" 같은 정보. 날짜별로 저장하면 나중에 "어제 왜 안 됐지?" 할 때 확인할 수 있어.
:::

MAX5에서는 하루 2번 (아침 8시, 저녁 8시) 돌려.

## 내 컴퓨터가 꺼져 있으면?

cron은 내 컴퓨터에서 돌아가. 컴퓨터가 꺼져 있으면 실행 안 돼.

24시간 돌리려면 인터넷에 있는 컴퓨터(서버)를 써야 해.

| 방법 | 특징 |
|------|------|
| GitHub Actions | 무료, 코드 저장소랑 같이 관리 |
| Cloudflare Workers | 무료 티어 있음, 설정 간단 |
| 저렴한 서버 (VPS) | 월 $5, 자유도 높음 |

GitHub Actions가 가장 쉬워:

```
> GitHub Actions로 매일 아침 7시(한국시간)에
> run-workflow.sh를 실행하는 설정 파일 만들어줘.
> .env에 있는 환경변수는 GitHub Secrets로 설정할 거야.
```

:::info GitHub Actions가 뭐야?
GitHub에서 코드를 올리면 자동으로 뭔가를 실행해주는 서비스야. "코드를 올리면 자동 테스트", "매일 아침 자동 실행" 같은 게 가능해. 무료야.
:::

:::info GitHub Secrets가 뭐야?
GitHub에 비밀번호(API 키, 토큰)를 안전하게 저장하는 곳이야. `.env` 파일에 적던 내용을 여기에 넣으면, GitHub Actions에서 자동으로 읽어서 써.
:::

MAX5는 한 발 더 나가서, API를 만들어서 외부에서 호출하는 방식도 써:

```bash
# 외부에서 URL만 호출하면 실행됨
curl -s -X POST "https://max5.ai/api/cron/fetch" \
  -H "Authorization: Bearer $SECRET"
```

이러면 서버에 스크립트를 안 올려도 돼. 하지만 이건 고급 단계야. 처음엔 GitHub Actions로 충분해.

## 전체 복습

```
1. RSS 수집        → fetch-all.js
2. 품질 필터       → 500자 미만 제거, 에러 페이지 제거
3. 카테고리 분류    → 키워드 매칭 (자동)
4. AI 리뷰 생성    → 유형별 프롬프트로 자동 생성
5. 뉴스레터 생성    → generate-newsletter.js
6. 발송            → 카톡/슬랙 API
7. 자동 실행       → cron 또는 GitHub Actions
```

이 7단계가 MAX5의 전부야.

실제 실행할 때는 이렇게 4개 스크립트로 돌아가:

| 스크립트 | 위의 단계 |
|---------|----------|
| `fetch-all.js` | 1번 (수집) |
| `generate-reviews.js` | 2~4번 (필터 + 분류 + 리뷰) |
| `generate-newsletter.js` | 5번 (뉴스레터) |
| `send-kakao.js` | 6번 (발송) |

이 구조는 네가 만들 뉴스봇에도 그대로 적용돼.

## 응용 정리

| 뭘 만들고 싶어 | 바꿀 부분 |
|---------------|----------|
| 경쟁사 모니터링 | 소스 → 경쟁사 블로그 RSS |
| 업계 트렌드 봇 | 필터 → 특정 키워드만 |
| 쓰레드 자동화 | 가공 → SNS 포맷 |
| 일일 리포트 | 소스 → 사내 DB |
| 채용 모니터링 | 소스 → 채용공고 RSS |

소스를 바꾸고, 리뷰어 톤을 바꾸고, 발송 채널을 바꾸면 — 전부 같은 코드야.

---

## 한 줄 정리

수집 → 필터 → 가공 → 발송. 이 파이프라인 하나 만들어두면 뭐든 자동화할 수 있어.
