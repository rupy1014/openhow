**RSS로 뉴스를 긁고, AI가 리뷰를 쓰고, 매일 카톡으로 보내는 봇.**

이걸 직접 만들어볼 거야. MAX5라는 실제 프로젝트를 보면서.

완성되면 이런 메시지가 매일 아침 카카오톡에 와:

```
안녕하세요. 3월 8일(토) 바이브 코더를 위한 AI 브리핑입니다.
오늘 당장 써먹을 수 있는 '실전 AI 기술' 15가지를 요약했습니다.

[Today's AI News]

1. Claude Code, 멀티에이전트 워크플로우 지원 추가
2. Perplexity, 제로에서 90억 달러까지
3. MCP 서버 디버깅 도구 Inspector 출시
...
```

## 왜 이걸 만들어?

이 프로젝트 하나에 지금까지 배운 게 다 들어가.

| 배운 것 | 어디서 쓰여 |
|---------|-----------|
| 글쓰기 | AI 에이전트가 한국어 리뷰 생성 |
| 리서치 | 10개 소스에서 RSS 수집 |
| 데이터 정리 | 카테고리 분류, 중복 제거 |
| 콘텐츠 제작 | 뉴스레터 형식으로 가공 |
| 내 도구 만들기 | 스크립트로 자동화 |
| 바이브코딩 | 전체 봇을 코드로 구현 |

총정리 프로젝트야.

---

## 이 구조를 바꾸면 뭐든 돼

RSS 소스를 바꾸고, 에이전트 톤을 바꾸고, 발송 채널을 바꾸면 — 전부 같은 코드야.

| 응용 | 바꾸는 것 |
|------|----------|
| 경쟁사 모니터링 봇 | 소스 → 경쟁사 블로그 |
| 업계 트렌드 봇 | 필터 → 특정 키워드만 |
| 쓰레드 자동화 | 발송 → SNS 포맷으로 |
| 일일 리포트 | 소스 → 사내 데이터 |

전부 같은 구조야. **수집 → 필터 → 가공 → 발송.**

## MAX5 프로젝트 구조

```
max5-clone.ai/
├── scripts/                       ← 실행 스크립트 (레시피 모음)
│   ├── fetch-all.js               ← RSS 수집
│   ├── generate-reviews.js        ← AI 리뷰 자동 생성 (llm-mux)
│   ├── generate-newsletter.js     ← 뉴스레터 생성
│   ├── deploy-content.js          ← DB 배포
│   └── run-workflow.sh            ← 전체를 한 번에
│
├── .agent/skills/                 ← AI 리뷰어 프롬프트
│   ├── dev_reviewer/              ← 개발 도구 리뷰
│   ├── product_reviewer/          ← 서비스 리뷰
│   └── story_reviewer/            ← 성공 사례 리뷰
│
├── content/                       ← 수집된 원문 (날짜별)
├── content-workflow/              ← AI가 만든 리뷰
└── CLAUDE.md                      ← 프로젝트 지침서
```

데이터 흐름은 이래:

```
RSS 피드들          fetch-all.js         generate-reviews.js       send-kakao.js
  ↓                    ↓                       ↓                       ↓
10개 소스  →  content/날짜/원문.md  →  content-workflow/리뷰.md  →  카카오톡
```

각 단계를 하나씩 만들어볼 거야.
