=== 여기서부터 복사 ===

이 프로젝트에 하네스 엔지니어링을 세팅해줘. 범용 템플릿이 아니라, **이 프로젝트의 실제 구조를 분석해서** 맞춤으로 만들어야 해.

## 1단계: 프로젝트 분석

먼저 아래를 파악해줘. 결과를 보여주고 다음 단계로 넘어가기 전에 내 확인을 받아.

- 디렉토리 구조 (주요 폴더만)
- 사용 중인 언어, 프레임워크, 빌드 도구
- 기존 CLAUDE.md / AGENTS.md 내용 (있으면)
- 기존 린터 설정 (ESLint, Biome, Ruff 등)
- 기존 pre-commit hook (husky, lefthook 등)
- 테스트 프레임워크와 현재 커버리지 상태

## 2단계: 컨텍스트 엔지니어링

분석 결과를 바탕으로:

**CLAUDE.md 보강** — 기존 내용은 유지하고, 빠져 있는 것만 추가:
- 이 프로젝트의 실제 레이어/모듈 구조와 의존성 방향
- 코딩 컨벤션 (이미 있는 린터 규칙에서 추출 + 암묵적 패턴 명문화)
- 에이전트가 지켜야 할 금지 사항

**docs/architecture.md 생성** (없으면):
- 실제 디렉토리 구조 트리
- 각 디렉토리의 역할 한 줄 설명
- 의존성 방향 다이어그램
- 주요 진입점 목록

## 3단계: 아키텍처 제약

프로젝트에 이미 있는 도구를 활용해서 제약을 추가해줘. 없는 도구를 새로 설치하지 말고.

- **린터가 있으면**: 레이어 역방향 import 차단 규칙 추가
- **pre-commit hook이 있으면**: 파일 크기 체크 추가
- **CI가 있으면**: 구조 테스트 단계 제안 (직접 추가하지 말고 제안만)
- **아무것도 없으면**: CLAUDE.md의 금지 사항에만 의존하고, "이런 도구를 추가하면 좋겠다"고 제안

## 4단계: 엔트로피 관리 제안

직접 세팅하지 말고, 이 프로젝트에 맞는 청소 전략을 **제안만** 해줘:
- 어떤 엔트로피가 쌓일 가능성이 높은지
- 어떤 도구로 잡을 수 있는지
- 주기적 실행을 어떻게 자동화할 수 있는지

## 완료 확인

모든 단계가 끝나면 요약해줘:
1. CLAUDE.md에 추가/수정한 내용
2. architecture.md 생성 여부와 핵심 내용
3. 추가한 린터/hook 규칙
4. 엔트로피 관리 제안 사항
