# 하네스 플러그인 설치 프롬프트

> 이 파일의 "=== 여기서부터 복사 ===" 부터 "=== 여기까지 복사 ===" 까지 복사해서 새 Claude Code 세션에 붙여넣어.

---

=== 여기서부터 복사 ===

Claude Code에서 revfactory/harness 플러그인을 설치하고 바로 쓸 수 있게 세팅해줘.

## 1. 플러그인부터 깔아

아래 순서대로 진행해:

```bash
/plugin marketplace add revfactory/harness
/plugin install harness@harness
```

설치가 막히면 이유를 먼저 말하고, 대체로 `skills/harness`를 `~/.claude/skills/harness`로 복사하는 방식도 안내해줘.

## 2. 에이전트 팀 기능 켜

`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` 이 필요한지 확인하고, 안 켜져 있으면 어떻게 켜는지 바로 적어줘.

## 3. 하네스가 뭘 만드는지 짚어

설치 뒤에 아래를 짧게 설명해줘:

- `.claude/agents/*.md`
- `.claude/skills/*/skill.md`
- 에이전트 팀 모드와 서브 에이전트 모드 차이
- 파이프라인, 팬아웃/팬인, 전문가 풀, 생성-검증, 감독자, 계층적 위임이 각각 언제 맞는지

## 4. 바로 첫 팀 짜

내 프로젝트를 보고 가장 맞는 하네스 패턴 하나를 골라서, 왜 그 패턴이 맞는지 쉬운 말로 설명한 뒤 초안까지 만들어줘.

트리거 문장도 같이 알려줘:

```text
하네스 구성해줘
하네스 설계해줘
```

## 5. 바로 써볼 예시도 줘

내 프로젝트에 맞는 실전 프롬프트 3개만 적어줘.

출력 형식은 이렇게 맞춰:
1. 설치 결과
2. 필요한 환경 변수
3. 추천 패턴과 이유
4. 바로 쓸 프롬프트 3개
