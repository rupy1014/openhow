**AI한테 뭘 시키든, 시키는 방법이 결과를 결정해.**

---

## 좋은 지시문이 뭐야?

세 가지만 기억해.

1. **명확함** — 뭘 해달라는 건지 한눈에
2. **구체적** — 범위, 조건, 형식을 명시
3. **예시 포함** — "이런 식으로 해줘"

| 구분 | 예시 |
|------|------|
| 나쁜 예 | "좋은 글 써줘" |
| 좋은 예 | "20대 직장인 대상, 친근한 말투로, 월요병 극복 팁 3가지 알려줘" |
| 나쁜 예 | "코드 짜줘" |
| 좋은 예 | "Python으로 1~10 출력하는 코드 짜줘. 주석도 달아줘" |

## 프롬프트 5요소

필요에 따라 조합해서 써. 매번 다 넣을 필요 없어.

| 요소 | 설명 | 예시 |
|------|------|------|
| 페르소나 | AI 역할 | "너는 10년차 마케터야" |
| 목적 | 뭘 해달라는 건지 | "블로그 글 써줘" |
| 배경 | 왜 필요한지 | "신제품 출시 홍보용" |
| 형식 | 결과물 모양 | "3문단, 각 100자 내외" |
| 제약사항 | 하면 안 되는 것 | "전문용어 쓰지 마" |

## 시스템 프롬프트가 뭐야?

AI의 "역할"을 미리 정해주는 거야.

```
너는 친절한 요리 선생님이야.
초보자도 이해할 수 있게 쉽게 설명해줘.
```

이렇게 정해두면 AI가 모든 대답을 그 느낌으로 해.

구체적인 페르소나를 주면 더 전문적인 답이 나와.

```
당신은 2번의 엑싯 경험이 있는 실리콘밸리 연쇄창업가입니다.
{첨부한 문서}를 분석하고 스타트업 초기 전략과 투자 유치에 대한 조언을 해주세요.
```

## 바로 답 구하지 말고, 질문하게 시켜

AI한테 바로 답을 요구하지 말고, 질문을 먼저 시켜봐.

```
나: "블로그 글 써줘"
AI: (대충 아무 글이나 씀)

vs

나: "블로그 글 쓰려는데, 뭘 알아야 잘 써줄 수 있는지 질문해줘"
AI: "어떤 주제야? 대상 독자는? 글 길이는? 말투는?"
나: (하나씩 답해줌)
AI: (맞춤형 좋은 글)
```

소크라테스 질문법도 돼.

```
소크라테스 질문법을 활용해서 내 요구사항을 파악해줘
```

내가 미처 생각 못한 것들을 AI가 물어봐줘.

## 1원칙 사고 활용하기

기존 방식에 의존하지 말고 근본부터 다시 생각하는 방법이야.

```
나: "마케팅 어떻게 해야 해?"
AI: (일반적인 팁 나열)

vs

나: "제1원칙 사고로 우리 서비스 마케팅 전략을 세워줘.
     기존 방식 말고, 고객이 본질적으로 원하는 게 뭔지부터 분석해줘."
AI: (근본 질문부터 시작해서 새로운 관점 제시)
```

복잡한 문제를 단순화하거나, 기존과 다른 접근이 필요할 때 써.
