---

## 계획 모드가 뭐야?

바로 "만들어줘"하면 AI가 자기 생각대로 쭉 진행해.
방향이 틀려도 되돌리기 어려워.

```
나: "쇼핑몰 만들어줘"   ← 너무 막연해

vs

나: "쇼핑몰 만들려는데,
     어떤 순서로 만들면 좋을지 계획 세워줘"
AI:
1. 기본 폴더 구조 생성
2. 상품 목록 페이지
3. 상품 상세 페이지
4. 장바구니 기능
5. 결제 페이지
```

계획 확인하고 마음에 들면 "진행해줘".
아니면 "이 부분은 이렇게 바꿔"라고 수정해.

`Shift + Tab`으로 계획 모드를 켤 수 있어. (Claude Code 설치 후 사용 가능)

| 모드 | 설명 |
|------|------|
| 일반 모드 | AI가 바로 실행 |
| 계획 모드 | 계획만 세우고, 실행은 승인 후 |

---

## 동시에 여러 개 하려면?

터미널 여러 개 열면 동시에 작업할 수 있어.

```
터미널 1: claude (코드 생성)
터미널 2: npm run dev (서버 실행)
터미널 3: npm test (테스트)
```

`Ctrl+Shift+`` 또는 `+` 버튼으로 터미널을 추가해.

기다릴 필요 없이 동시에 진행돼.
단, 같은 파일을 동시에 수정하면 충돌 날 수 있어.

---

## 다른 주제 섞이면 어떻게 돼?

다른 프로젝트 내용이 섞이면 AI가 혼란해.

**새 세션을 여는 게 좋은 경우:**
- 작업 **목적이 바뀔 때** (기능 개발 → 버그 수정)
- 완전히 다른 주제로 넘어갈 때

**오염돼도 괜찮은 경우:**
- 같은 프로젝트 내 연관된 작업
- 짧은 질문-응답

```
잘못된 예:
"쇼핑몰 결제 기능 만들어줘"
(5분 후)
"블로그 댓글 기능 만들어줘"
→ AI가 쇼핑몰이랑 블로그 섞어버려

올바른 예:
- 쇼핑몰 작업은 쇼핑몰 폴더에서
- 블로그 작업은 블로그 폴더에서
- 또는 새 대화 시작
```

파일을 **드래그앤드랍**으로 입력창에 넣으면
AI가 내용을 바로 참고해. 컨텍스트를 정확하게 줘.
