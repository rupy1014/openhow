**수집한 뉴스가 50개야. 다 리뷰할 거야? 아니지.**

분류하고, 쓸만한 것만 골라야지.

MAX5에서는 10개 소스에서 매일 수십 개가 들어와. 이 중에 리뷰할 가치가 있는 건 10~15개야.

---

## 카테고리 분류를 시켜

**Step 1.** 수집한 파일들을 분류해달라고 해

```
> 이 폴더에 있는 마크다운 파일 전부 읽고,
> 아래 카테고리 중 하나로 분류해줘:
>
> - ai_tools: AI 도구 소식
> - vibe_coding: AI 코딩, 프롬프트
> - indie_hacker: 1인 창업
> - case_study: 성공/실패 사례
>
> 파일명, 제목, 분류 결과를 표로 정리해줘.
```

이거 한마디면 50개 파일이 깔끔하게 정리돼.

MAX5에서는 이걸 키워드 기반으로 자동 분류해. 제목이나 본문에 특정 단어가 나오면 자동으로 카테고리를 매겨:

```
"코딩", "에이전트", "프롬프트" → vibe_coding
"매출", "MRR", "수익" → case_study
"마케팅", "SEO", "고객" → marketing
```

처음엔 AI한테 수동으로 시키고, 패턴이 보이면 나중에 자동화하는 거야.

---

## 통계를 뽑아봐

**Step 2.** 분류 결과를 분석해

```
> 오늘 수집한 콘텐츠 통계 정리해줘.
> 카테고리별 개수, 소스별 개수,
> 본문이 너무 짧은 것(1000자 미만)도 알려줘.
```

이런 결과가 나와:

| 카테고리 | 개수 |
|----------|------|
| ai_tools | 12 |
| vibe_coding | 8 |
| case_study | 5 |
| indie_hacker | 3 |

"오늘은 AI 도구 소식이 많네. 바이브코딩 리뷰를 우선 만들자." — 이런 판단을 할 수 있어.

---

## 중복도 찾아줘

같은 뉴스가 여러 소스에서 들어올 수 있어.

```
> 이 폴더에서 제목이 비슷한 파일 찾아줘.
> 같은 뉴스인데 다른 소스에서 가져온 거 있으면 알려줘.
```

MAX5에서는 파일명 앞에 소스 이름을 붙여서 구분해:

```
hacker-news-claude-code-multi-agent.md   ← HN에서 가져온 거
techcrunch-claude-code-multi-agent.md    ← 같은 뉴스, 다른 소스 → 중복!
```
