## 소스 바꾸기

지금 기본으로 10개 소스가 설정돼 있어:

| 소스 | 뭘 긁어오냐 |
|------|-----------|
| Hacker News | 테크 커뮤니티 인기 글 |
| HackerNoon | 테크 스타트업, AI 트렌드 |
| MarkTechPost | AI 연구, 스타트업 적용 사례 |
| The Bootstrapped Founder | 솔로프리너, SaaS 운영 |
| levels.io | 인디해커 실전기 |
| Trends.vc | AI 니치 시장, 성공 사례 |
| Product Hunt | 신제품 런칭 |
| Lenny's Newsletter | 프로덕트, 그로스 전략 |
| SaaS Weekly | SaaS 트렌드, 가격 전략 |
| Entrepreneurs Handbook | 창업 스토리 |

추가하고 싶으면:

```
> RSS 소스에 GeekNews 추가해줘. URL은 https://news.hada.io/rss
```

빼고 싶으면:

```
> marktechpost 소스는 빼줘
```

딸깍.

## 리뷰 톤 바꾸기

기본은 "MZ세대 친구에게 카톡하듯" 반말체야.

존댓말로 바꾸고 싶으면:

```
> 리뷰어들의 톤을 존댓말로 바꿔줘. "~했습니다", "~인데요" 스타일로.
```

새로운 리뷰어를 만들고 싶으면:

```
> YouTube 영상 리뷰용 리뷰어를 새로 만들어줘.
> 영상 요약 + 핵심 타임스탬프 + 한줄평 형식으로.
```

리뷰어 프롬프트는 `.agent/skills/` 폴더에 마크다운 파일로 있어. Claude Code가 알아서 찾아서 수정해주니까 네가 폴더를 뒤질 필요 없어.

딸깍.

## 발송 채널 추가하기

카톡, 슬랙, 디스코드, 이메일... 뭐든:

```
> 슬랙 웹훅으로 매일 뉴스레터를 보내는 기능 만들어줘
```

```
> 카카오톡 나에게 보내기로 전송하는 기능 추가해줘
```

:::info 웹훅이 뭐야?
"이 주소로 메시지를 보내면 슬랙/디스코드에 알림이 와" 하는 URL이야. 슬랙이나 디스코드 설정에서 만들 수 있어.
:::

딸깍.

## 웹사이트에 올리기

리뷰를 웹사이트에서 보고 싶으면 DB(Supabase)에 배포할 수 있어.

Claude Code에서:

```
/deploy-content
```

이것도 딸깍이야. 배포 명령어가 이미 만들어져 있거든.

근데 꼭 안 해도 돼. `npm run dev`로 로컬에서 바로 보이니까.

## 핵심

| 바꾸고 싶은 것 | 딸깍 |
|---------------|------|
| 소스 추가/제거 | "RSS 소스에 OOO 추가해줘" |
