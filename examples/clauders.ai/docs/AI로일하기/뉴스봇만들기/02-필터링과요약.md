**기본으로 돌아가는 건 확인했어. 이제 내 입맛에 맞게 바꿔보자.**

바꾸는 것도 Claude Code한테 시키면 돼. 직접 코드 수정할 필요 없어.

---

## 소스 바꾸기

기본 소스가 10개 있어. 여기에 추가하거나, 빼거나, 바꿀 수 있어.

Claude Code에서:

```
> RSS 소스에 GeekNews 추가해줘. URL은 https://news.hada.io/rss
```

```
> 경쟁사 블로그도 추가해줘. URL은 https://competitor.com/blog/feed
```

```
> marktechpost 소스는 빼줘. 너무 학술적이야.
```

이러면 `scripts/fetch-all.js`의 SOURCES 목록을 알아서 수정해줘.

---

## 리뷰 톤 바꾸기

기본은 "MZ세대 친구에게 카톡하듯" 반말체야. 이걸 바꾸고 싶으면:

```
> dev_reviewer의 톤을 존댓말로 바꿔줘.
> "~했습니다", "~인데요" 스타일로.
```

```
> story_reviewer에 "실패 교훈" 섹션을 추가해줘.
> 성공 사례만 다루지 말고 실패에서 배울 점도 써줘.
```

리뷰어 프롬프트는 `.agent/skills/` 폴더에 마크다운으로 있어. Claude Code가 알아서 찾아서 수정해줘.

새로운 리뷰어를 만들 수도 있어:

```
> YouTube 영상 리뷰용 리뷰어를 새로 만들어줘.
> 영상 요약 + 핵심 타임스탬프 + 한줄평 형식으로.
```

---

## 필터링 기준 바꾸기

기본으로 500자 미만은 버려. 이걸 바꾸고 싶으면:

```
> 수집 필터링 기준을 바꿔줘.
> 1000자 미만은 버리고, AI/코딩 관련 키워드가 없는 글도 스킵해.
```

---

## 발송 채널 추가하기

카톡, 슬랙, 디스코드, 이메일... 뭐든 가능해.

```
> 슬랙 웹훅으로 뉴스레터를 보내는 기능 추가해줘.
> 웹훅 URL은 .env의 SLACK_WEBHOOK_URL에서 읽어.
```

```
> 디스코드 채널에 매일 뉴스를 보내는 기능 만들어줘.
```

:::info 웹훅이 뭐야?
"이 주소로 메시지를 보내면 슬랙/디스코드에 알림이 와" 하는 URL이야. 슬랙이나 디스코드 설정에서 만들 수 있어.
:::

---

## 웹사이트로 보고 싶으면

리뷰를 Supabase(DB)에 올리면 웹사이트에서도 볼 수 있어.

```
> Supabase 설정하고 콘텐츠를 DB에 배포해줘.
```

Claude Code가 `.env` 설정부터 테이블 생성, 배포까지 안내해줄 거야.

근데 꼭 해야 하는 건 아니야. 카톡/슬랙 발송만으로도 충분해.

## 핵심 포인트

| 바꾸고 싶은 것 | Claude Code에게 이렇게 말해 |
|---------------|---------------------------|
| 소스 추가/제거 | "RSS 소스에 OOO 추가해줘" |
| 리뷰 톤 변경 | "리뷰어 톤을 OOO로 바꿔줘" |
| 필터 조건 변경 | "필터링 기준을 OOO로 바꿔줘" |
| 발송 채널 추가 | "슬랙/디스코드/이메일로 보내줘" |
| 새 리뷰어 생성 | "OOO용 리뷰어 만들어줘" |

**직접 코드를 건드릴 필요 없어.** Claude Code가 프로젝트 구조를 알고 있으니까, 뭘 바꿔야 하는지 알아서 찾아서 수정해.
