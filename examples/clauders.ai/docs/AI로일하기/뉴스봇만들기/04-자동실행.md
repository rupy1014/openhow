## 반자동: 내가 한 번만 치면 되는 상태

지금이 이 상태야. Claude Code 열고 `/workflow` 한 번이면 끝.

솔직히 이것만으로도 충분해. 매일 아침 터미널 열고 `/workflow` 한 번 치는 데 5초야.

---

## 완전 자동: 내가 아무것도 안 해도 되는 상태

매일 아침 8시에 자동으로 돌리고 싶으면? 이것도 Claude Code한테 시켜.

```
> 매일 아침 8시에 콘텐츠를 자동 수집하고 리뷰까지 생성하는 cron을 만들어줘.
> GitHub Actions로 하고 싶어.
```

:::info cron이 뭐야?
알람 시계야. 근데 알람 대신 프로그램을 실행해. "매일 아침 8시에 이 스크립트 돌려줘"를 설정하는 거야.
:::

:::info GitHub Actions가 뭐야?
GitHub에서 코드를 올리면 자동으로 뭔가를 실행해주는 서비스야. "매일 아침 자동 실행" 같은 게 가능해. 무료야.
:::

---

## 자동화 방법은 여러 가지야

| 방법 | 특징 | 난이도 |
|------|------|--------|
| 내 컴퓨터 cron | 컴퓨터 켜져 있어야 함 | 쉬움 |
| GitHub Actions | 무료, 24시간 동작 | 쉬움 |
| Cloudflare Workers | 무료 티어, 설정 간단 | 보통 |
| 서버 (VPS) | 월 $5, 자유도 높음 | 어려움 |

뭘 선택하든 Claude Code한테 물어보면 설정을 만들어줘.

**내 컴퓨터에서 돌리고 싶으면:**

```
> 매일 아침 8시에 수집 + 리뷰 생성을 자동으로 돌리는 cron 설정 만들어줘.
> 로그도 날짜별로 저장해줘.
```

**GitHub Actions로 하고 싶으면:**

```
> GitHub Actions로 매일 아침 8시(한국시간)에
> 콘텐츠 수집하고 리뷰 생성하는 워크플로우 만들어줘.
> .env에 있는 환경변수는 GitHub Secrets로 설정할 거야.
```

**핵심은 이거야**: 어떤 방법을 선택하든 너는 "이걸 자동으로 돌려줘"라고 말하기만 하면 돼. 설정 파일은 AI가 만들어줘.

---

## 이게 자동화의 본질이야

```
네가 하는 일:    "자동으로 돌려줘"
AI가 하는 일:    cron 설정, GitHub Actions YAML, 에러 핸들링, 로그 설정
```

코드를 읽거나 이해할 필요 없어. "뭘 원하는지"만 말하면 돼.

---

## 전체 복습

```
1단계: 프로젝트 클론          ← git clone (1분)
2단계: /workflow 실행         ← 수집 + 리뷰 + 퇴고 (자동)
3단계: 결과 확인              ← localhost에서 바로 확인
4단계: 커스터마이징           ← 소스, 톤, 채널 변경 (AI한테 시킴)
5단계: 자동화                ← cron 또는 GitHub Actions (AI한테 시킴)
```

1~3단계면 뉴스봇이 돌아가. 4~5단계는 선택이야.

## 응용 정리

| 뭘 만들고 싶어 | Claude Code에게 이렇게 말해 |
|---------------|---------------------------|
| 경쟁사 모니터링 | "소스를 경쟁사 블로그로 바꿔줘" |

| 쓰레드 자동화 | "리뷰를 트위터 쓰레드 형식으로 바꿔줘" |

| 채용 모니터링 | "채용공고 RSS를 수집하게 바꿔줘" |

소스를 바꾸고, 리뷰어 톤을 바꾸고, 발송 채널을 바꾸면 — 전부 같은 코드야.
