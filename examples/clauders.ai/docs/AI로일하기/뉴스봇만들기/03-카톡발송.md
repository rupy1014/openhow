**리뷰가 완성됐어. 이제 뉴스레터로 만들고 보내자.**

`content-workflow/` 폴더에 한국어 리뷰가 쌓여 있잖아. 이걸로 두 가지를 할 거야:

1. 리뷰들을 뉴스레터 텍스트로 변환
2. 카카오톡으로 발송

---

## 뉴스레터 텍스트 만들기

리뷰 파일이 10개 있어. 이걸 뉴스레터 형식으로 정리해야 해.

**Step 1.** AI한테 뉴스레터 생성 스크립트를 시켜

```
> content-workflow/ 폴더에 있는 리뷰 파일들을 읽고
> 뉴스레터 텍스트를 만드는 스크립트 만들어줘.
>
> 형식:
> 안녕하세요. {날짜} 바이브 코더를 위한 AI 브리핑입니다.
> 오늘의 실전 AI 기술 {N}가지를 요약했습니다.
>
> [Today's AI News]
> 1. {제목}
> 2. {제목}
> ...
>
> 중요한 순서대로 15개만 골라줘.
> content-workflow/newsletter.txt로 저장.
```

MAX5에서는 카테고리별 점수를 매겨서 정렬해:

- 바이브코딩 관련 → 점수 높음 (독자가 가장 관심 있는 주제)
- AI 도구 소식 → 중간
- 성공 사례 → 낮음

이렇게 우선순위를 정하면 뉴스레터가 매번 일관성 있게 나와.

실행:

```bash
node scripts/generate-newsletter.js
```

`content-workflow/newsletter.txt`가 생겨. 열어보면 카톡으로 보낼 메시지가 완성돼 있어.

---

## 카카오톡으로 보내기

카카오톡 "나에게 보내기" 기능을 쓰면, 내 카톡에 자동으로 메시지를 보낼 수 있어.

:::info API가 뭐야?
프로그램끼리 대화하는 방법이야. "카카오톡 API"는 프로그램이 카카오톡한테 "이 메시지 보내줘"라고 요청하는 통로야.
:::

**Step 2.** 카카오 API를 셋업해

```
> 카카오 디벨로퍼스에서 "나에게 보내기" API 쓰려면
> 어떤 설정이 필요한지 단계별로 알려줘.
```

핵심만:
1. [developers.kakao.com](https://developers.kakao.com)에서 앱 만들기
2. 카카오 로그인 켜기
3. "나에게 보내기" 권한 추가
4. 인증 토큰 발급

:::info 토큰이 뭐야?
API를 쓰기 위한 임시 비밀번호야. "나 인증된 사람이야"를 증명하는 거야. 일정 시간이 지나면 만료되니까 자동 갱신 로직을 넣어야 해.
:::

**Step 3.** 발송 스크립트를 만들어

```
> content-workflow/newsletter.txt를 읽고 카톡 나에게 보내기로 전송하는 스크립트 만들어줘.
> scripts/send-kakao.js 로 저장.
> .env 파일에서 토큰을 읽어서 써.
> 토큰 만료 시 자동 갱신해줘.
```

`.env` 파일에 토큰을 저장해:

```bash
# .env에 추가
KAKAO_REST_API_KEY=your_api_key
KAKAO_ACCESS_TOKEN=your_token
KAKAO_REFRESH_TOKEN=your_refresh_token
```

:::info .env 파일이 뭐야?
API 키나 비밀번호 같은 민감한 정보를 저장하는 파일이야. 코드에 직접 안 쓰고 따로 관리해. 이전 글에서 AI 서비스 설정할 때도 .env를 썼어.
:::

실행:

```bash
node scripts/send-kakao.js
```

카톡에 메시지가 오면 성공.

## DB에 올리기 (사이트용)

MAX5처럼 웹사이트도 운영하고 싶으면, 리뷰를 DB에도 올려.

:::info DB가 뭐야?
데이터베이스. 엑셀 시트를 인터넷에 올려놓고 프로그램에서 읽고 쓸 수 있게 한 거야. 웹사이트가 콘텐츠를 보여주려면 DB에서 데이터를 가져와야 해.
:::

MAX5에서는 Supabase라는 무료 서비스를 써:

```bash
# 미리보기 (실제로 저장 안 함)
./scripts/deploy-content.sh --dry-run

# 신규만 배포
./scripts/deploy-content.sh --new-only
```

꼭 해야 하는 건 아니야. 카톡 발송만으로도 충분해.

## 카톡 말고 다른 채널은?

같은 구조로 다른 곳에도 보낼 수 있어.

| 채널 | 방법 |
|------|------|
| 슬랙 | 웹훅 URL로 보내기 |
| 디스코드 | 웹훅 URL로 보내기 |
| 이메일 | 메일 발송 라이브러리 |
| 텔레그램 | Bot API |

:::info 웹훅이 뭐야?
"뭔가 생기면 여기로 알려줘" 하는 URL이야. 슬랙이나 디스코드에서 웹훅 URL을 만들면, 그 URL로 메시지를 보낼 수 있어.
:::

채널을 바꾸는 건 발송 부분만 바꾸면 돼. 수집-필터링-리뷰는 그대로야.

