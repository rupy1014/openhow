**해외 뉴스 원문 하나를 던지면, AI가 한국어 리뷰를 써.**

MAX5에서는 테크 뉴스를 "친구한테 카톡하듯" 설명하는 리뷰를 만들어.

사람이 하면 한 편에 30분. AI한테 시키면 2분.

---

## 어떻게 시키냐?

**Step 1.** 원문을 폴더에 넣어

MAX5에서 수집한 원문은 마크다운 파일이야. 마크다운은 `#`이 제목, `**`가 굵게 — 글을 쉽게 꾸미는 방법이야.

```markdown
# Claude Code adds multi-agent workflow support

> Source: [https://techcrunch.com/...](https://techcrunch.com/...)

Claude Code now supports multi-agent workflows...
```

**Step 2.** AI한테 리뷰를 시켜

```
> 이 원문을 읽고 한국어 리뷰를 써줘.
> 톤: 친구한테 카톡으로 설명하듯, 반말
> 어려운 용어는 쉽게 풀어서
> 형식은 이렇게:
>
> # {이모지} {한국어 제목}
> > 한줄 요약
> ## 이게 뭔데?
> ## 핵심 정리
> ## 우리는 이렇게 생각해
```

**Step 3.** 결과를 확인하고 다듬어

```
> "이게 뭔데?" 부분이 너무 길어. 3문단으로 줄여줘.
> "우리는 이렇게 생각해"에 바이브코딩 활용법도 넣어줘.
```

끝. 파일이 바로 수정돼.

---

## 뉴스 종류에 따라 다르게 시켜

모든 뉴스를 같은 방식으로 리뷰하면 밋밋해. MAX5에서는 종류별로 다르게 써.

| 종류 | 어떤 뉴스야 | 리뷰 방향 |
|------|-----------|-----------|
| 오픈소스 도구 | GitHub 링크, 코드, 설치법 | "이걸 내 작업에 어떻게 쓰냐" |
| 서비스 출시 | Product Hunt, 가격, 기능 | "이거 써보면 뭐가 좋으냐" |
| 성공 사례 | 매출 숫자, 인터뷰 | "여기서 뭘 배울 수 있냐" |

이걸 프롬프트에 반영하면 돼:

```
> 이 원문은 오픈소스 도구야.
> GitHub 스타 수, 설치 방법, 핵심 기능을 정리하고
> Claude Code에서 이 도구를 어떻게 쓸 수 있는지 넣어줘.
```

## 톤 맞추는 핵심: 예시를 줘

"친근하게 써줘"는 모호해. 사람마다 "친근함"이 다르니까.

**이전에 쓴 글을 보여주는 게 10배 정확해.**

```
> 이 폴더에 있는 기존 리뷰 3개 읽고, 같은 톤으로 써줘.
```

MAX5에서는 한 발 더 나가서, CLAUDE.md에 톤 가이드를 써뒀어:

```markdown
# 톤앤매너
- 반말체 ("~했대", "~인 거야")
- 비유 많이
- 금지: 번역체, 마케팅 문구, 과장
```

CLAUDE.md는 프로젝트의 지침서야. 한 번 써두면 매번 설명 안 해도 돼.
