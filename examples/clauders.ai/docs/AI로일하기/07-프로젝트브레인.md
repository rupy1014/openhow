**PRD 50페이지, 시장분석, 경쟁사 자료 — 이걸 AI가 통째로 이해하고 있으면?**

프로젝트에 대해 물어볼 때마다 매번 컨텍스트를 설명할 필요가 없어.

NotebookLM에 소스를 올리고, 더 찾게 시키고, MCP로 데이터를 땡겨와서 내 프로젝트만의 AI 두뇌를 만드는 거야.

---

## NotebookLM이 뭐냐

Google이 만든 AI 리서치 도구야. [notebooklm.google.com](https://notebooklm.google.com)

핵심은 **소스 기반 답변**이야. 네가 올린 문서만 근거로 답변해. 헛소리(할루시네이션)가 거의 없어.

| 일반 AI 채팅 | NotebookLM |
|-------------|------------|
| 학습된 지식으로 답변 | **네가 올린 문서**로만 답변 |
| 출처 불명확 | 출처(소스) 표시됨 |
| 프로젝트 맥락 모름 | 프로젝트 전체를 이해한 상태 |

PDF, Google Docs, 웹페이지 URL, 텍스트 — 다 올릴 수 있어. 올리면 AI가 전부 읽고, 그 안에서 답변해줘.

---

## 뭐가 만들어지냐

실제 프로젝트 하나를 예시로 보여줄게.

**맥비(macbe)** — 기획자용 큐레이션 매거진이야. 894개의 기획 자료를 카테고리별로 정리한 웹사이트.

```
프로젝트 문서들:
├── PRODUCT_SPEC.md     ← 제품 스펙 문서
├── README.md           ← 프로젝트 개요
├── data/               ← 894개 기획 자료 CSV
│   ├── gsheet_1024.csv         ← Google Sheets 원본 (4,525줄)
│   ├── cleaned_resources.csv   ← 정제된 데이터 (1,324건)
│   └── VALIDATION_REPORT.md    ← 링크 검증 리포트
└── scripts/            ← 데이터 파이프라인
    ├── clean_csv.py
    ├── validate_and_enrich.py
    └── csv_to_json.py
```

이 문서를 전부 NotebookLM에 올리면, AI가 프로젝트 전체를 이해한 상태에서 답변해.

"기획 자료 카테고리 중에서 가장 콘텐츠가 부족한 건 뭐야?"라고 물으면, 데이터 + 스펙 문서를 다 읽은 상태에서 대답해줘.

---

## 어떻게 NotebookLM에 올리냐

**Step 1.** NotebookLM에서 새 노트북을 만들어

```
> NotebookLM에 "macbe-brain"이라는 노트북 만들어줘.
```

**Step 2.** 프로젝트 핵심 문서를 소스로 추가해

```
> 이 파일들을 소스로 올려줘:
> - PRODUCT_SPEC.md (제품 스펙)
> - README.md (프로젝트 개요)
> - data/VALIDATION_REPORT.md (링크 검증 결과)
```

:::info 뭘 올려야 해?
코드 파일 말고, **프로젝트를 설명하는 문서**를 올려. 스펙, 기획서, 분석 자료, 데이터 리포트. AI가 "이 프로젝트가 뭘 하려는 건지" 이해할 수 있는 것들이야.
:::

**Step 3.** 올린 소스를 확인해

```
> macbe-brain 노트북에 소스가 잘 올라갔는지 확인해줘.
```

여기까지가 기본 세팅이야. 이제 NotebookLM이 너의 프로젝트를 "읽은" 상태야.

---

## 더 찾게 시키면 뭐가 달라지냐

소스만 올리면 NotebookLM은 올린 문서 안에서만 답해.

근데 **리서치 기능**을 쓰면 웹에서 관련 자료를 더 찾아와.

**Step 1.** 리서치를 시작해

```
> macbe-brain 노트북에서 리서치 시작해줘.
> 주제: "기획자/PM을 위한 큐레이션 서비스 사례와 리소스 매거진 운영 전략"
```

NotebookLM이 웹에서 관련 기사, 블로그, 논문을 찾아서 노트북에 추가해.

**Step 2.** 리서치 결과를 확인해

```
> 리서치 상태 확인해줘. 끝났으면 어떤 자료를 찾았는지 알려줘.
```

이제 너의 스펙 문서 + 데이터 리포트 + **웹에서 찾은 최신 자료**까지 합쳐진 상태야.

"해외에 비슷한 기획자 큐레이션 서비스가 있어?"라고 물으면, 내 문서 + 외부 사례를 같이 참고해서 답해줘.

## NotebookLM MCP가 뭐냐

여기가 핵심이야. NotebookLM에 쌓인 지식을 **내 작업 환경으로** 가져오는 거야.

[notebooklm-mcp](https://github.com/PleasePrompto/notebooklm-mcp) — NotebookLM을 Claude Code에서 직접 쓸 수 있게 해주는 MCP 서버야.

설치는 한 줄이야:

```bash
claude mcp add notebooklm -- npx -y notebooklm-mcp
```

이러면 Claude Code 안에서 노트북을 만들고, 소스를 올리고, 질문하고, 리서치를 시키는 게 전부 가능해져.

위에서 한 작업을 터미널 떠나지 않고 다 할 수 있어.

---

## MCP로 데이터를 어떻게 땡겨오냐

**Step 1.** 노트북에 질문해서 정리된 답변을 받아

```
> macbe-brain 노트북에 물어봐:
> "현재 894개 자료의 카테고리 분포를 분석하고,
> 콘텐츠가 부족한 카테고리 3개와 보강 방향을 정리해줘."
```

MCP가 NotebookLM에 쿼리를 날리고, 답변을 가져와.

**Step 2.** 답변을 프로젝트 파일로 저장해

```
> 방금 받은 답변을 docs/category-analysis.md로 저장해줘.
```

**Step 3.** 반복해서 지식을 쌓아

```
> 노트북에 물어봐:
> "링크 검증에서 깨진 260개 링크의 패턴이 뭐야?
> 어떤 카테고리에서 가장 많이 깨졌어?"
> 답변을 docs/broken-links-analysis.md로 저장해.
```

이렇게 하면 NotebookLM에 쌓인 지식이 프로젝트 안에 마크다운 파일로 정리돼.

## 전처리해서 RAG로 쓰려면

쌓인 문서를 프로젝트 전용 RAG로 만들면, AI가 매번 전체 문서를 다 읽지 않아도 돼.

맥비 프로젝트에서 실제로 한 방식이야:

```
원본 데이터 (Google Sheets CSV, 4,525줄)
  ↓ clean_csv.py (정제)
정리된 데이터 (1,324건)
  ↓ validate_and_enrich.py (검증 + 보강)
검증된 데이터 (894건, 100% 링크 확인)
  ↓ csv_to_json.py (변환)
resources.json (검색 가능한 형태)
```

이걸 NotebookLM 리서치 결과에도 똑같이 적용해:

```
NotebookLM 리서치 결과 (노트, 쿼리 답변)
  ↓ MCP로 추출
마크다운 파일들 (docs/ 폴더)
  ↓ 전처리 (중복 제거, 카테고리 분류)
정리된 지식베이스
  ↓ CLAUDE.md에 경로 등록
프로젝트 브레인 완성
```

**핵심은 CLAUDE.md야.** 여기에 지식베이스 경로를 적어두면, Claude Code가 프로젝트 시작할 때 자동으로 참고해.

```markdown
<!-- CLAUDE.md에 추가 -->
## 프로젝트 지식베이스
- docs/category-analysis.md — 카테고리 분포 분석
- docs/broken-links-analysis.md — 깨진 링크 패턴
- docs/curation-strategy.md — 큐레이션 전략 리서치
```
