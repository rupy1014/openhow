**리뷰 하나에서 뉴스레터, SNS, 블로그를 전부 뽑아.**

한 소스에서 여러 형태를 만드는 거야. 이걸 **원소스 멀티유즈**라고 해.

```
리뷰 원본 → 뉴스레터 요약
         → SNS 포스트
         → 블로그 글
```

---

## 뉴스레터 만들기

**Step 1.** 리뷰 파일들을 읽고 뉴스레터로 만들어

```
> 이 폴더에 있는 리뷰 전부 읽고,
> 오늘의 뉴스레터를 만들어줘.
>
> 형식:
> 안녕하세요. 3월 8일(토) AI 브리핑입니다.
> 오늘의 실전 AI 기술 N가지를 요약했습니다.
>
> 1. {제목}
> 2. {제목}
> ...
>
> 중요한 것부터 15개만 골라줘.
```

MAX5에서는 이 형식을 스크립트로 자동화했어. 처음엔 수동으로 하다가, 패턴이 잡히니까 코드로 만든 거야.

---

## SNS 콘텐츠도 뽑아

**Step 2.** 같은 리뷰에서 SNS용으로 변환

```
> 이 리뷰를 바탕으로 SNS 콘텐츠 만들어줘.
>
> - 트위터/X: 한 줄 임팩트 + 링크
> - 쓰레드: 5장짜리 텍스트 (1장당 3줄)
> - 링크드인: 전문적이면서 친근하게, 5줄
```

하나의 리뷰에서 3개 채널 콘텐츠가 나와.

---

## 톤 맞추기의 핵심: 에이전트

매번 톤을 설명하기 귀찮잖아. MAX5에서는 콘텐츠 종류별로 **담당자(에이전트)**를 만들었어.

자동화에서 배운 그 에이전트 개념이야. 역할을 정해두면 매번 동일한 품질이 나와.

```
.claude/agents/newneek-reviewer.md   ← 뉴닉 스타일 리뷰어
.agent/skills/dev_reviewer/          ← 개발 도구 전문
.agent/skills/product_reviewer/      ← 서비스 리뷰 전문
.agent/skills/story_reviewer/        ← 성공 사례 전문
```

각 에이전트 파일에 이렇게 써두면:

```markdown
말투:
  - 친근한 반말체 ("~했대", "~인 거야")
  - 비유 많이
금지:
  - 번역체 ("~하는 것이 중요합니다")
  - 마케팅 문구 그대로 옮기기
```

"이 에이전트로 써줘"만 하면 톤이 맞아.

---

## 대량 생산은 이렇게

```
> 이 폴더에 있는 원문 전부 읽고,
> 종류별로 분류한 다음,
> 각각에 맞는 형식으로 리뷰를 만들어줘.
> 오픈소스면 활용법 포함, 서비스면 사용 팁 중심.
> 결과는 content-workflow/ 폴더에 저장.
```

이러면 한 번에 10개 이상 리뷰가 만들어져.

처음엔 하나씩 확인하면서 톤을 잡고. 톤이 잡히면 한꺼번에 돌려.
