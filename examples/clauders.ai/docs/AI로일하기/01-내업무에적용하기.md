**뉴스봇은 예시야. 진짜 목표는 네 업무를 AI한테 시키는 거야.**

:::youtube WrETuMoY9yA
:::

"우리 업무는 다른데요" — 다 그렇게 말해. 근데 뜯어보면 패턴이 같아.

---

## 뭘 자동화할 수 있어?

네 하루를 돌아봐. 이 세 가지 중 하나에 해당하면 AI한테 시킬 수 있어.

**1. 반복하는 거**

매주 하는 보고서, 매일 확인하는 데이터, 매번 같은 포맷으로 정리하는 문서.

"또 이거야?" 싶은 순간이 신호야.

**2. 수집 → 정리 → 생성 패턴**

어딘가에서 정보를 모으고, 정리하고, 새로운 문서나 메시지를 만드는 일.

- 경쟁사 뉴스 모아서 요약 → 뉴스봇이랑 같은 패턴
- 회의록 정리해서 액션 아이템 추출 → 같은 패턴
- 고객 피드백 모아서 주간 리포트 → 같은 패턴

**3. 판단이 필요 없는 거**

"A이면 B를 해"처럼 규칙이 명확한 일. 판단이 필요한 건 아직 사람이 해야 해.

```
✅ 자동화하기 좋은 것:
- 매주 월요일 주간 보고서 템플릿 채우기
- 새 블로그 글이 올라오면 SNS 포스트 초안 만들기
- 엑셀 데이터를 정해진 포맷으로 변환하기

❌ 아직 사람이 해야 하는 것:
- 채용 합격 여부 결정
- 클라이언트 제안서 최종 톤 결정
- 위기 상황 대응 판단
```

## 어떻게 적용해?

:::steps

### 업무 하나를 골라

"가장 귀찮은 반복 업무" 하나만.

큰 거 말고 작은 거부터. "매주 경쟁사 블로그 확인하기", "회의 후 요약 메일 보내기" 같은 거.

### 수집 → 정리 → 생성으로 쪼개

아무리 복잡한 업무도 이 세 단계로 쪼갤 수 있어.

| 단계 | 하는 일 | 뉴스봇 예시 | 네 업무 |
|------|---------|------------|--------|
| 수집 | 어딘가에서 정보를 가져와 | RSS 피드 읽기 | ? |
| 정리 | 필요한 것만 골라서 정리해 | AI 요약 + 필터링 | ? |
| 생성 | 최종 결과물을 만들어 | 카톡 메시지 발송 | ? |

이 표를 채워봐. 빈칸이 채워지면 뭘 만들어야 하는지 보여.

### Claude한테 시켜

```text:Claude 프롬프트
나는 매주 [업무 내용]을 해.

수집: [어디서 정보를 가져와]
정리: [어떤 기준으로 골라]
생성: [최종 결과물이 뭐야]

이 과정을 자동화하는 스크립트 만들어줘.
```

:::

## 어떤 업무에 써?

**주간 경쟁사 모니터링**

```
수집: 경쟁사 3곳 블로그 RSS
정리: 새 글만 추리고 AI가 한 줄 요약
생성: Slack #competitor 채널에 포스트
```

**회의록 자동 정리**

```
수집: 회의 녹음 텍스트 (Clova Note, Notta 등)
정리: 핵심 논의 + 결정사항 + 액션아이템 추출
생성: Notion 회의록 페이지에 정리
```

**SNS 콘텐츠 제작**

```
수집: 이번 주 발행한 블로그 글
정리: 핵심 포인트 3개 추출
생성: 인스타/X/링크드인용 텍스트 각각 생성
```

어떤 업무든 수집 → 정리 → 생성으로 쪼개면 뉴스봇이랑 구조가 같아.

---

## 뭘 조심해?

**작은 것부터 해.** "전체 업무 자동화"가 아니라 "이 한 가지 귀찮은 거" 하나만.

**완벽하지 않아도 돼.** 80%만 자동화하고 나머지 20%는 사람이 확인해도 충분해. 0%에서 80%가 되는 것만으로 이미 큰 거야.

**자동화한 걸 기록해.** 뭘 자동화했는지, 어떤 프롬프트를 썼는지 적어둬. 다음 자동화가 2배 빨라져.

---

## 한 줄 정리

네 업무에서 "반복 + 수집→정리→생성" 패턴을 찾아. 그게 자동화 대상이야.
