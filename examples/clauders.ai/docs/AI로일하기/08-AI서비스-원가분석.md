**사주 리포트 하나 생성하는 데 1,500원.**

AI 서비스를 만들면 제일 먼저 드는 생각이 "이거 돈 얼마나 나가지?"야.

실제로 운영 중인 사주 서비스(운세냥)의 원가를 까볼게. LLM API 호출 몇 번에, 건당 얼마인지.

---

## 뭘 만드는 서비스냐

사주 팔자를 분석해서 **5만 자짜리 프리미엄 리포트**를 생성하는 서비스야.

생년월일시를 입력하면 AI가 17개 장(운명 코드, 대운, 연애, 재물 등)을 각각 써서, 하나로 합쳐줘.

```
사용자 입력: 1991년 2월 16일 0시 30분, 여성
    ↓
맥미니 워커가 13~17장을 병렬 생성
    ↓
5만 자 리포트 완성 (약 3~5분)
```

---

## 비용 구조가 어떻게 되냐

크게 두 가지야. **LLM 호출 비용** + **인프라 비용**.

### LLM 호출 (변동비 — 건당 발생)

| 항목 | 수치 |
|------|------|
| 모델 | `gpt-4.1-mini` (OpenRouter 경유) |
| 장당 서브파트 | 2~3개 |
| 총 LLM 호출 수 | **35~40회** |
| 호출당 비용 | ~$0.04 |
| **리포트 1건 합계** | **~$1.50 (약 2,000원)** |

짧은 장(3,500자 이하)은 한 번에 싱글패스로 생성하고, 긴 장은 서브파트를 나눠서 병렬로 생성해.
그래서 장이 17개여도 LLM 호출은 35~40회 정도.

### RAG (벡터 검색 — 거의 공짜)

| 항목 | 수치 |
|------|------|
| 임베딩 모델 | `text-embedding-3-small` (OpenAI) |
| 장당 RAG 쿼리 | 2~4회 |
| 총 임베딩 호출 | ~100회 |
| **리포트 1건 합계** | **~$0.002 (약 3원)** |

사주 지식을 벡터 DB(Qdrant)에 넣어놓고, 장마다 관련 지식을 검색해서 프롬프트에 넣어줘.
그런데 임베딩 비용이 워낙 싸서 사실상 무시할 수준이야.

### 인프라 (고정비 — 월 단위)

현재는 **맥미니 셀프호스팅**이야. 맥미니 하나에 워커 + Qdrant + 오케스트레이터 다 올라가.

| 항목 | 비용 |
|------|------|
| 맥미니 (이미 보유) | $0 |
| 전기세 | 월 ~$5 |
| Cloudflare Tunnel | 무료 |
| **월 고정비** | **~$5** |

## 클라우드로 옮기면 얼마냐

맥미니가 고장 나거나, 스케일이 필요하면 클라우드로 가야 해. 그때 비용이 어떻게 바뀌냐.

| 항목 | 맥미니 (현재) | 클라우드 |
|------|-------------|---------|
| LLM API | $1.50/건 | $1.50/건 (동일) |
| 임베딩 API | $0.002/건 | $0.002/건 (동일) |
| 벡터 DB | $0 (로컬 Qdrant) | ~$25/월 (Qdrant Cloud) |
| 서버 | $0 (맥미니) | ~$10/월 (fly.io 등) |
| **월 고정비** | ~$5 | **~$35~40** |

LLM 호출 비용은 어디서 돌리든 똑같아. 차이는 **월 고정비** $30~35 정도.

그래서 이렇게 돼:

| 월 생성량 | 맥미니 건당 | 클라우드 건당 |
|----------|-----------|-------------|
| 50건 | ~$1.60 | ~$2.30 (+$0.70) |
| 100건 | ~$1.55 | ~$1.85 (+$0.35) |
| 500건 | ~$1.51 | ~$1.57 (+$0.07) |

**월 500건 넘어가면 차이가 거의 없어.** 그 전까진 맥미니가 이득이야.

## 수익성은 어떻냐

원가를 알았으니까 마진을 볼 수 있어.

| 상품 | 판매가 | 원가 | 마진 |
|------|--------|------|------|
| 프리미엄 17장 | 14,900원 | ~2,000원 | **약 7배** |
| 라이트 7장 | 9,900원 | ~800원 | **약 12배** |
| 딥다이브 1장 | 4,900원 | ~60원 | **약 80배** |

LLM 서비스의 특징이야. 변동비(API 호출)는 싸고, 고정비(개발·운영)가 비싸.
**건당 마진이 높으니까 소량으로도 수익이 나.**

---

## 비용을 줄이려면 어떻게 하냐

세 가지 레버가 있어.

**1. 모델을 바꿔라**

`gpt-4.1-mini` 대신 더 싼 모델로 내리면 호출당 비용이 절반 이하로 떨어져.
대신 품질도 떨어지니까, 장별로 "싼 모델로 충분한 장"과 "비싼 모델이 필요한 장"을 나눌 수 있어.

**2. 캐싱을 써라**

같은 생년월일이면 대운·원국 분석이 동일하잖아. 이런 장은 한 번 생성하고 캐싱하면 재생성 비용이 $0이야.

**3. RAG로 프롬프트를 줄여라**

프롬프트에 지식을 다 넣으면 입력 토큰이 많아져서 비싸. RAG로 필요한 지식만 뽑아서 넣으면 입력 토큰이 줄어들어.
이 서비스에서 RAG를 쓰는 이유가 이거야. **품질을 올리면서 동시에 비용을 낮춰.**

## 한 줄 정리

AI 서비스 원가의 90%는 LLM API 호출이야. 인프라는 셀프호스팅이면 거의 공짜고, 클라우드여도 볼륨이 늘면 무시할 수준이 돼. **건당 2,000원으로 15,000원짜리 상품을 만들 수 있다** — 이게 AI 서비스의 매력이야.
