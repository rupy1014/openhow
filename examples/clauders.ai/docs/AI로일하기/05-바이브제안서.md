**마크다운으로 제안서 쓰고, AI한테 산출물까지 만들게 했더니 수주됐어.**

[실제 제안서 보기 (RxSight LAL 마케팅 제안서)](https://aracare-rxsight-proposal.vercel.app/)

---

## 뭐가 다르냐

보통 제안서는 이렇게 만들잖아.

```
기존 방식:
  내용 정리 → PPT 열기 → 디자인 → 산출물 목업 따로 → 수정 반복

바이브 방식:
  마크다운에 내용 정리 → AI가 HTML 제안서로 변환
  → 산출물도 AI가 프로토타입으로 만들어서 제안서 안에 내장
```

이 제안서는 PPT가 아니야. **HTML 단일 페이지**야. 사이드바에 목차가 있고, 스크롤하면서 읽어.

핵심은 — 제안서 안에서 **산출물을 바로 미리보기**할 수 있다는 거야.

---

## 어떻게 만들었냐

**Step 1.** 제안서 구조를 마크다운으로 잡아

```markdown
00. Cover — 표지
01. Executive Summary — 핵심 요약
02. Analysis & Strategy — 현황 분석, 전략 방향
03. About Us — 회사 소개
04. Deliverables — 산출물 구성 + 견적
05. Action Plan — 일정
06. Budget — 세부 견적
07. Client Checklist — 클라이언트 준비 사항
08. Contact — 연락처
```

내용만 정리하면 돼. 디자인은 AI가 해.

**Step 2.** AI한테 HTML 제안서로 변환 요청해

```
> 이 마크다운을 제안서 HTML로 만들어줘.
> 다크 테마, 사이드바 목차, 섹션별 스크롤.
> 각 산출물에 "미리보기" 버튼 달아줘.
```

바이브 PPT에서 했던 것과 같은 방식이야. 마크다운 → HTML 변환.

**Step 3.** 산출물 프로토타입을 AI로 만들어

여기가 제안서만의 차별점이야. **납품할 산출물의 시안을 미리 만들어서 제안서에 넣어.**

이 제안서에 들어간 프로토타입들:

| 산출물 | 형태 | 만든 방법 |
|--------|------|----------|
| 브랜딩 전략 가이드 | HTML 문서 | Claude Code |
| B2B 도입 제안서 | HTML 프레젠테이션 | Claude Code |
| 환자용 브로셔 (A5 8P) | HTML 레이아웃 | Claude Code + 나노바나나 |
| 영상 스토리보드 | HTML (감성/이성 2버전) | Claude Code |
| 포스터, 리플릿, X배너 | HTML 레이아웃 드래프트 | Claude Code + 나노바나나 |

전부 HTML이야. 포토샵, 피그마 안 열었어.

---

## 이미지는 어떻게 만들었냐

**나노바나나**를 썼어. AI 이미지 생성 도구야.

브로셔에 들어가는 제품 이미지, 배경 그래픽, 아이콘 — 전부 AI가 생성한 거야.

```
방법:
  나노바나나에서 이미지 생성 → 다운로드 → HTML에 삽입
```

제안 단계에서 완성도 높은 시안을 보여주는 게 핵심이야. 클라이언트가 "이게 실제로 나올 결과물이구나"를 바로 느끼게.

---

## 왜 수주됐냐

```
일반 제안서:  "브로셔 만들겠습니다" → 텍스트 설명
바이브 제안서: "브로셔 만들겠습니다" → [미리보기] 클릭 → 실제 시안이 보임
```

클라이언트 입장에서 생각해봐. 제안서를 읽다가 "미리보기" 버튼을 누르면 실제 브로셔 레이아웃이 나와. 영상 스토리보드도 감성 버전, 이성 버전 두 가지를 탭으로 비교할 수 있어.

**산출물을 말로 설명하지 않고, 보여준 거야.**

---

## 한 줄 정리

마크다운으로 쓰고, AI가 제안서 + 산출물 시안까지 만들어. 클라이언트한테 보여줄 수 있으면 수주 확률이 달라져.
