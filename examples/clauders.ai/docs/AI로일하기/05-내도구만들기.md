**매번 똑같은 프롬프트 치고 있어? 그걸 도구로 만들어.**

01~04에서 해본 걸 돌아봐. 글쓰기, 리서치, 데이터 정리, 콘텐츠 제작.

매주 반복되는 게 있으면 — 그건 도구로 만들 타이밍이야.

---

## 커스텀 커맨드로 만들기

자동화 파트에서 배운 커스텀 커맨드 기억나지? `.claude/commands/` 폴더에 파일을 만들면 `/명령어`로 실행할 수 있었어.

**Step 1.** 반복되는 작업을 커맨드로 만들어

MAX5에는 `/deploy` 커맨드가 있어:

```markdown
<!-- .claude/commands/deploy.md -->
content-workflow 폴더에 있는 마크다운 파일을 DB에 배포해.
deploy-content.sh 스크립트를 사용해.
신규만 배포하고 기존은 스킵.
결과 요약을 보여줘.
```

이제 `/deploy`만 치면 끝.

**Step 2.** 네 작업에 맞는 커맨드를 만들어봐

```
> .claude/commands/review.md 파일 만들어줘.
> 내용: 오늘 수집한 원문을 읽고,
> 종류별로 분류한 뒤, 한국어 리뷰를 만들어서
> content-workflow 폴더에 저장해.
```

## 바이브코딩으로 스크립트 만들기

커맨드는 "AI한테 시킬 내용을 글로 적어둔 것"이야. 대부분은 이걸로 충분해.

근데 가끔 이걸로 안 될 때가 있어. 파일을 가공해야 한다거나, 외부 서비스에서 데이터를 가져와야 할 때.

그러면 **스크립트**를 만들어. 스크립트는 컴퓨터가 순서대로 실행하는 명령어 모음이야. 레시피처럼 "이거 하고, 저거 하고, 마지막에 이거 해"를 파일로 저장해둔 거야.

MAX5에서 실제로 만든 스크립트들:

| 스크립트 | 하는 일 |
|---------|---------|
| `fetch-all.js` | 10개 소스에서 뉴스 수집 |
| `generate-newsletter.js` | 뉴스레터 텍스트 생성 |
| `deploy-content.js` | DB에 콘텐츠 배포 |
| `run-workflow.sh` | 위 전체를 한 번에 실행 |

이걸 어떻게 만들었냐?

```
> RSS 피드를 읽어서 마크다운으로 저장하는 스크립트 만들어줘.
> 피드 목록은 설정 파일에 정의하고,
> 중복은 파일명으로 체크해서 스킵.
> 날짜별 폴더에 저장.
```

이게 바이브코딩이야. 코드를 네가 짜는 게 아니라, AI한테 "이거 만들어줘"라고 시키는 거야.

## 파이프라인으로 묶어

파이프라인은 공장 조립 라인이야. 앞 단계가 끝나면 다음 단계로 넘어가.

MAX5의 워크플로우:

```
수집 → 리뷰 생성 → 뉴스레터 → DB 배포
```

이걸 하나의 스크립트로 묶으면:

```bash
./scripts/run-workflow.sh          # 전체 실행
./scripts/run-workflow.sh --dry-run   # 미리보기 (실제로 안 함)
```

처음부터 이렇게 만든 게 아니야. 하나씩 수동으로 하다가, 패턴이 보이니까 묶은 거야.

## 도구를 만드는 기준

모든 걸 도구로 만들 필요 없어.

| 만들어야 할 때 | 안 만들어도 될 때 |
|---------------|-----------------|
| 매주 반복돼 | 한 번만 할 일이야 |
| 형식이 정해져 있어 | 매번 다른 결과가 필요해 |
| 5분 이상 걸려 | 프롬프트 한 줄이면 끝나 |
