**AI는 금붕어야. 기억력이 짧아.**

컨텍스트를 관리하느냐 마느냐가 결과 품질을 결정해.

---

## 컨텍스트가 뭐야?

AI가 한 번에 "볼 수 있는" 텍스트의 양이야. 컨텍스트 윈도우라고 불러.

- Claude는 약 **200K 토큰** (책 1~2권 분량)
- 현재 대화에서만 유효해
- 대화가 길어지면 앞부분을 까먹어
- 새 대화 시작하면 완전 리셋돼

비유하면 화이트보드에 적은 메모야.
대화하는 동안만 유효하고, 지우면 끝이야.

이전 세션을 불러올 수도 있어.

```bash
claude -c    # 직전 세션 이어서 하기
claude -r    # 과거 세션 목록에서 선택
```

불러온다고 무한정 기억하는 건 아냐.
컨텍스트가 가득 차면 여전히 앞부분을 잊어버려.

---

## /compact가 뭐야?

대화가 길어지면 컨텍스트를 **요약/압축**하는 기능이야.

```bash
/compact                        # 기본 압축
/compact 리뷰한 사이트명만 유지  # 커스텀 지시
```

| 사용량 | 상태 | 뭘 해야 해? |
|--------|------|------------|
| 0~50% | 안전 | 자유롭게 작업 |
| 50~70% | 주의 | 모니터링 시작 |
| 70~85% | 경고 | **즉시 /compact** |
| 85%+ | 긴급 | 새 세션 시작 |

Auto-compact 기능이 있긴 한데, 70% 쯤에서 미리 정리하는 습관이 좋아.

## 메모리가 뭐야?

컨텍스트가 단기기억이라면, 메모리는 **장기기억**이야.
파일로 저장해두면 AI가 매번 참고해.

**CLAUDE.md**가 대표적이야.

```markdown
# 프로젝트 설명
이 프로젝트는 OOO

# 코딩 규칙
- 함수명은 소문자로
- 주석은 한글로
```

이렇게 적어두면 Claude가 매번 읽고 따라.

메모리도 결국 파일이라서 세션 시작할 때 **입력 토큰으로 소모**돼.
너무 길면 비효율적이야.

---

## HITL이 뭐야?

**Human In The Loop**. 사람이 중간에 개입한다는 뜻이야.

코드 작성 중에는 개입하기 어려워. 너무 빨리 진행되니까.
**기획 단계에서 개입이 효과적**이야.

AI가 내 의도와 다른 방향으로 가면 바로 `ESC`를 눌러.
그리고 프롬프트를 다시 써.

```
"잠깐, 내가 원한 건 이게 아닌데..."
"다시 해줘. 조건은 이거야..."
```

AI 맹신 금지. 중간중간 결과 확인해.

---

## 검증은 새 세션에서

AI가 만든 결과물을 **같은 대화에서 검증하면 안 돼**.

왜? AI는 자기가 만든 것에 긍정 편향이 있어.
같은 컨텍스트 안에서는 "맞다"고 답하기 쉬워.

검증하는 법:
1. **새 세션 열기** — 같은 AI라도 새 대화에서
2. **다른 LLM 사용** — Claude로 만들었으면 ChatGPT로 검토
3. **직접 확인** — 코드는 실행, 글은 직접 읽어보기

만든 AI와 검증하는 AI는 달라야 해.

## 한 줄 정리

컨텍스트는 단기기억, 메모리는 장기기억. 둘 다 잘 관리해야 AI가 똑똑해.
