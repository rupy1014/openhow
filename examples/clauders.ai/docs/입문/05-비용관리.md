**"갑자기 안 돼" — 그거 한도에 걸린 거야.**

:::youtube gLZ1wJUADqk
:::

Claude Code를 신나게 쓰다 보면 갑자기 멈춰. 고장 난 게 아니야. 한도에 걸린 거야.

아끼는 법만 알면 돼.

---

## 한도가 어떻게 돼?

Claude Code와 claude.ai(웹)는 **같은 한도를 공유해.** 웹에서 많이 쓰면 Claude Code도 빨리 막혀.

**5시간 단위 한도** — 최근 5시간 동안 쓴 양을 보고 제한해.

| 플랜 | 가격 | 5시간당 대략 |
|------|------|------------|
| Pro | $20/월 | 10~40번 요청 |
| Max 5x | $100/월 | 50~200번 요청 |
| Max 20x | $200/월 | 200~800번 요청 |

정확한 숫자는 공개 안 됐어. 요청마다 쓰는 양이 다르거든.

**주간 한도** — 7일 동안 쓸 수 있는 전체 양이 따로 있어. 이걸 넘으면 7일을 기다려야 해.

---

## Claude Code는 왜 빨리 막혀?

웹에서 "안녕"이라고 치면 메시지 1개야.

근데 Claude Code에서 "이 파일 고쳐줘"라고 하면 뒤에서 이런 일이 벌어져:

```
파일 읽기 → 분석 → 수정안 만들기 → 파일 쓰기 → 확인
```

**한 번 요청에 내부 호출이 5~30개 생겨.** 웹보다 5~10배 빠르게 한도를 먹는 거야.

---

## 한도에 걸리면?

"Rate limit reached" 메시지가 뜨면서 멈춰.

5시간 한도는 **롤링** 방식이야. 오전 9시에 막혔으면, 9시에 쓴 분량이 5시간 뒤인 오후 2시에 풀려. 한 번에 다 풀리는 게 아니라 조금씩 풀려.

주간 한도에 걸리면? Pro는 기다리는 수밖에 없어. Max는 추가 요금 내고 더 쓸 수 있어.

---

## 어떻게 아껴?

**1. Sonnet 모델을 써**

Claude Code에는 두 가지 모델이 있어.

| 모델 | 특징 | 언제 써 |
|------|------|--------|
| Sonnet | 빠르고 가벼워 | 대부분의 작업 |
| Opus | 똑똑하지만 무거워 | 복잡한 설계 결정 |

Sonnet만 써도 대부분의 작업은 충분해. Opus는 한도를 훨씬 빠르게 먹어.

```text:Claude 프롬프트
/model sonnet
```

이렇게 치면 모델이 바뀌어.

**2. /compact로 대화 압축해**

대화가 길어지면 Claude가 기억할 게 많아져서 한도를 더 먹어.

```text:Claude 프롬프트
/compact
```

대화 내용을 요약해서 60~70% 줄여줘. 1시간마다 한 번씩 해봐.

**3. 작업이 바뀌면 /clear**

"블로그 글 고치기"가 끝나고 "이력서 페이지 만들기"를 할 때, 대화를 이어가지 마.

```text:Claude 프롬프트
/clear
```

새 대화로 시작하면 이전 대화의 토큰을 안 써.

**4. 구체적으로 말해**

```
❌ "이 코드 개선해줘"
✅ "auth.ts의 로그인 함수에 입력 검증 추가해줘"
```

범위가 넓으면 Claude가 파일을 많이 읽어. 그만큼 한도를 먹어.

**5. 계획 먼저, 실행 나중에**

`Shift+Tab` 두 번으로 계획 모드를 켜. 이 모드에서는 파일을 읽고 분석만 해. 수정은 안 해.

계획이 맞으면 그때 실행. 잘못된 방향으로 코드를 짜고 되돌리는 것보다 훨씬 아껴.

## 언제 플랜을 올려?

| 상황 | 추천 |
|------|------|
| 한도에 자주 안 걸려 | Pro $20 유지 |
| 주 2~3번 한도에 걸려 | Max $100 고려 |
| 매일 한도에 걸려 | Max $200 추천 |

처음엔 Pro로 시작해. 한도에 걸리기 시작하면 그때 올려.
