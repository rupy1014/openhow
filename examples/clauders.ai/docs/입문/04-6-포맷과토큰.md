**AI에게 구조화된 형식으로 시키면 더 잘 알아먹어.**

---

## Markdown이 뭐야?

가장 많이 쓰게 될 문서 형식이야. README, 설명서, 노트에 다 써.

```markdown
# 제목
## 소제목

- 목록1
- 목록2

**굵게** *기울임*

| 열1 | 열2 |
|-----|-----|
| A   | B   |
```

`#`이 제목, `-`가 목록, `**`가 굵은 글씨. 이것만 알면 돼.

---

## JSON이 뭐야?

데이터 구조를 표현할 때 쓰는 형식이야. 설정 파일에 많이 나와.

```json
{
  "이름": "홍길동",
  "나이": 25,
  "취미": ["독서", "게임", "운동"]
}
```

키-값 쌍 `"키": "값"`, 중괄호 `{}`로 묶고, 배열은 `[]`.

---

## XML이 뭐야?

자주 쓰지는 않지만 가끔 나와. Claude는 XML을 특히 잘 이해해.

```xml
<사람>
  <이름>홍길동</이름>
  <나이>25</나이>
</사람>
```

여는 태그 `<태그>`, 닫는 태그 `</태그>`. HTML이랑 비슷해.

---

## 구조화가 왜 중요해?

프롬프트에 구조를 넣으면 AI가 더 정확하게 이해해.

```
나쁜 예:
"사용자 정보 받아서 저장해줘. 이름이랑 이메일이랑 전화번호"

좋은 예:
"사용자 정보 저장 기능 만들어줘.
입력 데이터:
{
  "name": "문자열",
  "email": "이메일 형식",
  "phone": "010-XXXX-XXXX 형식"
}"
```

꼭 외울 필요 없어.
필요할 때 AI한테 "이 내용을 마크다운으로 바꿔줘" 하면 돼.

---

## 토큰이 뭐야?

AI가 글을 처리하는 단위야.

- 한글 1글자 = 약 1~2 토큰
- 영어 1단어 = 약 1~2 토큰

```
"안녕하세요" = 약 5토큰
"Hello world" = 약 2토큰
```

---

## 입력 토큰이랑 출력 토큰이 뭐야?

| 구분 | 설명 | 비용 |
|------|------|------|
| 입력 토큰 | 내가 AI에게 보내는 글 | 저렴 |
| 출력 토큰 | AI가 생성하는 글 | 입력의 3~5배 |

- 긴 문서 요약: 입력 많고 출력 적어 → 저렴
- "자세히 설명해줘": 출력 많아 → 비싸
