**스크린샷 한 장이 설명 100줄보다 나아. Claude Code는 이미지를 볼 수 있어.**

:::youtube 4qeuIDqOFQs
:::

"이런 느낌으로 만들어줘"를 말로 설명하면 AI가 딴 걸 만들어. 사진을 보여주면 바로 알아들어.

---

## 어떻게 보여줘?

세 가지 방법이 있어.

**1. 붙여넣기 (가장 빠름)**

스크린샷을 찍고 Claude Code 터미널에 붙여넣어.

- **Mac**: `Cmd+Shift+4`로 스크린샷 → Claude Code에서 `Ctrl+V`
- **Windows**: `Win+Shift+S`로 스크린샷 → Claude Code에서 `Ctrl+V`

Mac에서 `Cmd+V`가 아니라 **`Ctrl+V`** 야. 헷갈리기 쉬워.

**2. 드래그 앤 드롭**

파일 탐색기에서 이미지를 끌어다가 터미널 창에 놓으면 돼.

**3. 파일 경로**

이미지가 프로젝트 폴더 안에 있으면 그냥 말해.

```text:Claude 프롬프트
temp/mockup.png 보고 이 디자인이랑 비슷하게 만들어줘.
```

---

## 어떤 이미지를 쓸 수 있어?

| 형식 | 쓸 수 있어? |
|------|------------|
| PNG | O (스크린샷에 좋아) |
| JPG | O |
| GIF | O |
| WebP | O |
| PDF | X (이미지로는 안 돼) |

한 번에 여러 장도 보낼 수 있어. "이 화면은 이렇고, 저 화면은 저렇게"처럼.

---

## 언제 쓰면 좋아?

**UI 디자인 피드백**

```text:Claude 프롬프트
[스크린샷 붙여넣기]
이 화면에서 버튼 색상을 파란색으로 바꿔줘.
```

만들고 → 스크린샷 찍고 → 붙여넣고 → 수정 요청. 이 루프가 제일 빨라.

**레퍼런스 전달**

```text:Claude 프롬프트
[비슷한 서비스 스크린샷 붙여넣기]
이 사이트랑 비슷한 레이아웃으로 만들어줘.
근데 색상은 초록 계열로.
```

**에러 화면 공유**

브라우저에서 에러가 나는데 텍스트로 복사가 안 될 때, 스크린샷을 찍어서 보내면 돼.

---

## 더 잘 쓰려면?

**상태별로 여러 장 보내.** 기본 화면, 로딩 중, 에러 상태 — 이렇게 여러 장 보여주면 Claude가 각 상태를 다 구현해.

**이미지 + 텍스트를 같이 줘.** 스크린샷만 보내면 Claude가 뭘 해야 할지 모를 수 있어. "이것처럼 만들어줘"를 같이 써.

**필요한 부분만 잘라서 보내.** 4K 전체 화면을 그대로 보내면 느려져. 필요한 영역만 캡처하면 더 빠르고 정확해.

---

## 한 줄 정리

스크린샷 찍고 `Ctrl+V`. "이거랑 비슷하게"가 최강의 프롬프트야.
