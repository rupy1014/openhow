**Claude Code는 폴더 단위로 일한다.**

폴더 안을 보고, 파일을 만들고, 설정을 읽는다. 이 폴더가 프로젝트다.

---

## .md가 뭐야?

`.md`는 마크다운(Markdown) 파일이야. 그냥 메모장 같은 텍스트 파일이다.

특별한 프로그램 없이 열 수 있고, 읽으면 그냥 글이야. `#`이 제목, `-`가 목록, `**굵게**`가 굵은 글씨 — 이 정도만 알면 된다.

Claude Code에서 `.md` 파일을 많이 쓰는 이유는 간단해. Claude가 텍스트를 잘 읽으니까.

---

## 폴더가 왜 프로젝트야?

폴더 안에 `CLAUDE.md`를 넣으면 Claude가 그걸 설정으로 읽는다.

```
my-blog/
├── CLAUDE.md   ← Claude 설정 (이 프로젝트에만 적용)
├── post-1.md
└── post-2.md
```

`CLAUDE.md`에 "말투는 구어체로, 서론 없이 시작해"라고 써두면
이 폴더에서 `claude`를 실행할 때마다 그게 자동으로 적용된다.

Claude가 실행될 때 이 파일을 자동으로 읽어. 매번 설명할 필요 없다.

---

## 웹 AI랑 뭐가 달라?

ChatGPT, Claude.ai, Gemini 같은 웹 AI도 설정이 있다.
GPT의 Custom Instructions, Claude의 프로젝트 기능, 파일 첨부, 도구 설정 등이다.

근데 한계가 있어.

- **계정 단위**다. 프로젝트마다 다른 설정을 쓰려면 매번 바꿔야 한다.
- **설정이 파일에 붙어있지 않다.** 설정은 웹 계정 안에, 파일은 내 컴퓨터에 — 폴더를 열어도 설정이 따라오지 않는다.
- **재현이 안 된다.** 같은 결과를 다시 내려면 같은 조건을 기억해서 맞춰야 한다.

**CLI 방식**은 다르다. 터미널에서 실행하고, 설정 파일이 프로젝트 폴더 안에 있다.

```
웹 AI:
  설정(웹) ↔ 파일(내 컴퓨터)  ← 분리돼 있음
  대화 시작 → 설명 → 작업 → 닫기 → 다시 설명 → 작업 → ...

CLI AI:
  설정 + 파일이 같은 폴더 안에 있음
  설정 작성 → 실행 → 작업 → 실행 → 작업 → ...
```

설정이 폴더에 붙어있으니까, 폴더를 열면 설정도 같이 불러온다.
블로그 폴더엔 블로그 설정, 업무 폴더엔 업무 설정 — 자동으로.

CLI 방식 도구는 여러 개다. Claude Code(Anthropic), Codex CLI(OpenAI), Gemini CLI(Google) 등이 있다.

## 왜 Claude를 써?

셋 다 비슷해 보이지만 Claude를 쓰는 이유가 있다.

**추론 능력이 가장 좋다.**
코딩 벤치마크(SWE-bench), 수학·추론 테스트에서 꾸준히 상위권이다.
긴 맥락에서 지시를 잘 따르는 능력이 다른 모델보다 낫다.

**컨텍스트 창이 크다.**
기본 200K 토큰 — 책 한 권 분량이야. 파일이 많아져도 Claude는 전체를 보고 답한다.
Max 플랜($100↑)에선 Opus 모델로 1M 토큰까지 늘어난다. 대형 프로젝트도 한 번에 다 넣을 수 있어.

**요금제가 현실적이다.**

| 플랜 | 가격 | 특징 |
|------|------|------|
| Pro | $20/월 | 입문. 감 잡는 단계 |
| Max (5x) | $100/월 | 본격 사용 |
| **Max (20x)** | **$200/월** | **목표 플랜** |

처음엔 **Pro ($20)** 로 시작한다. 사용법을 익히고, 한도에 걸리기 시작하면 올린다.

최종 목표는 **Max $200 플랜을 쓰는 것**이다.

$200을 쓴다는 건 그냥 많이 쓴다는 게 아니다.
나만의 자동화 봇, AI 비서, 소규모 서비스를 직접 만들어서
생산성이 10배가 되는 시점을 말한다. 충분히 가능하다.

$200 내고 $2,000짜리 일을 Claude가 대신 한다면 남는 장사다.

## 설정 파일이 뭐가 있어?

| 파일 | 위치 | 적용 범위 |
|------|------|----------|
| 전역 설정 | `~/.claude/CLAUDE.md` | 모든 프로젝트 |
| 프로젝트 설정 | `프로젝트폴더/CLAUDE.md` | 이 폴더에서만 |

**전역 설정** — 어디서나 항상 적용.
"항상 한국어로 답해", "코드에 주석 달지 마" 같은 공통 규칙.

**프로젝트 설정** — 이 폴더에서만 적용.
"이 블로그는 구어체로", "이 앱은 React 18 기준" 같은 프로젝트별 규칙.

두 파일이 모두 있으면 둘 다 읽는다. 프로젝트 설정이 우선이다.

하위 폴더에 `CLAUDE.md`를 넣을 수도 있어.
Claude가 그 폴더 안의 파일을 읽을 때 자동으로 불러온다.
예: `posts/CLAUDE.md`에 "이 폴더 글은 공개용. 존댓말 금지"라고 써두면 posts/ 작업할 때만 적용돼.

## VSCode에서 어떻게 열어?

**Step 1.** 폴더를 만들고 VSCode로 연다

```bash
mkdir my-blog
cd my-blog
code .
```

> **Mac:** `code .`가 안 되면 VSCode에서 ⌘+Shift+P → `Shell Command: Install 'code' command in PATH` 실행.

**Step 2.** VSCode 화면 구조를 알아둬

VSCode를 열면 세 영역이 보인다.

| 위치 | 이름 | 하는 일 |
|------|------|---------|
| **왼쪽** | 파일 탐색기 | 폴더 안의 파일 목록. 클릭하면 오른쪽에 열린다 |
| **오른쪽 위** | 에디터 | 파일 내용이 보이는 곳. 여기서 읽고 수정한다 |
| **아래쪽** | 터미널 | Claude Code를 실행하는 곳. 여기에 명령어를 친다 |

Claude한테 "파일 만들어줘"라고 하면 왼쪽 파일 탐색기에 새 파일이 실시간으로 나타나. 클릭하면 오른쪽 에디터에서 내용을 볼 수 있다.

왼쪽에 파란색 화살표 같은 아이콘이 보일 수 있어. Git이 변경된 파일을 표시하는 거야. 무시해도 된다.

VSCode 오른쪽에 Copilot Chat 대화창이 열려있을 수 있는데, 닫아도 돼. Claude Code는 터미널에서 실행한다.

**Step 3.** VSCode 터미널을 연다

- **Mac / Windows:** `` Ctrl+` `` (백틱 — Tab 위 또는 1 왼쪽)
- 또는 상단 `Terminal` → `New Terminal` (`Terminal`이 안 보이면 `...` 안에 있다)

화면 아래쪽에 터미널 패널이 열린다.

터미널 여러 개를 동시에 띄울 수도 있어. 터미널 패널 오른쪽 위 `+` 버튼을 누르면 새 터미널이 생긴다. 프로젝트 여러 개를 동시에 작업할 때 쓴다.

**Step 4.** Claude Code를 실행한다

```bash
claude
```

`>` 프롬프트가 뜨면 Claude가 `my-blog` 폴더 안에 들어온 것이다.

**Step 5.** CLAUDE.md를 만들어봐

`>` 프롬프트에 쳐봐:

```
CLAUDE.md 파일 만들어줘. 내용은 "말투는 구어체로. 서론 없이 바로 시작해."
```

VSCode 왼쪽 파일 탐색기에 `CLAUDE.md`가 나타난다. 클릭하면 오른쪽 에디터에서 내용을 확인할 수 있어.

이제 이 폴더에서 `claude`를 실행하면 그 설정이 자동으로 적용된다.

## 한 줄 정리

프로젝트 = 폴더 + CLAUDE.md. 설정 한 번 써두면 매번 설명 안 해도 된다. 다음은 이 설정으로 첫 대화를 시작한다.
