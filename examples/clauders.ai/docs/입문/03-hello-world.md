## 어떻게 요청해?

뭔가 특별한 명령어를 외워야 할 것 같지? 아니야. 그냥 말로 하면 돼.

ChatGPT한테 하듯 그대로 하면 돼. 특별한 문법 없어.

```
이 폴더 글 전부 읽고 각각 3줄 요약 만들어줘
```

```
내 이력서 페이지에 다크모드 토글 추가해줘
```

```
오타 있는 파일 찾아서 고쳐줘
```

ChatGPT랑 다른 점은 딱 하나야.
말로 요청하면 Claude가 파일을 **직접** 읽고, 만들고, 수정해.
대화로 알려주는 게 아니라 직접 손을 대.

실제로 일어나는 일:

1. 터미널에 요청을 적어 → "자기소개 페이지 만들어줘"
2. Claude가 파일을 만들어 → 터미널에 진행 상황이 나와
3. VSCode 왼쪽 파일 탐색기에 새 파일이 실시간으로 나타나
4. 클릭하면 오른쪽 에디터에서 Claude가 쓴 내용을 바로 볼 수 있어

내가 하는 건 1번뿐이야. 나머지는 Claude가 해.

## "해줘"만으론 부족할 때

"해줘"로 시작하는 건 맞아. 근데 복잡한 작업은 바로 "해줘"보다 **같이 기획하는 게** 결과가 훨씬 좋아.

---

**기획을 같이 해** — 복잡한 작업은 바로 시키지 말고 계획부터 세워.

왜? 복잡한 걸 한 번에 시키면 Claude가 자기 생각대로 다 해버려.
중간에 방향이 틀려도 되돌리기 어려워.

```
이력서 페이지에 프로젝트 섹션 추가하고 싶어. 일단 어떻게 할지 계획만 세워줘. 코드는 아직 짜지 마.
```

Claude가 계획을 보여줘. 마음에 들면 "진행해줘", 아니면 "이 부분은 이렇게 바꿔"라고 하면 돼.

`Shift+Tab` 두 번으로 "계획만 세우기" 모드를 켤 수도 있어. 이 모드에서 Claude는 파일을 읽고 분석만 해. 실제로 바꾸지 않아.

핵심은 이거야. Claude한테 다 맡기는 게 아니라, **내가 방향을 잡고 Claude가 실행하는** 구조.

---

**참고 자료를 던져** — "이것처럼"을 보여주면 결과가 완전히 달라져.

왜? 참고 없이 시키면 Claude가 처음부터 알아서 만들어.
"그냥 예쁘게"가 아니라 "이것처럼"을 보여줘야 원하는 게 나와.

```
참고: https://resume-three-virid.vercel.app/
이 스타일이랑 비슷하게 만들어줘
```

```
post-1.md 파일 말투 참고해서 나머지 글들도 같은 톤으로 바꿔줘
```

URL이든 파일이든 같이 넘기면 Claude가 보고 맞춰서 짜.

---

"해줘"로 시작하고, 복잡하면 기획을 같이 해. 참고 자료를 던지면 결과가 달라져.
