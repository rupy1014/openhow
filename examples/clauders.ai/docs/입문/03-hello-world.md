---

## 어떻게 요청해?

뭔가 특별한 명령어를 외워야 할 것 같지? 아니야. 그냥 말로 하면 돼.

ChatGPT한테 하듯 그대로 하면 된다. 특별한 문법 없다.

```
이 폴더 글 전부 읽고 각각 3줄 요약 만들어줘
```

```
내 이력서 페이지에 다크모드 토글 추가해줘
```

```
오타 있는 파일 찾아서 고쳐줘
```

ChatGPT랑 다른 점은 딱 하나야.
말로 요청하면 Claude가 파일을 **직접** 읽고, 만들고, 수정한다.
대화로 알려주는 게 아니라 직접 손을 댄다.

실제로 일어나는 일:

1. 터미널에 요청을 적는다 → "자기소개 페이지 만들어줘"
2. Claude가 파일을 생성한다 → 터미널에 진행 상황이 출력된다
3. VSCode 왼쪽 파일 탐색기에 새 파일이 실시간으로 나타난다
4. 클릭하면 오른쪽 에디터에서 Claude가 쓴 내용을 바로 볼 수 있다

내가 하는 건 1번뿐이야. 나머지는 Claude가 한다.

## 어떻게 더 잘 나올까?

일단 말 걸어봤으면 알 거야. 결과가 생각이랑 다를 때가 있어.

두 가지만 알면 훨씬 좋아진다.

---

**플랜 먼저 받기** — 복잡한 작업은 바로 시키지 말고 계획부터 받는다.

왜? 복잡한 걸 한 번에 시키면 Claude가 자기 생각대로 다 해버려.
중간에 방향이 틀려도 되돌리기 어렵다.
계획 단계에서 방향을 맞추는 게 훨씬 빠르다.

```
이력서 페이지에 프로젝트 섹션 추가하고 싶어. 일단 어떻게 할지 계획만 세워줘. 코드는 아직 짜지 마.
```

마음에 들면 "진행해줘"라고 하면 된다.
`Shift+Tab` 두 번으로 플랜 모드를 켤 수도 있다.

플랜 모드에서 Claude는 파일을 읽고 분석만 한다. 실제로 바꾸지 않는다.
계획을 보고 승인하면 그때 실행된다.

---

**레퍼런스 주기** — 참고할 URL이나 파일을 같이 넘기면 결과가 훨씬 좋다.

왜? 레퍼런스 없으면 Claude가 처음부터 설계한다.
"그냥 예쁘게"가 아니라 "이것처럼"을 알려줘야 원하는 게 나온다.

```
참고: https://resume-three-virid.vercel.app/
이 스타일이랑 비슷하게 만들어줘
```

```
post-1.md 파일 말투 참고해서 나머지 글들도 같은 톤으로 바꿔줘
```

URL이든 파일이든 같이 넘기면 Claude가 보고 맞춰서 짠다.

---
