> "제 컴퓨터에선 안 되는데요?"
> "버전이 달라서 에러가 나."

개발하다 보면 가장 짜증 나는 순간이 바로 **"내 컴퓨터에서만 안 될 때"**야.
서로 다른 도구, 다른 재료를 쓰고 있어서 그래.

이걸 해결해주는 마법사가 있어. 바로 **NPM (Node Package Manager)**이야.

"이 프로젝트엔 양파 3개, 당근 1개가 필요해"라고 적힌 **레시피(package.json)**만 있으면,
누구든 **버튼 하나(npm install)**로 똑같은 재료를 준비할 수 있지.

```
📚 이 글을 읽고 나면

✅ npm install이 뭔지 정확히 안다
✅ package.json이 레시피라는 걸 이해한다
✅ 환경 변수(.env)로 비밀 정보를 관리할 수 있다
```

---

## 1단계: 레시피 확인하기 (package.json)

남의 코드를 받아오면(Clone), 폴더 안에 `package.json`이라는 파일이 꼭 있어.
열어보면 대충 이렇게 생겼어.

```json
{
  "name": "vibe-project",
  "dependencies": {
    "react": "^18.2.0",
    "supabase-js": "^2.0.0"
  }
}
```

이게 바로 **"이 요리 하려면 리액트랑 수파베이스가 필요해"**라고 적어놓은 **장보기 목록**이야.

![레시피로 요리하기 vs NPM으로 코딩하기 비교](assets/09-npm-analogy.png)

이게 없으면 일일이 하나씩 다운로드 받아야 해. 끔찍하지?

---

## 2단계: 장보기 시작 (npm install)

터미널을 열고, 프로젝트 폴더에서 딱 한 마디만 외쳐.

```bash
npm install
```
(또는 줄여서 `npm i`)

그럼 터미널이 미친 듯이 글자를 뿜어내며 뭔가를 다운로드해.
끝나고 나면 `node_modules`라는 폴더가 생겼을 거야.

**여기에 모든 재료가 배달된 거야.**
이제 네 컴퓨터 환경은 옆자리 고수 개발자랑 완전히 똑같아졌어.

> **💡 꿀팁:**
> `node_modules` 폴더는 용량이 엄청 커.
> 그래서 보통 깃(Git)에는 안 올려. (`.gitignore`에 적혀 있음)
> 어차피 `npm install` 하면 언제든 다시 생기니까!

## 3단계: 비밀 재료 숨기기 (.env)

요리를 하는데 "우리 할머니만의 비법 소스"가 들어간다고 치자.
이 비법을 동네방네 소문내면 안 되겠지?

개발에서도 **API 키, 비밀번호** 같은 건 코드에 적으면 안 돼.
GitHub에 올리는 순간 해커들이 채가거든.

그래서 **환경 변수 파일(.env)**을 써.

1. 프로젝트 최상위 폴더에 `.env` 파일을 만들어. (이름 앞에 점 필수!)
2. 안에 비밀 정보를 적어.

```env
API_KEY=abcdefg12345
DB_PASSWORD=secret1234
```

3. 그리고 코드를 작성할 땐 이렇게 써.

```javascript
/* 코드에는 비밀번호가 안 보임! */
const apiKey = process.env.API_KEY;
```

이렇게 하면, 코드는 공유해도 **비밀번호는 내 컴퓨터에만** 남게 돼.
안전하지?

> **🚨 주의:**
> `.env` 파일도 깃에 올리면 안 돼!
> `.gitignore` 파일에 `.env` 한 줄을 꼭 추가해.

## 정리: 완벽한 바이브 코더로 거듭나기

지금까지 배운 걸 총정리해볼까?

1. **Terminal:** 컴퓨터랑 대화하고 (명령)
2. **VS Code:** 작업대에서 코드 짜고 (편집)
3. **Git:** 망칠까 봐 세이브하고 (저장)
4. **NPM:** 필요한 재료 한 방에 세팅 (준비)

이 4가지만 알면, 이제 세상의 어떤 튜토리얼도 따라 할 수 있어.

"근데 코딩은 언제 배워?"

놀라지 마.
**지금까지 배운 게 코딩의 절반이야.**
나머지 절반(코드 작성)은 AI가 옆에서 다 알려줄 거니까.

이제 진짜 **'바이브 코딩'**을 시작할 준비가 끝났어.
준비됐어?
