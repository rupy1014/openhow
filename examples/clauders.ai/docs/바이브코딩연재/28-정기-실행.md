> "코드도 다 짰고, 슬랙으로도 잘 와.
> 근데 매일 아침 9시에 컴퓨터 켜서 실행 버튼 누르는 게 귀찮아.
> 까먹고 늦잠 자면 뉴스 못 받는 거잖아."

맞아. 내가 직접 실행 버튼을 눌러야 한다면 그건 **반쪽짜리 자동화**야.
진짜 자동화는 **내 컴퓨터가 꺼져 있어도, 내가 여행을 가도 알아서 돌아가는 것**이지.

그러려면 내 코드를 **늘 켜져 있는 컴퓨터(서버)**로 옮겨야 해.
"서버를 사야 하나?" 아니!
우리에겐 **GitHub Actions**라는 공짜 클라우드 컴퓨터가 있거든.

---

## 이 글을 읽고 나면

- 내 컴퓨터가 꺼져 있어도 코드가 돌아가게 할 수 있어.
- **GitHub Actions**가 뭔지 이해할 수 있어.
- **크론(Cron)** 시계를 맞춰서 "매일 아침 9시에 실행해"라고 명령할 수 있어.

---

## 1. GitHub Actions: 깃허브가 빌려주는 컴퓨터

우리가 `git push`로 코드를 올리는 저장소(GitHub) 있잖아.
거기엔 단순히 코드 저장 기능만 있는 게 아니야.

**"네가 원하면 내 컴퓨터(서버)를 잠깐 빌려줄게. 거기서 코드 실행해도 돼."**
라는 기능이 숨어있어. 이게 바로 **GitHub Actions**야.

우리는 이 컴퓨터한테 **작업 지시서(Workflow)**를 딱 한 장 써주면 돼.
"매일 아침 9시에, 내 코드를 다운받아서, `node index.js`를 실행해줘."

![GitHub Actions 작동 원리](assets/28-github-actions.png)

---

## 2. 작업 지시서 만들기 (Workflow)

이 지시서는 `.github/workflows` 폴더에 `.yml` 파일로 만들어.
좀 복잡해 보이지만, AI한테 시키면 1초 만에 만들어줘.

### 크론(Cron) 시간 설정의 함정
여기서 주의할 점! GitHub 컴퓨터는 **영국 시간(UTC)**을 써.
한국(KST)은 영국보다 **9시간 빠르지**.

- 한국 아침 9시 = 영국 밤 12시 (00:00)
- 한국 저녁 6시 = 영국 아침 9시 (09:00)

그래서 "한국 아침 9시에 실행해줘"라고 하려면,
지시서엔 **"0시 0분(UTC)에 실행해"**라고 적어야 해.

---

## 3. 실전: 자동화 완성하기 (상황극)

이제 마지막 퍼즐을 맞춰보자.
AI에게 시켜서 지시서를 만들고, 비밀번호(환경변수)를 등록해보자.

### 1단계: 지시서 작성 요청

**나 (팀장)**
> "이 프로젝트를 GitHub Actions로 매일 실행하고 싶어.
> **한국 시간으로 아침 9시**마다 실행되게 워크플로우 파일을 만들어줘.
>
> *참고: 시간 설정할 때 UTC 변환해줘.*"

**AI (신입사원)**
> "좋아! `.github/workflows/daily.yml` 파일을 만들게.
> 한국 아침 9시는 UTC로 0시니까 `cron: '0 0 * * *'`로 설정했어.
> Node.js 환경에서 의존성 설치하고 실행하는 코드야."

(AI가 만들어준 파일을 그대로 저장하면 돼!)

<br/>

### 2단계: 비밀값 등록 (가장 중요)

여기서 그냥 Push 하면 에러가 나. 왜냐고?
`.env` 파일은 GitHub에 안 올렸잖아. (보안 때문에)
그래서 GitHub 컴퓨터는 **슬랙 웹훅 주소**를 모르는 상태야.

AI한테 물어보면 이렇게 알려줄 거야.

**나 (팀장)**
> "근데 `.env` 파일은 안 올렸는데, GitHub가 환경변수를 어떻게 알아?"

**AI (신입사원)**
> "좋은 지적이야! 코드는 공개되지만 비밀번호는 공개되면 안 되잖아.
> **GitHub 저장소 설정(Settings)**에 비밀번호를 따로 저장하는 금고가 있어."

<br/>

**[따라 해봐]**

1. GitHub 저장소 상단 메뉴의 **Settings** 클릭
2. 왼쪽 메뉴에서 **Secrets and variables** → **Actions** 클릭
3. **New repository secret** 초록색 버튼 클릭
4. **Name**: `SLACK_WEBHOOK_URL` (AI가 코드에 쓴 이름과 똑같이!)
5. **Secret**: `https://hooks.slack.com/...` (내 진짜 주소 붙여넣기)
6. **Add secret** 버튼 클릭

이제 GitHub 컴퓨터도 우리 비밀 주소를 알게 됐어! (근데 사람 눈에는 안 보여. 안전하지?)

<br/>

### 3단계: 배포 및 확인

이제 터미널에서 코드를 올리기만 하면 끝이야.

```bash
git add .
git commit -m "자동화 완성"
git push
```

GitHub 페이지에 가서 **Actions** 탭을 눌러봐.
설정한 시간이 되면 자동으로 리스트가 생기고 초록색 체크가 뜰 거야.

---

## 4. 드디어 완성!

이제 너의 컴퓨터가 꺼져 있어도, 여행을 가도, 잠을 자도
**너의 충실한 비서(봇)**는 매일 아침 9시에 일어나서
뉴스를 검색하고, 요약해서, 슬랙으로 배달해줄 거야.

이게 바로 개발자들이 말하는 **'서버리스(Serverless) 자동화'**야.
서버를 직접 관리하지 않으면서(less) 서버의 혜택을 누리는 거지.

---

## 오늘의 핵심 정리

**GitHub Actions**: 공짜로 빌려 쓰는 클라우드 컴퓨터.
**시간 설정**: **UTC(영국 시간)** 기준이라 **-9시간** 계산해야 해.
**Secrets**: 비밀번호(환경변수)는 코드가 아니라 **Settings 메뉴**에 저장해.
