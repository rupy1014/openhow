솔직하게 말할게.

**앞으로는 "나는 개발 몰라요"가 안 통해.**

그리고 그 격차는 점점 더 커질 거야.

## 왜 이런 차이가 생기는 걸까?

이상했어.
**같은 AI를 쓰는데, 어떤 사람은 완성하고 수익화까지 하고, 어떤 사람은 계속 폐기하고.**

뭐가 다른 걸까?

차이는 **"기술적으로 지시할 수 있느냐"**였어.

## 코드를 몰라도 되지만, "개념"은 알아야 해

여기서 중요한 포인트가 있어.

**코드를 직접 짤 줄 몰라도 돼.**
for문이 뭔지, if문을 어떻게 쓰는지... 처음엔 몰라도 돼.
그건 AI가 해줘.

<div style="background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); border-radius: 16px; padding: 3px; margin: 2em 0;">
<div style="background: #1a1a2e; border-radius: 14px; padding: 28px 32px; color: #e0e0e0; line-height: 1.7;">
  <h3 style="margin: 0 0 24px 0; color: #fff; font-size: 1.25em; text-align: center; border-bottom: 1px solid #444; padding-bottom: 18px;">💡 근데 솔직히 말하면</h3>
  <p style="margin: 0 0 16px 0;">바이브 코딩을 하면 할수록<br>코드가 조금씩 눈에 들어오기 시작해.</p>
  <p style="margin: 0 0 16px 0;">"아, 이게 함수구나"<br>"여기서 데이터를 가져오는구나"</p>
  <p style="margin: 0 0 0 0;">이게 보이기 시작하면 고도화가 훨씬 쉬워져.<br>처음엔 몰라도 되지만, 점점 알게 되는 거야.</p>
</div>
</div>

**근데 "개념"은 처음부터 알아야 해.**

<div style="background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); border-radius: 16px; padding: 3px; margin: 2em 0;">
<div style="background: #1a1a2e; border-radius: 14px; padding: 28px 32px; color: #e0e0e0; line-height: 1.7;">
  <h4 style="margin: 0 0 16px 0; color: #f87171; font-size: 1.05em;">❌ 이렇게 말하면 꼬여</h4>
  <p style="margin: 0 0 16px 0;">"결제 내역에서 환불 처리할 수 있게 해줘"<br>
  <span style="color: #a78bfa;">→</span> AI가 코드는 읽을 수 있지만, 서비스 정책은 몰라.</p>
  <ul style="margin: 0 0 16px 0; padding-left: 24px;">
    <li style="margin: 8px 0;">우리 서비스는 7일 이내만 환불 가능한데, AI는 그걸 모름</li>
    <li style="margin: 8px 0;">카드는 부분취소 되는데, 가상계좌는 취소 자체가 안 되고 환불 처리해야 하는 것도 모름</li>
    <li style="margin: 8px 0;">쿠폰 사용 건은 쿠폰 복원 로직도 필요한데, 그것도 모름</li>
  </ul>
  <p style="margin: 0 0 24px 0;"><span style="color: #a78bfa;">→</span> 결과: 일단 만들긴 하는데, 실제 운영하면 다 터져</p>

  <h4 style="margin: 24px 0 16px 0; color: #4ade80; font-size: 1.05em;">✅ 이렇게 말하면 돼</h4>
  <p style="margin: 0 0 16px 0;">"환불 기능 추가할 건데, 먼저 정책부터 정리하자.</p>
  <ul style="margin: 0 0 16px 0; padding-left: 24px;">
    <li style="margin: 8px 0;">환불 가능 기간: 결제 후 7일 이내</li>
    <li style="margin: 8px 0;">카드: 부분취소 가능, 가상계좌: 취소 안 됨(환불 계좌 받아서 송금)</li>
    <li style="margin: 8px 0;">쿠폰 사용 건은 쿠폰 복원 처리도 해야 해</li>
  </ul>
  <p style="margin: 0 0 0 0;">이 정책 기반으로 환불 로직 설계해줘."<br>
  <span style="color: #a78bfa;">→</span> 서비스 정책 + 결제수단별 기술 제약 명시 → 제대로 된 설계</p>
</div>
</div>

**"결제수단별 취소 정책? 쿠폰 복원 로직?"**

바로 이거야.
**코드 문법은 몰라도 되지만, 서비스 정책과 기술적 제약은 알려줘야 해.**
AI는 코드는 읽을 수 있어도, 우리 서비스가 어떻게 돌아가야 하는지는 몰라.

## 이 시리즈가 끝나면

<div style="background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); border-radius: 16px; padding: 3px; margin: 2em 0;">
<div style="background: #1a1a2e; border-radius: 14px; padding: 28px 32px; color: #e0e0e0; line-height: 1.7;">
  <h3 style="margin: 0 0 24px 0; color: #fff; font-size: 1.25em; text-align: center; border-bottom: 1px solid #444; padding-bottom: 18px;">🎯 목표 수준</h3>
  <p style="margin: 0 0 16px 0;"><strong>나:</strong> "이 프로젝트에 어울리는 아키텍처 추천해봐"</p>
  <p style="margin: 0 0 16px 0;"><strong>AI:</strong> "1. 레이어드 아키텍처 - 단순하고... 2. 클린 아키텍처 - 확장성..."</p>
  <p style="margin: 0 0 16px 0;"><strong>나:</strong> "우리는 규모가 작으니까 1번으로 가자.<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;근데 나중에 확장할 수 있게 서비스 레이어는 분리해줘."</p>
  <p style="margin: 0 0 0 0;"><span style="color: #a78bfa;">→</span> AI가 추천하면, 뭐가 왜 좋은지 판단할 수 있는 수준<br><span style="color: #a78bfa;">→</span> 기술 용어로 구체적인 방향을 지시할 수 있는 수준</p>
</div>

**"처음엔 되는데 고도화가 안 돼"에서**
**"고도화까지 할 수 있어"로.**

그게 이 시리즈의 목표야.

## 한 가지 약속

이 시리즈에서 AI를 마법처럼 포장하지 않을게.

<div style="background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); border-radius: 16px; padding: 3px; margin: 2em 0;">
<div style="background: #1a1a2e; border-radius: 14px; padding: 28px 32px; color: #e0e0e0; line-height: 1.7;">
  <h3 style="margin: 0 0 24px 0; color: #fff; font-size: 1.25em; text-align: center; border-bottom: 1px solid #444; padding-bottom: 18px;">⚠️ 솔직하게 말할게</h3>
  <ul style="margin: 0 0 16px 0; padding-left: 8px; list-style: none;">
    <li style="margin: 12px 0; position: relative; padding-left: 28px;"><span style="position: absolute; left: 0; color: #a78bfa;">→</span>AI가 다 해주지 않아</li>
    <li style="margin: 12px 0; position: relative; padding-left: 28px;"><span style="position: absolute; left: 0; color: #a78bfa;">→</span>처음엔 잘 되다가 꼬이는 건 당연해</li>
    <li style="margin: 12px 0; position: relative; padding-left: 28px;"><span style="position: absolute; left: 0; color: #a78bfa;">→</span>확인하고, 피드백하고, 다시 고치는 과정이 필요해</li>
    <li style="margin: 12px 0; position: relative; padding-left: 28px;"><span style="position: absolute; left: 0; color: #a78bfa;">→</span>네가 방향을 잡아야 해. AI는 실행만 해.</li>
  </ul>
  <p style="margin: 0 0 0 0;">"쉽다", "금방 된다"는 말 안 할게.<br>대신 "이렇게 하면 된다"를 알려줄게.</p>
</div>

**그리고 가장 중요한 것:**

<div style="background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); border-radius: 16px; padding: 3px; margin: 2em 0;">
<div style="background: #1a1a2e; border-radius: 14px; padding: 28px 32px; color: #e0e0e0; line-height: 1.7;">
  <h3 style="margin: 0 0 24px 0; color: #fff; font-size: 1.25em; text-align: center; border-bottom: 1px solid #444; padding-bottom: 18px;">🎯 AI가 만들면 끝이 아니야</h3>
  <p style="margin: 0 0 16px 0;">모든 코드를 한 줄 한 줄 읽을 필요는 없어.<br>근데 "기능 단위"로는 반드시 확인해야 해.</p>
  <ul style="margin: 0 0 16px 0; padding-left: 8px; list-style: none;">
    <li style="margin: 12px 0; position: relative; padding-left: 28px;"><span style="position: absolute; left: 0; color: #a78bfa;">→</span>버튼 만들었으면 → 눌러보기</li>
    <li style="margin: 12px 0; position: relative; padding-left: 28px;"><span style="position: absolute; left: 0; color: #a78bfa;">→</span>저장 기능 만들었으면 → 저장해보기</li>
    <li style="margin: 12px 0; position: relative; padding-left: 28px;"><span style="position: absolute; left: 0; color: #a78bfa;">→</span>삭제 기능 만들었으면 → 삭제해보기</li>
  </ul>
  <p style="margin: 0 0 0 0;">이 "확인 → 피드백" 과정이 없으면 나중에 다 꼬여.</p>
