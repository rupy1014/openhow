> "DB 테이블은 다 만들었어.
> 이제 버튼 누르면 저장이 되게 하고 싶은데...
> API를 만들어야 한다고?"

지난 시간에 우리는 튼튼한 **창고(DB 테이블)**를 지었어.
근데 창고만 있으면 뭐해? 물건을 넣고 빼는 사람이 없잖아.

이제 이 창고를 관리할 **4명의 일꾼**을 고용해야 해.
개발자들은 이 일꾼들을 묶어서 **CRUD**라고 불러.

1.  **C**reate (넣기)
2.  **R**ead (꺼내기)
3.  **U**pdate (고치기)
4.  **D**elete (버리기)

**회사 업무로 치면 '휴가 신청'과 똑같아.**
- 휴가 신청서 제출 (Create)
- 남은 휴가 며칠인지 확인 (Read)
- 날짜 변경 (Update)
- 휴가 취소 (Delete)

이 4가지만 할 줄 알면, 인스타그램(사진 올리고, 보고, 수정하고, 삭제)부터 배달앱(주문하고, 조회하고, 변경하고, 취소)까지 세상 모든 서비스를 만들 수 있어.

이 글을 읽고 나면:
- **CRUD**가 왜 서비스의 기본인지 이해할 수 있어.
- Next.js에서 **API Route**로 백엔드(일꾼)를 만드는 법을 배울 수 있어.
- **유효성 검사(Validation)**라는 '문지기'를 세우는 법을 익힐 수 있어.

---

## 1. API Route: 일꾼들의 사무소

"프론트엔드랑 백엔드를 따로 만들어야 해?"
전통적으로는 그래. 근데 **Next.js**를 쓰면 한 프로젝트 안에서 다 할 수 있어.

`app/api/` 폴더 안에 파일을 만들면, 그게 바로 정보가 드나드는 **뒷문(Backend API)**이 돼.
우리는 `app/api/expenses/route.ts`라는 파일을 만들어서 일꾼들을 배치할 거야.

> **13편에서 배운 내용 기억나?**
> 식당 주방(백엔드)에 손님(프론트)이 아무나 들어오면 안 되잖아?
> 그래서 **점원(API)**에게 주문서를 줘야 한다고 했잖아.
> 이 파일이 바로 그 **점원** 역할을 하는 거야.

-   **프론트엔드**: "저장해줘!" (주문서 제출)
-   **API Route**: "확인했어. 창고에 넣을게." (실제 작업)
-   **DB**: (데이터 저장됨)

## 2. 실전: 지출 추가하기 (Create)

첫 번째 일꾼, **Create(생성)** 담당을 만들어보자.
근데 무턱대고 아무거나 저장해주면 안 돼.

> **상황:**
> 누군가가 금액에 `-5000`원을 적거나, 카테고리도 없이 저장을 요청했어.
> 이걸 그대로 저장하면 나중에 정산할 때 대참사가 나.

그래서 우리는 **유효성 검사(Validation)**라는 **문지기**를 세워야 해.

![API 유효성 검사 (문지기)](assets/30-api-gatekeeper.png)

### AI에게 지시하기 (상황극)

**Tech Lead(나):** "지출 내역을 저장하는 API를 만들자. 근데 이상한 데이터는 절대 받으면 안 돼."

┌───────────────────────────────────────────────────────────────┐
│  1단계: API 코드 작성 (문지기 포함) 요청                        │
├───────────────────────────────────────────────────────────────┤
│                                                               │
│  나: "Next.js App Router로 `POST` API를 만들어줘.               │
│                                                               │
│  **요구사항:**                                                │
│  1. `supabase.from('expenses').insert()` 사용                  │
│  2. **유효성 검사 (필수):**                                    │
│     - `amount`(금액)가 0보다 커야 해.                          │
│     - `category_id`가 반드시 있어야 해.                        │
│     - 조건 안 맞으면 `400 Bad Request` 에러 반환.               │
│                                                               │
│  **파일명:** `app/api/expenses/route.ts`                      │
│                                                               │
│  AI: "알겠어! `request.json()`으로 데이터를 까보고,              │
│      조건이 안 맞으면 바로 `return Response.json(...)`으로       │
│      돌려보내는 코드를 짤게."                                    │
│                                                               │
└───────────────────────────────────────────────────────────────┘

AI가 작성해준 코드는 대략 이런 모양일 거야. (주석을 잘 봐!)

```typescript
// app/api/expenses/route.ts

import { createClient } from '@/utils/supabase/server';

export async function POST(request: Request) {
  // 1. 주문서(데이터) 읽기
  const body = await request.json();

  // 🛑 2. 문지기 (유효성 검사)
  if (!body.amount || body.amount <= 0) {
    return Response.json(
      { error: '금액은 0보다 커야 해.' },
      { status: 400 } // 탈락!
    );
  }

  // 3. 통과하면 창고(DB)에 저장
  const supabase = createClient();
  const { data, error } = await supabase
    .from('expenses')
    .insert(body)
    .select();

  if (error) {
    return Response.json({ error: error.message }, { status: 500 });
  }

  return Response.json(data);
}
```

이제 "마이너스 금액"이 들어오면, DB 근처에도 못 가고 문지기 선에서 컷(Cut) 당해.

## 3. 실전: 지출 목록 조회하기 (Read)

두 번째 일꾼, **Read(조회)** 담당이야.
저장된 데이터를 화면에 보여주려면 꺼내와야겠지?

여기서 중요한 건 **조인(Join)**이야.
DB에는 `category_id: 1`처럼 숫자만 저장돼 있어.
이걸 그대로 보여주면 사용자는 "1번이 뭔데?"라고 하겠지.
그래서 **"1번은 '식비'야"**라고 연결해서 가져와야 해. 이게 지난 시간에 배운 **1:N 관계의 핵심**이야.

이 코드가 완성되면, 프론트엔드에서 `/api/expenses`로 요청을 보낼 때마다 정리된 장부(JSON 데이터)를 받을 수 있게 돼.

## 오늘의 핵심 정리

✅ **CRUD**: 만들고(C), 보고(R), 고치고(U), 지우는(D) 기능은 서비스의 알파이자 오메가야.
✅ **API Route**: Next.js에서는 `app/api/` 폴더에 파일을 만들면 그게 바로 백엔드야.
✅ **문지기(Validation)**: 이상한 데이터가 DB를 더럽히지 않도록 입구에서 막아야 해.
