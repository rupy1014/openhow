**AI한테 기술적으로 지시하고, 결과를 판단할 수 있는 바이브 코더가 되는 법.**

> 코딩 문법이 아니라, AI한테 "잘 시키는 법"을 배우는 39편짜리 시리즈야.

이 시리즈를 마치면 이런 대화가 가능해져:

```
나: "이 프로젝트에 맞는 아키텍처 추천해봐"
AI: "1. 레이어드 - 단순하고... 2. 클린 - 확장성..."
나: "우리는 규모가 작으니까 1번으로 가자.
    근데 서비스 레이어는 분리해줘."
```

AI가 추천하면, 뭐가 왜 좋은지 판단할 수 있는 수준.
기술 용어로 방향을 지시할 수 있는 수준.

## 구성

| 파트 | 편 | 배우는 것 |
|------|-----|----------|
| 마인드셋 | 1~4 | AI를 어떻게 다뤄야 하는지 |
| 도구 세팅 | 5~9 | Claude Code, 터미널, VS Code, Git |
| 큰 그림 | 10~15 | 프론트/백엔드, DB, API, 프로젝트 구조 |
| 웹 서비스 | 16~19 | 할 일 관리 앱 만들기 |
| 설계 | 20~24 | 아키텍처, 디자인 패턴, 상태 관리 |
| 자동화 | 25~28 | 뉴스 요약 봇 만들기 |
| 데이터 서비스 | 29~32 | 지출 대시보드 만들기 |
| 독립 성장 | 33~35 | 에러 읽기, 디버깅, 혼자 성장하기 |
| AI 서비스 | 36~39 | AI API 연결, 챗봇, 분석 서비스, 카카오톡 봇 |
