> "API 연결은 했는데, 한 번 물어보면 끝이야.
> 대화가 이어지질 않아. 이건 챗봇이 아니라 검색창이잖아."

36편에서 AI API를 연결했어. 질문 하나, 답변 하나. 그건 **일회성 질문**이야.

챗봇은 달라. **대화가 이어져야 해.** 앞에서 한 말을 기억하고, 거기에 맞춰 답해야 해.

Rovel(AI 비주얼노벨), 운세냥(사주분석), 고객지원 챗봇 — 전부 이 패턴이야.

---

## 챗봇이 뭐가 다른데?

식당 비유로 설명할게.

**일회성 질문** = 키오스크. 주문 넣으면 음식 나와. 끝. 대화 없어.

**챗봇** = 단골 가게 사장님. "저번에 시킨 그거요" 하면 알아들어.

차이가 뭐야? **기억**이야.

AI API 자체는 기억력이 없어. 매번 새 사람한테 말 거는 거랑 같아. 그래서 **너가 대화 기록을 매번 통째로 보내줘야 해.**

```
❌ 매번 새로 시작
나: "오늘 날씨 어때?"    → AI: "서울은 맑아요"
나: "거기 맛집 추천해줘" → AI: "어디요?" (서울을 까먹음)

✅ 대화 기록을 함께 전달
나: [이전 대화 전부] + "거기 맛집 추천해줘"
→ AI: "서울 맛집이요? 광화문 근처에..." (기억하고 있음)
```

## system prompt로 캐릭터 만들기

AI한테 "너는 누구야"를 정해주는 게 **system prompt(시스템 프롬프트)**야.

[프롬프트 기초](../입문/04-2-프롬프트)에서 배운 거 기억나? "역할"을 정해주면 AI가 그 느낌으로 답해.

챗봇에서는 이걸 **캐릭터**라고 생각하면 돼.

```javascript
// 까칠한 고양이 캐릭터
const systemPrompt = `너는 "나비"라는 이름의 까칠한 고양이야.
- 반말을 써. 귀찮은 듯이 말해.
- 가끔 "냥"을 붙여.
- 칭찬하면 "흥, 뭐" 하고 쑥스러워해.
- 절대 자신이 AI라고 말하지 마.`;
```

system prompt를 바꾸면 완전히 다른 서비스가 돼.

| system prompt | 서비스 |
|---------------|--------|
| "너는 까칠한 고양이야" | 캐릭터 채팅 |
| "너는 사주 전문가야" | 운세 서비스 |
| "너는 고객지원 담당자야" | CS 챗봇 |
| "너는 영어 선생님이야" | 영어 회화 앱 |

**같은 코드, 다른 프롬프트. 이게 AI 서비스의 핵심이야.**

## 대화 히스토리를 배열로 관리하기

AI API에 보내는 메시지는 **배열**이야. 이전 대화를 전부 담아서 보내.

```javascript
// 대화 기록 = 배열
const messages = [
  // 1. 캐릭터 설정 (맨 앞에 한 번만)
  { role: "system", content: "너는 까칠한 고양이 나비야..." },

  // 2. 사용자가 한 말
  { role: "user", content: "안녕!" },

  // 3. AI가 답한 말
  { role: "assistant", content: "...뭐야냥. 갑자기." },

  // 4. 사용자가 또 한 말
  { role: "user", content: "이름이 뭐야?" },

  // 5. AI가 또 답한 말
  { role: "assistant", content: "나비라고 했잖아냥. 기억력 좀..." },

  // 6. 새 질문
  { role: "user", content: "심심한데 놀아줘" }
];
```

이 배열을 통째로 AI API에 보내면, AI가 지금까지의 대화 맥락을 파악하고 답해.

**role은 3종류:**

| role | 누구 | 용도 |
|------|------|------|
| `system` | 감독 | 캐릭터 설정. 맨 앞에 한 번 |
| `user` | 사용자 | 사용자가 보낸 메시지 |
| `assistant` | AI | AI가 답한 메시지 |

새 메시지가 올 때마다 배열에 추가하고, 전체를 보내. 그게 전부야.

## 스트리밍 응답 — 글자가 하나씩 나오는 거

ChatGPT 써봤으면 알 거야. 답변이 한꺼번에 뜨는 게 아니라 글자가 하나씩 타닥타닥 나와.

이게 **스트리밍**이야.

왜 스트리밍을 쓸까? AI 답변은 길어지면 5~10초 걸려. 그동안 화면에 아무것도 안 뜨면 "멈춘 건가?" 싶잖아.

스트리밍을 쓰면 첫 글자가 0.5초 안에 나와. 체감 속도가 완전히 달라져.

```javascript
// 스트리밍 요청 (OpenRouter 기준)
const response = await fetch("https://openrouter.ai/api/v1/chat/completions", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${apiKey}`,
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    model: "anthropic/claude-sonnet-4-20250514",
    messages: messages,
    stream: true  // 이 한 줄이 핵심
  })
});

// 글자가 도착할 때마다 화면에 붙이기
const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  const chunk = decoder.decode(value);
  // chunk에서 텍스트를 꺼내서 화면에 추가
  displayText(chunk);
}
```

`stream: true` 한 줄만 추가하면 돼. 나머지는 받아서 화면에 붙이는 거야.

## 실습: 캐릭터 채팅 페이지 만들기

이제 AI한테 시켜서 진짜 만들어보자.

### AI한테 이렇게 지시해

```text
캐릭터 채팅 페이지를 만들어줘.

## 요구사항
1. 캐릭터 3개 중 하나를 선택하는 화면
   - 까칠한 고양이 "나비"
   - 다정한 할머니 "순자"
   - 열혈 체육교사 "김코치"
2. 선택하면 채팅 화면으로 이동
3. 말풍선 UI (사용자는 오른쪽, AI는 왼쪽)
4. 스트리밍 응답 (글자가 하나씩 나오게)
5. 대화 히스토리 유지 (이전 대화를 기억)

## 기술
- Next.js + API Route
- OpenRouter API (환경변수: OPENROUTER_API_KEY)
- 각 캐릭터마다 다른 system prompt

## 안 만들 것
- 로그인
- 대화 저장 (새로고침하면 초기화)
- 캐릭터 편집
```

### 핵심 구조

AI가 만들어주는 코드는 이런 구조야.

```
app/
├── page.tsx              ← 캐릭터 선택 화면
├── chat/[id]/page.tsx    ← 채팅 화면
├── api/chat/route.ts     ← AI API 호출 (서버)
└── lib/characters.ts     ← 캐릭터 정보 + system prompt
```

`characters.ts`가 핵심이야. 여기서 캐릭터마다 다른 성격을 넣어.

```typescript
// lib/characters.ts
export const characters = [
  {
    id: "nabi",
    name: "나비",
    description: "까칠한 고양이",
    avatar: "🐱",
    systemPrompt: `너는 "나비"라는 까칠한 고양이야.
      반말 써. 귀찮은 듯이 대답해. 가끔 "냥" 붙여.
      칭찬하면 쑥스러워해. 절대 AI라고 밝히지 마.`
  },
  {
    id: "sunja",
    name: "순자",
    description: "다정한 할머니",
    avatar: "👵",
    systemPrompt: `너는 "순자 할머니"야. 70대 한국 할머니.
      따뜻하게 말해. "아이고~", "그래그래" 자주 써.
      뭘 물어봐도 밥 먹었냐고 먼저 물어봐.`
  },
  {
    id: "coach",
    name: "김코치",
    description: "열혈 체육교사",
    avatar: "💪",
    systemPrompt: `너는 "김코치". 열혈 체육교사야.
      항상 에너지 넘치게 말해. "파이팅!", "할 수 있어!" 자주 써.
      뭘 하든 운동에 비유해서 설명해.`
  }
];
```

**캐릭터를 추가하고 싶으면?** 이 배열에 하나 더 넣으면 끝이야. 코드 한 줄도 안 건드려도 돼.

## 한 줄 정리

챗봇 = system prompt(캐릭터) + messages 배열(기억) + stream(체감 속도). 이 세 개면 AI 대화 서비스가 돼.
