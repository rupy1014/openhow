> "생년월일 넣으면 5만자 사주 분석이 나와."
> "재배 환경 넣으면 맞춤 농업 컨설팅이 나와."
> "신체 정보 넣으면 골프 클럽 피팅 제안이 나와."

쇼케이스에서 가장 많이 보이는 패턴이 이거야.
**사용자가 뭔가를 넣으면, AI가 분석해서 결과를 돌려주는 서비스.**

운세냥, 팜가이드, AiGOLF LAB, 애드플래너...
전부 구조가 같아. **입력 폼 → AI 분석 → 결과 표시.** 이 세 단계뿐이야.

---

## 왜 이 패턴이 많냐

만들기 쉬워서야.

DB도 필요 없어. 로그인도 필요 없어.
폼 하나, API 호출 하나, 결과 화면 하나. 끝.

근데 사용자 입장에서는 "와, AI가 나를 분석해줬어"라는 느낌을 받아.
**투자 대비 임팩트가 가장 큰 구조**야.

---

## 전체 구조가 어떻게 생겼냐

```
[사용자] → 폼 입력 → [내 서버] → 프롬프트 조립 → [AI API] → 분석 결과 → [화면]
```

세 덩어리로 나뉘어.

1. **폼** — 사용자한테 정보를 받아
2. **프롬프트 조립** — 입력값을 AI 프롬프트에 끼워넣어
3. **결과 표시** — AI 응답을 예쁘게 보여줘

하나씩 만들어보자.

---

## 1단계: 폼으로 입력 받기

실습으로 **"직업 + 관심사 → AI가 사이드프로젝트 아이디어 3개 추천"** 서비스를 만들 거야.

AI에게 이렇게 시켜.

┌───────────────────────────────────────────────────────────────┐
│  나: "사용자한테 직업과 관심사를 입력받는 폼을 만들어줘.         │
│      직업은 텍스트 입력, 관심사는 쉼표로 구분해서 여러 개 넣게.  │
│      '분석하기' 버튼 누르면 서버로 전송."                       │
│                                                               │
│  AI: "알겠어. form 컴포넌트 만들고 state로 관리할게."            │
└───────────────────────────────────────────────────────────────┘

```tsx
// components/AnalysisForm.tsx

export default function AnalysisForm({ onSubmit, loading }) {
  const [job, setJob] = useState('');
  const [interests, setInterests] = useState('');

  return (
    <form onSubmit={(e) => {
      e.preventDefault();
      onSubmit({ job, interests });
    }}>
      <input
        placeholder="직업 (예: 마케터)"
        value={job}
        onChange={(e) => setJob(e.target.value)}
      />
      <input
        placeholder="관심사 (예: 여행, 요리, 독서)"
        value={interests}
        onChange={(e) => setInterests(e.target.value)}
      />
      <button type="submit" disabled={loading}>
        {loading ? '분석 중...' : '분석하기'}
      </button>
    </form>
  );
}
```

폼은 이게 전부야. 복잡할 거 없어.

## 2단계: 프롬프트 템플릿에 끼워넣기

핵심은 여기야. 사용자 입력을 **프롬프트 템플릿의 빈칸**에 넣는 거야.

┌───────────────────────────────────────────────────────────────┐
│  나: "서버 액션을 만들어줘. 직업과 관심사를 받아서              │
│      OpenAI API에 프롬프트를 보내고 결과를 받아와.              │
│                                                               │
│      프롬프트 구조:                                            │
│      - 역할: '사이드프로젝트 컨설턴트'                          │
│      - 입력: 직업과 관심사                                     │
│      - 출력: 아이디어 3개, 각각 제목/설명/난이도 포함            │
│      - 포맷: 마크다운으로 출력                                  │
│                                                               │
│      API Key는 .env에서 가져와."                               │
│                                                               │
│  AI: "알겠어. 프롬프트 템플릿을 만들고 입력값을 끼워넣을게."      │
└───────────────────────────────────────────────────────────────┘

```typescript
// app/actions/analyze.ts

export async function analyzeSideProject(job: string, interests: string) {
  const prompt = `
너는 사이드프로젝트 컨설턴트야.

사용자 정보:
- 직업: ${job}
- 관심사: ${interests}

이 사람에게 딱 맞는 사이드프로젝트 아이디어 3개를 추천해줘.
각 아이디어마다 아래 형식으로 작성해:

## 아이디어 1: {제목}
- **한 줄 설명:** ...
- **왜 이 사람에게 맞냐:** ...
- **난이도:** 상/중/하
- **예상 개발 기간:** ...
`;

  const res = await fetch('https://api.openai.com/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${process.env.OPENAI_API_KEY}`,
    },
    body: JSON.stringify({
      model: 'gpt-4o-mini',
      messages: [{ role: 'user', content: prompt }],
    }),
  });

  const data = await res.json();
  return data.choices[0].message.content;
}
```

포인트는 **템플릿 리터럴** (`${}`)이야.
`${job}`, `${interests}` 자리에 사용자 입력이 들어가.

운세냥이면 `${생년월일}`, 팜가이드면 `${재배환경}`, AiGOLF LAB이면 `${신체정보}`.
서비스는 달라도 구조는 똑같아.

## 3단계: 결과를 마크다운으로 표시하기

AI 응답은 보통 마크다운 형태로 와. `##`, `**`, `-` 같은 게 섞여있어.
이걸 그냥 텍스트로 뿌리면 `## 아이디어 1` 이런 게 그대로 보여서 보기 싫어.

┌───────────────────────────────────────────────────────────────┐
│  나: "AI 응답이 마크다운으로 오는데,                            │
│      이걸 예쁘게 렌더링해서 보여줘.                             │
│      react-markdown 라이브러리 써줘."                           │
│                                                               │
│  AI: "알겠어. react-markdown 설치하고 결과 영역에 적용할게."     │
└───────────────────────────────────────────────────────────────┘

```tsx
// components/ResultView.tsx
import ReactMarkdown from 'react-markdown';

export default function ResultView({ result }) {
  if (!result) return null;

  return (
    <div className="prose max-w-none">
      <ReactMarkdown>{result}</ReactMarkdown>
    </div>
  );
}
```

`prose`는 Tailwind CSS의 타이포그래피 플러그인이야.
마크다운을 넣으면 제목, 목록, 볼드가 자동으로 예쁘게 스타일링돼.

---

## 로딩 상태는 왜 중요하냐

AI API 호출은 느려. 보통 5~15초 걸려.
그 사이에 화면이 아무 반응 없으면 사용자는 "먹통인가?" 하고 떠나.

**버튼 텍스트를 바꾸는 것만으로도 충분해.**

```tsx
// 페이지 컴포넌트
const [loading, setLoading] = useState(false);
const [result, setResult] = useState('');

async function handleSubmit({ job, interests }) {
  setLoading(true);
  setResult('');

  try {
    const analysis = await analyzeSideProject(job, interests);
    setResult(analysis);
  } catch (error) {
    setResult('분석에 실패했어. 다시 시도해봐.');
  } finally {
    setLoading(false);
  }
}
```

`loading`이 `true`면 버튼이 "분석 중..."으로 바뀌고 비활성화돼.
이것만 해도 사용자는 "아, 일하고 있구나" 하고 기다려.

더 멋지게 하고 싶으면 스켈레톤 UI나 프로그레스 바를 넣으면 돼.
근데 첫 버전에서는 버튼 텍스트 변경이면 충분해.

---

## 실제 쇼케이스는 뭐가 다르냐

기본 구조는 같아. 차이점은 **프롬프트의 깊이**와 **입력 방식**이야.

| 서비스 | 입력 방식 | 프롬프트 특징 |
|--------|----------|-------------|
| 운세냥 | 생년월일 | 사주명리학 전문 지식이 프롬프트에 담겨있어 |
| 팜가이드 | 재배환경 폼 | 농업 도메인 지식 + 지역별 기후 데이터 |
| AiGOLF LAB | 신체정보 | 골프 피팅 전문가 역할 + 클럽 스펙 데이터 |
| 애드플래너 | PDF 업로드 | 파일 파싱 → 프롬프트 조립 (입력이 파일) |

결국 **도메인 지식을 프롬프트에 얼마나 잘 녹이느냐**가 서비스의 품질을 결정해.
코드 구조는 다 똑같아.

---

## 한 줄 정리

**폼 → 프롬프트 템플릿 → AI API → 마크다운 표시.** 이 네 줄이면 AI 분석 서비스가 완성돼. 차이를 만드는 건 코드가 아니라 프롬프트에 담긴 도메인 지식이야.
