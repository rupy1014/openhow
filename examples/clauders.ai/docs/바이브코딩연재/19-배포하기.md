> "제 컴퓨터에서는 잘 되는데, 친구한테 어떻게 보여주죠?"
> "주소창에 localhost라고 치니까 안 나온대요."

지금 너가 만든 서비스는 **너의 노트북(Localhost)** 안에 갇혀 있어.
마치 집 안에서 혼자 부르는 노래와 같지.
친구들이 내 노래를 들으려면, 앨범을 내서 전 세계에 배포해야겠지?

우리는 오늘 **Vercel**이라는 무료 배포소를 통해, 너의 서비스를 전 세계 어디서든 접속할 수 있게 만들 거야.
클릭 몇 번이면 돼. 진짜야.

---

## 이 글을 읽고 나면

- 내 컴퓨터에 있는 코드를 인터넷상에 올릴 수 있어 (GitHub)
- **Vercel**을 통해 나만의 `.vercel.app` 웹사이트 주소를 가질 수 있어
- **환경변수**를 설정해서 배포된 사이트와 DB(Supabase)를 연결할 수 있어

---

## 1. 배포의 3단계 흐름

복잡해 보이지만 딱 3단계만 기억하면 돼.

![배포 3단계 흐름](assets/19-deployment-flow.png)

우리가 할 일은 **GitHub에 코드를 올리고**, **Vercel 버튼을 누르고**, **비밀번호(환경변수)를 알려주는 것**뿐이야.

---

## 2. 실전: 따라 하기만 하면 배포 끝

### 1단계: GitHub에 코드 올리기

아직 코드가 내 컴퓨터에만 있다면, GitHub라는 도서관에 꽂아야 해.
(터미널에 입력해봐)

```bash
git add .
git commit -m "배포 준비 완료"
git push
```

이제 내 코드가 GitHub에 올라갔어.

<br/>

### 2단계: Vercel에서 가져오기

1. [Vercel.com](https://vercel.com)에 접속해서 (GitHub 아이디로) 가입해.
2. **Add New...** → **Project** 클릭
3. 왼쪽 목록에서 방금 올린 프로젝트(`smart-todo` 등)가 보일 거야. **Import** 버튼을 눌러.

<br/>

### 3단계: 환경변수 설정 (여기가 제일 중요!)

Import를 누르면 설정 화면이 나와.
여기서 바로 `Deploy`를 누르면? **100% 에러 나.**

왜냐하면 Vercel은 우리집 금고 비밀번호(**Supabase URL, API Key**)를 모르거든.
우리가 `.env.local` 파일에 숨겨놨잖아? 그걸 Vercel에게도 알려줘야 해.

1. **Environment Variables** 메뉴를 클릭해서 펼쳐.
2. `.env.local` 파일에 있는 내용을 하나씩 복사해서 붙여넣어.

| Key | Value |
|-----|-------|
| `NEXT_PUBLIC_SUPABASE_URL` | (너의 Supabase URL) |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | (너의 Supabase Anon Key) |

다 넣었으면 **Deploy** 버튼을 눌러!

## 3. 에러가 났어? 당황하지 마

배포를 처음 하면 **Build Failed**(빌드 실패)가 뜰 확률이 높아.
주로 TypeScript 오류 때문인데. AI가 있으니 걱정 마.

### AI에게 해결 요청하기

**상황**: Vercel 로그에 빨간 글씨로 에러가 잔뜩 떴다.

**🧑🏻‍💻 나 (사용자)**
> "Vercel 배포 중에 에러가 났어. 로그를 복사해줄게. 원인이랑 해결 방법 알려줘.
>
> (로그 붙여넣기: `Type error: Property 'id' does not exist on type 'Todo'...`)"

**🤖 AI**
> `types/todo.ts` 파일이랑 실제 코드에서 쓰는 타입이 달라서 생긴 문제야.
> `TodoList.tsx` 파일의 23번째 줄을 이렇게 고쳐...

AI가 시키는 대로 코드를 고치고, 다시:
1. `git add .`
2. `git commit -m "타입 에러 수정"`
3. `git push`

하면 Vercel이 **자동으로 감지하고 다시 배포**를 시작해. (이게 진짜 편한 점이야!)

---

## 4. 마지막 관문: Supabase 허용 주소 추가

드디어 배포가 성공해서 `https://smart-todo.vercel.app` 같은 멋진 주소가 생겼어!
접속해서 **로그인**을 눌러봐.

어라? 로그인이 안 되거나 에러 페이지가 뜰 거야.
Supabase 입장에서는 "어? 처음 보는 사이트`(.vercel.app)`에서 로그인을 요청하네? 수상한데?" 하고 막는 거지.

**🧑🏻‍💻 나 (사용자)**
> "배포 주소가 나왔어. `https://smart-todo.vercel.app` 이야.
> Supabase에서 이 주소를 허용하게 설정하려면 어디로 가야 해?"

**🤖 AI**
> Supabase 대시보드에서 **Authentication → URL Configuration**으로 가.
> **Site URL**이나 **Redirect URLs**에 방금 받은 Vercel 주소를 추가하고 저장해.

<br/>

### 최종 확인 (휴먼터치)

이제 진짜 끝났어. 핸드폰을 꺼내.
친구한테 카톡으로 링크를 보내고, 접속해봐.

1. **핸드폰으로 접속**: 잘 뜨나?
2. **로그인**: 구글 로그인이 되나?
3. **글 쓰기**: "핸드폰으로 작성함"이라고 써봐.
4. **컴퓨터 확인**: 컴퓨터에서도 그 글이 보이나?

이게 된다면, 너는 이제 **Full-Stack 개발자**야.

## 오늘의 핵심 정리

✅ **배포 흐름**: 내 컴퓨터 → GitHub → Vercel → 전 세계
✅ **환경변수**: `.env`에 있는 비밀번호는 Vercel에도 똑같이 등록해야 해.
✅ **재배포**: 코드를 수정하고 `git push`만 하면 Vercel이 알아서 다시 배포해.
✅ **URL 허용**: 배포된 주소를 Supabase(백엔드)에도 알려줘야 로그인이 돼.
