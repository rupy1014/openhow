27편에서 슬랙 봇을 만들었어. 웹훅 URL 하나로 메시지를 쏘는 구조.

근데 현실적으로 생각해봐.
너의 고객, 너의 팀원, 너의 가족 중에 슬랙 쓰는 사람이 몇 명이야?

**한국에서 메신저 = 카카오톡이야.**
오늘은 27편의 슬랙 봇을 카카오톡 봇으로 바꿔볼 거야.

---

## 카카오톡 자동화, 두 가지 방법이 있어

| 방법 | 뭐냐 | 난이도 | 용도 |
|------|------|--------|------|
| **나에게 보내기** | 내 카톡에 메시지를 보내는 API | 쉬움 | 알림, 리마인더, 뉴스봇 |
| **카카오 채널 챗봇** | 카카오톡 채널에서 고객과 대화하는 봇 | 어려움 | 고객 응대, 예약, CS |

오늘은 **"나에게 보내기"**를 해볼 거야.
내 카톡에 뉴스 요약이 딱 오게 만드는 거지.

카카오 채널 챗봇은 마지막에 간단히 소개할게.

---

## 1. 카카오 개발자 앱 만들기

슬랙은 웹훅 URL 하나면 끝났어.
카카오는 살짝 다르다. **앱을 만들고, 토큰을 발급받아야** 해.

:::info 토큰이 뭐야?
"나 인증된 사람이야"를 증명하는 임시 비밀번호야. 슬랙의 웹훅 URL이랑 역할이 같아.
:::

**Step 1.** [Kakao Developers](https://developers.kakao.com/) 접속 → 로그인

**Step 2.** 상단 **내 애플리케이션** → **애플리케이션 추가하기** 클릭

**Step 3.** 앱 이름 입력 (예: `뉴스봇`), 사업자명은 본인 이름으로 → 저장

**Step 4.** 만들어진 앱 클릭 → 왼쪽 메뉴에서 **카카오 로그인** → **활성화 설정** ON

**Step 5.** 왼쪽 메뉴 **카카오 로그인 → 동의항목** → `카카오톡 메시지 전송` 항목을 **선택 동의**로 설정

여기까지가 앱 세팅이야. 5분이면 끝나.

---

## 2. 토큰 발급받기

앱을 만들었으면 이제 **토큰**을 받아야 해.
이게 메시지를 보낼 수 있는 열쇠야.

**Step 1.** 앱 설정 페이지에서 **앱 키** 메뉴 → **REST API 키**를 복사해

**Step 2.** 브라우저에서 아래 URL을 열어 (REST API 키 부분만 바꿔)

```
https://kauth.kakao.com/oauth/authorize?client_id={REST_API_KEY}&redirect_uri=http://localhost:3000/callback&response_type=code&scope=talk_message
```

**Step 3.** 동의 화면이 나오면 **동의하고 계속하기** 클릭

**Step 4.** 리다이렉트된 URL에서 `code=` 뒤의 값을 복사해. 이게 **인가 코드**야.

**Step 5.** AI한테 이렇게 시켜

```
> 카카오 인가 코드로 access_token을 발급받는 스크립트 만들어줘.
> REST API 키: {네 REST API 키}
> 인가 코드: {방금 복사한 코드}
> redirect_uri: http://localhost:3000/callback
```

AI가 `curl` 명령어나 Node.js 스크립트를 만들어줄 거야.
실행하면 `access_token`이 나와. 이걸 `.env`에 저장해.

```
KAKAO_ACCESS_TOKEN=여기에_토큰_붙여넣기
```

---

## 3. 메시지 보내기

토큰이 있으면 메시지 보내는 건 간단해.

```javascript
const axios = require('axios');
require('dotenv').config();

async function sendToKakao(message) {
  const url = 'https://kapi.kakao.com/v2/api/talk/memo/default/send';

  await axios.post(url, new URLSearchParams({
    template_object: JSON.stringify({
      object_type: 'text',
      text: message,
      link: { web_url: 'https://developers.kakao.com' }
    })
  }), {
    headers: {
      'Authorization': `Bearer ${process.env.KAKAO_ACCESS_TOKEN}`,
      'Content-Type': 'application/x-www-form-urlencoded'
    }
  });

  console.log('카톡 전송 완료!');
}
```

실행하면 **너의 카카오톡**에 메시지가 와.
슬랙 웹훅이랑 구조가 똑같아. URL에 데이터를 쏘는 것뿐이야.

---

## 4. 뉴스봇을 카카오톡으로 바꾸기

27편에서 만든 슬랙 뉴스봇 기억나? 핵심만 바꾸면 돼.

```
> 기존 슬랙 발송 코드에서 sendToSlack 함수를
> 카카오톡 나에게 보내기로 바꿔줘.
> KAKAO_ACCESS_TOKEN은 환경변수에서 읽어와.
```

바뀌는 건 **발송 함수 하나**야.
뉴스 수집, 요약 로직은 그대로. 출구만 슬랙에서 카톡으로 바뀐 거야.

28편에서 만든 GitHub Actions 설정도 그대로 쓸 수 있어.
**Secrets**에 `KAKAO_ACCESS_TOKEN`만 추가하면 끝.

```
GitHub 저장소 Settings → Secrets and variables → Actions
→ New repository secret
→ Name: KAKAO_ACCESS_TOKEN
→ Secret: 너의 토큰 값
```

이제 매일 아침, 슬랙이 아니라 **카카오톡**으로 뉴스가 와.

---

## 5. 카카오 채널 챗봇은 뭐가 다르냐

"나에게 보내기"는 **나한테** 보내는 거야.
고객한테 보내려면 **카카오 채널 챗봇**이 필요해.

| | 나에게 보내기 | 카카오 채널 챗봇 |
|---|---|---|
| 대상 | 나 | 카카오 채널 구독자 |
| 방향 | 일방향 (발송만) | 양방향 (대화 가능) |
| 서버 | 필요 없음 | 필요함 (Webhook 받을 서버) |
| 심사 | 없음 | 카카오 심사 필요 |
| 용도 | 개인 알림 | 고객 응대, 예약, CS |

카카오 채널 챗봇을 제대로 만들면 이런 게 가능해.

> **톡대리**: 카카오톡에서 고객 응대와 예약 관리, 중요 이슈 보고까지 자동 처리하는 AI 운영 에이전트

고객이 카톡으로 "예약하고 싶어요"라고 말하면 AI가 자동으로 응대하는 거야.
이건 서버 구축, 카카오 비즈니스 채널 개설, 스킬 서버 연동까지 필요해서 별도로 다룰게.

---

## 한 줄 정리

슬랙이든 카톡이든, 메시지를 쏘는 구조는 같아. URL + 토큰 + 데이터. 출구만 바꾸면 어디든 보낼 수 있어.
