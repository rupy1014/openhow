> "AI한테 시키면 뭔가 나오긴 해. 근데 거기서 끝이야."

맞아. 시작은 누구나 해.
유튜브 보고 따라하면 뭔가 돌아가는 건 만들 수 있어.

**근데 거기서 기능 하나 추가하려니까 꼬이지?**
**에러 나면 뭐가 문제인지 모르겠지?**
**결국 반쯤 만들다 포기하게 되지?**

그게 "그냥 시키는 것"과 **"바이브 코딩"**의 차이야.
그 차이가 뭔지 얘기해볼게.

```
📚 이 글을 읽고 나면

✅ 바이브 코딩이 뭔지 설명할 수 있어
✅ 코드 문법 대신 개념으로 AI와 대화한다는 걸 이해해
```

## 전통적인 코딩 vs 바이브 코딩

먼저 큰 그림부터 볼게.

```
┌───────────────────────────────────────────────────────────────────────┐
│                     프로그램을 만드는 두 가지 방법                        │
├─────────────────────────────┬─────────────────────────────────────────┤
│       전통적인 코딩          │           바이브 코딩                    │
├─────────────────────────────┼─────────────────────────────────────────┤
│  사람이 직접 코드를 짠다      │   AI가 코드를 짜고, 사람이 방향을 잡는다   │
│  문법, 규칙을 외워야 한다     │   의도와 결과를 명확하게 지시한다         │
│  오류 하나에 몇 시간 고민     │   AI한테 물어보면 바로 해결책 제시        │
│  혼자 모든 걸 알아야 한다     │   AI와 대화하면서 같이 만든다            │
└─────────────────────────────┴─────────────────────────────────────────┘
```

전통적 코딩 vs 바이브 코딩: 직접 쌓는 것과 지휘하는 것의 차이
![전통적 코딩 vs 바이브 코딩: 직접 쌓는 것과 지휘하는 것의 차이](assets/traditional_vs_vibe_coding.png)

**잠깐, "바이브"면 대충 느낌만 전달하면 되는 거 아니야?**

아니야. 여기서 오해하면 안 돼.

```
⚠️ 바이브 코딩 ≠ 대충 말하기

"바이브"라는 단어 때문에 헷갈릴 수 있어.
"느낌만 전달하면 AI가 알아서 해주겠지?" → 이게 안 돼서 다들 막히는 거야.

바이브 코딩의 진짜 의미:
- 코드 문법은 몰라도 됨
- 대신 "뭘 만들지, 왜 만들지, 어떻게 동작해야 하는지"는 명확히 알아야 함
- AI한테 기술적인 지침을 정확하게 내려야 함
```

```
🎯 바이브 코딩이란?

"코드 대신, 명확한 의도와 예상 결과로 AI를 지휘하는 것"

코드 한 줄 한 줄 짜는 게 아니라,
"이 버튼 누르면 이렇게 동작해야 해"라고 구체적으로 지시하는 방식이야.
```

## 구체적으로 어떻게 다르냐?

예를 들어 **할 일 관리 앱**을 만든다고 해볼게.

### 전통적인 코딩

```
1. 프로그래밍 언어 선택 (JavaScript? Python? Swift?)
2. 문법 공부 (변수, 함수, 조건문, 반복문...)
3. 프레임워크 선택 (React? Vue? Flutter?)
4. 프레임워크 사용법 공부
5. 직접 코드 작성 시작
6. 에러 발생 → 구글 검색 → 스택오버플로우 뒤지기
7. 몇 시간 삽질 후 겨우 버튼 하나 완성
8. ...반복...
```

**6개월~1년 공부해야 간단한 앱 하나 만들 수 있어.**

### 바이브 코딩

```
1. AI에게 말하기: "할 일 관리 앱 만들고 싶어"
2. AI가 물어봄: "어떤 기능이 필요해?"
3. 대화하며 기획: "추가, 삭제, 완료 체크. 스마트폰에서 쓸 거야"
4. AI가 코드 작성 → 결과 확인
5. 피드백: "버튼이 너무 작아. 더 크게 해줘"
6. AI가 수정 → 다시 확인
7. 반복하면서 완성
```

**대화할 줄 알면 시작할 수 있어.**

처음부터 술술 되진 않아. 연습이 필요하지.
근데 6개월~1년 공부하는 것보다는 훨씬 빨리 결과물을 볼 수 있어.

## "그럼 코딩 공부 안 해도 되는 거야?"

앞서 말했듯이, "대충 시키면 다 해줌"은 아니야.
바이브 코딩도 **알아야 할 게 있어.**

```
┌───────────────────────────────────────────────────────────────────────┐
│                    바이브 코딩에서 필요한 것들                           │
├───────────────────────────────────────────────────────────────────────┤
│                                                                       │
│  ❌ 몰라도 되는 것                                                     │
│     - 프로그래밍 언어 문법                                              │
│     - 복잡한 알고리즘                                                   │
│     - 코드를 직접 타이핑하는 것                                          │
│                                                                       │
│  ✅ 알아야 하는 것                                                      │
│     - 뭘 만들고 싶은지 (목적)                                           │
│     - 프로그램의 종류가 뭐가 있는지 (웹? 앱? 자동화?)                     │
│     - AI가 만든 걸 확인하고 피드백하는 법                                │
│     - 에러가 나면 AI한테 어떻게 설명하는지                                │
│                                                                       │
└───────────────────────────────────────────────────────────────────────┘
```

**코드를 "짜는 것"과 "보는 것"은 달라.**

짜는 건 AI가 하지만, **보고 판단하는 건 네 몫이야.**

```
💡 비유하자면

자동차를 직접 만들 줄 몰라도 운전은 할 수 있어.
근데 운전하려면 적어도 이건 알아야 해:
- 엑셀이 뭐고 브레이크가 뭔지
- 신호등이 뭘 의미하는지
- 어디로 가고 싶은지

바이브 코딩도 마찬가지야.
AI가 차를 만들어주지만, 운전은 네가 해.
![바이브 코딩 운전 비유: 엔진을 몰라도 운전은 할 수 있어요](assets/vibe_coding_driving_analogy.png)
```

지금은 "운전을 배우면 된다"는 것만 기억해둬.
차를 직접 조립하는 법은 몰라도 괜찮아.

## 왜 지금 바이브 코딩이냐?

**"AI로 코딩하는 거야 예전에도 있었잖아. 뭐가 달라진 건데?"**

달라졌어. 많이.

```
┌───────────────────────────────────────────────────────────────────────┐
│                       AI 코딩의 발전 과정                               │
├───────────────────────────────────────────────────────────────────────┤
│                                                                       │
│  🔹 2020년 이전                                                        │
│     "if문 예제 알려줘" → 구글 검색 수준                                  │
│     코드 조각만 찾아줌, 조립은 직접 해야 함                               │
│                                                                       │
│  🔹 2022년 (ChatGPT 등장)                                              │
│     "로그인 기능 만들어줘" → 코드 덩어리 생성                             │
│     복사-붙여넣기 해야 함, 파일 관리는 직접                               │
│                                                                       │
│  🔹 2024년~현재 (Claude Code 등)                                       │
│     "할 일 앱 만들어줘" → 프로젝트 전체 생성                              │
│     파일 구조, 코드, 실행, 에러 수정까지 가능                             │
│                                                                       │
└───────────────────────────────────────────────────────────────────────┘
```

**근데 서두에서 말한 것처럼, 여기서 멈추는 사람이 대부분이야.**

처음엔 되는데, 기능 추가하면 꼬이고, 결국 포기하게 되지.

**그래서 "바이브 코딩"이 필요한 거야.**

```
🎯 결국 차이는 "명령하는 사람"

- AI 성능은 다 똑같음
- 같은 AI 써도 어떤 사람은 완성하고, 어떤 사람은 포기함
- 차이는 "어떻게 시키느냐"

그 "어떻게 시키느냐"를 이 시리즈에서 알려줄 거야.
```

## 바이브 코딩으로 뭘 만들 수 있냐?

실제로 개발 경험 없는 사람들이 바이브 코딩으로 만든 것들이야.

```
┌───────────────────────────────────────────────────────────────────────┐
│                 비개발자들이 만든 실제 사례                               │
├───────────────────────────────────────────────────────────────────────┤
│                                                                       │
│  💼 업무 자동화                                                         │
│     - 매일 아침 뉴스 요약해서 슬랙에 보내주는 봇                          │
│     - 엑셀 파일 수십 개를 자동으로 합쳐주는 프로그램                       │
│     - 이메일 왔을 때 내용 분석해서 분류해주는 도구                         │
│                                                                       │
│  🌐 웹사이트/서비스                                                     │
│     - 개인 포트폴리오 웹사이트                                           │
│     - 동네 스터디 모임 관리 페이지                                        │
│     - 소규모 쇼핑몰                                                     │
│                                                                       │
│  📱 간단한 앱                                                           │
│     - 습관 트래커                                                       │
│     - 가계부 앱                                                         │
│     - 운동 기록 앱                                                      │
│                                                                       │
│  📊 데이터 분석                                                         │
│     - 매출 데이터 시각화 대시보드                                         │
│     - 설문 결과 자동 분석                                                │
│     - 경쟁사 가격 모니터링                                               │
│                                                                       │
└───────────────────────────────────────────────────────────────────────┘
```

**이 사람들이 코딩을 배웠을까?**

아니야. **AI와 대화하는 법**을 배웠어.

시행착오도 있었고, 몇 번이고 AI한테 다시 시킨 것도 있어.
근데 그 과정에서 점점 **"AI한테 어떻게 말해야 하는지"**를 알게 됐어.

## 바이브 코딩의 핵심은 "대화"

1편에서 말한 "유능한 신입사원" 기억하지?
코딩 실력은 뛰어나지만, 뭘 만들어야 하는지는 모르는 사람.

**바이브 코딩 = 이 신입사원과 "대화"하면서 만드는 것**

앞서 본 것처럼: 말하기 → 확인 → 피드백 → 반복.

**한 번에 완성품이 나오지 않아.**
대화하면서, 확인하면서, 고치면서 만들어가는 거야.

그래서 "바이브"야. 완벽한 설계도 없이, **대화하면서 방향을 잡아가는** 거니까.

---

## 바이브 코딩식 대화법 맛보기

바이브 코딩에서 AI한테 어떻게 말하면 좋을까?
맛보기로 하나만 보여줄게.

```
🎯 AI한테 말할 때

❌ "투두 앱 만들어줘"
   → AI: 뭔가 만들긴 하는데... 이게 내가 원하던 거야?

✅ "할 일 관리 앱을 만들고 싶어.
   일단 어떤 기능들이 필요할지 같이 정리해볼까?"
   → AI: "네, 먼저 핵심 기능부터 정리해볼게. 추가, 삭제, 완료 체크가 기본이고..."
   → 대화하면서 기획이 잡힘
```

**차이가 보여?**

"만들어줘"가 아니라 "같이 정리해보자"야.
이렇게 하면 AI가 네 의도를 더 잘 이해할 수 있어.

이런 대화법은 나중에 더 자세히 다룰게.
지금은 "바이브 코딩 = 대화"라는 것만 기억해둬.

## 바이브 코딩이 안 되는 것도 있어

솔직하게 말할게.

```
⚠️ 바이브 코딩의 한계

모든 걸 바이브 코딩으로 만들 수 있는 건 아니야.

❌ 어려운 것들
- 수백만 명이 동시에 쓰는 대규모 서비스
- 은행, 병원 같이 보안이 아주 중요한 시스템
- 게임 엔진 같은 고성능 프로그램
- 기존에 있는 복잡한 시스템에 기능 추가

✅ 잘 되는 것들
- 개인용, 소규모 팀용 도구
- 프로토타입, MVP
- 반복 업무 자동화
- 간단한 웹사이트, 앱
- 데이터 분석, 시각화

💡 MVP가 뭐냐?
   Minimum Viable Product, 최소 기능 제품이야.
   "일단 핵심 기능만 있는 버전"을 빨리 만들어서
   사람들 반응을 보는 거야. 창업할 때 많이 써.
```

**근데 대부분의 사람들에게는 "잘 되는 것들"이면 충분해.**

회사에서 쓸 자동화 도구, 사이드 프로젝트, 창업 MVP...
이런 건 바이브 코딩으로 충분히 만들 수 있어.

## "나도 할 수 있을까?"

**할 수 있어.** 근데 조건이 있어.

```
┌───────────────────────────────────────────────────────────────────────┐
│              바이브 코딩을 잘하기 위한 3가지 마인드셋                      │
├───────────────────────────────────────────────────────────────────────┤
│                                                                       │
│  1️⃣ "한 번에 안 돼도 괜찮아"                                           │
│     처음부터 완벽하게 안 나와.                                           │
│     대화하면서 고쳐나가면 돼.                                            │
│                                                                       │
│  2️⃣ "내가 방향을 잡아야 해"                                            │
│     AI가 다 해주는 게 아니야.                                            │
│     뭘 만들지, 어떻게 동작해야 하는지는 네가 정해.                         │
│                                                                       │
│  3️⃣ "조금씩 배워가면 돼"                                               │
│     처음엔 모르는 용어가 많아.                                           │
│     하나씩 만들다 보면 자연스럽게 알게 돼.                                │
│                                                                       │
└───────────────────────────────────────────────────────────────────────┘
```

**이 시리즈가 그 과정을 도와줄 거야.**

그리고 재밌는 게 하나 있어.

```
💡 바이브 코딩을 하다 보면

처음엔 AI가 뭘 하는지 하나도 몰라.
"뭔가 막 만드네..." 수준이야.

근데 몇 번 하다 보면?
"아, 지금 로그인 부분 고치고 있구나"
"여기서 에러 난 건 이것 때문이겠네"

코드를 **짤 줄은 몰라도, 볼 줄은 알게 돼.**

1편에서 말한 "유능한 매니저" 기억하지?
바이브 코딩을 하면서 자연스럽게 그렇게 성장하게 돼.
```

## 오늘의 핵심 정리

```
✅ 바이브 코딩 = 코드 대신 명확한 지침으로 AI를 지휘하는 것
   → 코드 문법은 몰라도, 의도와 결과는 정확히 전달해야 해

✅ 코드를 "짜는 것"은 몰라도 되지만, "보는 것"은 배워야 해
   → AI가 만든 걸 확인하고 피드백하는 건 네 몫

✅ 지금이 타이밍인 이유
   → AI가 코드 조각이 아니라 프로젝트 전체를 다룰 수 있게 됐어

✅ 바이브 코딩의 핵심은 "대화 → 확인 → 피드백 → 반복"
   → 한 번에 완성 X, 같이 만들어 나가기 O
```
