> "코딩 안 배워도 된다면서? 그런데 뭔가 알아야 한다고?"

맞아. 헷갈리지.
"코드 몰라도 된다"면서 "뭔가는 알아야 한다"니.

**사실 이게 바이브 코딩의 핵심이야.**

코드 **문법**은 몰라도 돼.
근데 **개념**은 알아야 해.

그 차이를 명확하게 정리해줄게.

**바이브 코딩 학습**은 달라:

  <p style="margin: 0; line-height: 1.8;">프로그램이 뭔지 → 웹/앱이 어떻게 다른지 → 프론트/백엔드가 뭔지 → ...<br>→ 개념 알면 AI한테 "로그인 기능 만들어줘" 할 수 있음</p>
</div>
</div>

문법은 AI가 처리해.
너는 **"뭘 만들지"와 "이게 맞는지"**만 판단하면 돼.

"AI가 알아서 해주면 되는 거 아니야?"

## "코드를 볼 줄 알아야 한다"는 게 무슨 뜻이냐?

2편에서 "코드를 짜는 것과 보는 것은 다르다"고 했지.
이게 무슨 뜻인지 예시로 보여줄게.

```javascript
// AI가 만든 코드 예시
function addTodo(text) {
  const newTodo = {
    id: Date.now(),
    text: text,
    completed: false
  };
  todos.push(newTodo);
  saveTodos();
}
```

**짜는 것 (몰라도 됨):**
- `function`이 뭔지
- `const`가 뭔지
- `Date.now()`가 뭔지
- 중괄호 `{}`를 언제 쓰는지

**보는 것 (알아야 함):**
- "아, 이건 할 일을 추가하는 기능이구나"
- "새 할 일에 id, 텍스트, 완료 여부가 들어가는구나"
- "추가하고 나서 저장도 하는구나"

## 오늘의 핵심 정리

<div style="background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); border-radius: 16px; padding: 3px; margin: 2em 0;">
<div style="background: #1a1a2e; border-radius: 14px; padding: 28px 32px; color: #e0e0e0; line-height: 1.7;">
  <h3 style="margin: 0 0 24px 0; color: #fff; font-size: 1.25em; text-align: center; border-bottom: 1px solid #444; padding-bottom: 18px;">✅ 오늘의 핵심 정리</h3>
  <ul style="margin: 0; padding-left: 8px; list-style: none;">
    <li style="margin: 12px 0;"><span style="color: #a78bfa;">→</span> <strong>전통 코딩은 "문법"부터, 바이브 코딩은 "개념"부터</strong><br><span style="color: #888; margin-left: 20px;">배우는 순서와 내용이 다름</span></li>
    <li style="margin: 12px 0;"><span style="color: #a78bfa;">→</span> <strong>코드 문법은 AI가 처리, 너는 "뭘 만들지"와 "이게 맞는지"를 판단</strong><br><span style="color: #888; margin-left: 20px;">문법 몰라도 됨, 개념은 알아야 함</span></li>

    <li style="margin: 12px 0;"><span style="color: #a78bfa;">→</span> <strong>이 시리즈에서 배울 것</strong><br><span style="color: #888; margin-left: 20px;">프로그램 종류 → 구조 → 설계 개념 → 도구 → 실전</span></li>
  </ul>
