# 용어 사전

---

### 개발자 도구 (Developer Tools)
```
브라우저에서 F12 누르면 나오는 창.
웹페이지의 코드를 볼 수 있음.
에러를 확인할 때 사용.
```

### 깃 (Git)
```
코드의 변경 내역을 추적하는 도구.
"저장" 버튼의 강화판이라고 생각하면 됨.
언제 뭘 바꿨는지 기록이 남음.
```

### 깃허브 (GitHub)
```
Git으로 관리하는 코드를 인터넷에 올리는 곳.
코드 저장소.
```

### 노드 (Node.js)
```
JavaScript를 컴퓨터에서 실행하게 해주는 프로그램.
Claude Code 설치에 필요함.
한 번만 설치하면 됨.
```

### npm
```
Node.js의 패키지 관리자.
프로그램을 설치할 때 사용.
예: npm install -g @anthropic-ai/claude-code
```

### 대시보드 (Dashboard)
```
관리 화면.
Vercel 대시보드 = Vercel 관리 페이지
```

### 도메인 (Domain)
```
웹사이트 주소.
예: google.com, naver.com
구매해서 내 사이트에 연결 가능.
```

### 도커 (Docker)
```
프로그램을 "컨테이너"라는 상자에 담아서 실행하는 도구.
어디서든 바로 실행 가능.
"내 컴퓨터에선 되는데..." 문제를 해결.
```

### 디렉토리 (Directory)
```
폴더와 같은 말.
"이 디렉토리로 이동" = "이 폴더로 이동"
```

### 디버깅 (Debugging)
```
버그(오류)를 찾아서 고치는 과정.
AI에게 에러 메시지를 보여주면 잘 해결해줌.
```

### 디펜던시 (Dependency)
```
프로그램이 동작하려면 필요한 다른 프로그램.
npm install 하면 디펜던시들이 설치됨.
```

### 레포지토리 (Repository)
```
저장소. 줄여서 "레포"
GitHub에서 프로젝트 하나 = 레포지토리 하나
```

### 로컬 (Local)
```
내 컴퓨터.
"로컬에서 실행" = "내 컴퓨터에서 실행"
반대말: 서버, 클라우드
```

### 리팩토링 (Refactoring)
```
코드를 더 깔끔하게 정리하는 것.
기능은 그대로, 구조만 개선.
```

### 마크다운 (Markdown)
```
글을 쉽게 꾸미는 방법.
# = 제목, ** ** = 굵게, - = 목록.
이 문서도 마크다운으로 작성됨.
```

### 명령어 (Command)
```
컴퓨터에게 내리는 지시.
터미널에서 입력하는 것들.
예: cd, npm, git, claude
```

### 배포 (Deploy)
```
인터넷에 올리기.
내 컴퓨터에만 있던 걸 전 세계가 볼 수 있게 하기.
```

### 백엔드 (Backend)
```
사용자에게 보이지 않는 서버 쪽 코드.
데이터 저장, 계산, API 처리 등.
반대: 프론트엔드
```

### 빌드 (Build)
```
코드를 실행 가능한 형태로 만드는 것.
```

### 보일러플레이트 (Boilerplate)
```
프로젝트 시작할 때 쓰는 기본 템플릿.
매번 반복되는 설정이 미리 되어 있음.
```

### 서버 (Server)
```
24시간 켜져 있는 컴퓨터.
인터넷에 파일을 제공하는 역할.
```

### 터미널 (Terminal)
```
명령어를 입력하는 검은 창.
맥: Terminal 앱
윈도우: 명령 프롬프트 또는 PowerShell
VS Code 안에도 있음.
```

### 에디터 (Editor)
```
코드를 작성하는 프로그램.
VS Code가 대표적. 메모장의 고급 버전.
```

### 에러 (Error)
```
프로그램이 실행 중 문제가 생긴 것.
빨간 글씨로 나오면 대부분 에러.
AI에게 에러 메시지를 복붙하면 해결책을 알려줌.
```

### 커밋 (Commit)
```
Git에서 변경사항을 저장하는 것.
"저장"과 비슷하지만 기록이 남음.
```

### 클라우드 (Cloud)
```
인터넷 어딘가에 있는 컴퓨터.
내 컴퓨터가 아닌 인터넷 상의 서버.
```

### 프레임워크 (Framework)
```
개발을 쉽게 해주는 도구 모음.
React, Vue, Next.js 등.
```

### 프롬프트 (Prompt)
```
AI에게 하는 요청/지시.
"프롬프트를 잘 짜야 한다" = "AI에게 요청을 잘 해야 한다"
```

### 프롬프트 엔지니어링 (Prompt Engineering)
```
AI에게 효과적으로 요청하는 기술.
같은 질문도 어떻게 하냐에 따라 결과가 다름.
CLAUDE.md도 프롬프트 엔지니어링의 일종.
```

### 프론트엔드 (Frontend)
```
사용자에게 보이는 화면 쪽 코드.
HTML, CSS, JavaScript로 만드는 부분.
반대: 백엔드
```

### 푸시 (Push)
```
GitHub에 올리기.
내 컴퓨터의 코드를 GitHub으로 보내는 것.
반대: 풀 (Pull) = 가져오기
```

### 호스팅 (Hosting)
```
웹사이트를 인터넷에 올려두는 것.
Vercel = 호스팅 서비스
```

### 환경변수 (Environment Variable)
```
프로그램에 전달하는 설정 값.
API 키 같은 민감한 정보를 코드에 직접 안 쓰고 따로 저장.
.env 파일에 저장하거나 Vercel 설정에서 입력.
```

### 컨텍스트 (Context)
```
AI에게 주는 맥락 정보.
대화 내역, 파일 내용, 지침 등.
컨텍스트가 많으면 AI가 더 정확히 이해함.
```

### 컨텍스트 윈도우 (Context Window)
```
AI가 한 번에 볼 수 있는 양.
책상 위 서류처럼, 너무 많으면 맨 밑에 있는 건 안 보임.
```

### 커스텀 커맨드 (Custom Commands)
```
Claude Code에서 만드는 나만의 명령어.
.claude/commands/ 폴더에 저장.
/내명령어 형태로 실행.
```

### Agent (에이전트)
```
여러 단계를 자동으로 처리하는 AI 워크플로우.
사람이 일일이 지시하지 않아도 알아서 다음 단계로 넘어감.
커스텀 커맨드로 나만의 에이전트를 만들 수 있음.
```

### API
```
프로그램끼리 대화하는 방법.
"날씨 API" = 날씨 정보를 가져오는 통로
AI도 API로 사용할 수 있음.
```

### API Key (API 키)
```
API를 사용하기 위한 비밀번호 같은 것.
절대 다른 사람에게 공유하면 안 됨.
```

### CLAUDE.md
```
Claude Code에게 프로젝트를 설명하는 파일.
프로젝트 루트에 두면 자동으로 읽음.
"이 프로젝트는 이렇게 작업해줘" 하는 지침서.
```

### Claude Code
```
터미널에서 Claude와 대화하는 CLI 도구.
코드 작성, 파일 수정, 명령어 실행까지 가능. 
```

### CSS
```
웹페이지의 디자인을 담당.
색상, 크기, 위치 등.
```

### HTML
```
웹페이지의 구조를 담당.
텍스트, 이미지, 버튼 등의 배치.
```

### Hallucination (할루시네이션)
```
AI가 거짓말하는 것.
없는 정보를 진짜인 것처럼 말함.
그래서 항상 검증이 필요.
```

### Hook (훅)
```
특정 이벤트가 발생할 때 자동으로 실행되는 코드.
Claude Code에서 커밋 전/후에 자동으로 뭔가 하기.
```

### JavaScript (JS)
```
웹페이지의 동작을 담당.
버튼 클릭, 계산, 애니메이션 등.
```

### LLM (Large Language Model)
```
대규모 언어 모델.
ChatGPT, Claude 같은 AI의 기반 기술.
```

### MCP (Model Context Protocol)
```
AI에게 "손"을 달아주는 기술.
외부 도구를 AI가 직접 사용할 수 있게 해줌.
파일 읽기, 웹 검색, DB 조회 등.
```

### MCP 서버
```
MCP로 연결하는 외부 도구.
예: 파일시스템, 브라우저, 데이터베이스 등.
AI가 이 서버를 통해 실제 작업을 수행.
```

### MVP
```
Minimum Viable Product.
최소 기능 제품. 핵심 기능만 있는 첫 버전.
```

### Sub-agent (서브 에이전트)
```
메인 에이전트가 생성하는 작은 에이전트.
복잡한 작업을 분업할 때 사용.
```

### Token (토큰)
```
AI가 텍스트를 처리하는 단위.
대략 한글 1글자 = 1~2토큰.
API 비용이 토큰 단위로 계산됨.
```

### Vercel
```
웹사이트를 무료로 배포해주는 서비스.
GitHub 연결하면 자동 배포.
```

### VS Code
```
Visual Studio Code.
코드 편집 프로그램. 무료, 인기 많음.
```

### Vibe Coding (바이브 코딩)
```
AI와 대화하면서 개발하는 방식.
코드를 직접 안 쳐도 AI가 대신 작성.
"분위기(vibe)만 전달하면 코딩이 된다"는 의미.
```
