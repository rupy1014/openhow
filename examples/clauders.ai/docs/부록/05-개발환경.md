**코드를 짜든 글을 쓰든, 작업하는 공간을 알아야 해.**

---

## GUI랑 CLI가 뭐야?

| 약어 | 뜻 | 방식 |
|------|-----|------|
| GUI | Graphic User Interface | 마우스로 클릭 |
| CLI | Command Line Interface | 글자로 명령 입력 |

GUI는 식당 메뉴판 보고 주문하는 거.
CLI는 주방에 직접 말로 주문하는 거.

CLI가 무섭다고? 복사-붙여넣기만 해도 돼.
AI가 명령어 다 알려줘. 한번 익숙해지면 오히려 빨라.

---

## IDE가 뭐야?

IDE(작업실)는 코드 작성에 필요한 도구가 다 있는 곳이야.

메모장 + 터미널 + 파일관리 = 올인원.

```
┌──────────────────────────────────────────┐
│  파일/폴더 목록   │      편집 영역        │
│  (왼쪽 사이드바)  │  (코드 보고 수정)     │
│                   │                       │
│  📁 프로젝트명    │   여기서 파일 봐      │
│    📁 src        │                       │
│    📄 index.html │                       │
├───────────────────┴───────────────────────┤
│  터미널 (아래쪽)                           │
│  $ 여기에 명령어 입력                      │
└──────────────────────────────────────────┘
```

왼쪽은 파일 목록, 오른쪽은 편집, 아래는 터미널.

---

## 터미널이 뭐야?

컴퓨터에게 **글로 명령**하는 창이야.

대부분 AI가 명령어 알려줘. 복사해서 붙여넣기만 하면 돼.
잘못해도 컴퓨터 안 망가져 (대부분).

자주 쓰는 명령어 (외울 필요 없어):

```bash
cd 폴더명     # 폴더로 이동
ls           # 파일 목록 보기 (맥)
dir          # 파일 목록 보기 (윈도우)
npm install  # 패키지 설치
```

---

## 폴더 구조가 뭐야?

폴더 = 정리함이야. 관련 있는 것끼리 묶어놔.

**코딩 프로젝트:**
```
내프로젝트/              ← 집
  ├── src/               ← 거실 (메인 코드)
  │   ├── index.js       ← TV (메인 파일)
  │   └── utils.js       ← 리모컨 (보조 파일)
  ├── public/            ← 현관 (외부 공개)
  │   └── index.html     ← 현관문
  ├── package.json       ← 집 설명서
  └── README.md          ← 방문자 안내문
```

**기획/글쓰기 프로젝트:**
```
내기획프로젝트/
  ├── PRD/               ← 기획서 모음
  ├── Research/          ← 리서치 자료
  ├── Drafts/            ← 초안들
  └── CLAUDE.md          ← Claude 설정
```

## 한 줄 정리

IDE는 작업실, 터미널은 명령 창구, 폴더는 정리함. 이 세 개만 알면 돼.
