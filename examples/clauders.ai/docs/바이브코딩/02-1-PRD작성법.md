**PRD 한 장이면 AI가 딴짓을 안 해.**

"기획해줘"라고 했는데 AI가 쓸데없는 걸 잔뜩 넣어?

PRD를 제대로 쓰면 그런 일이 없어.

---

## PRD가 뭐야?

Product Requirements Document. 거창하게 들리지만 별거 아니야.

**"이 서비스는 뭐고, 뭘 해야 하고, 뭘 안 해야 하는지"** 적은 문서야.

A4 한 장이면 돼. 10장짜리 기획서가 아니야.

---

## 좋은 PRD에 뭐가 있어?

6가지만 있으면 돼:

| 항목 | 쉽게 말하면 | 예시 |
|------|-----------|------|
| 한 줄 설명 | 이 서비스는 뭐야? | "AI 캐릭터 5명과 대화하는 웹앱" |
| 대상 사용자 | 쓰는 사람이 누구야? | "AI 캐릭터에 관심 있는 사람" |
| 핵심 기능 | MVP에 꼭 필요한 것만 | 캐릭터 선택, 채팅, 한국어 |
| 안 만들 것 | 이번에 안 할 것 | 로그인, 대화 저장, 관리자 |
| 사용자 흐름 | 어떤 순서로 쓰냐 | 캐릭터 선택 → 대화 → 종료 |
| 성공 기준 | 어떻게 되면 완성이야 | "캐릭터 골라서 10번 대화 가능" |

더 쓸 필요 없어. 이 6개면 AI가 방향을 잡아.

---

## AI한테 PRD를 쓰게 하는 법

**Step 1.** 아이디어를 말로 써

완벽하지 않아도 돼. 두세 줄이면 충분해.

```
나 AI 캐릭터 채팅 서비스 만들고 싶어.
- 캐릭터 5명이 있어 (각각 성격이 달라)
- 하나를 골라서 대화할 수 있어
- 한국어로 대화해
- AI는 OpenRouter API 써
```

**Step 2.** PRD를 써달라고 해

```
위 내용으로 PRD.md를 만들어줘.
한 줄 설명, 대상 사용자, 핵심 기능, 안 만들 것,
사용자 흐름, 성공 기준 — 이 6가지를 포함해서.
코드는 아직 짜지 마.
```

**Step 3.** 받은 PRD를 검토해

AI가 PRD를 내놓으면 이 3가지를 확인해:

- **핵심 기능이 3~5개야?** → 많으면 줄여
- **"안 만들 것"이 있어?** → 없으면 추가 요청
- **사용자 흐름이 단순해?** → 5단계 넘으면 줄여

```
핵심 기능에서 "사용자 프로필"은 빼줘. MVP에 필요 없어.
"안 만들 것"에 다크모드, 다국어 지원 추가해줘.
```

**Step 4.** 확정하면 저장해

```
이 PRD 최종 확정이야. PRD.md로 저장해줘.
```

이제 이 PRD를 기준으로 코드를 짜게 할 수 있어.

---

## 복사해서 바로 쓰는 프롬프트

```
[여기에 네 아이디어를 2~3줄로 써]

위 내용으로 PRD.md를 만들어줘.
다음 6가지를 포함해서:

1. 한 줄 설명
2. 대상 사용자
3. 핵심 기능 (최대 5개)
4. 안 만들 것 (최소 3개)
5. 사용자 흐름 (최대 5단계)
6. 성공 기준

코드는 아직 짜지 마.
```

이 프롬프트를 복사해서 아이디어만 바꾸면 돼.

---

## 한 줄 정리

아이디어 2줄 + PRD 프롬프트 = AI가 기획서를 써. 코드는 그 다음이야.
