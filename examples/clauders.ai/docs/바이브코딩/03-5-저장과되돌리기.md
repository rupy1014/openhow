게임에서 보스전 전에 세이브하잖아. 코딩도 똑같아.

---

## git이 뭐야?

**git**은 파일의 "변경 기록"을 저장하는 도구야.

워드에서 Ctrl+S 누르면 이전 내용이 사라지잖아. git은 달라. 매번 "언제, 뭘 바꿨는지" 기록이 남아.

세이브 슬롯이 무한 개인 거야. 잘못되면 이전 세이브로 돌아갈 수 있어.

Claude Code를 설치할 때 git도 같이 깔렸어. 따로 할 거 없어.

---

## GitHub은 뭐야?

**GitHub**은 git 기록을 인터넷에 올리는 곳이야.

내 컴퓨터에만 있으면 나만 봐. GitHub에 올리면:

- 다른 사람도 볼 수 있어
- 배포 서비스(Vercel 등)랑 연결할 수 있어
- 컴퓨터가 고장 나도 코드가 안 사라져

둘 다 Claude한테 시키면 돼. 명령어를 외울 필요 없어.

| | git | GitHub |
|---|---|---|
| 어디에 있어? | 내 컴퓨터 안 | 인터넷 |
| 뭘 해? | 변경 기록 저장 | 기록을 온라인에 백업 |
| 비유 | 게임 세이브 | 클라우드 세이브 |

---

## 어떻게 저장해?

기능 하나가 끝나면 Claude한테 말해.

```text:Claude 프롬프트
지금까지 만든 거 git으로 저장해줘.
"캐릭터 선택 화면 완성"이라고 메시지 남기고.
```

이게 **commit**이야. "이 시점에서 세이브"라는 뜻이야.

Claude가 알아서 해줘. `git add`, `git commit` 같은 명령어는 몰라도 돼.

---

## 어떻게 되돌려?

만들다 망가졌어. 에러를 3번 고쳤는데 계속 나와.

```text:Claude 프롬프트
마지막으로 잘 되던 시점으로 되돌려줘.
```

Claude가 git 기록을 보고 되돌려. 내가 할 건 "되돌려줘" 한마디뿐이야.

이래서 기능마다 저장이 중요한 거야. 저장 안 했으면 돌아갈 곳이 없어.

---

## GitHub에 올리려면?

```text:Claude 프롬프트
이 프로젝트를 GitHub에 올려줘.
리포지토리 이름은 my-project로.
```

처음 한 번만 하면 돼. 그 다음부터는:

```text:Claude 프롬프트
git으로 저장하고 GitHub에도 올려줘.
```

이 한마디면 저장 + 업로드가 끝나.

---

## 다른 컴퓨터에서 이어서 하려면?

회사에서 작업하다가 집에서 이어서 하고 싶어. GitHub에 올려뒀으면 가능해.

**처음 가져올 때:**

```text:Claude 프롬프트
GitHub에 있는 내 프로젝트 가져와줘.
주소는 https://github.com/내아이디/my-project
```

이게 **clone**이야. GitHub에서 내 컴퓨터로 복사하는 거야.

**이미 가져온 적 있으면:**

```text:Claude 프롬프트
GitHub에서 최신 내용 가져와줘.
```

이게 **pull**이야. 다른 데서 올린 내용을 내 컴퓨터에 반영하는 거야.

흐름은 이래:

```
회사: 작업 → 저장 → GitHub에 올리기(push)
집:   최신 내용 가져오기(pull) → 작업 → 저장 → GitHub에 올리기(push)
회사: 최신 내용 가져오기(pull) → 작업 이어서...
```

어디서든 GitHub만 거치면 이어서 할 수 있어.

---

## 다른 사람이랑 같이 하려면?

GitHub에 올린 프로젝트는 다른 사람을 초대할 수 있어.

```text:Claude 프롬프트
GitHub 리포지토리에 collaborator 추가하는 방법 알려줘.
```

초대받은 사람도 똑같이 clone → 작업 → push 하면 돼. 같은 프로젝트를 여러 사람이 동시에 작업할 수 있어.

근데 같은 파일을 동시에 고치면 **충돌**이 나. 이건 자연스러운 거야.

```text:Claude 프롬프트
git 충돌이 났어. 해결해줘.
```

Claude가 양쪽 변경 내용을 보여주고, 어떤 걸 남길지 물어봐. 선택하면 끝이야.

협업할 때 충돌을 줄이는 방법:

- **같은 파일을 동시에 안 고쳐.** "나는 이 파일, 너는 저 파일"로 나눠
- **자주 올려.** 하루치 쌓아놓고 올리면 충돌이 커져. 기능 하나 끝날 때마다 올려
- **시작 전에 최신 내용 가져와.** 작업 시작할 때 pull부터 해

---

## 얼마나 자주 저장해?

```
기능 만들기 → 확인 → 저장(commit) → 다음 기능
```

이 리듬을 지켜. 저장 없이 3개 기능을 연달아 만들면, 2번째에서 망가졌을 때 전부 날려야 해.

한 기능 끝날 때마다 세이브. 이게 전부야.

---

## 한 줄 정리

git은 무한 세이브 슬롯, GitHub은 클라우드 백업. 기능 끝나면 저장, 망가지면 되돌려.
