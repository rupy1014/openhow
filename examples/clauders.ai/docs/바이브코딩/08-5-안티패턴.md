"만들어줘"라고 말하면 뭔가 나오니까 잘 되는 것 같아. 근데 3일 뒤에 터져.

---

## 한 번에 다 시키면?

```
❌ "쇼핑몰 만들어줘. 상품 목록, 장바구니, 결제, 회원가입 다 넣어."
```

AI가 한 번에 코드를 쏟아내. 뭐가 뭔지 모르겠어. 에러가 나도 어디를 고쳐야 하는지 모르겠어.

Google의 Addy Osmani가 이걸 **70/30 문제**라고 불러.

AI가 70%까지는 마법처럼 빨라. 근데 나머지 30%에서 점점 느려져.

70%는 "동작하는 것처럼 보이는" 단계야. 에러 처리, 보안, 실제 사용자 대응 — 이 30%가 진짜 힘든 거야.

**해결**: [하나씩 만들기](./04-하나씩만들기). 한 기능 끝내고 확인하고 다음.

---

## 고치다가 더 망가지면?

A를 고쳤더니 B가 터져. B를 고쳤더니 A가 다시 터져.

두더지 잡기 게임이야. 하나 잡으면 다른 데서 올라와.

왜 이렇게 돼? 대화가 길어지면 Claude가 이전에 고친 내용과 현재 에러를 동시에 기억해야 해. 15번째 메시지쯤 되면 앞뒤가 모순되기 시작해.

**해결**: 3번 시도해도 안 되면 되돌려. 잘 되던 시점으로 돌아가서 새 대화로 다시 시작해.

---

## AI가 말하면 다 맞아?

2025년 Stack Overflow 설문에서 **66%의 개발자**가 "거의 맞지만 완전히 맞지는 않은" AI 코드를 경험했어.

AI가 없는 라이브러리를 추천하는 경우도 있어. 연구에 따르면 AI가 추천하는 패키지 이름 중 **21.7%가 실제로 없어.** 이름을 지어낸 거야.

이런 걸 **환각(hallucination)** 이라고 해. AI가 자신있게 틀린 말을 하는 거야.

**해결**:
- 브라우저에서 직접 열어보고 확인해
- "이거 진짜 있는 라이브러리야?" 하고 물어봐
- 에러가 이상하면 AI가 지어낸 건 아닌지 의심해봐

---

## 대화를 안 끊으면?

Claude Code에서 2시간 연속 작업하면 대화 내용이 엄청 쌓여.

그러면 AI가 초반에 한 결정을 잊어. 같은 파일을 다른 방식으로 두 번 고쳐서 충돌이 나기도 해.

**해결**:
- 기능 하나 끝나면 `/compact`로 대화 압축
- 작업이 바뀌면 `/clear`로 새로 시작
- [비용 관리](../입문/06-비용관리)에서 자세히 다뤄

---

## 보안은 안 챙겨도 돼?

바이브코딩으로 만든 앱 5,600개를 스캔한 결과:

- **2,000개 이상의 보안 취약점**
- **400개 이상의 노출된 비밀 키**
- **175건의 개인정보 노출**

AI가 만든 코드에 API 키를 하드코딩하거나, 데이터베이스 보안을 꺼두거나, 입력 검증을 빼먹는 경우가 많아.

**해결**:
- API 키는 반드시 환경변수로 (`process.env.API_KEY`)
- `.env` 파일은 `.gitignore`에 추가
- Claude한테 "보안 취약점 없는지 확인해줘"라고 한 번 더 물어봐
