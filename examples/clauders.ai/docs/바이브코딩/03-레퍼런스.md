**"예쁘게 해줘"보다 "이거랑 비슷하게 해줘"가 10배 빨라.**

AI한테 맥락을 잘 주면 결과가 달라져.

맥락 없이 "만들어줘"하면 AI가 혼자 상상해서 만들어. 그게 네 생각이랑 다를 확률이 높아.

---

## 레퍼런스가 뭐야?

AI한테 "이런 느낌으로 해줘"라고 보여주는 참고 자료야.

사진, 스크린샷, 비슷한 서비스 URL, 와이어프레임 — 다 레퍼런스야.

말로 100줄 설명하는 것보다 스크린샷 1장이 낫다.

---

## 어떤 레퍼런스를 줘?

| 레퍼런스 종류 | 언제 쓰면 좋아 | 예시 |
|-------------|-------------|------|
| 스크린샷 | UI 디자인을 정하고 싶을 때 | "이 사이트 캡처해서 비슷하게" |
| 비슷한 서비스 | 전체 느낌을 전달할 때 | "당근마켓 채팅처럼" |
| 와이어프레임 | 레이아웃을 직접 그렸을 때 | 손그림이라도 돼 |
| 색상/폰트 | 브랜드 느낌을 잡을 때 | "#FF6B35, 나눔고딕" |
| 데이터 구조 | 어떤 정보를 다루는지 | "상품엔 이름, 가격, 이미지가 있어" |

---

## 스크린샷이 제일 강력해

:::steps

### 비슷한 서비스를 찾아

네가 만들고 싶은 것과 비슷한 서비스를 브라우저에서 열어.

### 스크린샷을 찍어

- **Windows**: `Win + Shift + S` → 영역 선택
- **Mac**: `Cmd + Shift + 4` → 영역 선택

### Claude한테 보여줘

```
이 스크린샷처럼 만들어줘.
근데 색상은 파란색 계열로, 폰트는 좀 더 둥글게.
```

Claude Code는 이미지를 읽을 수 있어. 스크린샷을 보고 비슷한 레이아웃을 만들어.

글로 "헤더가 있고, 왼쪽에 사이드바가 있고, 가운데에 카드가 3열로..."라고 설명하는 것보다 훨씬 정확해.

:::

## 규칙 파일로 맥락을 박아둬

레퍼런스를 매번 말로 주면 피곤해. 반복되는 규칙은 파일로 만들어두면 돼.

**CLAUDE.md** — 프로젝트의 규칙서야.

이 파일에 적어두면 Claude가 매번 읽어. 대화할 때마다 설명 안 해도 돼.

```markdown
# 프로젝트 규칙

## 기술 스택
- HTML + CSS + 바닐라 JS
- 외부 라이브러리 사용 금지

## 디자인 규칙
- 색상: #2563EB (파란), #F3F4F6 (배경)
- 폰트: 시스템 기본 폰트
- 모바일 우선

## 금지 사항
- React, Vue 등 프레임워크 사용 금지
- npm, 빌드 도구 사용 금지
```

이렇게 해두면 Claude가 React를 꺼내들지 않아.

"바닐라 JS로 해줘"를 매번 말할 필요 없어.

## "이거 아닌데"를 줄이는 팁

**구체적으로 말해.**

```
❌ "좀 예쁘게 해줘"
✅ "카드 사이 간격 넓히고, 그림자 추가해줘"
```

**하나만 바꿔.**

```
❌ "색도 바꾸고 레이아웃도 바꾸고 폰트도 바꿔줘"
✅ "배경색 #F9FAFB로 바꿔줘" → 확인 → "폰트 크기 16px로" → 확인
```

**안 할 것을 말해.**

```
❌ "로그인 페이지 만들어줘"
✅ "로그인 페이지 만들어줘. 소셜 로그인은 넣지 마. 이메일/비밀번호만."
```

AI는 시키지 않아도 기능을 붙여. "이건 하지 마"를 말하는 게 "이걸 해줘"보다 중요할 때가 있어.

---

## 한 줄 정리

스크린샷 1장이 설명 100줄보다 낫고, CLAUDE.md 한 번이면 매번 설명 안 해도 돼.
