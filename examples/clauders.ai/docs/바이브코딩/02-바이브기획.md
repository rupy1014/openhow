**코드부터 짜면 안 돼. 기획부터야.**

"AI 캐릭터 채팅 만들어줘"라고 하면 뭐가 나올까?

AI가 알아서 기술을 골라. 화면 구조도 정해. 기능도 넣어.

근데 그게 네가 원한 거야? 아마 아닐 거야.

---

## 왜 기획 없이 시작하면 망해?

실제로 해보면 이래:

```
나: "AI 캐릭터 채팅 서비스 만들어줘"
```

AI가 뭘 하냐:

- Next.js + TypeScript + Prisma + PostgreSQL을 깔아
- 로그인 시스템을 만들어
- 관리자 페이지를 만들어
- 다크모드를 넣어
- 1,500줄짜리 코드가 나와

너는 그냥 캐릭터 5개랑 대화하고 싶었을 뿐인데.

**AI가 네 대신 기획을 해버린 거야.** 네 의도랑 전혀 다른 방향으로.

---

## 제1원칙: "코드는 짜지 마"

이 한 줄이 전부야:

```
코드는 아직 짜지 마. 먼저 계획을 보여줘.
```

이 말을 먼저 하면:

- AI가 복잡한 기술을 안 골라
- 화면 구조를 네가 정할 수 있어
- 한 번에 수백 줄이 안 쏟아져

Peter Yang(바이브코딩 12원칙 저자)이 이렇게 말했어:

> "이 한마디가 90%의 과잉 설계를 막아준다."

계획을 보고, 마음에 드는 부분만 "이거 실행해줘"라고 하면 돼.

---

## 거창한 기획도 함정이야

여기서 많이들 실수해.

"기획부터 하라니까" → 로그인, 결제, 관리자 페이지, 알림, 통계... 10페이지짜리 기획서를 써.

이것도 똑같이 망해. 왜?

**만들기 전에는 뭐가 중요한지 모르거든.**

머릿속에선 "결제 기능이 필수야"라고 생각했는데, 실제로 만들어보니 "캐릭터 성격 설정"이 훨씬 중요할 수 있어.

안 만들어보면 몰라. 진짜야.

---

## 린 스타트업이 뭐야?

에릭 리스(Eric Ries)가 만든 개념이야. 핵심은 간단해:

> "가장 작은 걸 먼저 만들고, 써보고, 고쳐."

이걸 **만들기 → 써보기 → 고치기** 루프라고 해.

스타트업도 이렇게 해. 처음부터 완벽한 서비스 안 만들어. 핵심 하나만 빠르게 만들어서 사람들한테 보여주고, 반응 보고 방향을 잡아.

바이브코딩이랑 완벽하게 맞아. AI한테 시키면 30분이면 뭔가 돌아가거든. 10페이지 기획서 쓰는 동안 바이브코딩으로는 이미 3번 만들고 고칠 수 있어.

---

## MVP로 시작해

**MVP**(Minimum Viable Product) = 핵심 기능 딱 하나만 돌아가는 최소 버전.

린 스타트업의 핵심 도구야.

AI 캐릭터 채팅의 MVP는 뭘까?

- ❌ 로그인, 결제, 관리자 페이지, 다크모드
- ✅ **캐릭터 1개랑 대화할 수 있는 화면 하나**

이것만 있으면 돼. 나머지는 **써보고 나서** 붙여.

MVP의 기준은 하나야: **"이걸로 핵심 경험을 해볼 수 있어?"**

대화가 되면 OK. 로그인은 나중에. 결제는 더 나중에.

---

## 기획 = MVP PRD 한 장

그래서 기획이라고 거창한 게 아니야.

**PRD**(Product Requirements Document) = "내가 뭘 만들고 싶은지 정리한 문서".

A4 한 장이면 돼. 10장짜리 기획서가 아니야.

이걸 AI한테 먼저 만들어달라고 해.

```
나 AI 캐릭터 채팅 서비스 만들고 싶어.
MVP 기준으로 PRD.md 정리해줘. 코드는 아직 짜지 마.
핵심 기능 하나만 먼저 돌아가게 해줘.
```

포인트는 **"MVP 기준으로"**야. 이 말 하나가 AI의 과잉 기획을 막아줘.

AI가 코드 대신 문서를 내놓아.

그 문서를 읽고, 마음에 안 드는 건 바꾸라고 하면 돼.

**코드는 PRD가 확정된 다음에.**

---

## 같은 프로젝트, 세 가지 결과

| | 기획 없이 | 거창한 기획 | MVP 기획 |
|---|---|---|---|
| 첫 프롬프트 | "채팅 서비스 만들어줘" | "전체 기능 기획서 만들어줘" | "MVP PRD 만들어줘" |
| AI가 하는 일 | 기술 선택 + 코드 생성 | 10페이지 문서 작성 | 핵심 1개 정리 |
| 결과물 | 1,500줄 코드 | 안 만들어지는 기획서 | 바로 만들 수 있는 1장 |
| 첫 동작까지 | 모름 | 한참 | 30분 |
| 수정 비용 | 처음부터 다시 | 기획서부터 다시 | 고치면서 진행 |

**작게 만들고, 써보고, 고치는 게 이겨.**

---

## 한 줄 정리

"코드는 짜지 마"를 먼저 말하고, 기획은 MVP 한 장으로 시작해. 나머지는 만들면서 붙여.
