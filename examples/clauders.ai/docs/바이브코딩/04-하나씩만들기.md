**한 번에 다 만들지 마. 한 기능씩 만들고 확인해.**

"전체 서비스 만들어줘"라고 하면 뭐가 나올까?

코드가 수백 줄 쏟아져. 어디가 뭔지 모르겠어. 에러가 나도 어디를 고쳐야 하는지 모르겠어.

이걸 피하는 방법이 있어. **한 입씩 만들기.**

---

## 한 입씩이 뭐야?

케이크를 한 번에 삼키지 않잖아. 한 조각씩 먹지.

코드도 똑같아. 기능 하나를 끝까지 만들고, 확인하고, 다음으로 넘어가.

이걸 **수직 슬라이싱**이라고 해.

수직이 뭐냐? 화면부터 데이터까지, 한 기능을 위아래로 쭉 관통해서 끝내는 거야.

```
❌ 수평 슬라이싱 (전체를 한꺼번에)
화면 전부 만들기 → 기능 전부 연결 → 데이터 전부 처리

✅ 수직 슬라이싱 (기능별로 하나씩)
기능 A: 화면 + 기능 + 데이터 → 확인 ✅
기능 B: 화면 + 기능 + 데이터 → 확인 ✅
기능 C: 화면 + 기능 + 데이터 → 확인 ✅
```

## 어떻게 해?

:::steps

### PRD에서 기능 목록 뽑기

```
PRD.md 기반으로 구현 순서를 정해줘.
가장 핵심 기능부터 순서대로 목록으로.
한 번에 하나씩 만들 거야.
```

AI가 이런 목록을 줘:

```
1. 캐릭터 선택 화면 (카드 목록)
2. 채팅 화면 (말풍선 UI)
3. OpenRouter API 연결 (AI 대화)
4. 대화 히스토리 유지
5. 모바일 대응
```

### 1번부터 시작

```
1번 "캐릭터 선택 화면"부터 만들어줘.
나머지는 아직 하지 마.
```

"나머지는 아직 하지 마"가 핵심이야. 안 그러면 AI가 알아서 2, 3, 4까지 같이 만들어.

### 브라우저에서 확인

Claude가 알려주는 주소(보통 `http://localhost:숫자`)를 브라우저에서 열어봐.

됐으면 다음으로.

```
캐릭터 선택 화면 확인했어. 좋아.
2번 "채팅 화면" 만들어줘.
```

안 됐으면 여기서 고쳐.

```
카드 크기가 너무 작아. 좀 키워줘.
```

한 기능이 끝나야 다음으로 넘어가.

:::

## 왜 한 입씩이야?

**에러를 잡기 쉬워.**

전체를 한 번에 만들면 에러가 어디서 난 건지 모르겠어. 기능 하나만 만들면 에러가 그 안에 있어.

**되돌리기가 쉬워.**

기능 하나가 망가져도 그것만 되돌리면 돼. 전체를 날릴 필요 없어.

**방향 수정이 쉬워.**

2번까지 만들고 보니 "이거 아닌데?"

3, 4, 5를 아직 안 만들었으니까 방향을 틀기 쉬워.

전체를 다 만든 뒤에 방향을 틀면? 처음부터 다시 해야 해.

## 저장하고 정리하는 습관

기능 하나가 끝나면 두 가지를 해.

**1. 저장 (git commit)**

```text:Claude 프롬프트
지금까지 만든 거 git으로 저장해줘.
"캐릭터 선택 화면 완성"이라고 메시지 남기고.
```

왜 저장해? 다음 기능 만들다 망가지면, 이 시점으로 되돌아올 수 있으니까.

git이 뭔지 모르겠으면 [저장과 되돌리기](./03-5-저장과되돌리기)를 먼저 읽어봐.

**2. 정리 (문서화)**

저장한 다음에 이 한마디를 해봐:

```text:Claude 프롬프트
지금까지 만든 기능을 정리해줘.
뭘 만들었고, 어떤 파일이 뭘 하는지, 핵심 로직이 뭔지.
```

왜? AI는 대화가 길어지면 앞에서 한 걸 잊어.

근데 이렇게 정리해두면, 다음 기능을 만들 때 AI가 이 문서를 읽고 맥락을 파악해.

정리 안 하면? 3번째 기능쯤 되면 AI가 1번 기능이랑 충돌하는 코드를 짜. 뭘 만들었는지 까먹으니까.

```
기능 끝 → 저장 → 정리 → 다음 기능
```

이 리듬을 지켜.

## 한 입 크기는 얼마야?

**10분 안에 확인할 수 있는 크기.**

너무 크면 "전부 만들어줘"랑 다를 게 없어. 너무 작으면 진도가 안 나가.

| 크기가 맞아 | 크기가 커 |
|------------|----------|
| "로그인 폼 만들어줘" | "회원가입, 로그인, 비밀번호 찾기 만들어줘" |
| "상품 카드 하나 만들어줘" | "쇼핑몰 전체 만들어줘" |
| "다크모드 토글 추가해줘" | "디자인 전체 리뉴얼해줘" |

확인하고, 되면 다음. 안 되면 고쳐. 이 리듬이 바이브코딩의 핵심이야.

---

## 한 줄 정리

만들고, 확인하고, 저장하고, 정리하고, 다음. 이 리듬이 전부야.
