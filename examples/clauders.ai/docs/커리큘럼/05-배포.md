# PART 5: 배포 — 세상에 공개하기

> "내 컴퓨터에서만 보이는 서비스 → 전 세계 어디서나 접속 가능한 서비스로"

**⏱ 예상 시간**: 1시간
**🎯 목표**: `https://내이름.vercel.app` 주소 획득

---

## 개념 먼저: 배포가 뭐예요?


> Vercel이 24시간 서버를 대신 운영해줘요. 무료로!


**배포 3단계:**
```
내 코드 → GitHub (코드 저장소) → Vercel (자동 서버 운영)
```

왜 GitHub을 거치나요? → Vercel이 GitHub에서 코드를 가져다 자동으로 배포해줘요.
한 번 연결해두면 `git push` 할 때마다 자동으로 업데이트돼요!

---

## ✅ Task 5-1: GitHub 레포지토리 생성

1. `github.com` 접속 → 로그인
2. 오른쪽 상단 **`+`** → **New repository**
3. Repository name: `my-ai-chat`
4. **Private** 선택 (API 키 관련 코드니까 비공개로)
5. **Create repository** 클릭
6. 생성된 페이지에서 HTTPS 주소 복사 (예: `https://github.com/내이름/my-ai-chat.git`)

---

## ✅ Task 5-2: 코드를 GitHub에 올리기

Claude Code에 입력하세요 (GitHub URL을 내 것으로 교체):


:::tip 📋 Claude Code에 붙여넣기 (★ GitHub URL을 내 것으로 교체 ★)
이 프로젝트를 GitHub에 올려줘.

원격 저장소 URL: https://github.com/내이름/my-ai-chat.git
(← 위에서 복사한 내 GitHub URL로 교체)

주의사항:
- .env.local 파일은 절대 올리면 안 돼 (.gitignore 확인)
- node_modules도 올리지 마
커밋 메시지: "첫 배포: AI 캐릭터 채팅 서비스"
:::


터미널에서 GitHub 로그인을 요구하면 → GitHub 계정으로 로그인하세요.

**확인:**
GitHub에서 내 레포지토리 페이지를 새로고침하면 파일들이 올라와 있어야 해요.
(`.env.local` 파일은 보이면 안 돼요 — 정상입니다)

---

## ✅ Task 5-3: Vercel 배포

이제 Vercel에서 GitHub 레포지토리를 연결해요.

1. `vercel.com` 접속
2. **Sign Up** → **Continue with GitHub** (GitHub으로 로그인)
3. **Add New** → **Project**
4. GitHub 레포 목록에서 `my-ai-chat` 찾아서 **Import**


:::info
Import 화면에서 Environment Variables 섹션을 찾아서 펼치세요.

아래 내용을 추가하세요:

왜 이게 필요한가요? .env.local 파일은 GitHub에 올라가지 않아요.

  그래서 Vercel 서버에서 쓸 API 키를 여기서 별도로 알려줘야 해요.
:::


환경변수 추가 후 → **Deploy** 클릭!

배포 중... 1~3분 기다리세요. (초록색 진행 바가 채워지는 게 보여요)

---

## 배포 에러가 났을 때

**"Build Failed"** 화면이 나왔나요? 당황하지 마세요.

1. Vercel 대시보드 → 내 프로젝트 → **Deployments** 탭
2. 실패한 배포 클릭 → **Logs** 탭
3. 빨간 에러 메시지 전체 복사
4. Claude Code에 붙여넣기 + "Vercel 배포 에러야. 이 로그 보고 고쳐줘"
5. 수정 후 터미널에서:

```bash
git add .
git commit -m "배포 에러 수정"
git push
```

Vercel이 자동으로 다시 배포를 시작해요! (2~3분 기다리기)

## 이제부터는 자동 배포!

한 번 연결해두면, 앞으로는 이렇게만 하면 돼요:

```bash
# 코드 수정 후
git add .
git commit -m "변경사항 설명"
git push
```

→ Vercel이 자동으로 감지 → 자동으로 다시 배포 → 2~3분 후 업데이트 완료!
