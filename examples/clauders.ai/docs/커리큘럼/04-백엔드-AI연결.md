# PART 4: 백엔드 — AI 연결하기

> "화면은 만들었어요. 이제 진짜 AI와 대화할 수 있게 연결합니다."

**⏱ 예상 시간**: 1시간 30분
**🎯 목표**: OpenRouter를 통해 실제 AI가 채팅에 대답하도록 만들기

---

## 개념 먼저: API가 뭐예요?


> 식당에서 주문서를 쓰는 것처럼, 정해진 양식이 있어요:


### 왜 중간에 우리 서버가 필요한가?

```
❌ 직접 연결하면:
채팅 화면 → OpenRouter API 키가 브라우저에 노출 → 해킹 위험!

✅ 우리 서버를 거치면:
채팅 화면 → [우리 API 서버] → OpenRouter → AI 응답
                       ↑
              API 키는 서버에서만 관리. 화면에 안 보임!
```

Next.js의 **API Routes**가 이 중간 서버 역할을 해줘요.
`app/api/chat/route.ts` 파일을 만들면 자동으로 `/api/chat` 주소가 생겨요.

---

## ✅ Task 4-1: API 라우트 만들기

Claude Code에 다음을 붙여넣기하세요. **시스템 프롬프트 부분을 내 캐릭터로 수정하세요!**


:::tip 📋 Claude Code에 붙여넣기 (★ 시스템 프롬프트를 내 캐릭터로 수정 ★)
app/api/chat/route.ts 파일을 새로 만들어줘.

역할:
- POST 요청을 받는 Next.js API Route Handler
- 요청 body: { messages: Array&lt;{role: string, content: string}&gt; }
- OpenRouter API 호출 후 AI 응답 텍스트 반환

OpenRouter 설정:
- Base URL: https://openrouter.ai/api/v1/chat/completions
- Model: openai/gpt-4o-mini
- API 키: 환경변수 OPENROUTER_API_KEY에서 읽기
- Headers에 HTTP-Referer와 X-Title도 포함

시스템 프롬프트 (이 부분을 내 캐릭터로 교체):
"당신은 도비입니다. Harry Potter에 나오는 충직한 하우스엘프예요.
다음 규칙을 반드시 지키세요:
1. 항상 3인칭으로 '도비'라고 자신을 지칭해요 ('저'는 절대 사용 금지)
2. 문장 마지막에 '주인님'을 붙여요
3. 도움을 주지 못하면 자책해요 ('도비가 못났습니다!')
4. 극도로 공손하고 헌신적으로 행동해요
5. 절대 캐릭터에서 벗어나지 마세요"

에러 처리:
- OPENROUTER_API_KEY 없으면 500 에러
- OpenRouter 호출 실패 시 에러 메시지 포함한 500 응답
- TypeScript 타입 안전하게 작성
:::

## ✅ Task 4-2: 환경변수 설정

API 키를 코드에 직접 쓰면 위험해요. `.env.local` 파일에 저장해야 해요.

Claude Code에 입력:
```
프로젝트 루트에 .env.local 파일을 만들어줘.
내용:
OPENROUTER_API_KEY=여기에_실제_키_넣기
```

그런 다음 **VS Code**에서 `.env.local` 파일을 열어서,
`여기에_실제_키_넣기` 부분을 PART 1에서 저장해둔 OpenRouter 키로 교체하세요:

```
OPENROUTER_API_KEY=sk-or-v1-xxxxxxxxxxxx...
```


:::info
= API 키 같은 비밀 설정을 저장하는 금고 파일이에요.

자동으로 .gitignore에 추가되어서 GitHub에 올라가지 않아요.

나중에 배포할 때는 Vercel에서 별도로 입력할 거예요 (PART 5에서).
:::

## ✅ Task 4-3: 채팅 화면 → API 연결

이제 채팅 화면에서 실제 API를 호출하도록 연결할 거예요.

Claude Code에 입력:


:::tip 📋 Claude Code에 붙여넣기
app/page.tsx를 수정해줘.

지금 전송 버튼은 "..." 로딩만 보이고 끝나.
이걸 실제 API 호출로 바꿔줘:

전송 버튼 클릭 시 흐름:
1. 내 메시지를 messages에 추가
2. 입력창 초기화
3. isLoading = true → "..." 로딩 말풍선 표시
4. /api/chat으로 POST 요청
   - body: { messages: 전체 대화 내역 }
5. 응답 받으면:
   - AI 메시지를 messages에 추가
   - isLoading = false
6. 에러 처리:
   - fetch 실패 시 "죄송합니다. 오류가 발생했어요." 메시지 표시
   - isLoading = false

UX 개선:
- 전송 중에는 버튼/입력창 비활성화 (disabled)
- 새 메시지 오면 자동으로 스크롤 맨 아래로
- Enter로도 전송 가능 (Shift+Enter는 줄바꿈)
:::
