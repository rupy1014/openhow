**에이전트 패턴은 세 가지다. 직렬, 병렬, 대립.**

조합하면 웬만한 작업은 자동화된다.

---

## 직렬은 언제 쓰냐

앞 에이전트 결과가 다음 에이전트 입력이 될 때. 순서가 중요하다.

**자기소개서 작성**

```
> 내 경력 파일 보고 자기소개서 써줘.

[profiler]     경력 읽기 → 핵심 강점 3개 추출
    ↓
[storyteller]  강점 기반 스토리 초안 작성
    ↓
[tone-editor]  문체 다듬기 — 공문투 제거
    ↓
[fit-checker]  JD와 매칭 검수 → 88점 통과

완료. cover-letter.md 저장
```

앞 단계 결과가 없으면 다음 단계를 시작할 수 없다. 순서를 바꾸면 안 된다.

---

## 병렬은 언제 쓰냐

서로 안 기다려도 되는 작업일 때. 5개를 순서대로 하면 5배 걸린다. 동시에 하면 1배다.

**SNS 포스팅 5개 동시 생산**

```
> 이 제품 SNS 포스팅 5개 동시에 써줘. 각각 다른 방향으로.

[에이전트-1] 기능 소개 포스팅 작성 중...
[에이전트-2] 고객 후기 포스팅 작성 중...
[에이전트-3] 비하인드 스토리 포스팅 작성 중...
[에이전트-4] 수치·데이터 포스팅 작성 중...
[에이전트-5] 질문형 훅 포스팅 작성 중...

완료. posts/sns-01~05.md 저장
```

**시장조사 → 통합 분석**

조사는 병렬, 분석은 직렬. 두 패턴을 이어 붙인다.

```
> 국내 SaaS 시장 3개 섹터 조사하고 통합 분석해줘.

[researcher-1] HR SaaS 조사 중...
[researcher-2] 회계·세무 SaaS 조사 중...    ← 3개 동시
[researcher-3] 마케팅 SaaS 조사 중...
    ↓
[analyst]      3개 섹터 데이터 통합 분석 중...

완료. analysis/market-report.md 저장
```

---

## 대립은 언제 쓰냐

한쪽만 보면 편향된다. 찬성·반대를 동시에 돌리고 합쳐서 균형 잡힌 판단을 낸다.

**사업 기획서 검토**

```
> 이 기획서 다각도로 검토해줘.

[blue-team]   강점·기회 분석 중...
[red-team]    약점·리스크 분석 중...    ← 동시
    ↓
[synthesizer] 양측 의견 통합 → 최종 판단 작성

완료. review/decision-report.md 저장
```

**코드 PR 리뷰**

```
> 이 PR 다각도로 리뷰해줘.

[security-reviewer]    보안 취약점 검토 중...
[performance-reviewer] 성능 병목 검토 중...    ← 동시
[style-reviewer]       코드 스타일 검토 중...
    ↓
[report-writer]        리뷰 통합 리포트 작성

완료. review/pr-123.md 저장
```

---

## 한 줄 정리

직렬은 순서, 병렬은 속도, 대립은 균형. 세 패턴을 조합하면 복잡한 작업도 에이전트로 설계할 수 있다.
