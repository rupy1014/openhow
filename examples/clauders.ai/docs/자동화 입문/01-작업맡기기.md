**구조가 정해진 작업은 Claude에게 맡기면 된다.**

블로그 연재가 딱 그 예시다. 목차 잡고, 회차별로 쓰고, 퇴고하는 패턴 — 이걸 Claude가 맡는다.

---

## 어떻게 세팅해?

**Step 1.** `my-blog` 폴더에서 `dsclaude` 실행

```bash
dsclaude
```

**Step 2.** 아래 세팅 파일을 복사해서 붙여넣는다

:::copy-embed _embeds/setup-blog 블로그 연재 자동화 세팅 파일
:::

처음엔 뭘 보내는 건지 다 읽기 어려울 수 있어. 읽지 않아도 돼.
통째로 복사해서 붙여넣으면 Claude가 파일 구조를 알아서 만든다.

---

## 뭐가 만들어져?

```
my-blog/
├── CLAUDE.md                     ← 블로그 규칙 (말투, 분량, 독자)
├── series-plan.md                ← 연재 목차 (기획 후 채워짐)
├── docs/
│   └── tone-and-manner.md        ← 글쓰기 스타일 기준
├── posts/                        ← 완성된 글
├── drafts/                       ← 작업 중인 초안
├── reviews/                      ← 퇴고 기록
└── .claude/
    ├── commands/
    │   ├── plan.md               ← /plan 명령어
    │   ├── write.md              ← /write 명령어 (파이프라인 실행)
    │   └── review.md             ← /review 명령어
    └── agents/
        ├── series-manager.md     ← 시리즈 컨텍스트 파악
        ├── outline-writer.md     ← 회차 개요 설계
        ├── writer.md             ← 초안 작성
        ├── style-touch.md        ← 문체 다듬기
        └── proofreader.md        ← 기준 검수
```

왜 이렇게 파일을 쪼개놨냐?

한 파일에 규칙, 목차, 말투 기준을 다 넣으면 Claude가 대화가 길어질수록 앞부분을 잊어버려. 역할별로 파일을 나누면 각 담당자가 자기한테 필요한 것만 읽고 집중한다. 사람도 메모 한 장에 열 가지 적어놓으면 헷갈리는 거랑 같아.

**commands/** — `/명령어`로 직접 실행하는 단축키 파일이다.

**agents/** — commands가 순서대로 부르는 담당자 파일이다. 각 담당자는 이름, 역할 설명, 어떤 AI를 쓸지가 적혀있고, 자기 몫만 처리한다.

담당자는 Claude를 시작할 때 자동으로 불러온다. 파일을 추가했는데 안 보이면 `/agents`를 치면 바로 인식돼.

| 명령어 | 하는 일 |
|--------|---------|
| `/plan` | 시리즈 제목·편 수를 받아 전체 목차를 `series-plan.md`에 저장 |
| `/write` | 목차 기반으로 해당 회차를 series-manager → outline-writer → writer → style-touch → proofreader 순서로 처리 |
| `/review` | 완성된 글을 발행 전 최종 검토 |

---

## 세팅 끝나면 뭘 해?

**Step 1.** `CLAUDE.md`에서 블로그 이름, 주제, 누구한테 쓰는 글인지를 실제 내용으로 채운다

**Step 2.** `docs/tone-and-manner.md`를 내 말투로 고친다

금지 표현, 글 시작 방식, 말투 기준을 실제 내가 쓰는 방식으로 바꿔야 `/write`가 제대로 작동한다. 이 두 파일이 모든 담당자의 기준이 된다.

**Step 3.** 첫 기획 실행

```
/plan "카페 창업 이야기" 10편
```

세팅 한 번으로 환경이 잡힌다. 그 다음부터는 `/write 3화` 한 줄이면 된다.
