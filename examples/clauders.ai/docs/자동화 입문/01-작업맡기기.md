**구조가 정해진 작업은 Claude에게 맡기면 돼.**

블로그 연재가 딱 그 예시야. 목차 잡고, 회차별로 쓰고, 퇴고하는 패턴 — 이걸 Claude가 맡아.

---

## 어떻게 세팅해?

**Step 1.** `my-blog` 폴더를 만들고 VS Code로 열어

바탕화면에 `my-blog` 폴더를 만들어 → VS Code → `File` → `Open Folder` → 선택

**Step 2.** 터미널에서 `dsclaude` 실행

```bash
dsclaude
```

**Step 3.** 아래 세팅 파일을 복사해서 붙여넣어

:::copy-embed _embeds/setup-blog 블로그 연재 자동화 세팅 파일
:::

처음엔 뭘 보내는 건지 다 읽기 어려울 수 있어. 읽지 않아도 돼.
통째로 복사해서 붙여넣으면 Claude가 파일 구조를 알아서 만들어.

## 뭐가 만들어져?

```
my-blog/
├── CLAUDE.md                     ← 블로그 규칙 (말투, 분량, 독자)
├── series-plan.md                ← 연재 목차 (기획 후 채워짐)
├── docs/
│   └── tone-and-manner.md        ← 글쓰기 스타일 기준
├── posts/                        ← 완성된 글
├── drafts/                       ← 작업 중인 초안
├── reviews/                      ← 퇴고 기록
└── .claude/
    ├── commands/
    │   ├── write.md              ← /write 명령어 (파이프라인 실행)
    │   └── review.md             ← /review 명령어
    └── agents/
        ├── series-manager.md     ← 시리즈 컨텍스트 파악
        ├── outline-writer.md     ← 회차 개요 설계
        ├── writer.md             ← 초안 작성
        ├── style-touch.md        ← 문체 다듬기
        └── proofreader.md        ← 기준 검수
```

왜 이렇게 파일을 쪼개놨냐?

Claude는 대화가 길어질수록 앞에서 한 말을 잊어. 이걸 **기억 한계**라고 해. 사람이 메모 한 장에 열 가지 적어놓으면 헷갈리는 거랑 같아. Claude도 한 대화에 너무 많은 걸 넣으면 앞부분이 흐려져.

그래서 **팀장-담당자 구조**로 나누는 거야.

회사 팀장을 생각해봐. 좋은 팀장은 기획서를 직접 쓰고, 디자인도 하고, 검수까지 혼자 하지 않아. 방향을 잡고, 적절한 사람한테 일을 넘기고, 결과를 확인해.

Claude도 똑같아. 우리가 대화하는 Claude가 **팀장**이야.

- **팀장** — 나랑 대화하면서 방향을 잡고, 맥락을 유지해. "3화 분위기는 좀 더 밝게"라고 하면 기억하고 있어.
- **담당자(agents)** — 실제 작업을 해. 초안 쓰기, 문체 다듬기, 검수 — 각자 맡은 것만 집중해.

팀장이 직접 다 하면 기획하다가 말투 기준을 잊고, 검수하다가 원래 방향을 놓쳐. 담당자한테 나눠주면 각자 깨끗한 상태에서 자기 일만 해.

**commands/** — `/명령어`로 직접 실행하는 단축키 파일이야.

**agents/** — commands가 순서대로 부르는 담당자 파일이야. 각 담당자는 이름, 역할 설명, 어떤 AI를 쓸지가 적혀있고, 자기 몫만 처리해.

담당자는 Claude를 시작할 때 자동으로 불러와. 파일을 추가했는데 안 보이면 `/agents`를 치면 바로 인식돼.

| 명령어 | 하는 일 |
|--------|---------|
| `/write` | 목차 기반으로 해당 회차를 series-manager → outline-writer → writer → style-touch → proofreader 순서로 처리 |
| `/review` | 완성된 글을 발행 전 최종 검토 |

## 세팅 끝나면 뭘 해?

**Step 1.** `CLAUDE.md`에서 블로그 이름, 주제, 누구한테 쓰는 글인지를 실제 내용으로 채워

**Step 2.** `docs/tone-and-manner.md`를 내 말투로 고쳐

금지 표현, 글 시작 방식, 말투 기준을 실제 내가 쓰는 방식으로 바꿔야 `/write`가 제대로 작동해. 이 두 파일이 모든 담당자의 기준이 돼.

**Step 3.** Claude랑 대화하면서 기획을 잡아

```
> 나 카페 창업 블로그 쓰고 싶은데, 어떻게 구성하면 좋을까?

10편 정도면 좋겠어. 이런 구성은 어때?
1화: 왜 카페를 하려는지
2화: 상권 분석
3화: 인테리어 콘셉트
...

> 좋아. 근데 2화랑 3화 순서 바꾸자. 인테리어부터 보여주는 게 재밌을 것 같아.

알겠어. 수정해서 목차 정리해둘게.

> 응.
```

이렇게 대화하면서 기획이 잡혀. 그냥 말하면 Claude가 같이 기획해줘.

**Step 4.** 1화 실행

```
/write 1화
```

## 한 줄 정리

세팅 한 번으로 환경이 잡혀. 기획은 대화로, 실행은 `/write 3화` 한 줄이면 돼.
