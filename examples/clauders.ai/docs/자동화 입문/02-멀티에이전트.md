`/write 3화`를 치면 한 명이 아니라 다섯 명이 일한다.

---

## 에이전트가 뭐냐

앞 글에서 만든 `.claude/agents/` 안의 파일들이다.

각각 역할이 고정된 작업자다. `model`, `description`이 지정돼 있고, `/write`가 실행되면 순서대로 넘겨받아 처리한다.

```
/write 3화

series-manager   → 3화 컨텍스트 파악 (series-plan.md + 직전 편 읽기)
    ↓
outline-writer   → 서두 훅 2개 + 본문 섹션 설계
    ↓
writer           → 초안 작성 → drafts/03-초안.md 저장
    ↓
style-touch      → 금지 표현 제거, 문장 다듬기
    ↓
proofreader      → 검수 80점 이상 → posts/03-제목.md 저장
```

각 에이전트는 `CLAUDE.md`, `docs/tone-and-manner.md`, `series-plan.md`를 공통으로 참조한다.

---

## 왜 나눠놨냐

우리가 대화하는 Claude가 마스터 에이전트다. `/write`를 치면 마스터가 직접 글을 쓰는 게 아니라 담당자들에게 일을 넘긴다.

담당자(sub-agent)는 자기 역할만 처리하고 끝낸다. 마스터의 컨텍스트는 오염되지 않는다.

마스터가 직접 다 하면 기획, 초안, 퇴고가 한 대화에 뒤섞인다. 담당자에게 시키면 각자 깨끗한 컨텍스트로 집중한다.

80점 미만이면 `proofreader`가 결과를 마스터에게 돌려준다. 마스터가 `style-touch`를 다시 실행한다. 담당자끼리 직접 소통하지 않는다 — 항상 마스터를 거친다.

---

## 한 줄 정리

`/write`는 다섯 에이전트가 순서대로 처리하는 파이프라인이다. 기준 미달이면 자동으로 재실행된다.
