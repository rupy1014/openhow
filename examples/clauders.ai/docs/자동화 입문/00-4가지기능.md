**Claude Code에는 커스터마이징 레이어가 네 가지다.**

이 네 개를 알고 조합하면, 나에게 맞는 자동화 환경을 직접 설계하고 계속 개선해나갈 수 있다.

---

## 네 가지가 뭐야?

| 기능 | 위치 | 하는 일 |
|------|------|---------|
| **commands** | `.claude/commands/` | `/명령어` 하나로 워크플로우 실행 |
| **agents** | `.claude/agents/` | 역할 고정된 서브 에이전트 |
| **skills** | `.claude/skills/` | Claude가 스스로 판단해 꺼내 쓰는 행동 패키지 |
| **hooks** | `settings.json` | 이벤트 발생 시 자동 실행되는 트리거 |

---

## 서로 어떻게 달라?

**commands** — 내가 직접 부른다.
`/write`를 치는 순간 실행된다. 복잡한 워크플로우를 한 줄로 묶어둔 단축키야.

**agents** — commands가 부르는 담당자들이야.
각자 역할이 고정돼 있고, 독립된 컨텍스트에서 자기 몫만 처리하고 끝낸다.

**skills** — Claude가 상황 보고 스스로 꺼낸다.
내가 부를 필요 없어. "이 작업에 이 skill이 맞겠다"고 판단하면 자동으로 적용돼.
commands는 내가 실행, skills는 Claude가 실행.

**hooks** — 이벤트에 붙어있다.
파일이 수정된 후, 명령어 실행 전, 대화가 끝날 때 — 정해둔 스크립트가 자동으로 돌아간다.
Claude가 모르는 사이에 실행된다.

---

## 왜 배워야 해?

Anthropic이 설계한 구조인데, 다른 AI 코딩 도구들도 비슷한 레이어로 수렴하고 있어.
Codex CLI도, Gemini CLI도 같은 방향이야.

이 네 개를 한 번 익히면, 어떤 AI 도구를 써도 "어떻게 설정하면 되겠구나"가 보인다.

그리고 이게 진짜 차이를 만드는 지점이야.

처음 설정 그대로 쓰는 사람과, 계속 개선해나가는 사람은 6개월 뒤 출력물이 다르다.
바이브코딩할 때, 업무 자동화할 때 — 같은 Claude인데 쓰는 방식이 달라지면 효율이 확 벌어진다.

---

## 이 섹션에서 뭘 배워?

commands + agents로 블로그 연재 자동화 환경을 직접 만든다.
`/write 3화` 한 줄이면 다섯 에이전트가 순서대로 돌아가는 파이프라인.

skills와 hooks는 어떤 패턴에서 쓰이는지 사례로 보여준다.

commands, agents, skills, hooks — 이 네 개가 Claude Code 커스터마이징의 전부다.
