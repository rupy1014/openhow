**AI는 패턴이 있는 반복 작업을 잘한다.**

구조가 정해진 작업은 한 번 세팅해두면 그다음부터는 명령어 하나로 끝난다.

블로그 연재가 딱 그 예시다. 목차 잡고, 회차별로 쓰고, 퇴고하는 패턴 — 이걸 Claude가 맡는다.

---

## 세팅

블로그 연재 환경을 통째로 잡아주는 프롬프트다. `dsclaude` 치고, 아래 내용을 붙여넣으면 된다.

````
아래 파일들을 생성해줘. 내 블로그 연재 자동화 환경 설정 파일들이야. 각 파일의 내용을 그대로 만들어줘. 경로는 현재 프로젝트 루트 기준이야.

---

## 파일 1: `CLAUDE.md`

```markdown
# 블로그 프로젝트 규칙

## 프로젝트 개요

[블로그 이름, 주제, 타깃 독자 여기 작성]

## 글쓰기 원칙

- 톤앤매너: `docs/tone-and-manner.md` 참조
- 독자 수준: 비개발자, 실무 중심
- 서론 없이 바로 본론으로
- 글 길이: 800~1500자

## 파일 구조

- `series-plan.md` — 전체 연재 기획
- `docs/tone-and-manner.md` — 말투와 스타일 기준
- `posts/` — 완성된 글
- `drafts/` — 작업 중인 초안
- `reviews/` — 퇴고 기록
```

## 파일 2: `docs/tone-and-manner.md`

```markdown
# 톤앤매너 가이드

## 말투
- 구어체. 독자에게 직접 말하는 느낌.
- "저는", "제가" 대신 "나는", "내가" — 편안한 거리감.

## 문체 원칙
- 짧은 문장 우선. 한 문장에 한 가지만.
- 전문용어는 첫 등장 시 괄호로 설명.

## 금지 표현
- "상기와 같이", "전술한 바와 같이" — 공문투 금지
- "~하겠습니다" — 과도한 경어 금지
- "~인 것 같습니다" — 불필요한 완화 표현 금지

## 서두
- 질문으로 시작하거나, 결론부터 먼저.
- "안녕하세요, N번째 글입니다" 형식 금지.

## 마무리
- 다음 편 예고 한 줄 (있으면).
- 독자 행동 유도 한 줄 (있으면).
```

## 파일 3: `.claude/skills/plan/SKILL.md`

```
---
name: plan
description: 블로그 연재 기획 에이전트. 시리즈 주제와 편 수를 받아 전체 목차와 회차별 개요를 작성. "/plan [시리즈 제목] [N편]"으로 호출.
user-invocable: true
allowed-tools: "Read, Write, AskUserQuestion"
---
# 시리즈 기획 (/plan)

1. CLAUDE.md 읽기 — 프로젝트 개요, 독자 수준 파악
2. 기존 series-plan.md 있으면 읽기 (업데이트 모드)
3. 시리즈 구조 설계: 도입 → 심화 → 마무리, 각 회차 제목·핵심 포인트 2~3개
4. series-plan.md 저장
```

## 파일 4: `.claude/skills/write/SKILL.md`

```
---
name: write
description: 블로그 포스트 작성 에이전트. series-plan.md의 해당 회차 개요 기반으로 초안 작성. "/write [N화]"로 호출.
user-invocable: true
allowed-tools: "Read, Write, Glob"
---
# 포스트 작성 (/write)

1. series-plan.md 읽기 → 해당 회차 개요 파악
2. docs/tone-and-manner.md 읽기
3. posts/ 안 기존 글 1~2개 읽기 (문체 일관성 확인)
4. 초안 작성 — 금지 표현 즉시 체크
5. drafts/0N-[제목]-draft.md 저장
```

## 파일 5: `.claude/skills/review/SKILL.md`

```
---
name: review
description: 블로그 퇴고 에이전트. 초안을 docs/tone-and-manner.md 기준으로 검토하고 수정본을 posts/에 저장. "/review [N화]"로 호출.
user-invocable: true
allowed-tools: "Read, Write, Edit, Glob"
---
# 퇴고 (/review)

1. drafts/0N-*-draft.md 읽기
2. docs/tone-and-manner.md 읽기
3. 검토: 톤앤매너 / 서두 / 흐름 / 독자 수준 / 분량(800~1500자)
4. 리뷰 기록 → reviews/review-0N.md
5. 수정 적용 → posts/0N-[제목].md 저장
```

---

이제 위 파일들을 모두 생성해줘. 생성 후:
1. `posts/`, `drafts/`, `reviews/` 폴더 생성 (각각 `.gitkeep` 파일 하나씩)
2. `series-plan.md` — 빈 파일로 생성
3. 어떤 파일을 만들었는지 목록만 간단히 알려줘.
````

## 세팅 결과

붙여넣으면 이런 파일 구조가 만들어진다.

```
my-blog/
├── CLAUDE.md               ← 블로그 규칙 (말투, 분량, 독자 수준)
├── series-plan.md          ← 연재 목차 (기획 후 채워짐)
├── docs/
│   └── docs/tone-and-manner.md  ← 글쓰기 스타일 기준
├── posts/                  ← 완성된 글
├── drafts/                 ← 작업 중인 초안
├── reviews/                ← 퇴고 기록
└── .claude/
    └── skills/
        ├── plan/SKILL.md   ← /plan 명령어 정의
        ├── write/SKILL.md  ← /write 명령어 정의
        └── review/SKILL.md ← /review 명령어 정의
```

`.claude/skills/` 안에 있는 게 **스킬(skills)**이다. `dsclaude` 안에서 `/명령어` 형태로 쓸 수 있는 작업 단위다. 이 세팅에서 만들어지는 세 개는 이렇다.

| 명령어 | 하는 일 |
|--------|---------|
| `/plan` | 시리즈 제목과 편 수를 받아 전체 목차를 `series-plan.md`에 저장 |
| `/write` | 목차 기반으로 해당 회차 초안을 `drafts/`에 저장 |
| `/review` | 초안을 톤앤매너 기준으로 퇴고해 `posts/`에 저장 |

스킬 외에 `.claude/` 안에 들어갈 수 있는 것들이 더 있다.

**commands** — `/명령어`와 같지만 마크다운 파일 하나로 정의한다. 스킬보다 간단한 작업에 쓴다.

**agents** — 독립적으로 작업을 수행하는 서브에이전트. 긴 작업을 병렬로 돌리거나, 역할을 분리할 때 쓴다.

**hooks** — 특정 이벤트(파일 저장, 작업 완료 등)에 자동으로 실행되는 스크립트. 저장할 때마다 포맷 맞추기, 완료 시 검증 실행 같은 자동화에 쓴다.

---

## 내 스타일로 고치기

세팅이 완료되면 두 파일을 먼저 열어서 수정한다.

**`CLAUDE.md`** — 블로그 이름, 주제, 타깃 독자를 실제 내용으로 채운다. 글 길이 기준도 내 스타일에 맞게 바꾼다.

**`docs/tone-and-manner.md`** — 기본 가이드가 들어있지만 내 말투가 아니다. 금지 표현, 서두 패턴, 말투 기준을 실제로 내가 쓰는 방식으로 고쳐야 제대로 작동한다.

이 두 파일이 `/write`와 `/review`의 기준이 된다. 대충 두면 대충 나온다.

---

## 한 줄 정리

붙여넣기 한 번으로 환경이 잡힌다. CLAUDE.md와 docs/tone-and-manner.md만 내 스타일로 고치면 된다.
