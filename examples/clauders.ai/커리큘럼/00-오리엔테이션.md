## 바이브코딩이란?

<div style="background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); border-radius: 16px; padding: 3px; margin: 2em 0;">
<div style="background: #1a1a2e; border-radius: 14px; padding: 28px 32px; color: #e0e0e0; line-height: 1.7;">
  <h3 style="margin: 0 0 20px 0; color: #fff; font-size: 1.1em;">바이브코딩 = "AI에게 코드를 시키는 것"</h3>
  <p style="margin: 0 0 16px 0;"><strong style="color: #f87171;">기존 방식:</strong> 파이썬/자바스크립트 문법 수백 시간 공부 → 3년 후 첫 서비스</p>
  <p style="margin: 0 0 16px 0;"><strong style="color: #4ade80;">바이브코딩:</strong> AI에게 "이거 만들어줘" → 8시간 후 첫 서비스</p>
  <p style="margin: 0 0 0 0; color: #94a3b8; font-size: 0.95em;">Andrej Karpathy (OpenAI 공동창업자)가 2025년에 명명한 개념.<br>
  "코드를 직접 쓰지 않고, AI와 대화하면서 소프트웨어를 만드는 것."</p>
</div>
</div>

### 레스토랑 사장님 비유

바이브코딩을 한 마디로 표현하면:

> **"레스토랑 사장님처럼 일하는 것"**

레스토랑 사장님은 요리를 직접 하지 않아요.
- 메뉴를 기획하고
- 주방장(AI)에게 "오늘 메뉴는 이렇게 만들어줘"라고 말하고
- 나온 음식을 맛보고 "이건 좀 더 맵게, 저건 플레이팅 다시"라고 피드백하죠.

**코드를 아는 것 ≠ 서비스를 만들 수 있는 것**
**서비스를 만드는 능력 = 무엇을 만들지 알고 + AI에게 제대로 시키는 것**
