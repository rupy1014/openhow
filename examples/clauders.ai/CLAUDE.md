# clauders.ai — 콘텐츠 가이드

## 톤앤매너 참조

글을 쓸 때는 반드시 `rules/tone.md`를 읽고 따른다.

핵심 요약:
- 말투: 평어 (~이야, ~해, ~거야). 존댓말 금지.
- 인칭: 나 / 너. "여러분" 금지.
- 문장: 15자 이내, 1~3문장 후 빈 줄.
- 시작: 서론 없이 결론 또는 질문으로 바로.
- 제목: 질문형 구어체. "개요", "방법" 같은 레이블 금지.

---

## 철학: 원포인트 아티클

**하나의 아티클 = 하나의 포인트.**

강의를 쪼갠 게 아니라, 처음부터 하나를 위해 설계된 콘텐츠.
스낵처럼 30초~2분 안에 소비되고, 하나만 기억해도 충분해.

---

## 말투: 구어체

모든 콘텐츠는 **구어체(~해, ~돼, ~야, ~거야)**로 써. 친구한테 말하듯이.

미사여구 없이 정보만 전달해.

| 금지 | 대체 |
|------|------|
| ~습니다, ~합니다 | ~해, ~돼 |
| ~됩니다, ~입니다 | ~돼, ~야 |
| ~한다, ~된다, ~이다 (서술체) | ~해, ~돼, ~야, ~거야 |
| ~해요, ~예요 | ~해, ~야 |
| ~해주세요, ~하세요 | ~해봐 |

```
❌ "스크립트 한 줄 붙여넣으시면 설치됩니다."
❌ "스크립트 한 줄 붙여넣으면 설치된다."
✅ "스크립트 한 줄 붙여넣으면 설치돼."
```

---

## 구조: 결과 → 과정 → 라이브 → 다시

### 1. 결과 보여주기

완성본 먼저. 서론 없음.

```
❌ "이번 시간에는 Claude Code의 커스텀 커맨드를 배워보겠습니다."
✅ [스크린샷/코드 출력] "한 줄로 블로그 글이 나왔어."
```

- 이미지, 코드 출력, 터미널 결과 중 하나
- 텍스트만으로 설명 가능하면 굵은 한 문장
- **3줄 이내**

---

### 2. 과정 설명

결과가 어떻게 나왔는지 단계별로.

```
Step 1. ...
Step 2. ...
Step 3. ...
```

- 화면 캡처 + 한 줄 설명 조합이 이상적
- 핵심 단계만, **최대 5단계**
- 각 단계는 **1~2줄**

---

### 3. 라이브

"지금 바로 해보세요" 포인트.

- 실행 가능한 최소 코드 블록
- 복사해서 바로 쓸 수 있는 프롬프트 / 커맨드
- 환경 설정이 필요하면 한 줄로 링크만

---

### 4. 다시 설명

핵심 한 문장 + 다음이 궁금해지는 암시.

```
✅ "커스텀 커맨드가 있으면 매번 설명 안 해도 돼. 다음은 이걸 자동화하는 방법."
```

- **2줄 이내**
- 요약이 아니라 "한 줄 기억"
- 다음 아티클로 자연스럽게 이어지면 좋음

---

## 분량 기준

| 구간 | 목표 분량 |
|------|----------|
| 결과 | 3줄 이내 |
| 과정 | 단계 × 1~2줄, 최대 5단계 |
| 라이브 | 코드 블록 1~2개 |
| 다시 | 2줄 이내 |
| **전체** | **짧을수록 좋지만, 내용이 필요하면 넘어가도 돼** |

## 파일 네이밍

```
{그룹}-{회차}-{주제}.md

예:
강의-1회차-커스텀커맨드.md   ← 강의 1회차의 주제
강의-1회차-CLAUDE파일.md     ← 강의 1회차의 다른 주제
강의-2회차-멀티에이전트.md   ← 강의 2회차의 주제
숙제-1회차-실습.md           ← 숙제
특강-콘텐츠자동화-파이프라인.md  ← 특강 (회차 대신 대주제)
```

**구조 원칙:**
- `{그룹}` → 사이드바 1단 그룹 (강의 / 숙제 / 특강)
- `{회차}` → 사이드바 2단 서브그룹 (1회차 / 2회차 / ...)
- `{주제}` → 아티클 제목 (자유롭게)

회차 없이 주제만 있으면 서브그룹 없이 그룹 바로 아래에 위치.

## 아티클 템플릿

### frontmatter 규칙

- `title` — 주제 이름만. 파일명처럼 짧게. 설명 붙이지 말 것.
- `nav` — 사이드바에 표시되는 이름. title과 같거나 더 짧게.

```
❌ title: "프로젝트 = 폴더. Claude는 이 안에서만 일한다."
✅ title: "프로젝트"

❌ title: "커스텀 커맨드로 블로그 글 자동 생성하기"
✅ title: "커스텀 커맨드"
```

### 섹션 구조

```markdown
---
slug: {그룹}-{회차}-{주제}
title: "{주제 이름}"
nav: "{사이드바 짧은 이름}"
order: {숫자}
---

{결과물 이미지 or 코드 출력 or 굵은 한 문장}

---

## {질문형 제목 — 어떻게 ~하냐}

**Step 1.** ...

**Step 2.** 코드가 필요하면 Step 안에 바로 넣는다

```bash
...
```

**Step 3.** ...

---

## {질문형 제목 — 뭐가 ~냐 / 왜 ~냐}

단계 밖에서 설명할 내용. 없으면 생략.

---

## 한 줄 정리

{핵심 한 문장. 다음이 궁금해지게.}
```

### 섹션 제목 규칙

고정 제목을 쓰지 않아. 주제에 맞는 **구어체 질문형**으로 써. "한 줄 정리"만 고정.

| 금지 | 대체 예시 |
|------|----------|
| "과정 설명" | "어떻게 설치하냐" |
| "방법" | "어떻게 만드냐" |
| "개요" / "소개" | "뭐가 달라지냐" |
| "직접 해보기" | Step 안에 코드 블록으로 통합 |

```
❌ ## 과정 설명
✅ ## 어떻게 설치하냐

❌ ## 개요
✅ ## 뭐가 설치됐냐
```

### 코드 블록 위치

코드 블록은 **Step 안에** 넣어. 별도 "직접 해보기" 섹션을 만들지 않아.

```
❌
**Step 2.** 스크립트를 실행한다.

## 직접 해보기
```bash
curl ...
```

✅
**Step 2.** 아래 스크립트를 복사해서 붙여넣고 Enter

```bash
curl ...
```
```

### 실제 예시

`docs/튜토리얼/01-설치.md` 참고.

## 금지 패턴

| 하면 안 되는 것 | 이유 |
|----------------|------|
| "이번 시간에는..." | 결론부터 말해라 |
| 3줄 이상의 서론 | 독자는 이미 이탈 |
| 한 아티클에 두 가지 이상 | 포인트가 흐려짐 |
| "정리하자면..." + 요약 반복 | 다시 설명은 새로운 말로 |
| 단계가 6개 이상 | 쪼개서 두 아티클로 |
| 고정 섹션 제목 ("과정 설명", "직접 해보기") | 질문형으로 바꿔라 |
| ~습니다, ~합니다, ~해요 | 구어체(~해, ~돼)로 써 |
| ~한다, ~된다 (서술체) | ~해, ~돼로 바꿔 |
| 미사여구, 감탄사 | 정보만 남겨 |
