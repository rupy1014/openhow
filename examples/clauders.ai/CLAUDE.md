# clauders.ai — 콘텐츠 가이드

## 철학: 원포인트 아티클

**하나의 아티클 = 하나의 포인트.**

강의를 쪼갠 게 아니라, 처음부터 하나를 위해 설계된 콘텐츠.
스낵처럼 30초~2분 안에 소비되고, 하나만 기억해도 충분하다.

---

## 구조: 결과 → 과정 → 라이브 → 다시

### 1. 결과 보여주기

완성본 먼저. 서론 없음.

```
❌ "이번 시간에는 Claude Code의 커스텀 커맨드를 배워보겠습니다."
✅ [스크린샷/코드 출력] "한 줄로 블로그 글이 나왔습니다."
```

- 이미지, 코드 출력, 터미널 결과 중 하나
- 텍스트만으로 설명 가능하면 굵은 한 문장
- **3줄 이내**

---

### 2. 과정 설명

결과가 어떻게 나왔는지 단계별로.

```
Step 1. ...
Step 2. ...
Step 3. ...
```

- 화면 캡처 + 한 줄 설명 조합이 이상적
- 핵심 단계만, **최대 5단계**
- 각 단계는 **1~2줄**

---

### 3. 라이브

"지금 바로 해보세요" 포인트.

- 실행 가능한 최소 코드 블록
- 복사해서 바로 쓸 수 있는 프롬프트 / 커맨드
- 환경 설정이 필요하면 한 줄로 링크만

---

### 4. 다시 설명

핵심 한 문장 + 다음이 궁금해지는 암시.

```
✅ "커스텀 커맨드가 있으면 매번 설명 안 해도 됩니다. 다음은 이걸 자동화하는 방법."
```

- **2줄 이내**
- 요약이 아니라 "한 줄 기억"
- 다음 아티클로 자연스럽게 이어지면 좋음

---

## 분량 기준

| 구간 | 목표 분량 |
|------|----------|
| 결과 | 3줄 이내 |
| 과정 | 단계 × 1~2줄, 최대 5단계 |
| 라이브 | 코드 블록 1~2개 |
| 다시 | 2줄 이내 |
| **전체** | **A4 한 장 이내** |

---

## 파일 네이밍

```
{그룹}-{주제}-{번호}.md

예:
강의-claude-code-1.md      ← 강의 시리즈
숙제-claude-code-1.md      ← 과제/실습
특강-콘텐츠자동화-1.md     ← 특강 시리즈
```

번호는 시리즈 순서. 독립적인 아티클이면 번호 없이 `{그룹}-{주제}.md`.

---

## 금지 패턴

| 하면 안 되는 것 | 이유 |
|----------------|------|
| "이번 시간에는..." | 결론부터 말해라 |
| 3줄 이상의 서론 | 독자는 이미 이탈 |
| 한 아티클에 두 가지 이상 | 포인트가 흐려짐 |
| "정리하자면..." + 요약 반복 | 다시 설명은 새로운 말로 |
| 단계가 6개 이상 | 쪼개서 두 아티클로 |
