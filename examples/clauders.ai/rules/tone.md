# 톤앤매너 가이드

## 쉬운 말: 초등학생한테 설명하듯이

**전문 용어를 쓰지 마.** 써야 하면 바로 옆에 쉬운 말로 풀어줘.

| 이렇게 쓰지 마 | 이렇게 써 |
|----------------|-----------|
| 런타임 | 엔진 같은 거야. 프로그램을 돌려주는 프로그램 |
| 컨텍스트 | 대화 내용, 기억 |
| 토큰 | 글자 수 (대충 한글 1글자 = 2~3토큰) |
| CLI | 터미널에서 글자로 명령하는 방식 |
| 프론트엔드 | 눈에 보이는 화면 |
| 트리거 | 자동 실행 조건 |
| alias | 단축 명령어 |
| 프롬프트 | AI한테 하는 말 (요청) |

**비유를 써.** 처음 나오는 개념은 일상 비유로 한 번 풀어줘.

```
❌ "서브 에이전트에게 독립된 컨텍스트가 할당된다."
✅ "담당자마다 새 메모장을 받게 돼. 다른 사람 메모는 안 보여."
```

**영어 단어는 처음 나올 때 한글 설명을 붙여.**

```
❌ "SWE-bench에서 상위권이다."
✅ "코딩 실력 테스트(SWE-bench)에서 꾸준히 상위권이야."
```

---

## 말투: 친구한테 말하듯이

초등학생 친구한테 설명하는 말투야. 딱딱한 서술체가 아니라 자연스러운 구어체로 써.

| 이렇게 쓰지 마 (서술체) | 이렇게 써 (구어체) |
|--------------------------|---------------------|
| ~한다, ~된다, ~이다 | ~해, ~돼, ~야, ~거야 |
| ~않는다, ~없다 | ~않아, ~없어 |
| ~읽는다, ~만든다 | ~읽어, ~만들어 |
| ~시작한다, ~적용된다 | ~시작해, ~적용돼 |
| ~해라, ~써라 | ~해봐, ~써봐 |

```
❌ "파일을 직접 읽고, 만들고, 수정한다."
✅ "파일을 직접 읽고, 만들고, 수정해."

❌ "이렇게 설정하면 된다."
✅ "이렇게 설정하면 돼."

❌ "이 폴더에서만 적용된다."
✅ "이 폴더에서만 적용돼."

❌ "설정 한 번 써두면 매번 설명 안 해도 된다."
✅ "설정 한 번 써두면 매번 설명 안 해도 돼."

❌ "Claude Code는 내 컴퓨터 안에서 직접 일하는 AI다."
✅ "Claude Code는 내 컴퓨터 안에서 직접 일하는 AI야."
```

**존댓말은 절대 금지.**

| 금지 | 대체 |
|------|------|
| ~습니다, ~합니다 | ~해, ~돼 |
| ~됩니다, ~입니다 | ~돼, ~야 |
| ~해요, ~예요 | ~해, ~야 |
| ~해주세요 | ~해봐 |
| ~이잖아요, ~거죠 | ~이잖아, ~거야 |

## 인칭: 나 / 너

필자는 "나", 독자는 "너".

```
❌ "여러분"
✅ "너"

❌ "사용자는 이렇게 할 수 있습니다"
✅ "이렇게 하면 돼"
```

## 문장 길이와 호흡

**한 문장은 짧게.** 15자 이내가 이상적이야.

짧은 문장이 목표지, 짧은 글이 목표가 아니야. 설명은 빼지 마.

1~3문장 쓰고 빈 줄을 넣어. 빈 줄이 호흡이야. 덩어리지면 안 읽혀.

```
❌ 한 문장에 다 쑤셔넣기
AI에게 일을 시키는 방법을 배우면 나중에 자동화도 가능하고, 반복 작업을 줄일 수 있으며, 더 중요한 일에 집중할 수 있게 됩니다.

✅ 짧게 쪼개서, 여러 줄로
AI한테 일 시키는 법을 배우면 반복 작업이 사라져.

중요한 일에 집중할 수 있게 돼.
```

## 첫 문장

서론 없이 바로 시작해. **결론이나 질문으로 바로 들어가.**

```
❌ "이번 시간에는 CLAUDE.md에 대해 배워볼 거야."
✅ "CLAUDE.md가 있으면 매번 설명 안 해도 돼."

❌ "오늘 다룰 내용은 다음과 같아."
✅ "한 줄로 블로그 글이 나왔어."
```

---

## 강조 표현

중요한 한 줄은 **볼드**로.

```
**핵심이 여기 있어.**
```

인상적인 한 문장은 blockquote로.

```
> "코딩을 몰라도 돼. 시키는 법만 알면 돼."
```

과도하게 쓰지 마. 볼드가 많으면 아무것도 강조 안 한 거랑 같아.

## 구어체 연결어

자연스러운 구어체 연결어를 써.

| 써도 되는 것 | 쓰지 말 것 |
|------------|-----------|
| 근데, 그래서, 그러면, 왜냐면 | 그러나, 따라서, 이에 따라, 또한 |
| 어차피, 결국, 솔직히 | 결론적으로, 요컨대, 즉 |
| 그거, 이거, 저거 | 해당, 본, 상기의 |

```
❌ "따라서 이 방법을 적용하면 효율이 향상됩니다."
✅ "그래서 이렇게 하면 훨씬 빨라."
```

## 질문형 제목

섹션 제목은 질문형이야. 고정 레이블을 쓰지 마.

친구한테 말하듯이. `~냐`는 상대를 내려다보는 느낌이야.
`~해?`, `~야?`, `~어?`, `~ㄹ까?`로 써.

| 금지 | 대체 예시 |
|------|----------|
| "개요" / "소개" | "이게 뭐야?" / "왜 이걸 써?" |
| "과정 설명" | "어떻게 해?" |
| "직접 해보기" | Step 안에 코드 블록으로 통합 |
| "정리" | "한 줄 정리" (아티클 마지막에만) |

```
❌ ## 어떻게 하냐
✅ ## 어떻게 해?

❌ ## 뭘 만드냐
✅ ## 뭘 만들까?

❌ ## 개요
✅ ## 이게 뭐야?
```

---

## 코드 블록과 예시

코드 블록은 Step 안에 넣어. 별도 섹션 만들지 마.

**"이렇게 해봐" 다음에 바로 코드.**

```
✅
**Step 2.** 아래를 복사해서 터미널에 붙여넣어.

```bash
claude
```
```

## 풀어서 설명해

**"왜?"를 달아줘.**

설명하고 끝내지 말고, 왜 그게 중요한지까지 써줘.
안 그러면 독자가 혼자 채우다가 이탈해.

```
❌ "담당자에게 지침을 전달해."

✅ "담당자에게 지침을 전달해.
왜? 팀장은 '이미 안다'고 넘어가버리니까."
```

짧은 문장 한 줄만 더 붙이면 돼. 길어지는 게 아니야.

## 금지 패턴

| 하면 안 되는 것 | 이유 |
|----------------|------|
| "이번 시간에는..." | 서론 없이 시작해 |
| "~해볼 거야" | 바로 말해 |
| "~한다", "~된다" (서술체) | "~해", "~돼"로 써 |
| "정리하자면..." + 긴 요약 | 한 줄로 끝내 |
| 감탄사 ("정말!", "놀랍게도") | 정보만 남겨 |
| 완곡 표현 ("어떨까요?", "해보실 수 있어요") | 직접 말해 |
| 단계가 6개 이상 | 아티클 두 개로 쪼개 |

## 실제 예시

**짧고 직접적인 문장:**
```
체중계가 무서웠어.
아니, 진짜 무서웠어.
```

**독자 직접 호칭:**
```
자, 상상해봐.
너 웹소설 하나 써보려고 해.
```

**구어체로 설명:**
```
왜? 대화가 길어지면 Claude Code는 자동 요약을 해.
이 과정에서 이전에 읽은 파일 내용이 압축되거나 손실돼.
```

**결론 먼저:**
```
이게 핵심이야.

| 방법 | 결과 |
|------|------|
| 그냥 "글 써줘" | 평균적인 글 |
| 구체적으로 시키기 | 원하는 스타일 |
```
