# 톤앤매너 가이드

## 쉬운 말: 초등학생도 이해할 수 있게

**전문 용어를 쓰지 않는다.** 써야 하면 바로 옆에 쉬운 말로 풀어준다.

| 이렇게 쓰지 마 | 이렇게 써 |
|----------------|-----------|
| 런타임 | 엔진 같은 거야. 프로그램을 돌려주는 프로그램 |
| 컨텍스트 | 대화 내용, 기억 |
| 토큰 | 글자 수 (대충 한글 1글자 = 2~3토큰) |
| CLI | 터미널에서 글자로 명령하는 방식 |
| 프론트엔드 | 눈에 보이는 화면 |
| 트리거 | 자동 실행 조건 |
| alias | 단축 명령어 |
| 프롬프트 | AI한테 하는 말 (요청) |

**비유를 써라.** 처음 나오는 개념은 일상 비유로 한 번 풀어준다.

```
❌ "서브 에이전트에게 독립된 컨텍스트가 할당된다."
✅ "담당자마다 새 메모장을 받는다. 다른 사람 메모는 안 보여."
```

**영어 단어는 처음 나올 때 한글 설명을 붙인다.**

```
❌ "SWE-bench에서 상위권이다."
✅ "코딩 실력 테스트(SWE-bench)에서 꾸준히 상위권이다."
```

---

## 말투: 평어

모든 글은 **평어**로 쓴다.

평어 ≠ 반말. 위아래 없이 동등하게 대화하는 말투다.

| 금지 | 대체 |
|------|------|
| ~습니다, ~합니다 | ~다, ~한다 |
| ~됩니다, ~입니다 | ~된다, ~이다 |
| ~해요, ~예요 | ~해, ~야, ~한다 |
| ~해주세요 | ~해라, ~해봐 (또는 명령형 생략) |
| ~이잖아요, ~거죠 | ~이잖아, ~거야 |

```
❌ "이렇게 설정하시면 됩니다."
✅ "이렇게 설정하면 돼."

❌ "어렵지 않아요."
✅ "어렵지 않아."
```

---

## 인칭: 나 / 너

필자는 "나", 독자는 "너".

```
❌ "여러분"
✅ "너"

❌ "사용자는 이렇게 할 수 있습니다"
✅ "이렇게 하면 돼"
```

---

## 문장 길이와 호흡

**한 문장은 짧게.** 15자 이내가 이상적이다.

단, 짧은 문장이 목표지 짧은 글이 아니다. 설명은 빼지 마라.

1~3문장 쓰고 빈 줄을 넣는다. 빈 줄이 호흡이다. 덩어리지면 안 읽힌다.

```
❌ 한 문장에 다 쑤셔넣기
AI에게 일을 시키는 방법을 배우면 나중에 자동화도 가능하고, 반복 작업을 줄일 수 있으며, 더 중요한 일에 집중할 수 있게 됩니다.

✅ 짧게 쪼개서, 여러 줄로
AI에게 일 시키는 방법을 배우면 반복 작업이 사라진다.

중요한 일에 집중할 수 있게 된다.
```

## 첫 문장

서론 없이 시작한다. **결론 또는 질문으로 바로 들어간다.**

```
❌ "이번 시간에는 CLAUDE.md에 대해 배워볼 거야."
✅ "CLAUDE.md가 있으면 매번 설명 안 해도 돼."

❌ "오늘 다룰 내용은 다음과 같아."
✅ "한 줄로 블로그 글이 나왔다."
```

---

## 강조 표현

중요한 한 줄은 **볼드**로.

```
**핵심이 여기 있다.**
```

인상적인 한 문장은 blockquote로.

```
> "코딩을 몰라도 돼. 시키는 법만 알면 돼."
```

과도하게 쓰지 않는다. 볼드가 많으면 아무것도 강조 안 한 거랑 같다.

## 구어체 연결어

자연스러운 구어체 연결어를 쓴다.

| 써도 되는 것 | 쓰지 말 것 |
|------------|-----------|
| 근데, 그래서, 그러면, 왜냐면 | 그러나, 따라서, 이에 따라, 또한 |
| 어차피, 결국, 솔직히 | 결론적으로, 요컨대, 즉 |
| 그거, 이거, 저거 | 해당, 본, 상기의 |

```
❌ "따라서 이 방법을 적용하면 효율이 향상됩니다."
✅ "그래서 이렇게 하면 훨씬 빠르다."
```

## 질문형 제목

섹션 제목은 질문형이다. 고정 레이블을 쓰지 않는다.

친구에게 말하듯이. `~냐`는 상대를 내려다보는 억양이다.
`~해?`, `~야?`, `~어?`, `~ㄹ까?`로 쓴다.

| 금지 | 대체 예시 |
|------|----------|
| "개요" / "소개" | "이게 뭐야?" / "왜 이걸 써?" |
| "과정 설명" | "어떻게 해?" |
| "직접 해보기" | Step 안에 코드 블록으로 통합 |
| "정리" | "한 줄 정리" (아티클 마지막에만) |

```
❌ ## 어떻게 하냐
✅ ## 어떻게 해?

❌ ## 뭘 만드냐
✅ ## 뭘 만들까?

❌ ## 개요
✅ ## 이게 뭐야?
```

---

## 코드 블록과 예시

코드 블록은 Step 안에 넣는다. 별도 섹션 만들지 않는다.

**"이렇게 해봐" 다음에 바로 코드.**

```
✅
**Step 2.** 아래를 복사해서 터미널에 붙여넣어.

```bash
claude
```
```

실제 결과물도 코드 블록으로 보여줄 수 있다.

```
AI가 뭔가 써줬어:

```
"체중계가 무서웠다."
```

오, 짧고 공감되지?
```

---

## 비교/대조

"이렇게 하면 안 되고, 이렇게 해야 한다"는 표로 보여주면 명확하다.

```
| 이렇게 하면 | 이렇게 해봐 |
|------------|------------|
| "글 써줘" | 구체적으로 뭘 원하는지 말해줘 |
| 매번 규칙 반복 | CLAUDE.md에 저장 |
```

---

## 풀어서 설명해라

**"왜?"를 달아줘.**

설명하고 끝내지 말고, 왜 그게 중요한지 / 어떤 상황인지까지 써줘.
독자가 스스로 채우다가 이탈한다.

```
❌ "서브 에이전트에게 지침을 전달한다."

✅ "서브 에이전트에게 지침을 전달한다.
왜? 마스터는 '이미 안다'고 넘어가버려서."
```

설명이 길어지는 게 아니다. 짧은 문장이 한 줄 더 붙는 거다.

---

## 금지 패턴

| 하면 안 되는 것 | 이유 |
|----------------|------|
| "이번 시간에는..." | 서론 없이 시작해라 |
| "~해볼 거야" | "~한다"로 직접 말해라 |
| "정리하자면..." + 긴 요약 | 한 줄로 끝내라 |
| 감탄사 ("정말!", "놀랍게도") | 정보만 남겨라 |
| 완곡 표현 ("어떨까요?", "해보실 수 있어요") | 직접 말해라 |
| 단계가 6개 이상 | 아티클 두 개로 쪼개라 |

## 실제 예시

2기 1주차 강의에서 뽑은 패턴들:

**짧고 직접적인 문장:**
```
체중계가 무서웠다.
아니, 진짜 무서웠다.
```

**독자 직접 호칭:**
```
자, 상상해봐.
너 웹소설 하나 써보려고 해.
```

**구어체로 설명:**
```
왜? 대화가 길어지면 Claude Code는 자동 요약을 해.
이 과정에서 이전에 읽은 파일 내용이 압축되거나 손실돼.
```

**결론 먼저:**
```
이게 핵심이야.

| 방법 | 결과 |
|------|------|
| 그냥 "글 써줘" | 평균적인 글 |
| 구체적으로 시키기 | 원하는 스타일 |
```
