# 도서 톤앤매너 — 딸깍! 시리즈

## 한 줄 원칙

**훅은 과감하게, 내용은 정직하게.**

---

## 훅과 본문의 역할 분리

| 위치 | 역할 | 톤 |
|------|------|-----|
| 제목·장 제목 | 강력한 훅. 기대감을 건드려 | 과감. 자극적이어도 됨 |
| 부제·도입부 | 반전. "거의 맞아, 근데..." | 솔직. 현실을 인정 |
| 본문 | 그 '거의'를 채워주는 과정 | 정직. 되는 것과 안 되는 것을 구분 |
| 장 마무리 | 감 잡힘 → 응용 가능 | 자신감. "이만큼 하면 이만큼 된다" |

훅 예시:
```
❌ "AI 코딩 에이전트의 기본 사용법을 알아보겠습니다"
✅ "'해줘' 한마디면 코드가 나와."

뒤이어 본문:
✅ "근데 그게 바로 서비스가 되진 않아. 여기서부터가 진짜야."
```

---

## 3가지 금지

1. **허황된 약속 금지** — "코딩 없이 CTO 되기", "누구나 억대 매출 앱" 같은 소리 안 함
2. **겁주기 금지** — "어려우니까 각오하세요", "개발은 원래 힘듭니다" 안 함
3. **얼버무리기 금지** — "상황에 따라 다릅니다"로 도망치지 않음. 되는 건 되고, 안 되는 건 안 된다고 말함

---

## 저자의 위치

저자는 **10년차 개발자**야. 비개발자한테 말을 걸되, 개발자의 눈으로 판단해.

- "이건 AI한테 시키면 진짜 돼" → 개발자가 검증한 사실
- "이건 아직 안 돼, 직접 해야 해" → 개발자니까 말할 수 있는 솔직함
- "이 정도면 충분해" → 과하지도 부족하지도 않은 현실적 기준선

```
❌ "AI가 모든 걸 해줍니다!" (비개발자가 쓴 것 같은 환상)
❌ "근본적으로 한계가 있으므로..." (개발자 특유의 방어적 태도)
✅ "여기까진 AI가 해. 여기부터는 네가 해. 그거면 돼."
```

---

## 독자에게 주는 감정 흐름

각 장을 읽으면 이런 감정이 남아야 해:

1. **"오 진짜?"** — 훅에서 기대감
2. **"아 그렇구나"** — 현실 인정 (근데 실망 아님)
3. **"이 정도면 할 수 있겠다"** — 감 잡힘
4. **"다음 건 뭐지?"** — 응용 욕구

```
❌ "대단하죠? 여러분도 할 수 있습니다!" (감탄 강요)
❌ "지금까지 배운 내용을 정리하면..." (교과서)
✅ "이제 이게 되니까, 저것도 되겠다는 감이 올 거야."
```

## 말투

`rules/tone.md`의 구어체 규칙을 그대로 따라.

단, 도서이므로 아래만 추가:

- 온라인 아티클보다 **설명이 길어져도 됨** — 책은 2분이 아니라 한 챕터를 읽는 거니까
- **비유와 예시를 아끼지 마** — 비개발자가 처음 접하는 개념은 비유 하나가 설명 열 줄을 이김
- **"왜"를 반드시 설명해** — 온라인은 "이렇게 해"로 끝나도 되지만, 책은 "왜 이렇게 하냐면"까지 가야 함

---

## 장(chapter) 구조

온라인 아티클의 "결과 → 과정 → 라이브 → 다시"를 도서에 맞게 확장:

1. **완성 화면** — 이 장이 끝나면 뭐가 되는지 먼저 보여줘
2. **"거의"의 정체** — 왜 바로 안 되는지, 뭘 알아야 하는지
3. **따라하기** — 단계별 실습. 스크린샷 + 코드 + 한 줄 설명
4. **안 되면?** — 자주 막히는 지점, 에러 대응 (4기 운영 데이터 기반)
5. **감 잡기** — 핵심 한 문장 + "이게 되면 이것도 돼" 응용 암시
