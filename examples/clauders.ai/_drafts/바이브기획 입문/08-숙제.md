**아이디어 하나를 끝까지 기획 문서로 묶어봐.**

주제는 자유야.

대신 순서는 고정해.

---

## 이번 숙제는 어떻게 해?

**Step 1.** 주제 하나를 고른다.

예: 동네 클래스 예약, 소상공인 주문 관리, 반려동물 일정 관리.

**Step 2.** 메뉴 설계 1장을 만든다.

1Depth/2Depth/3Depth와 URL 초안을 적어.

**Step 3.** 서비스 정책 1장을 만든다.

책임 주체, 처리 기한, 자동 처리 규칙을 넣어.

**Step 4.** 화면설계 2장을 만든다.

핵심 화면 2개만 골라 상태와 이벤트를 적어.

**Step 5.** 요구사항 정의 10개를 만든다.

Must 6개, Should 3개, Could 1개로 나눠.

**Step 6.** 제안서 5슬라이드 분량으로 요약해.

문제, 해결, 범위, 일정, KPI 3개까지 넣어.

**Step 7.** 프롬프트와 회고를 같이 제출해.

결과물만 내면 다음 사람이 재현을 못 해.

---

## 제출 전에 뭘 체크해?

| 항목 | 체크 |
|---|---|
| 메뉴 이름과 화면 이름이 일치해 | |
| 정책에 시간 기준이 숫자로 있어 | |
| 화면설계에 에러 상태가 있어 | |
| 요구사항마다 수용 기준이 있어 | |
| 제안서 KPI가 3개 이하야 | |
| 프롬프트 파일을 같이 제출했어 | |
| 회고에 다음 액션 3개를 적었어 | |

전부 체크되면 숙제 완료야.
