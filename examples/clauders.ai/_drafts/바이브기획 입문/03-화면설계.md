**화면설계는 와이어가 아니라 동작 명세야.**

기존 HTML 산출물에서 반복된 칸이 있었어.

`화면 메타 -> 화면 정보 -> UI 컴포넌트 -> 인터랙션 -> API`

---

## 화면설계에서 꼭 적어야 할 건 뭐야?

**Step 1.** 화면 메타를 먼저 고정해.

화면 ID, 우선순위, 난이도부터 적어.

**Step 2.** 화면 정보 4칸을 채워.

화면명, 진입 경로, 화면 목적, 갱신 주기.

**Step 3.** 이벤트를 액션-결과로 분리해.

클릭, 입력, 스크롤 같은 행동마다 결과를 붙여.

**Step 4.** API 칸을 반드시 붙여.

Endpoint, Request, Response, Error까지 적어.

---

## AI로 화면설계 문서를 어떻게 뽑아?

**Step 1.** 아래 포맷으로 화면별로 생성해.

```text
[화면명] 화면설계서를 작성해줘.

필수 항목:
- 화면 메타(화면ID/우선순위/난이도)
- 화면 정보(진입경로/목적/갱신주기)
- UI 컴포넌트 목록
- 상태 정의(기본/로딩/빈상태/에러)
- 인터랙션(사용자행동 -> 시스템반응)
- API 명세(Endpoint/Request/Response/Error)
```

**Step 2.** 통과 기준 3개로 검사해.

- 에러 상태가 있는지
- 진입 경로가 있는지
- API 에러 코드가 있는지
