**두 개의 콘텐츠 자산이 있어. 이걸 하나의 채널로 엮으면 국내에서 이 포지션을 가진 사람이 없어.**

---

## 우리가 가진 것

| 자산 | 성격 | 핵심 |
|------|------|------|
| **clauders.ai** | 4주 입문 과정 | "시키는 법"을 배워서 바로 결과물을 내는 과정 |
| **바이브코딩 북** (35편) | 체계적 심화 시리즈 | "AI한테 기술적으로 지시하는 법" — 개념, 구조, 설계까지 |
| **졸업생 50명** | 실제 증거 | 비개발자가 4주 만에 만든 결과물들 |
| **커뮤니티** | 매일 사례 | 매일 올라오는 use case, 실패, 성공 |

이 네 개를 따로 쓰면 각각 약해. 합치면 **프레임 + 증거 + 깊이 + 신선함**이 다 갖춰져.

---

## 채널 한 줄 정의

**"AI한테 일 시키는 법을 가장 많이 연구한 사람이, 인사이트를 풀고 사례로 증명하는 채널"**

```
신사임당 = 사업의 원리를 풀고, 실제 사장님이 증명
우리     = AI 활용의 원리를 풀고, 실제 비개발자가 증명
```

---

## 핵심 프레임: 바이브코딩 북에서 가져와

바이브코딩 북의 마인드셋 파트가 채널의 **세계관**이 돼.

| 프레임 | 출처 | 영상에서 어떻게 쓰여 |
|--------|------|-------------------|
| **AI = 유능한 신입사원** | 1편 | "AI가 못하는 게 아니야. 시키는 법이 잘못된 거야" |
| **유능한 상사 되기** | 4편 | "알아서 해줘"에서 "추천해봐 → 판단 → 이렇게 해줘"로 |
| **막연 → 구체 → 기술적** | 4편 | 수준별 비교로 시청자가 자기 위치를 알게 됨 |
| **한 입 크기** | 1편 | "한 번에 다 시키면 왜 망하는지" 반복 강조 |
| **70/30 문제** | 안티패턴 | "70%까지는 마법, 30%가 지옥인 이유" |

이 프레임들이 매 영상에서 반복돼. 시청자가 이 용어를 쓰기 시작하면 채널의 세계관에 들어온 거야.

---

## 두 콘텐츠의 역할 분담

```
바이브코딩 북 (35편)          clauders.ai (4주)
   │                            │
   │  프레임 / 인사이트           │  실전 / 증거
   │  "왜 이렇게 해야 하는지"      │  "실제로 이렇게 됐어"
   │                            │
   └──────────┬─────────────────┘
              │
         유튜브 채널
     인사이트 + 사례 증명
```

| | 바이브코딩 북 | clauders.ai |
|--|-------------|-------------|
| 영상에서의 역할 | **"왜"를 설명하는 프레임** | **"진짜"를 보여주는 증거** |
| 인사이트 영상 | 핵심 개념 출처 | 졸업생 사례로 뒷받침 |
| 사례 영상 | 수준별 비교 프레임 적용 | 졸업생이 직접 등장 |
| 데모 영상 | "한 입 크기" 원칙 시연 | 커리큘럼 기반 소재 |

## 콘텐츠 세 축 (구체화)

### 1. 인사이트 (40%) — "이 사람 안목이 있다"

**바이브코딩 북의 프레임 + 50명 데이터 + 커뮤니티 사례**

이게 채널의 핵심 경쟁력이야. 다른 AI 유튜버는 도구를 소개해. 우리는 **원리를 설명하고 사례로 박아.**

| 소재 | 프레임 (북에서) | 증거 (clauders에서) |
|------|---------------|-------------------|
| AI한테 왜 "알아서 해줘"가 안 먹히는지 | "AI = 신입사원" 프레임 | 수강생 1주차 실패 사례 |
| 되는 사람 vs 안 되는 사람 | "막연→구체→기술적" 3단계 | 50명 중 패턴 분석 |
| 바이브코딩의 진짜 한계 | 70/30 문제 | 커뮤니티에서 올라온 실패 |
| "한 번에 다 시키기"가 왜 망하는지 | "한 입 크기" 원칙 | Before/After 비교 |
| 프론트/백엔드를 왜 알아야 하는지 | 기술 개념 시리즈 (10~15편) | "이 용어 하나로 결과가 달라진" 사례 |
| AI한테 설계를 지시하는 법 | 설계 시리즈 (20~24편) | 졸업생 4주차 대화 로그 |

**포맷:**

```
[프리훅] 강렬한 한 마디나 결과
        "코딩 경험 0인 마케터한테 '알아서 해줘'라고 시켰더니 이렇게 됐어."
        → 결과물 화면 (엉망)

[셋업]  "50명 가르치면서 같은 패턴을 봤어."

[본론]  인사이트 (북 프레임) + 사례 (커뮤니티/졸업생)
        "왜 이렇게 되냐면..." → 북 프레임으로 설명
        "실제로 이 사람은..." → 사례로 증명
        "그래서 이렇게 바꿨더니..." → After

[클로징] 핵심 한 문장
```

### 2. 사례 증명 (40%) — "진짜 되는구나"

**졸업생 스토리 + 커뮤니티 하이라이트**

신사임당의 게스트 인터뷰와 같은 역할. 근데 우리 게스트는 "사업가"가 아니라 **"코딩 모르던 직장인"**이야.

**졸업생 스토리 포맷:**

```
[프리훅] 결과물 화면
        "이 사람은 디자이너야. 코딩 한 줄도 몰라."

[바닥]  시작점 — "터미널이 뭔지도 몰랐어"

[발견]  전환점 — 북의 프레임으로 설명
        "2주차에 '한 입 크기'를 알게 된 거야"
        "그전까지 맨날 '전체 만들어줘' 했대"

[폭발]  구체적 숫자 + 결과물 시연
        "매주 3시간 → 5분"
        실제 봇/사이트/자동화 화면

[클로징] 본인 한마디 + 교훈
```

**커뮤니티 하이라이트:**

매일 올라오는 사례 중 강한 것을 모아서 "이번 주 커뮤니티 베스트" 같은 시리즈도 가능해.

```
"이번 주 커뮤니티에서 올라온 것 중에 이게 대박이었어."
→ 사례 3개 소개 + 각각 인사이트 한 줄
```

### 3. 라이브 데모 (20%) — "와 이게 돼?"

**clauders.ai 커리큘럼 기반 + 북의 원칙 시연**

데모를 할 때도 그냥 만드는 게 아니라, **북의 원칙을 실시간으로 적용**하면서 만들어.

```
[만드는 중] "지금 '한 입 크기'로 하는 거야.
            전체를 시키면 안 돼. 이 기능 하나만 먼저."

[중간 확인] "여기서 코드를 확인해. 한 파일에 다 때려박았는지.
            방금 북에서 말한 '역할 분리' 이거야."

[완성]     "10분 만에 됐지? 근데 핵심은 시간이 아니라
            '한 입씩 + 확인 + 다음' 이 리듬이야."
```

데모가 단순 "와 신기하다"로 끝나지 않고, **원리를 체화하는 영상**이 돼.

## 시청자 여정 (퍼널)

```
[발견]  유튜브 검색 / 추천
        "바이브코딩", "AI 코딩", "코딩 없이 앱 만들기"
            │
[관심]  데모 영상으로 "와"
        "10분 만에 이력서 사이트 만들기"
            │
[신뢰]  인사이트 + 사례 영상 3~5개 소비
        "이 사람 50명 가르쳤대. 프레임이 있네"
            │
[학습]  바이브코딩 북 시리즈 진입
        "제대로 배워보고 싶다"
            │
[전환]  clauders.ai 수강 신청
        "혼자 하기엔 한계. 커리큘럼 + 커뮤니티가 필요해"
```

유튜브 영상이 직접 파는 게 아니야. **관심 → 신뢰 → 학습 → 전환** 흐름을 만드는 거야.

## 제목 패턴

인사이트/사례/데모 각각 제목 톤이 달라.

**인사이트:** 도발 + 근거

```
"AI한테 '알아서 해줘'라고 하면 안 되는 진짜 이유"
"비개발자 50명 가르쳤더니 보이는 것"
"바이브코딩이 70%에서 멈추는 이유"
"코딩 모르면서 AI 잘 쓰는 사람들의 공통점"
"ChatGPT만 쓰는 사람이 모르는 것"
```

**사례:** 직업 + 숫자 + 결과물

```
"마케터가 4주 만에 만든 경쟁사 모니터링 봇"
"디자이너가 코딩 없이 만든 포트폴리오 사이트"
"3시간 → 5분. 기획자의 AI 자동화 실화"
"엑셀만 쓰던 사람이 대시보드를 만들었어"
```

**데모:** 시간 + 결과물

```
"코딩 모르는데 10분 만에 이력서 사이트 만들기"
"매일 아침 AI가 뉴스 요약해서 카톡으로 보내줌"
"마크다운으로 쓰면 발표자료가 나옴"
```

## 장기 시리즈 구조

초기 10개가 끝나면 이렇게 확장해:

| 시리즈 | 소스 | 주기 |
|--------|------|------|
| **원리 시리즈** | 북 마인드셋 (1~4편) | 월 1~2회 |
| **개념 시리즈** | 북 기술 개념 (10~24편) | 월 2회 |
| **졸업생 시리즈** | clauders 졸업생 | 격주 |
| **커뮤니티 하이라이트** | 커뮤니티 사례 | 주 1회 (숏폼 가능) |
| **라이브 데모** | clauders 커리큘럼 | 월 2회 |

북 35편만으로도 인사이트/개념 영상 소재가 **1년 이상** 나와. 커뮤니티 사례는 매일 새로 쌓이니까 소재가 마르지 않아.

## 경쟁 없는 포지션

| 기존 AI 유튜버 | 우리 |
|---------------|------|
| 도구 소개 ("이 도구 써보세요") | 원리 설명 ("왜 이렇게 시켜야 하는지") |
| 개발자 → 개발자 | 비개발자 → 비개발자 |
| 본인 경험 1명 | 50명 + 커뮤니티 데이터 |
| 감 잡기 | 체계적 프레임 (35편 시리즈) |
| 단발 영상 | 세계관 (AI=신입, 유능한상사, 한입크기) |

**"비개발자 50명을 가르치고, 35편짜리 체계까지 만든 사람"** — 이 포지션은 국내에 없어.

---

## 한 줄 정리

북이 프레임을 주고, clauders가 증거를 주고, 커뮤니티가 신선한 소재를 준다. 이 세 개가 합쳐진 채널은 대체 불가야.
