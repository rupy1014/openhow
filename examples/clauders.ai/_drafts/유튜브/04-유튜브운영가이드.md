**채널 핵심: "비개발자 50명을 가르치고, 매일 사례가 쌓이는 사람이 인사이트를 푸는 채널"**

튜토리얼 채널이 아니야. 쇼케이스 나열도 아니야.

신사임당처럼 **인사이트 + 실제 사례로 쐐기를 박는 채널**이야.

---

## 왜 이 방향이야?

| 채널 유형 | 구독 이유 | 한계 |
|----------|----------|------|
| 튜토리얼 | "따라해야지" | 한 번 보고 끝. 재방문 없음 |
| 쇼케이스 나열 | "와 대단하네" | 남의 이야기. 감정 투자 없음 |
| **인사이트 + 사례** | "이 사람 말 들으면 건진다" | 매번 와. 신뢰 쌓임. 전환 자연스러움 |

신사임당이 강한 이유: 주장만 하는 게 아니라 **실제 사람이 실제 숫자로 증명**해. "월 3천만원 번다"가 아니라 통장 캡처를 보여줘.

우리한테 그 증거가 **졸업생 50명 + 커뮤니티에서 매일 올라오는 사례**야.

---

## 포지셔닝: 신사임당 × 빌더조쉬

| | 신사임당 | 빌더조쉬 | 우리 |
|--|---------|---------|------|
| 소재 | 타인의 성공 사례 | 본인의 빌딩 과정 | **둘 다** |
| 신뢰 근거 | 게스트가 증명 | 본인이 증명 | **50명이 증명** |
| 정보 공개 | 게스트 의존 | 도구/방법 전부 공개 | **커리큘럼까지 공개** |
| 핵심 감정 | "대리 만족" | "나도 해볼까" | **"이 사람들이 했으면 나도 돼"** |

**차별점: 아무도 이 포지션에 없어.**

AI 유튜버는 많은데 대부분:
- 개발자가 개발자한테 말하거나
- AI 뉴스 읽어주거나
- "이 도구 써보세요" 나열하거나

**비개발자 50명을 직접 가르치고, 커뮤니티에서 매일 사례가 쌓이고, 그 데이터로 인사이트를 뽑는 사람**은 없어.

## 신사임당 스토리텔링 공식 적용

### 영상 구조: 5막

```
[0:00~0:15] 프리훅 — 가장 강렬한 결과 먼저
             "코딩 경험 0인 마케터가, 4주 만에 경쟁사 모니터링 봇을 만들었어."

[0:15~1:00] 셋업 — 왜 이 이야기를 하는지
             "50명을 가르치면서 패턴이 보였어. 되는 사람과 안 되는 사람의 차이."

[1:00~중반] 바닥 — 공감 구간
             "처음엔 다 똑같아. 터미널이 뭔지도 몰라. 근데..."

[중반~후반] 발견 + 폭발 — 인사이트 + 구체적 증거
             "이걸 알고 나서 달라졌어." + 실제 결과물 화면, 수치

[마지막 2분] 교훈 + 여운
             핵심 한 문장. CTA 없이 여운으로.
```

### 핵심 공식: "바닥 → 발견 → 폭발"

| 단계 | 신사임당 | 우리 |
|------|---------|------|
| 바닥 | "빚 3억이었어요" | "엑셀도 겨우 쓰는 사람이었어" |
| 발견 | "이 시장의 빈틈을 찾았어요" | "AI한테 시키는 법을 알게 됐어" |
| 폭발 | "월 매출 3천만원" | "3시간 걸리던 일을 5분에 끝내" |

바닥이 낮을수록 폭발의 임팩트가 커.

## 1. 인사이트 영상

**50명 데이터에서 뽑은 패턴, 교훈, 발견을 공유해.**

신사임당이 "사업의 원리"를 말하듯, 우리는 **"AI 활용의 원리"**를 말해.

근거는 항상 실제 사례야. 주장만 하면 블로거고, 사례로 박으면 전문가야.

**포맷:**

```
[프리훅] 강렬한 한 마디
         "바이브코딩으로 만든 앱 5,600개를 스캔했더니, 2,000개에 보안 구멍이 있었어."

[셋업]   왜 이 이야기를 하는지
         "50명 가르치면서 같은 실수를 반복하는 걸 봤어. 3가지 패턴이야."

[본론]   인사이트 + 사례로 증명
         패턴 1: [인사이트] + [커뮤니티에서 올라온 실제 사례]
         패턴 2: [인사이트] + [졸업생 결과물 화면]
         패턴 3: [인사이트] + [구체적 숫자]

[클로징] 핵심 한 문장
         "결국 이거야. [한 줄 교훈]"
```

**소재 후보:**

| 소재 | 훅 | 근거 |
|------|-----|------|
| 되는 사람 vs 안 되는 사람 | "50명 가르쳤더니 패턴이 보여" | 수강생 데이터 |
| 바이브코딩의 진짜 한계 | "70%까지는 마법, 나머지 30%는 지옥" | 커뮤니티 사례 + 외부 연구 |
| AI한테 시키는 법의 원리 | "프롬프트가 아니야. 구조가 중요해" | 커리큘럼 설계 경험 |
| ChatGPT vs Claude Code | "웹 AI랑 터미널 AI는 완전히 다른 도구야" | 수강생 Before/After |
| 비개발자가 빠지는 함정 3가지 | "매번 같은 데서 막혀" | 50명 데이터 |
| AI 자동화의 ROI | "시간을 돈으로 환산하면 이 정도야" | 졸업생 실제 절감 수치 |

**핵심 원칙:**
- **주장 → 사례 → 숫자** 순서. 주장만 하면 안 돼. 사례가 쐐기야.
- **"우리가 가장 많이 봤다"** 뉘앙스. "50명 가르치면서", "커뮤니티에서 매일 올라오는데". 이게 자연스러운 권위야.
- **정보를 숨기지 마.** 다 줘. 신사임당도 빌더조쉬도 다 줘. 정보를 주면 줄수록 신뢰가 쌓이고, 신뢰가 쌓이면 커리큘럼은 자연스럽게 팔려.

## 2. 사례 증명 영상

**졸업생 또는 커뮤니티 멤버의 실제 이야기.**

신사임당의 게스트 인터뷰와 같은 역할이야. 근데 우리는 "사업 성공 사례"가 아니라 **"AI로 뭔가 해낸 비개발자"** 야.

**포맷:**

```
[프리훅] 결과물 화면 + 한 마디
         "이 사람은 디자이너야. 코딩 한 줄도 몰라. 근데 이걸 만들었어."

[바닥]   이 사람의 출발점
         "터미널이 뭔지도 몰랐어. Claude Code 처음 열었을 때 멘붕이었대."

[발견]   전환점
         "근데 2주차에 이걸 알게 된 거야. [핵심 인사이트]"

[폭발]   결과 + 구체적 수치
         결과물 시연 + "매주 3시간 걸리던 게 5분으로 줄었어"

[클로징] 본인 한마디 + 교훈
```

**강한 스토리가 되는 사람:**

- 출발점이 낮은 사람 ("엑셀도 겨우 쓰는데")
- 구체적 수치가 있는 사람 ("3시간 → 5분", "외주비 200만원 절약")
- 지금도 계속 쓰고 있는 사람 (일회성이 아닌)

**"돈 벌었냐" 대응:**

피하지 마. 프레임을 바꿔.

| 프레임 | 예시 |
|--------|------|
| 비용 절감 | "외주 안 맡기고 직접 만들었더니 200만원 아꼈어" |
| 시간 절감 | "3시간 걸리던 보고서를 5분에 끝내" |
| 역할 확장 | "기획서만 쓰던 사람이 프로토타입까지 직접" |
| 기회 생성 | "이 스킬 하나로 팀에서 포지션이 바뀌었어" |

이게 다 돈이야. 말하는 방식이 다를 뿐이야.

## 3. 라이브 데모

**이건 신사임당에 없는 우리만의 무기야.**

신사임당은 "말로" 증명해. 우리는 **화면으로** 증명할 수 있어. 10분 안에 뭔가 만들어지는 걸 눈으로 보여주는 건 어떤 스토리텔링보다 강해.

비율은 20%로 적지만, **신규 유입 영상**으로 제일 강해. 인사이트 영상은 구독자가 봐. 데모 영상은 비구독자가 봐.

**포맷:**

```
[0:00] 완성된 결과물 먼저 (5초)
[0:05] "코딩 한 줄 안 쳤어" (10초)
[0:15] 실시간 만드는 과정 (7~8분)
[8:30] 완성 → 배포 → URL (1분)
[9:30] 인사이트 한 마디 — "이게 가능한 이유는..."
```

**소재 후보:**

| 소재 | 훅 |
|------|-----|
| 이력서 사이트 | "10분 만에 포트폴리오 완성" |
| 뉴스봇 | "매일 아침 AI가 뉴스 요약해서 카톡으로" |
| 바이브 PPT | "마크다운으로 쓰면 발표자료가 나와" |
| 경쟁사 모니터링 봇 | "매주 경쟁사 블로그 자동 요약" |

## 신뢰 구축 전략

### 1. 증거 먼저, 주장 나중

```
❌ "AI를 배우면 생산성이 올라가요" → 설명
✅ 졸업생 결과물 화면 / 실제 자동화 로그 → "이 사람이 어떻게 했는지 얘기해볼게"
```

### 2. "우리가 가장 많이 봤다"

직접 "전문가입니다"라고 말하지 마. 대신:
- "50명 가르치면서 봤는데..."
- "커뮤니티에서 매일 사례가 올라오거든..."
- "3기까지 운영하면서 패턴이 보여..."

사례 수가 권위야. 숫자가 쌓일수록 자연스럽게 "이 분야에서 가장 많이 본 사람"이 돼.

### 3. 숫자의 구체성

```
❌ "시간이 많이 줄었어"
✅ "매주 3시간 걸리던 보고서가 5분으로 줄었어"

❌ "많은 사람이 성공했어"
✅ "50명 중 38명이 데모나잇에서 결과물을 발표했어"
```

### 4. 과정의 투명성

실패 사례, 시행착오도 공유해. 성공만 보여주면 "광고"가 돼. 실패도 보여줘야 "진짜"가 돼.

- "1기 때는 이게 안 됐어"
- "이 방법은 생각보다 안 먹혀"
- "수강생 중에 포기한 사람도 있어. 이유는..."

## 커뮤니티 = 콘텐츠 엔진

커뮤니티에서 매일 올라오는 사례가 **영상 소재의 원천**이야.

```
커뮤니티 사례 공유 → 패턴 발견 → 인사이트 영상
                  → 강한 사례 → 사례 증명 영상
                  → 신기한 결과물 → 라이브 데모 영상
```

커뮤니티가 활발할수록 영상 소재가 무한해져. 영상이 좋으면 신규 수강생이 오고, 커뮤니티가 더 커지고, 소재가 더 쌓여. **플라이휠**이야.

1번(데모)으로 "와" → 2번(인사이트)으로 "이 사람 누구지?" → 3번(사례)으로 "진짜네" → 이 루프 반복.

---

## 제작 원칙

**길이:** 인사이트/사례는 10~15분. 데모는 10분 이내.

**편집:** 신사임당처럼 핵심 숫자는 화면 가득 자막. 정보 나열 3분 이상 연속 금지. 중간에 결과물 화면, 리액션, 반전 질문으로 패턴 브레이크.

**톤:** 겸손한 관찰자. "전문가입니다"가 아니라 "50명 보면서 느낀 건데". 시청자가 "이 사람이랑 나랑 비슷하다"고 느끼게.

**업로드:** 주 1회. 무리하면 안 돼. 꾸준함이 퀄리티보다 중요해.

**Remotion 활용:** [영상 파이프라인](./03-remotion-video-pipeline)으로 인트로/아웃트로 자동 생성. 본편은 화면 녹화(OBS 등)를 붙여.
