## 3. 책 소개(범위)

### 책 소개

"해줘" 한마디면 될 것 같지? 거의 맞습니다.

AI 코딩 에이전트에게 말로 시키면 진짜 코드가 나옵니다. 하지만 그게 바로 서비스가 되려면 기획·디버깅·배포라는 '거의'를 채워야 합니다. 이 책은 코딩을 한 줄도 모르는 사람이 그 '거의'를 채워서, 아이디어 단계부터 실제 서비스 배포까지 혼자 해내는 과정을 단계별로 안내합니다. 나아가 맥미니 한 대로 24시간 돌아가는 자동화 시스템과 AI 비서를 직접 구축하는 데까지 나아갑니다.

저자는 지난 10년간 2,000여 명의 개발자에게 기술 안내를 해온 검증된 강사이며, 현재 비개발자 대상 바이브코딩 입문 스터디 '클로더즈'를 매달 기수제로 운영하며 누적 100명 이상의 비개발자를 가르쳐온 커리큘럼이 이 책의 토대입니다.

### 다루는 내용

1. **AI 코딩 에이전트 입문** — Claude Code 설치·설정, 프롬프트 작성법, 비용 관리. Codex·Cursor 등 대안 도구 비교
2. **바이브 코딩 프로세스** — 아이디어 → PRD 기획 → 기술 선택 → 하나씩 만들기 → 디버깅 → 배포 → 사용자 피드백
3. **맥미니 AI 자동화** — Lindy 같은 유료 AI 비서 SaaS의 구성요소(cron 스케줄링, 웹 스크래핑, LLM 호출, 메신저 발송)를 해부하고, 맥미니 한 대로 직접 구현
4. **실전 프로젝트** — 캐릭터 AI 챗(첫 번째 앱 만들기), 뉴스 요약 봇(RSS→AI 요약→슬랙/카톡 발송), 맥미니 AI 비서(24시간 자동화 종합). 앞 프로젝트의 결과물이 다음 프로젝트의 재료가 되는 연결 구조

### 대상 독자

- 코딩은 모르지만 "내 아이디어를 직접 만들어보고 싶은" 기획자, 마케터, 창업자, 직장인
- ChatGPT는 써봤지만 "채팅창 밖에서 AI를 쓰는 법"이 궁금한 사람
- 노코드 툴의 한계를 느꼈지만 개발을 배우기엔 부담스러운 사람
- 사이드 프로젝트로 서비스를 만들어보고 싶은 1인 창업 준비자

### 대상 독자는 이 책에서 무얼 얻어야 할까요?

- AI에게 "유능한 상사처럼" 지시하는 구체적인 방법론
- 아이디어를 30일 안에 동작하는 웹 서비스로 만드는 실전 프로세스
- "이건 AI한테 시키면 되겠다"는 판단력 — 무엇이 가능하고 무엇이 아직 안 되는지
- 반복 업무를 자동화하는 실용적인 스킬 (뉴스봇, 슬랙봇, 정기 리포트)
- 맥미니 한 대로 월 구독료 없이 24시간 돌아가는 나만의 AI 비서 시스템

### 대상 독자에게 어떻게 효과적으로 정보를 제공하는지요? (이 책만의 특징/차별점)

1. **100명+ 비개발자를 가르치며 검증된 커리큘럼** — 매달 새로운 기수를 운영하며, 수강생이 막히는 지점(Git 개념, 에러 대응, 비용 공포)을 매 기수마다 파악하고 보강했습니다. "저자가 쓰고 싶은 것"이 아니라 "독자가 실제로 필요한 순서"로 설계되어 있습니다.
2. **도구 중립, 실습은 구체적** — "AI 코딩 에이전트"라는 범용 개념을 가르치면서, 실습은 Claude Code로 통일. 각 장마다 "Codex에서는" 박스를 넣어 Codex가 더 나은 지점을 안내하고, Cursor 등 대안도 함께 소개해 독자가 자기 환경에 맞게 선택할 수 있습니다.
3. **결과 먼저, 과정은 그 다음** — 매 장마다 "완성 화면"을 먼저 보여주고, 그걸 만드는 과정을 따라가는 구조. 독자가 목표를 눈으로 확인한 뒤 움직입니다.
4. **70/30 법칙** — "70%는 기획, 30%만 AI 실행"이라는 원칙을 반복 강조. 코딩 없이 만들 수 있다는 환상이 아니라, 기획력이 핵심이라는 현실적 관점을 제시합니다.
5. **SaaS를 해부해서 직접 만든다** — Lindy($59/월~) 같은 유료 AI 비서 SaaS를 소개한 뒤 "굳이 돈 내고 쓸 필요 없다"고 선언합니다. 그 기능을 구성요소(cron, 스크래핑, LLM API, 메신저 연동)로 해체하고, 오픈소스 프로젝트 OpenClaw의 설계를 참고하면서 맥미니 한 대로 전부 직접 구현하는 과정을 보여줍니다. 독자는 "서비스를 소비하는 사람"에서 "서비스를 만드는 사람"으로 관점이 바뀝니다.
6. **프로젝트가 이어지는 구조** — 캐릭터 AI 챗(첫 앱 체험) → 뉴스 요약 봇(자동화 입문) → 슬랙/카톡 발송 → 맥미니 24시간 자동화. 첫 프로젝트에서 "나도 만들 수 있구나"를 느끼고, 최종적으로 "나만의 AI 비서 시스템"이 완성됩니다.
7. **140개 용어 사전** — 비개발자가 AI와 대화할 때 필요한 기술 용어를 한국어 설명으로 정리. 모르는 단어가 나오면 바로 찾을 수 있습니다.

## 4. 경쟁서 비교

### 『Do It! 챗GPT&파이썬으로 AI 직원 만들기』 (이지스퍼블리싱, 2024)

- **장점** : ChatGPT + Python 조합으로 실습 위주, Do It! 시리즈의 검증된 학습 구조
- **단점** : Python 코딩 학습이 전제. 코딩을 아예 모르는 독자에겐 진입장벽이 높음. 기획→배포 전체 프로세스가 아닌 파편적 실습

### 『비전공자를 위한 이해할 수 있는 IT 지식』 (최원영, 티더블유아이지)

- **장점** : IT 개념을 쉽게 풀어냄, 베스트셀러 검증
- **단점** : "이해"에 초점. 읽고 나면 지식은 늘지만 직접 만드는 건 여전히 못함. 본서는 "만들기"까지 도달

### 『노코드 풀스택 개발 with 버블』 (이명성, 비제이퍼블릭, 2024)

- **장점** : 국내 최초 Bubble 입문서, 약 550페이지의 풍부한 실습
- **단점** : 특정 노코드 플랫폼(Bubble)에 종속. 커스터마이징 한계, 플랫폼 정책 변경 시 서비스 중단 리스크. 본서는 실제 코드 기반이므로 자유도가 압도적

### 해외 바이브 코딩 콘텐츠 (블로그, 강의, 유튜브)

- **장점** : 영문권에서 바이브코딩 개념 빠르게 확산 중
- **단점** : 한국어 서적 전무. 한국 독자의 실무 맥락(카카오톡 봇, 네이버 연동 등)을 다루지 않음

### 『프롬프트 엔지니어링』류 (다수 출간)

- **장점** : 프롬프트 작성 기법을 체계적으로 다룸
- **단점** : 프롬프트만 다루고, 그걸로 실제 소프트웨어를 만드는 과정이 없음. 본서는 프롬프트를 "수단"으로 쓰되 기획→배포 전체를 커버

### Lindy·Zapier Agents 등 AI 비서 SaaS

- **장점** : 설정 없이 바로 사용 가능, 뉴스 요약·일정 알림 등 편의 기능 제공
- **단점** : 월 구독료 발생(Lindy $59/월~, Zapier 유료 플랜), 커스터마이징 한계, 서비스 종료 시 데이터·워크플로 소실. 본서는 이런 서비스의 구성요소를 해부하고, 맥미니 한 대로 동일한 기능을 직접 구축하는 방법을 다룸 — "소비자"에서 "만드는 사람"으로의 전환

**핵심 차별점** : 국내에 "비개발자가 AI 코딩 에이전트로 실제 동작하는 서비스를 처음부터 끝까지 만드는" 책이 아직 없습니다. 노코드 책은 플랫폼 종속, 프롬프트 책은 채팅 수준에 머무르고, 코딩 입문서는 코딩 학습 자체가 목적입니다. 이 책은 "코딩을 배우지 않고 코딩의 결과물을 얻는" 새로운 카테고리를 개척하며, 100명+ 비개발자를 가르치며 실전 검증된 커리큘럼을 기반으로 합니다.
