## 0:15 – 0:40 | Claude Projects란?

### 1회차의 문제


:::info
1회차 방식의 한계:

새 대화 → 또 설명. 새 대화 → 또 설명.

"나는 마케터야, 우리 회사는 B2B SaaS야, 항상 간결하게..." 매번 반복.

Projects로 해결:

한 번 세팅해두면 이 Project 안에서 하는 모든 대화에 적용됨.

  내 업무 맥락, 내 말투 취향, 내 자주 쓰는 포맷 — 다 기억하는 AI 비서.
:::


### Projects = 나만의 작업 공간

| 일반 대화 | Projects |
|----------|----------|
| 매번 처음부터 설명 | 한 번 세팅, 계속 유지 |
| 대화마다 맥락 리셋 | 같은 Project 안에서 누적 |
| 파일 공유 불가 | 참고 문서 올려두기 가능 |

---

### ✅ Task: Project 만들기

**Claude Pro 플랜이 있다면:**

1. claude.ai 왼쪽 사이드바 → **Projects** → **New Project**
2. 이름 입력 (예: "업무 비서", "마케팅 도우미", "내 글쓰기 파트너")
3. Project 생성 완료

:::info 💡 무료 플랜 대안
Projects 기능은 유료 플랜 전용이에요.

무료라면 → 새 대화마다 아래 "컨텍스트 블록"을 첫 줄에 붙여넣기.

`[내 컨텍스트]: 나는 B2B SaaS 마케터야. 항상 간결하게, 불릿 포인트로 써줘. 전문 용어 최소화.`
:::

### 실습: 내 시스템 프롬프트 작성

아래 템플릿을 내 상황에 맞게 채워서 Project Instructions에 붙여넣기:


:::tip 📋 시스템 프롬프트 템플릿
## 나에 대해
나는 [직업/역할]이야. 주로 하는 일은 [주요 업무].

## 결과물 스타일
- [선호하는 길이: 짧게 / 길게 / 상황에 따라]
- [선호하는 형식: 불릿 / 문단 / 표]
- [톤: 격식체 / 캐주얼 / 상황에 따라]
- 서론 없이 결론부터 시작해줘

## 내가 자주 요청하는 것
- [이메일: 3문단 이내, 정중하게]
- [요약: 핵심 3가지, 불릿 포인트]
- [기획: 목적-대상-내용-일정 순서]

## 하지 말아야 할 것
- "물론이죠!", "훌륭한 질문이에요!" 같은 과한 칭찬 금지
- 불필요한 주의사항 달지 말 것
- [기타 내가 싫어하는 것]
:::

### 업종별 예시

**직장인 (팀장급):**
```
나는 IT 서비스 회사 팀장이야. 주로 팀 커뮤니케이션, 보고서 작성, 회의 정리가 업무.
결과물은 항상 간결하게. 팀원들이 읽기 쉽도록 불릿 포인트 선호.
이메일은 용건만 간단히, 3문단 이내. 보고서는 결론 → 이유 → 다음 액션 순서.
"물론이죠" 같은 표현 쓰지 말고, 바로 결과물 줘.
```

**프리랜서 디자이너:**
```
나는 UX/UI 프리랜서 디자이너야. 클라이언트 제안, 피드백 정리, SNS 운영이 주 업무.
클라이언트 커뮤니케이션은 정중하지만 전문적으로. 과도한 존댓말 지양.
제안서는 문제-해결책-기대효과 순서. SNS 글은 200자 이내, 해시태그 3개.
결과물 먼저 보여주고 설명은 나중에.
```

**소상공인:**
```
나는 카페를 운영 중이야. 고객 응대, SNS, 메뉴 기획이 주 업무.
친근하고 따뜻한 톤. 어려운 말 쓰지 말고 쉽게.
SNS 글은 짧고 임팩트 있게. 이벤트 공지는 날짜/혜택을 눈에 잘 띄게.
```

### ✅ Task: 파일 올리기

Project 화면 → **Add content** 또는 **Upload files** → 파일 첨부

파일이 없으면 직접 텍스트로 붙여넣기도 됩니다:


:::tip 📋 텍스트로 올리는 방법
Project에 아래 내용 참고자료로 추가해줘:

[회사/서비스 소개:
우리 회사는 ...]

[자주 쓰는 이메일 형식:
제목: ...
내용: ...]
:::
