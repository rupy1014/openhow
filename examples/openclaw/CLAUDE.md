# openclaw — 콘텐츠 가이드

## 톤앤매너 참조

글을 쓸 때는 반드시 `rules/tone.md`를 읽고 따른다.

핵심 요약:
- 말투: 평어 (~이야, ~해, ~거야). 존댓말 금지.
- 인칭: 나 / 너. "여러분" 금지.
- 문장: 15자 이내, 1~3문장 후 빈 줄.
- 시작: 서론 없이 결론 또는 질문으로 바로.
- 제목: 질문형 구어체. "개요", "방법" 같은 레이블 금지.

## 주제: OpenClaw (오픈클로)

**오픈소스 AI 에이전트 프레임워크.**

메신저(텔레그램, 디스코드, 슬랙 등)로 AI한테 명령하면 **직접 실행**하는 도구야.
ChatGPT처럼 "이렇게 하세요"가 아니라 "했어요"가 되는 거야.

핵심 키워드:
- 만든 사람: Peter Steinberger (@steipete)
- GitHub: github.com/openclaw/openclaw
- 사이트: openclaw.ai
- 특징: 멀티 플랫폼, 멀티 모델, 플러그인, 로컬 실행

## 철학: 원포인트 아티클

**하나의 아티클 = 하나의 포인트.**

강의를 쪼갠 게 아니라, 처음부터 하나를 위해 설계된 콘텐츠.
스낵처럼 30초~2분 안에 소비되고, 하나만 기억해도 충분해.

---

## 말투: 구어체

모든 콘텐츠는 **구어체(~해, ~돼, ~야, ~거야)**로 써. 친구한테 말하듯이.

미사여구 없이 정보만 전달해.

| 금지 | 대체 |
|------|------|
| ~습니다, ~합니다 | ~해, ~돼 |
| ~됩니다, ~입니다 | ~돼, ~야 |
| ~한다, ~된다, ~이다 (서술체) | ~해, ~돼, ~야, ~거야 |
| ~해요, ~예요 | ~해, ~야 |
| ~해주세요, ~하세요 | ~해봐 |

## 구조: 결과 → 과정 → 라이브 → 다시

### 1. 결과 보여주기

완성본 먼저. 서론 없음. **3줄 이내.**

### 2. 과정 설명

결과가 어떻게 나왔는지 단계별로. **최대 5단계, 각 1~2줄.**

### 3. 라이브

복사해서 바로 쓸 수 있는 코드 블록. **1~2개.**

### 4. 다시 설명

핵심 한 문장 + 다음 궁금증. **2줄 이내.**

## 아티클 템플릿

### frontmatter 규칙

- `title` — 주제 이름만. 짧게.
- `nav` — 사이드바 이름. title과 같거나 더 짧게.

```markdown
---
slug: 시작하기-OpenClaw
title: "OpenClaw"
nav: OpenClaw
order: 0
---

{결과물 or 굵은 한 문장}

---

## {질문형 제목}

**Step 1.** ...

**Step 2.** ...

---

## 한 줄 정리

{핵심 한 문장. 다음이 궁금해지게.}
```

## 금지 패턴

| 하면 안 되는 것 | 이유 |
|----------------|------|
| "이번 시간에는..." | 결론부터 말해라 |
| 한 아티클에 두 가지 이상 | 포인트가 흐려짐 |
| 단계가 6개 이상 | 쪼개서 두 아티클로 |
| ~습니다, ~한다 | 구어체로 써 |
| 감탄사, 미사여구 | 정보만 남겨 |
