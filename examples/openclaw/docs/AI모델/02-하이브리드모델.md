*검증 기준일: 2026-03-03 (공식 문서·릴리즈 기준)*

**번역은 로컬 무료 모델, 코딩은 클라우드 모델. 나눠서 써.**

이걸 하이브리드 모델 전략이라고 해.
작업 성격에 따라 돈이 안 드는 모델과 잘하는 모델을 섞는 거야.

---

## 왜 나눠 써?

| 항목 | 로컬 모델 (vLLM) | 클라우드 모델 (Codex/ChatGPT) |
|------|-------------------|-------------------------------|
| 비용 | 전기세만 | 토큰당 과금 |
| 프라이버시 | 데이터가 내 서버에만 | 외부로 전송 |
| 코딩 | 모델에 따라 다름 | Codex 계열이 강함 |
| 번역/요약 | Qwen 같은 다국어 모델이 강함 | 비용 대비 과할 수 있음 |

번역/요약/분류 같은 반복 작업은 무료인 로컬에.
코딩/복잡한 추론은 클라우드에 맡기면 돼.

> 로컬 모델 연결이 아직이면 [vLLM 연결](../AI모델/03-vLLM연결)부터 봐.

---

## 전략 1: 에이전트별로 나누기

가장 실용적이야.
"번역 에이전트는 Qwen, 코딩 에이전트는 Codex" 이런 식이야.

```jsonc
{
  "agents": {
    "list": [
      {
        "id": "main",
        "name": "메인 어시스턴트",
        "model": {
          "primary": "openai-codex/gpt-5.3-codex",
          "fallbacks": ["openai/gpt-4o"]
        }
      },
      {
        "id": "translator",
        "name": "번역봇",
        "model": {
          "primary": "vllm/Qwen/Qwen3.5-122B-A10B"
        }
      },
      {
        "id": "summarizer",
        "name": "요약봇",
        "model": {
          "primary": "vllm/Qwen/Qwen3.5-122B-A10B",
          "fallbacks": ["openai/gpt-4o-mini"]
        }
      }
    ]
  }
}
```

텔레그램 채널별로 에이전트를 바인딩하면 돼:

```jsonc
{
  "bindings": [
    {
      "agentId": "translator",
      "match": {
        "channel": "telegram",
        "peer": { "kind": "group", "id": "-100123456789" }
      }
    },
    {
      "agentId": "main",
      "match": { "channel": "telegram" }
    }
  ]
}
```

번역 그룹에 보내면 Qwen이, DM으로 보내면 Codex가 답해.

## 전략 2: 크론 작업별로 나누기

정기 작업(예약 실행)에서 작업마다 다른 모델을 지정해:

```bash
# 번역 → 로컬 (비용 $0)
openclaw cron add \
  --name "뉴스번역" \
  --every 2h \
  --model "vllm/Qwen/Qwen3.5-122B-A10B" \
  --message "content/의 새 영문 기사를 한국어로 번역해서 translated/에 저장해줘"

# 코드 리뷰 → 클라우드 (코딩 특화)
openclaw cron add \
  --name "코드리뷰" \
  --cron "0 9 * * *" \
  --tz "Asia/Seoul" \
  --model "openai-codex/gpt-5.3-codex" \
  --message "어제 커밋된 PR을 리뷰해줘"
```

## 전략 3: llm-task로 실시간 위임

가장 유연한 방법이야.
메인 에이전트(Codex)가 작업 도중에 서브태스크를 로컬 모델에 맡겨.

```
너: "이 영문 기사를 번역하고 코드 예제도 만들어줘"

메인 에이전트 (Codex):
  1. 번역은 로컬이 적합하겠다
  2. llm-task로 Qwen한테 번역 맡김
  3. 번역 결과 받아옴
  4. 코드 예제는 직접 작성
  5. 합쳐서 응답
```

**Step 1.** 설정 파일에서 llm-task 플러그인을 켜:

```jsonc
{
  "plugins": {
    "entries": {
      "llm-task": { "enabled": true }
    }
  }
}
```

**Step 2.** 에이전트 워크스페이스에 규칙 파일(`AGENTS.md`)을 만들어:

```markdown
# 모델 사용 규칙

## 로컬 모델 (llm-task → vllm/Qwen)
다음 작업은 llm-task로 vllm 프로바이더를 사용해:
- 번역 (temperature: 0.3)
- 요약 (temperature: 0.2)
- 분류/태깅 (temperature: 0.1)

## 직접 수행
다음은 네가 직접 해:
- 코드 작성/수정
- 복잡한 추론
- 파일 작업
```

에이전트가 "스스로 판단"하는 게 아니야.
**규칙을 써줘야** 그대로 따라.

## 실전 예시: 뉴스 번역 파이프라인

```
[RSS 수집]          [번역]              [편집/발행]
Node.js 스크립트 → Qwen 로컬 (무료) → Codex (품질 보증) → 웹사이트
```

```bash
# 매 2시간: 번역 (로컬, $0)
openclaw cron add \
  --name "뉴스번역" \
  --every 2h \
  --model "vllm/Qwen/Qwen3.5-122B-A10B" \
  --tz "Asia/Seoul" \
  --message "content/에서 새 영문 기사를 찾아 한국어로 번역해서 translated/에 저장해줘"

# 매 3시간: 편집+발행 (클라우드)
openclaw cron add \
  --name "뉴스편집" \
  --every 3h \
  --model "openai-codex/gpt-5.3-codex" \
  --tz "Asia/Seoul" \
  --message "translated/의 번역 기사를 검수해서 제목/요약/태그 추가하고 배포해줘"
```

| 단계 | 모델 | 기사당 비용 | 월 100편 |
|------|------|-----------|---------|
| 번역 | Qwen 로컬 | $0 | $0 |
| 편집 | Codex 클라우드 | ~$0.05 | ~$5 |
| **합계** | | | **~$5/월** |

전부 클라우드로 하면 ~$10/월이야.
반으로 줄어.

## 어떤 전략을 써?

| 전략 | 설정 난이도 | 유연성 | 이럴 때 써 |
|------|-----------|--------|-----------|
| 에이전트별 분리 | 중간 | 높음 | 역할이 확실히 나뉠 때 |
| 크론별 지정 | 쉬움 | 중간 | 정기 배치 작업 위주 |
| llm-task 위임 | 중간 | 최고 | 한 대화에서 모델을 섞고 싶을 때 |

처음이면 **에이전트별 분리**로 시작해봐.
가장 직관적이야.

---

## 한 줄 정리

반복 작업(번역/요약)은 로컬 $0 모델, 코딩/추론은 클라우드 모델. 에이전트별로 나누는 게 가장 쉬워.
