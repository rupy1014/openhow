## 바로 적용할 보안 설정

```json5
{
  gateway: {
    mode: "local",
    bind: "loopback",                    // 127.0.0.1만 허용
    auth: {
      mode: "token",
      token: "긴-랜덤-토큰-여기에"       // 직접 생성해서 넣어
    }
  },
  tools: {
    exec: { security: "deny", ask: "always" },   // 명령어 실행 전 항상 확인
    fs: { workspaceOnly: true },                  // 파일 접근을 workspace로 제한
    elevated: { enabled: false }                   // 관리자 권한 차단
  },
  channels: {
    telegram: {
      dmPolicy: "pairing",                        // 모르는 사람 차단
      groups: { "*": { requireMention: true } }    // 그룹에선 멘션해야 반응
    }
  }
}
```

이 설정만 해도 대부분의 위험은 막아.

## 페어링 — 누가 내 봇을 쓸 수 있어?

페어링은 "내 봇에 누가 말 걸 수 있는지"를 승인하는 절차야.

두 가지가 있어:
- **DM 접근 승인**: 어떤 사용자가 봇에게 메시지를 보낼 수 있는지
- **노드/디바이스 승인**: 어떤 기기가 게이트웨이에 연결될 수 있는지

```bash
# 대기 중인 페어링 요청 보기
openclaw pairing list telegram

# 특정 요청 승인
openclaw pairing approve telegram <CODE>
```

페어링/허용 목록은 `~/.openclaw/credentials/` 아래에 저장돼.
이 폴더에는 민감한 인증 정보가 있으니 **절대 외부에 공유하지 마**.

## 대시보드 노출 금지

대시보드(Control UI)는 관리자용 컨트롤 화면이야.
WebSocket 핸드셰이크에서 토큰/패스워드로 인증해.

| 접근 방식 | 보안 수준 | 권장? |
|-----------|----------|-------|
| localhost (127.0.0.1) | 높음 — 내 컴퓨터에서만 | 권장 |
| Tailscale VPN | 높음 — 인증된 기기만 | 권장 |
| SSH 터널 | 높음 — 암호화된 터널 | 권장 |
| 공개 인터넷 직접 노출 | 매우 낮음 | **금지** |

## 보안 점검 명령어

```bash
openclaw security audit          # 기본 점검
openclaw security audit --deep   # 심층 점검
openclaw security audit --fix    # 자동 수정
```

정기적으로 돌려봐. 설정 빠진 거 알려줘.

## 꼭 지켜야 할 것들

| 항목 | 왜? |
|------|-----|
| **root로 실행하지 마** | 전용 사용자 만들어 |
| **API 키를 설정 파일에 직접 넣지 마** | 환경변수나 `.env` 파일 써 |
| **Gateway를 외부에 노출하지 마** | `bind: "loopback"`. 외부 접근은 Tailscale |
| **파일 권한 설정** | `chmod 600 ~/.openclaw/openclaw.json` |
| **항상 최신 버전** | 보안 패치가 자주 나와. 주기적으로 업데이트해 |
| **브라우저 자동화는 마지막에** | 가장 위험한 기능이야 |

## API 키가 채팅에 노출되면?

메신저로 API 키를 보내지 마.
채팅 로그가 수십 개 파일에 저장돼. 한번 보내면 완전 삭제가 어려워.

```bash
# 안전한 방법
openclaw config set env.ANTHROPIC_API_KEY "sk-ant-새키"
openclaw gateway restart
```

## 포트 확인

```bash
# Linux
ss -tlnp

# macOS
lsof -iTCP -sTCP:LISTEN -n -P
```

이 포트들이 외부에 열려 있으면 안 돼:

| 포트 | 용도 |
|------|------|
| 18789 | OpenClaw Gateway |
| 5432 | PostgreSQL |
| 3306 | MySQL |
| 9229 | Node.js 디버그 |
| 18800 | OpenClaw 내부 |

VPS에서 돌린다면 방화벽 필수.

## 처음 시작할 때 이렇게 해

1. **민감한 데이터가 없는 환경**에서 시작
2. 연결은 **이메일 한 폴더만**부터
3. 안전 확인 후 범위 확장
4. 위험한 기능 순서: 셸 명령 → 파일시스템 → 브라우저 (마지막)

## VPS vs Mac: 보안 관점

| | Mac (집) | VPS |
|---|---|---|
| 해킹당하면? | **홈 네트워크 전체** 위험 | 빈 리눅스 박스만 |
| 물리 보안 | 좋음 | 데이터센터가 관리 |
| 추천 | 테스트용 | 상시 운영용 |

상시 운영은 VPS가 더 안전해.
Mac은 Docker 컨테이너 안에서 돌려.
