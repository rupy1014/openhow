*검증 기준일: 2026-03-03 (소스 코드·공식 문서·실사용자 Q&A 기준)*

**담당자마다 기억이 완전히 분리돼. 근데 백업은 마크다운만으로 안 돼.**

---

## 담당자 하나가 뭘 갖고 있어?

담당자(에이전트)를 만들면 자동으로 두 개의 폴더가 생겨.

| 폴더 | 뭐가 들어있어? |
|------|--------------|
| **워크스페이스** | 성격(SOUL.md), 규칙(AGENTS.md), 기억(MEMORY.md), 메모 |
| **상태 폴더** | 대화 기록(SQLite), 세션 정보, 인증 프로필 |

```
~/.openclaw/agents/secretary/
├── workspace-secretary/    ← 마크다운 파일들 (성격, 규칙, 기억)
│   ├── SOUL.md
│   ├── AGENTS.md
│   ├── MEMORY.md
│   └── memory/
│       └── 2026-03-03.md
└── agent/                  ← 상태 데이터
    ├── sessions/           ← SQLite (대화 기록)
    └── auth-profiles.json  ← 인증 정보
```

**핵심: 담당자마다 폴더가 완전히 따로야.**

`secretary`의 MEMORY.md와 `developer`의 MEMORY.md는 별개의 파일이야.
서로 뭘 기억하는지 모르고, 대화 기록도 안 보여.

---

## 기억은 어디에 저장돼?

두 곳이야.

### 1. 마크다운 파일 — 영구 기억

담당자가 "이건 기억해야겠다" 싶은 건 마크다운에 써.

```
MEMORY.md       → 핵심 사실 (항상 읽어)
memory/날짜.md  → 그날의 기록 (필요할 때 읽어)
SOUL.md         → 성격, 가치관
AGENTS.md       → 규칙, 우선순위
USER.md         → 너에 대한 정보
```

### 2. SQLite DB — 대화 기록

대화 내용, 세션 인덱스, 도구 실행 기록이 여기에 들어가.
마크다운에는 안 남는 세부 대화까지 전부 저장돼.

**그래서 마크다운만 백업하면 절반만 백업하는 거야.**

---

## 백업은 어떻게 해?

### 완전 백업 (가장 안전)

```bash
cp -r ~/.openclaw ~/.openclaw-backup-$(date +%Y%m%d)
```

이러면 설정, 기억, 대화 기록, 인증 정보까지 전부 복사돼.

### 뭘 포함해야 해?

| 뭘? | 경로 | 빠뜨리면? |
|-----|------|----------|
| 설정 파일 | `~/.openclaw/openclaw.json` | 담당자 구성이 날아가 |
| 워크스페이스 | `~/.openclaw/agents/*/workspace-*/` | 성격, 규칙, 기억이 사라져 |
| SQLite DB | `~/.openclaw/agents/*/sessions/` | 대화 기록이 없어져 |
| 인증 정보 | `~/.openclaw/credentials/` | 메신저 재연결해야 해 |

### 마크다운만 교체하면?

성격이랑 규칙은 바뀌어.
근데 **대화 기록은 SQLite에 있어서 안 바뀌어.**

SQLite 인덱스가 마크다운이랑 안 맞으면 이상하게 동작할 수 있어.
스냅샷을 만들 거면 **SQLite도 같이 백업해.**

## CLI 토큰 vs API 키 — 파일 범위가 달라

어떤 방식으로 AI 모델을 연결하느냐에 따라 데이터가 저장되는 범위가 달라져.

| 연결 방식 | 데이터 범위 | 백업 범위 |
|----------|-----------|----------|
| API 키 (일반적) | `.openclaw/` 안에만 | `.openclaw/` 폴더면 충분 |
| Claude CLI 토큰 | `.openclaw/` + `.claude/` | 더 넓은 범위 필요 |
| OpenAI Codex CLI | `.openclaw/` + 별도 폴더 | 더 넓은 범위 필요 |

API 키 방식이면 `.openclaw/` 폴더 하나로 끝나.
CLI 토큰 방식이면 다른 폴더에도 에이전트 파일이 생기니까 주의해.

## 텔레그램 봇을 여러 사람이 쓰면?

**채팅은 분리돼.** 텔레그램 DM은 사람별로 다른 세션이야.

근데 **기억(MEMORY.md)은 공유돼.** 같은 담당자를 쓰니까.

A한테 들은 얘기를 B한테 꺼낼 수도 있어.

### 채팅만 분리하면 충분할 때

민감한 정보가 없고, 기억이 섞여도 괜찮으면:

```json5
{
  channels: {
    telegram: {
      dmPolicy: "allowlist",
      allowFrom: ["alice-chat-id", "bob-chat-id"]
    }
  },
  session: {
    dmScope: "per-peer"   // 사람마다 세션 분리
  }
}
```

### 완전히 격리해야 할 때

사람마다 **다른 담당자를 배정**해.

```json5
{
  agents: {
    list: [
      { id: "for-alice", workspace: "~/.openclaw/workspace-alice" },
      { id: "for-bob", workspace: "~/.openclaw/workspace-bob" }
    ]
  },
  bindings: [
    {
      agentId: "for-alice",
      match: { channel: "telegram", peer: { kind: "direct", id: "alice-id" } }
    },
    {
      agentId: "for-bob",
      match: { channel: "telegram", peer: { kind: "direct", id: "bob-id" } }
    }
  ]
}
```

이러면 기억, 대화, 성격까지 전부 분리.

---

## 이전 상태로 돌아갈 수 있어?

**마크다운 교체만으로는 부족해.**

OpenClaw는 마크다운 + SQLite를 같이 써.
SQLite 인덱스가 깨지면 이전 상태 복원이 안 돼.

**완전한 스냅샷:**

```bash
# 저장
tar czf openclaw-snapshot-$(date +%Y%m%d).tar.gz ~/.openclaw

# 복원
tar xzf openclaw-snapshot-20260303.tar.gz -C ~/
openclaw gateway restart
```

설정 파일, 워크스페이스, SQLite, 인증 정보까지 전부 포함해야 "그때 그 상태"로 돌아갈 수 있어.

---

## 한 줄 정리

담당자마다 워크스페이스가 분리돼서 기억이 안 섞여. 백업할 땐 `.openclaw` 폴더 전체를 복사해야 완전해.
