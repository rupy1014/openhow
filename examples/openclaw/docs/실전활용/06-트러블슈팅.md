---

## 진단 3단계

```bash
# 1. 전체 상태
openclaw status

# 2. Gateway 상태
openclaw gateway status

# 3. 자동 진단
openclaw doctor
```

`openclaw doctor`가 대부분 원인을 알려줘.
`--fix` 붙이면 자동으로 고쳐줘.

---

## 자주 나는 문제들

### 메신저에서 답이 안 와

```bash
# 채널 연결 상태 확인
openclaw channels status --probe

# 페어링 대기 중인 거 있는지 확인
openclaw pairing list telegram
openclaw pairing approve telegram 코드
```

페어링 승인 안 해서 안 되는 경우가 제일 많아.

---

### Gateway가 안 떠

```bash
# 포트 충돌 확인
lsof -i :18789

# 다른 프로세스가 쓰고 있으면 죽이거나 포트 변경
openclaw config set gateway.port 18790
openclaw gateway restart
```

---

### 설정 바꿨는데 적용이 안 돼

**`openclaw gateway restart` 했어?**

설정 변경 후 Gateway 재시작은 필수야. 이걸 빼먹는 게 가장 흔한 실수.

---

### 401 Unauthorized (API 키 오류)

```bash
# 키 확인
openclaw config get env.ANTHROPIC_API_KEY

# 키 재설정
openclaw config set env.ANTHROPIC_API_KEY "sk-ant-새키"
openclaw gateway restart
```

---

### 스킬이 "not eligible"로 뜨

opencode-controller 같은 스킬이 작동 안 할 때:

```bash
# OpenCode가 돌아가는지 확인
ps aux | grep opencode

# 포트 확인
lsof -i :4096

# API 연결 테스트
curl -u opencode:비밀번호 http://127.0.0.1:4096/api/status

# 설정 검증
openclaw config validate

# 둘 다 재시작
openclaw gateway restart
```

---

### 크론 작업이 안 돌아

```bash
# 크론 활성화 확인
openclaw config get cron.enabled

# 목록 확인
openclaw cron list
```

Gateway가 **계속 실행 중이어야** 크론도 돌아가.
Gateway 꺼지면 크론도 멈춰.

---

### Docker에서 OpenCode 연결 안 돼

Docker 컨테이너 안에서 `localhost:4096`은 호스트가 아니라 컨테이너 자신을 가리켜.

```yaml
# docker-compose.yml
services:
  openclaw:
    image: openclaw/openclaw:latest
    network_mode: host
    extra_hosts:
      - "host.docker.internal:host-gateway"
    environment:
      - OPENCODE_API_URL=http://host.docker.internal:4096
```

---

### 메모리 부족 (npm install 실패)

저사양 서버에서 설치할 때:

```bash
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

스왑 파일 만들어주면 돼.

---

### 업그레이드 후 이상할 때

```bash
openclaw doctor
openclaw gateway install --force
openclaw gateway restart
```

업그레이드 직후에는 `openclaw doctor` 한 번 돌려봐.

---

## 실시간 디버깅

```bash
# 실시간 로그
openclaw logs --follow

# 디버그 모드로 실행
openclaw gateway --verbose
```

`--verbose`로 실행하면 메시지가 어디서 막히는지 정확히 보여.

---

## 한 줄 정리

`openclaw doctor`가 만능이야. 안 되면 `gateway restart`. 그래도 안 되면 `--verbose`로 로그 봐.
