---

## 진단 3단계

```bash
# 1. 전체 상태
openclaw status

# 2. Gateway 상태
openclaw gateway status

# 3. 자동 진단
openclaw doctor
```

`openclaw doctor`가 대부분 원인을 알려줘.
`--fix` 붙이면 자동으로 고쳐줘.

---

## 자주 나는 문제들

### 메신저에서 답이 안 와

```bash
# 채널 연결 상태 확인
openclaw channels status --probe

# 페어링 대기 중인 거 있는지 확인
openclaw pairing list telegram
openclaw pairing approve telegram 코드
```

페어링 승인 안 해서 안 되는 경우가 제일 많아.

---

### Gateway가 안 떠

```bash
# 포트 충돌 확인
lsof -i :18789

# 다른 프로세스가 쓰고 있으면 죽이거나 포트 변경
openclaw config set gateway.port 18790
openclaw gateway restart
```

---

### 설정 바꿨는데 적용이 안 돼

**`openclaw gateway restart` 했어?**

설정 변경 후 Gateway 재시작은 필수야. 이걸 빼먹는 게 가장 흔한 실수.

---

### 401 Unauthorized (API 키 오류)

```bash
# 키 확인
openclaw config get env.ANTHROPIC_API_KEY

# 키 재설정
openclaw config set env.ANTHROPIC_API_KEY "sk-ant-새키"
openclaw gateway restart
```

---

### 스킬이 "not eligible"로 뜨

opencode-controller 같은 스킬이 작동 안 할 때:

```bash
# OpenCode가 돌아가는지 확인
ps aux | grep opencode

# 포트 확인
lsof -i :4096

# API 연결 테스트
curl -u opencode:비밀번호 http://127.0.0.1:4096/api/status

# 플러그인/환경 진단
openclaw plugins doctor
openclaw doctor

# 둘 다 재시작
openclaw gateway restart
```

---

### 크론 작업이 안 돌아

```bash
# 크론 활성화 확인
openclaw config get cron.enabled

# 목록 확인
openclaw cron list
```

Gateway가 **계속 실행 중이어야** 크론도 돌아가.
Gateway 꺼지면 크론도 멈춰.

---

### Docker에서 OpenCode 연결 안 돼

Docker 컨테이너 안에서 `localhost:4096`은 호스트가 아니라 컨테이너 자신을 가리켜.

```yaml
# docker-compose.yml
services:
  openclaw:
    image: openclaw/openclaw:latest
    network_mode: host
    extra_hosts:
      - "host.docker.internal:host-gateway"
    environment:
      - OPENCODE_API_URL=http://host.docker.internal:4096
```

### 메모리 부족 (npm install 실패)

저사양 서버에서 설치할 때:

```bash
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

스왑 파일 만들어주면 돼.

---

### 업그레이드 후 이상할 때

```bash
openclaw doctor
openclaw gateway install --force
openclaw gateway restart
```

업그레이드 직후에는 `openclaw doctor` 한 번 돌려봐.

---

### 대화가 갑자기 "기억상실"처럼 보일 때 (컨텍스트 압축)

장시간 한 세션에서 여러 주제를 섞으면 컨텍스트 압축(compaction)으로 맥락이 날아갈 수 있어.

```bash
# 현재 컨텍스트 사용량/상태 확인
/status
```

**대응 순서**
1. 컨텍스트가 50% 이상 차기 시작하면 새 세션으로 분리
2. 주제별 채널/토픽으로 나눠서 맥락 오염 방지
3. 중요한 결정은 채팅에만 두지 말고 markdown 파일에 기록

---

### 크론이 "조용히" 멈췄을 때

버전 업그레이드나 명령/경로 변경 후 크론이 실패해도 알림이 안 오는 경우가 있어.

```bash
# 크론 목록/활성 상태 확인
openclaw cron list

# 실시간 로그에서 cron 관련 에러 확인
openclaw logs --follow | rg -i "cron|error|failed"
```

주요 원인:
- 오래된 명령어/패키지명을 계속 사용
- Gateway 재시작 누락
- 장시간 작업이 한 번 죽고 재시작 로직이 없음

---

### Claude Code/OpenCode에서 권한 프롬프트 때문에 멈출 때

실행 도중 권한 확인 대기 상태가 되면 OpenClaw 세션이 타임아웃으로 끝날 수 있어.

대응:
1. 자주 필요한 권한은 미리 허용(pre-allow) 설정
2. 위험 작업은 여전히 수동 승인 유지
3. 장시간 작업은 heartbeat에서 "진행 중/실패/완료"를 주기적으로 확인

---

### 장시간 작업이 중간에 사라질 때 (`/tmp` 이슈)

임시 디렉터리에 생성된 세션/산출물은 정리(cleanup)로 사라질 수 있어.

대응:
1. 코드 작업 경로를 영구 workspace로 고정
2. 시작 시 작업 디렉터리와 세션 ID를 로그에 남기기
3. heartbeat에서 세션 생존 여부를 점검하고 실패 시 1회 자동 재시도

---

### 모든 설정을 초기화하고 싶을 때

처음부터 다시 하고 싶으면:

```bash
# 설정 디렉토리 백업 후 삭제
mv ~/.openclaw ~/.openclaw.bak

# 처음부터 다시
openclaw onboard --install-daemon
```

### 대시보드가 안 열려

```bash
# Gateway가 돌고 있는지 확인
openclaw gateway status

# 포트 확인
lsof -i :18789

# Gateway 시작
openclaw gateway start
```

브라우저에서 `http://127.0.0.1:18789/` 직접 접속해봐.
`localhost:18789`가 아니라 `127.0.0.1:18789`로.

---

### 재부팅 후 게이트웨이가 자동 시작 안 돼

데몬 설치가 안 된 거야.

```bash
openclaw onboard --install-daemon
```

macOS는 LaunchAgent, Linux/WSL2는 systemd user unit으로 설치돼.

---

### Node.js 버전이 22보다 낮아

```bash
# 현재 버전 확인
node --version

# nvm으로 업그레이드
nvm install 22
nvm use 22
```

nvm이 없으면 [nvm 설치](https://github.com/nvm-sh/nvm)부터 해.
아니면 설치 스크립트가 자동으로 깔아줘: `curl -fsSL https://openclaw.ai/install.sh | bash`

## 실시간 디버깅

```bash
# 실시간 로그
openclaw logs --follow

# 디버그 모드로 실행
openclaw gateway --verbose
```

`--verbose`로 실행하면 메시지가 어디서 막히는지 정확히 보여.
