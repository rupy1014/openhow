## 1. 세컨드 브레인 (개인 지식 관리)

텔레그램에 아무거나 보내. 메모, 아이디어, 링크, 할 일.
나중에 "지난주에 봤던 그 기사 뭐였지?" 하면 찾아줘.

**메모리 3단계 구조:**

| 단계 | 파일 | 역할 |
|------|------|------|
| 항상 로드 | `MEMORY.md` | 핵심 사실 (~100줄) |
| 일별 기록 | `memory/YYYY-MM-DD.md` | 그날의 작업 노트 |
| 주제별 폴더 | `memory/people/`, `projects/` | 시맨틱 검색용 |

한 개발자는 Next.js로 대시보드를 만들어서 Cmd+K로 전체 메모리를 검색해.

**팁**: `MEMORY.md`는 100줄 이내로 유지해. 길어지면 토큰 낭비야.

## 2. 스마트홈 제어 (Home Assistant 연동)

한 개발자(Dan Malone)는 **"Claudette"**라는 너구리 캐릭터 에이전트를 만들었어.

**셋업:**
- Proxmox 서버, Home Assistant VM
- Ubuntu LXC 컨테이너에 OpenClaw (RAM 2GB, CPU 2코어)
- ha-mcp로 Home Assistant REST API 연결
- ElevenLabs "Lily" 음성 → Google Home 스피커

**텔레그램에서 이렇게 써:**

```
"아기 낮잠 모드 켜줘"
→ 모든 조명 1% 밝기, 2000K 색온도
→ 주말 낮 시간에만 자동 활성화

"거실 불 켜줘"
→ Home Assistant API 호출 → 즉시 실행
```

**효과**: Home Assistant UI에서 60분 걸리던 자동화 설정이 텔레그램으로 **5분**.

**비용**: Claude Max 구독 $200/월 (업무용이라 이미 쓰고 있었음)

## 3. 밤새 코딩 시키기 ("킬러 피처")

자기 전에 이렇게 시켜:

```
이 GitHub 이슈 분석하고, 코드 읽고, 수정해서 PR 올려놔.
```

아침에 일어나면 PR이 올라와 있어. 리뷰만 하면 돼.

**셋업:**
- opencode-controller 스킬
- 모델: Claude Opus (복잡한 코드) or Sonnet (단순 수정)
- 타임아웃 넉넉하게: `timeoutMs: 600000` (10분)

**주의**: 명확한 요구사항이 있을 때만 써. "뭔가 좋은 거 만들어줘"는 안 돼.

## 4. 제로 직원 회사 (멀티 에이전트)

한 창업자가 에이전트 6개로 **1주일간 회사를 혼자 운영**했어.

| 에이전트 | 역할 |
|---------|------|
| Morgan | 총괄 코디네이터 |
| Atlas | IT 인프라 |
| Ada | 운영 |
| Linus | 개발 |
| James | 리서치 |
| (본인) | 최종 의사결정만 |

**모델 전략 (비용 최소화):**
- 메인: Kimi K2.5 (무료, 128K 컨텍스트)
- 폴백: GLM 4.7 (무료)
- 로컬: Ollama qwen2.5:7b (무료)
- 하트비트: Ollama (무료)

**월 비용: ~$50** (대부분 로컬 추론)

**배운 것들:**
1. **모델 폴백은 필수**. 클라우드 API는 자주 타임아웃 남
2. **에이전트당 Gateway 하나**. 공유하면 연쇄 장애
3. **메모리 영속성 필수**. 재시작하면 맥락 사라져
4. **하트비트가 장애의 90%를 잡아줘**. 로컬 모델로 비용 $0

## 5. GitHub 이슈 자동 처리 (Slack 연동)

Slack에서 GitHub 이슈 알림이 오면:
1. OpenClaw이 이슈 분석
2. 로컬 코드 확인
3. 수정 구현
4. 이슈에 업데이트 코멘트

도구 전환(IDE ↔ 브라우저 ↔ 터미널) 없이 Slack에서 끝나.

**PR 모니터링 크론:**

```bash
openclaw cron add --name "PR체크" --cron "0 9 * * 1-5" \
  --session isolated --announce --channel slack \
  --message "열린 PR 중 머지 가능한 거 확인해. 충돌 없고, CI 통과, 승인 완료된 것만"
```

## OpenClaw 10개 설정 파일

개발자들이 쓰는 파일 시스템이야. 전부 마크다운이야.

| 파일 | 역할 | 팁 |
|------|------|-----|
| `AGENTS.md` | 우선순위, 경계, 워크플로우 | 영구 규칙은 여기에 |
| `SOUL.md` | 성격, 가치관, 말투 | 50~150줄. 임시 할일 넣지 마 |
| `USER.md` | 사용자 선호 (포맷, 톤) | |
| `IDENTITY.md` | 이름, 역할, 목표 | |
| `TOOLS.md` | 환경 정보 (경로, 별칭, 위험 명령어) | |
| `HEARTBEAT.md` | 주기적 자동 체크 항목 | 5~10개 이내. 토큰 절약 |
| `MEMORY.md` | 영구 기억 (~100줄) | |
| `memory/YYYY-MM-DD.md` | 일별 작업 노트 | 자동 생성 |
| `BOOT.md` | 시작 시 1회 실행 | |
| `BOOTSTRAP.md` | 최초 온보딩 스크립트 | |

**핵심**: 영구 규칙은 `AGENTS.md`, 성격은 `SOUL.md`, 취향은 `USER.md`. 섞지 마.

## 잘 되는 것 vs 안 되는 것

| 잘 되는 것 | 안 되는 것 |
|-----------|-----------|
| 정형화된 반복 작업 | 모호한 탐색적 작업 |
| 이메일 정리, 파일 분류 | 캘린더 관리 (불안정) |
| 모닝 브리핑, 예약 보고서 | 복잡한 UI 브라우저 자동화 (깨지기 쉬워) |
| 밤새 코딩 | 확인 없이 실행 (위험) |
| 세컨드 브레인 | 무한 루프 (가끔 빠짐) |
| 스마트홈 제어 | 긴 침묵 (생각 중 몇 분) |

**Hacker News 커뮤니티 평가:**
> "기술적인 사람이 어차피 만들었을 것을 대신 만들어주는 도구. 배관 공사를 줄여주는 거지, 마법은 아니야."
