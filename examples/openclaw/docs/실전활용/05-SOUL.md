**OpenClaw 에이전트는 마크다운 파일 10개로 성격과 행동을 정의해.**

전부 `~/.openclaw/workspace/`에 들어가.

---

## 워크스페이스 구조

```
~/.openclaw/workspace/
├── AGENTS.md       ← 운영 규칙 (최우선)
├── SOUL.md         ← 성격, 말투
├── USER.md         ← 사용자 선호
├── IDENTITY.md     ← 이름, 역할
├── TOOLS.md        ← 환경 정보
├── HEARTBEAT.md    ← 주기 체크 항목
├── MEMORY.md       ← 영구 기억
├── BOOT.md         ← 시작 시 1회 실행
├── BOOTSTRAP.md    ← 최초 온보딩
└── memory/
    ├── 2026-02-25.md  ← 일별 노트 (자동 생성)
    └── people/        ← 주제별 폴더
```

---

## 각 파일이 뭘 해?

### AGENTS.md — 운영 규칙

에이전트의 **우선순위, 경계, 워크플로우**를 정의해.
영구적인 규칙은 전부 여기에 써.

```markdown
# 규칙
- 이메일 발송은 항상 확인받고 해
- 파일 삭제는 절대 자동으로 하지 마
- 코드 수정 후 반드시 테스트 실행해

# 우선순위
1. 사용자 안전
2. 정확성
3. 속도
```

### SOUL.md — 성격과 말투

에이전트의 **가치관, 톤, 판단 기준**이야.
없어도 돌아가지만, 있으면 **완전히 다른 에이전트**가 돼.

4가지 섹션으로 구성해:

```markdown
# Core Truths (핵심 가치)
- 간결하게 답해. 불필요한 설명 하지 마.
- 모르면 모른다고 해. 추측하지 마.

# Boundaries (경계)
- 개인정보는 외부에 공유하지 마.

# Vibe (분위기)
- 친근하지만 프로페셔널하게.
- 한국어로 대화해.

# Continuity (연속성)
- 이전 대화에서 배운 것을 기억해.
```

**50~150줄이 적당해.** 길면 매 대화마다 토큰을 먹어.

### USER.md — 사용자 정보

사용자의 선호, 커뮤니케이션 스타일 등을 적어.

### IDENTITY.md — 정체성

에이전트의 이름, 역할, 목표.

```json5
// openclaw.json에서도 설정 가능
{
  identity: {
    name: "Samantha",
    emoji: "🤖"
  }
}
```

### TOOLS.md — 환경 정보

호스트 환경의 특이사항, 경로, 별칭, 위험한 명령어를 적어.

```markdown
# 환경
- OS: Ubuntu 22.04
- Node: 22.x (nvm 사용)
- Docker: /var/run/docker.sock

# 위험 명령어 (실행 전 확인)
- rm -rf
- docker system prune
```

## HEARTBEAT.md — 주기적 자동 체크

**30분마다 에이전트가 이 파일을 읽고 체크할 일이 있으면 실행해.**

할 일이 없으면 `HEARTBEAT_OK`만 보내고 끝 (비용 $0).

```markdown
## 아침 (8:00-8:30)
- Gmail에서 긴급 메일 확인 (제목에 "URGENT" 또는 상사 메일)
- 오늘 캘린더 일정 가져오기
- 텔레그램으로 모닝 브리핑 보내기

## 낮 (9:00-18:00)
- 30분마다 VIP 메일만 알려주기

## 저녁 (18:00)
- Slack #engineering 안 읽은 메시지 요약

## 항상
- 출발 12시간 전이면 항공편 상태 알려주기
- 밤 10시~아침 8시에는 URGENT 아니면 메시지 보내지 마
```

**팁:**
- **4KB 이내**로 유지해. 프롬프트에 들어가니까
- API 키나 비밀번호 넣지 마
- 비워두면 LLM 호출 자체를 건너뛰어 → 비용 $0
- 로컬 Ollama 모델로 하트비트 돌리면 비용 절약

```json5
// 하트비트 간격 설정
{
  agents: {
    defaults: {
      heartbeat: {
        every: "30m",      // 기본 30분
        activeHours: "08:00-22:00"  // 이 시간에만 실행
      }
    }
  }
}
```

## MEMORY.md — 영구 기억

에이전트가 항상 읽는 메모리 파일이야. **100줄 이내로 유지해.**

일별 노트는 `memory/YYYY-MM-DD.md`에 자동 생성돼.
주제별 폴더(`memory/people/`, `memory/projects/`)를 만들면 시맨틱 검색도 가능해.

---

## 메모리 컴팩션

대화가 길어지면 자동으로 **컴팩션**이 일어나.

1. 오래된 대화를 **요약본으로 압축**
2. 최근 메시지는 **원본 그대로 유지**
3. 핵심 결정사항, 파일 위치, 중요 맥락은 보존
4. 장황한 중간 과정은 삭제

컴팩션 직전에 에이전트가 자동으로 중요 정보를 메모리 파일에 저장해.

수동으로 하려면 `/compact` 명령어.

**주의**: 컴팩션 자체도 토큰을 써. 자주 하면 오히려 비용 증가.

## 실전 팁

**구체적으로 써.**

```
❌ "도움이 되는 어시스턴트가 되어줘"
✅ "답변은 3문장 이내로. 코드 블록은 실행 가능한 것만"
```

**만든 사람(Steinberger)의 팁:**

> "항상 모델한테 '질문 있어?' 라고 물어봐. 모델은 즉시 풀려고 하면서 추측하거든.
> 그 추측이 틀릴 때가 많아. 이 한마디가 품질을 확 올려."

**파일 분리 원칙:**
- 영구 규칙 → **AGENTS.md**
- 성격 → **SOUL.md**
- 취향 → **USER.md**
- **절대 섞지 마.** 각 파일의 역할이 명확해야 에이전트가 혼동 안 해.

## 자기 수정 에이전트

소스에서 설치했으면(`git clone`), 에이전트가 **자기 코드를 알고 있어**.

"이 동작이 마음에 안 들어. 바꿔줘"라고 하면
에이전트가 자기 소스 코드를 수정해.

에이전트한테 "HEARTBEAT.md가 낡았으면 알아서 업데이트해"라고 써두면,
체크리스트도 자동으로 진화해.

## 한 줄 정리

10개 마크다운 파일이 에이전트의 모든 걸 정의해. AGENTS.md에 규칙, SOUL.md에 성격, 나머지는 역할별로 분리.
