*검증 기준일: 2026-02-26 (공식 문서·릴리즈 기준)*

**설정은 `~/.openclaw/openclaw.json` 하나로 다 해.**

AI 모델, 메신저 연결, 예약 작업 — 전부 이 파일에 들어가.

---

## 기본 구조

```
~/.openclaw/
├── openclaw.json    ← 메인 설정 파일
├── workspace/       ← 에이전트가 파일 작업하는 공간
├── skills/          ← 설치한 스킬(플러그인)
├── logs/            ← 실행 로그
└── credentials/     ← 메신저 인증 정보
```

---

## 최소 설정

AI 모델만 연결하면 바로 쓸 수 있어.

```json5
{
  agent: {
    model: "anthropic/claude-sonnet-4-5"
  },
  channels: {
    telegram: {
      botToken: "여기에-BotFather가-준-토큰"
    }
  }
}
```

> 파일 형식은 **JSON5**야. 주석도 쓸 수 있고, 따옴표 없는 키도 돼.

---

## 설정 바꾸는 4가지 방법

| 방법 | 언제 써? |
|------|---------|
| `openclaw onboard` | 처음 설정할 때 (마법사) |
| `openclaw config set` | 한 줄 바꿀 때 |
| 대시보드 UI | 브라우저에서 편하게 |
| 직접 편집 | 파일 열어서 수정 |

```bash
# CLI로 모델 바꾸기
openclaw config set agent.model "anthropic/claude-opus-4-6"

# 설정 확인
openclaw config get agent.model

# 전체 진단
openclaw doctor
```

대시보드는 이렇게 열어:

```bash
openclaw dashboard
# http://127.0.0.1:18789/ 에서 확인
```

## Gateway 명령어

설정을 바꿨으면 **Gateway를 재시작**해야 적용돼.

```bash
openclaw gateway restart    # 재시작 (설정 변경 후 필수!)
openclaw gateway status     # 상태 확인
openclaw gateway --verbose  # 디버그 모드로 실행
```

Gateway가 메신저랑 AI 사이의 다리야.
이게 꺼져 있으면 아무것도 안 돼.

**팁**: `openclaw onboard --install-daemon` 했으면 Gateway가 자동으로 떠.
컴퓨터 껐다 켜도 알아서 실행돼.

## 환경변수로 키 관리하기

API 키를 설정 파일에 직접 넣으면 위험해.
환경변수를 쓰는 게 안전해.

```json5
{
  env: {
    ANTHROPIC_API_KEY: "sk-ant-너의-키"
  }
}
```

또는 `.env` 파일로 관리:

```bash
# ~/.openclaw/.env
ANTHROPIC_API_KEY=sk-ant-너의-키
TELEGRAM_BOT_TOKEN=1234567890:ABCdef...
```

## 설정 모듈화 (파일 나누기)

설정이 커지면 파일을 나눌 수 있어.

```json5
{
  agents: { $include: "./agents.json5" },
  channels: { $include: "./channels.json5" },
  cron: { $include: "./cron.json5" }
}
```

각 파일에 해당 설정만 넣으면 깔끔해.
