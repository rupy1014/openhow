## 공통 설치 (curl 한 줄)

**Step 1.** 터미널 열기

- **Mac**: `Cmd + Space` → "터미널" 검색
- **Windows**: WSL2 터미널 권장 (아래 Windows 섹션 참고)
- **Linux**: 터미널 열기

**Step 2.** 설치

```bash:터미널
# Mac / Linux
curl -fsSL https://openclaw.ai/install.sh | bash
```

```powershell:관리자 PowerShell
# Windows (PowerShell)
iwr -useb https://openclaw.ai/install.ps1 | iex
```

npm으로도 돼 (Node.js 22 이상 필요):

```bash
npm install -g openclaw@latest
```

**Step 3.** 설치 확인

```bash
openclaw --version
```

`2026.2.24` 같은 버전이 나오면 성공이야.

## 설치 후 바로 해야 할 것

**온보딩 마법사를 실행해.**

```bash
openclaw onboard --install-daemon
```

이거 하나면 다 잡아줘:
1. AI 모델 선택 (Anthropic 추천, 저렴하게 가려면 Gemini Flash)
2. API 키 입력
3. 메신저 채널 선택 (텔레그램, 왓츠앱 등)
4. 채널별 설정 안내
5. Gateway를 백그라운드 서비스로 등록 (`--install-daemon`)

끝나면 바로 메신저에서 AI한테 말 걸 수 있어.

## Windows에서 설치

**Windows에서는 WSL2로 설치하는 게 제일 편해.**
PowerShell 실행 정책 이슈를 피할 수 있고 안정성이 높아.

### WSL2로 설치 (권장)

**Step 1.** 관리자 PowerShell 열기

Windows 키 → "PowerShell" 검색 → 마우스 우클릭 → **관리자 권한으로 실행**

```powershell:관리자 PowerShell
wsl --install
```

설치 끝나면 **컴퓨터 재시작**해.
재시작하면 Ubuntu 터미널이 자동으로 열리고, 사용자 이름/비밀번호를 설정하라고 나와.

> 이미 WSL2가 설치돼 있으면 "이미 설치됨" 메시지가 뜨는데, 무시하고 다음으로 넘어가.

**Step 2.** Ubuntu 터미널 열기

Windows 키 → "Ubuntu" 검색 → 클릭

이제부터 모든 명령어는 이 Ubuntu 터미널에서 실행해.

**Step 3.** OpenClaw 설치

```bash:Ubuntu 터미널
curl -fsSL https://openclaw.ai/install.sh | bash
```

Node.js랑 Git이 없으면 자동으로 설치돼.
설치 끝나면 **터미널을 닫고 다시 열어야** `openclaw` 명령어가 인식돼.

**Step 4.** 온보딩 실행

```bash
openclaw onboard --install-daemon
```

마법사가 AI 모델 선택, 채널 연결, 데몬 설치를 단계별로 안내해.
끝에 "Gateway is running" 또는 "Daemon installed"가 뜨면 성공이야.

### PowerShell 네이티브 설치 (대안)

WSL2 설치가 어려운 환경(회사 PC, 보안 정책)에서만 써.
PowerShell 실행 정책 이슈가 발생할 수 있어.

```powershell:관리자 PowerShell
iwr -useb https://openclaw.ai/install.ps1 | iex
openclaw onboard --install-daemon
```

### Windows에서 자주 나는 오류

**"openclaw is not recognized" — 명령어를 못 찾아**

npm 전역 설치 경로가 PATH에 없는 거야.

```powershell
npm config get prefix
```

출력된 경로를 PATH에 추가하거나, WSL2로 전환하는 게 더 편해.

**"spawn git ENOENT" — Git이 없어**

[git-scm.com](https://git-scm.com)에서 Git for Windows를 설치하고 PowerShell을 재시작해.

**"스크립트 실행이 비활성화" — npm.ps1 차단**

PowerShell 실행 정책 제한이야.
**추천**: WSL2로 전환하면 이 문제가 안 생겨.

대안: 현재 사용자 범위에서만 실행 정책을 조정해 (회사 PC에서는 IT 정책에 따라 제한될 수 있어):

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

## macOS에서 설치

**macOS는 터미널 CLI가 제일 빨라. 앱 온보딩도 있어.**

### CLI 설치 (권장)

**Step 1.** 터미널 열기

`Cmd + Space` → "터미널" 검색 → 실행

**Step 2.** OpenClaw 설치

```bash:터미널
curl -fsSL https://openclaw.ai/install.sh | bash
```

Node.js 22+와 Git을 자동으로 확인하고 없으면 설치해.
끝나면 **터미널을 닫고 다시 열어야** `openclaw` 명령어가 인식돼.

**Step 3.** 온보딩

```bash
openclaw onboard --install-daemon
```

macOS에서는 LaunchAgent로 데몬이 설치돼.
재부팅해도 자동 실행돼.

**Step 4.** 대시보드 열기

```bash
openclaw dashboard
```

또는 브라우저에서 직접: `http://127.0.0.1:18789/`

브라우저에서 대시보드(Control UI)가 열리면 설치 완료야.

### 앱 온보딩 (GUI 방식)

터미널이 무서우면 앱으로도 할 수 있어.
GUI로 온보딩을 진행하는 방식이야.

**Step 1.** 첫 실행 시 macOS 경고 승인

"확인되지 않은 개발자" 경고가 뜨면:
시스템 설정 → 개인정보 보호 및 보안 → **허용** 클릭

**Step 2.** 로컬 네트워크 접근 허용

팝업에서 "허용" 클릭

**Step 3.** 보안 안내 확인

기본은 "개인용 1인 운영자" 모델이야.
여러 명이 쓰는 환경에서는 추가 락다운 설정이 필요해.

**Step 4.** Local vs Remote 선택

Gateway를 이 Mac에 둘지, 원격 서버에 둘지 선택해.
처음이면 Local로 시작해.

**Step 5.** 권한 요청 허용

Automation, Notifications, Accessibility 등 필요한 권한을 허용해.

### macOS에서 자주 나는 오류

**sharp 빌드 오류 (Homebrew libvips 충돌)**

Homebrew로 libvips가 설치돼 있으면 sharp 패키지 빌드가 충돌해.

```bash
SHARP_IGNORE_GLOBAL_LIBVIPS=1 npm install -g openclaw@latest
```

prebuilt 바이너리를 강제로 쓰게 하면 해결돼.

**sudo 권한 문제**

```bash
sudo npm install -g openclaw@latest
```

그래도 안 되면 nvm으로 Node.js를 다시 깔아봐.

## 설치 확인 체크리스트

설치 끝나면 이 4가지만 확인해.

**1. Gateway 상태 확인**

```bash
openclaw gateway status
```

"running" 또는 "reachable"이 나오면 성공이야.

**2. 전체 상태 확인**

```bash
openclaw status
```

Gateway, 채널, 데몬 상태가 다 보여.
빨간 글씨가 없으면 돼.

**3. 자동 진단**

```bash
openclaw doctor
```

문제가 있으면 자동으로 감지하고 수정 방법을 알려줘.
`--fix` 붙이면 자동으로 고쳐줘.

**4. 대시보드 열기**

```bash
openclaw dashboard
```

또는 브라우저에서 `http://127.0.0.1:18789/` 직접 접속해.

> 브라우저에서 Control UI가 열리고, 채팅 입력창이 보이면 **완전히 설치된 거야.**

### 온보딩이 설정하는 것들

| 단계 | 뭘 해? |
|------|--------|
| Model / Auth | AI 모델 선택 (Anthropic 권장) + API 키 입력 |
| Workspace | 기본 작업 공간 설정 (`~/.openclaw/workspace`) |
| Gateway | 포트, 바인드, 인증, 노출 방식 설정 |
| Channels | WhatsApp / Telegram / Discord 채널 연결 |
| Daemon | 백그라운드 서비스 등록 (재부팅 후 자동 실행) |
| Health check | 설치 상태 자동 점검 |
| Skills | 웹검색, 파일 관리 등 기능 모듈 설치 |

### 채널 연결 안 했어?

온보딩 때 건너뛰었으면 지금 해도 돼:

```bash
# Telegram 봇 연결
openclaw channels add telegram

# 연결된 채널 확인
openclaw channels list
```

## 문제가 생기면?

```bash
# 상태 확인
openclaw status

# 설정 진단
openclaw doctor

# 자동 수리
openclaw doctor --fix

# 실시간 로그
openclaw logs --follow
```

`openclaw doctor`가 대부분 문제를 알려줘. `--fix` 붙이면 자동으로 고쳐줘.

## 24시간 돌리고 싶으면?

| 방법 | 비용 | 특징 |
|------|------|------|
| **Mac에서 직접** | 전기세 월 ~500~2,000원 | M시리즈는 유휴 3~10W. 사용량에 따라 차이 |
| **DigitalOcean** | $6~20/월 | 원클릭 배포 지원 |
| **Oracle Cloud** | **무료** | ARM 4코어, 24GB RAM. 무기한 프리티어 |
| **Docker** | 서버 비용만 | `docker pull openclaw/openclaw:latest` |

> Mac Mini M시리즈 24시간 전기세 계산:
> 유휴 ~5W × 24h × 30일 = 3.6kWh/월 × 약 120원/kWh = **약 430원/월**.
> Node.js 프로세스 돌리면 ~10W 정도. 그래도 **월 ~1,000원** 수준이야.

서버 권장: CPU 2코어+, RAM 8GB, Ubuntu 22.04 LTS.

**주의**: Mac에서 돌리면 편하지만, 해킹당하면 홈 네트워크가 뚫려.
VPS가 더 안전해. 털려도 빈 리눅스 박스만 있으니까.
