*검증 기준일: 2026-02-26 (공식 문서·릴리즈 기준)*

**"매일 아침 6:30에 해줘" — 한 번 설정하면 매일 알아서 해.**

크론(cron)이라고 해. 예약 실행이야.

---

## CLI로 추가하기

```bash
# 매일 아침 7시 모닝 브리핑
openclaw cron add \
  --name "모닝브리핑" \
  --cron "0 7 * * *" \
  --tz "Asia/Seoul" \
  --session isolated \
  --message "서울 날씨, 오늘 일정 3개, 중요 이메일 요약해서 보내줘" \
  --announce \
  --channel telegram
```

```bash
# 등록된 작업 확인
openclaw cron list

# 삭제
openclaw cron remove 작업ID
```

---

## 스케줄 패턴

| 패턴 | 의미 |
|------|------|
| `0 7 * * *` | 매일 아침 7시 |
| `30 6 * * *` | 매일 아침 6:30 |
| `0 9 * * 1-5` | 평일 오전 9시 |
| `0 */2 * * *` | 2시간마다 |
| `0 18 * * 5` | 매주 금요일 오후 6시 |

외울 필요 없어. AI한테 "매일 아침 7시 패턴 알려줘"하면 돼.

---

## 세션 타입

| 타입 | 설명 | 언제 써? |
|------|------|---------|
| `isolated` | 매번 새 대화. 이전 대화 안 봐 | 대부분의 크론 (추천) |
| `main` | 네 메인 대화에 이어서 | 맥락이 필요한 작업 |

`isolated`가 기본이야. 이전 대화가 섞이지 않아서 깔끔해.

---

## 배달 방식

| 방식 | 설명 |
|------|------|
| `announce` | 메신저로 결과 전송 (기본) |
| `webhook` | 외부 URL로 POST |
| `none` | 내부 실행만, 알림 없음 |

```bash
# 텔레그램 특정 토픽으로 전송
openclaw cron add \
  --name "야간요약" \
  --cron "0 22 * * *" \
  --session isolated \
  --message "오늘 처리한 이메일, 완료한 작업, 내일 일정 요약해줘" \
  --announce \
  --channel telegram \
  --to "-1001234567890:topic:123"
```

## 실전 조합 예시

```bash
# 아침 브리핑
openclaw cron add --name "아침" --cron "0 7 * * *" --tz "Asia/Seoul" \
  --session isolated --announce --channel telegram \
  --message "날씨, 일정, 중요메일 요약. 150단어 이내"

# 평일 퇴근 리포트
openclaw cron add --name "퇴근" --cron "0 18 * * 1-5" --tz "Asia/Seoul" \
  --session isolated --announce --channel telegram \
  --message "오늘 처리한 작업, 내일 일정, 미완료 항목 정리"

# 서버 모니터링 (2시간마다)
openclaw cron add --name "서버체크" --cron "0 */2 * * *" \
  --session isolated --announce --channel discord \
  --to "channel:monitoring" \
  --message "서버 CPU, 메모리, 디스크 확인. 이상 있으면 알려줘"
```

## 비용 주의

크론 작업은 **실행할 때마다 토큰을 써**.

단순 셸(터미널 명령어)만 실행하면 토큰 안 들어.
근데 AI가 생각해야 하는 작업은 대화 한 번과 같은 비용이야.

```
5분마다 × AI 추론 = 하루 288번 = 월 ~$128 (GPT-4o 기준)
```

*비용 예시는 2026-02 공개 요금 기준이야.*

**팁**: 빈도를 낮추고 모델을 가볍게 설정해.
- 매시간 → 하루 2번
- Opus → Gemini Flash
