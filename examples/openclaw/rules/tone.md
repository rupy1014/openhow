# 톤앤매너 가이드

## 쉬운 말: 초등학생한테 설명하듯이

**전문 용어를 쓰지 마.** 써야 하면 바로 옆에 쉬운 말로 풀어줘.

| 이렇게 쓰지 마 | 이렇게 써 |
|----------------|-----------|
| 에이전트 | AI 비서. 시킨 일을 직접 하는 AI |
| 게이트웨이 | 중간 다리. 메신저랑 AI를 연결하는 통로 |
| 플러그인 | 확장 기능. 필요한 것만 골라 붙이는 부품 |
| 인스턴스 | 실행 중인 프로그램 하나 |
| SSRF | 서버가 엉뚱한 곳에 요청 보내는 보안 허점 |
| 크론 | 예약 실행. "매일 아침 7시에 해줘" |
| 토큰 | 글자 수 (대충 한글 1글자 = 2~3토큰) |
| 프롬프트 | AI한테 하는 말 (요청) |
| 웹훅 | 자동 알림. 뭔가 생기면 알아서 알려주는 연결선 |

**비유를 써.** 처음 나오는 개념은 일상 비유로 한 번 풀어줘.

```
❌ "게이트웨이가 멀티 플랫폼 메시지를 라우팅한다."
✅ "중앙 우체국 같은 거야. 텔레그램이든 디스코드든 메시지가 오면 AI한테 전달해줘."
```

**영어 단어는 처음 나올 때 한글 설명을 붙여.**

```
❌ "config.json을 수정한다."
✅ "설정 파일(config.json)을 수정해."
```

## 말투: 친구한테 말하듯이

초등학생 친구한테 설명하는 말투야. 딱딱한 서술체가 아니라 자연스러운 구어체로 써.

| 이렇게 쓰지 마 (서술체) | 이렇게 써 (구어체) |
|--------------------------|---------------------|
| ~한다, ~된다, ~이다 | ~해, ~돼, ~야, ~거야 |
| ~않는다, ~없다 | ~않아, ~없어 |
| ~읽는다, ~만든다 | ~읽어, ~만들어 |
| ~시작한다, ~적용된다 | ~시작해, ~적용돼 |
| ~해라, ~써라 | ~해봐, ~써봐 |

```
❌ "텔레그램 봇을 생성하고 토큰을 입력한다."
✅ "텔레그램 봇을 만들고 토큰을 넣어."

❌ "설정이 완료되면 메시지를 보낼 수 있다."
✅ "설정 끝나면 메시지 보낼 수 있어."
```

**존댓말은 절대 금지.**

| 금지 | 대체 |
|------|------|
| ~습니다, ~합니다 | ~해, ~돼 |
| ~됩니다, ~입니다 | ~돼, ~야 |
| ~해요, ~예요 | ~해, ~야 |
| ~해주세요 | ~해봐 |

## 인칭: 나 / 너

필자는 "나", 독자는 "너".

```
❌ "여러분"
✅ "너"

❌ "사용자는 이렇게 할 수 있습니다"
✅ "이렇게 하면 돼"
```

---

## 문장 길이와 호흡

**한 문장은 짧게.** 15자 이내가 이상적이야.

1~3문장 쓰고 빈 줄을 넣어. 빈 줄이 호흡이야. 덩어리지면 안 읽혀.

```
❌ 한 문장에 다 쑤셔넣기
OpenClaw를 설치하면 텔레그램이나 디스코드 같은 메신저에서 AI한테 말로 명령을 내려서 이메일 보내기, 파일 정리, 서버 모니터링 같은 일을 직접 시킬 수 있습니다.

✅ 짧게 쪼개서, 여러 줄로
OpenClaw를 설치하면 메신저에서 AI한테 일을 시킬 수 있어.

이메일 보내기, 파일 정리, 서버 감시 — 다 돼.
```

## 첫 문장

서론 없이 바로 시작해. **결론이나 질문으로 바로 들어가.**

```
❌ "이번 시간에는 OpenClaw에 대해 배워볼 거야."
✅ "텔레그램에서 '이메일 정리해줘'라고 치면 진짜 정리해."
```

---

## 강조 표현

중요한 한 줄은 **볼드**로.

인상적인 한 문장은 blockquote로.

```
> "말로 시키면 진짜 한다. 그게 OpenClaw이야."
```

과도하게 쓰지 마. 볼드가 많으면 아무것도 강조 안 한 거랑 같아.

## 구어체 연결어

| 써도 되는 것 | 쓰지 말 것 |
|------------|-----------|
| 근데, 그래서, 그러면, 왜냐면 | 그러나, 따라서, 이에 따라, 또한 |
| 어차피, 결국, 솔직히 | 결론적으로, 요컨대, 즉 |
| 그거, 이거, 저거 | 해당, 본, 상기의 |

---

## 질문형 제목

섹션 제목은 질문형이야. 고정 레이블을 쓰지 마.

`~해?`, `~야?`, `~어?`, `~ㄹ까?`로 써.

| 금지 | 대체 예시 |
|------|----------|
| "개요" / "소개" | "이게 뭐야?" / "왜 이걸 써?" |
| "과정 설명" | "어떻게 해?" |
| "직접 해보기" | Step 안에 코드 블록으로 통합 |
| "정리" | "한 줄 정리" (아티클 마지막에만) |

---

## 코드 블록과 예시

코드 블록은 Step 안에 넣어. 별도 섹션 만들지 마.

**"이렇게 해봐" 다음에 바로 코드.**

```
✅
**Step 2.** 아래를 복사해서 터미널에 붙여넣어.

```bash
npm install -g openclaw@latest
```
```

## 금지 패턴

| 하면 안 되는 것 | 이유 |
|----------------|------|
| "이번 시간에는..." | 서론 없이 시작해 |
| "~해볼 거야" | 바로 말해 |
| "~한다", "~된다" (서술체) | "~해", "~돼"로 써 |
| "정리하자면..." + 긴 요약 | 한 줄로 끝내 |
| 감탄사 ("정말!", "놀랍게도") | 정보만 남겨 |
| 단계가 6개 이상 | 아티클 두 개로 쪼개 |
