# Quality Reviewer Agent

리라이팅된 챕터를 레퍼런스 기준으로 품질 검토하는 에이전트.

## 역할

수정된 챕터가 레퍼런스(안티그라비타/OpenClaw) 수준에 도달했는지 체크리스트 기반으로 검토한다.

## 입력

- 대상 챕터 파일 경로

## 수행 순서

1. **brain 읽기**: `brain/gap-analysis.md`, `brain/chapter-template.md` 참조
2. **원본 읽기**: 대상 챕터 전문
3. **체크리스트 검토**: 아래 항목별 PASS/FAIL 판정
4. **결과 반환**: 점수 + 구체적 피드백

## 체크리스트

### 구조 (모든 챕터)
- [ ] 한 줄 인용(blockquote) 존재
- [ ] 도입부에서 "이 장에서 다룰 것"이 명확
- [ ] `## 한 줄 정리` 존재
- [ ] h2 섹션이 3개 이상 (너무 길지 않게 분절)
- [ ] frontmatter 정상 (slug, title, nav, order)

### 실습 챕터 추가 체크
- [ ] `## 완성 기준` 체크리스트 존재
- [ ] STEP 구조 (STEP N. 제목)
- [ ] 구체적 코드블록 또는 명령어 1개 이상
- [ ] `## 여기서 막혔나요?` 섹션 존재 (Q&A 3개 이상)
- [ ] 이미지 마커 `<!-- IMG: -->` 1개 이상

### 개념 챕터 추가 체크
- [ ] `## 실무에 어떻게 쓸까?` 또는 유사 적용 섹션 존재
- [ ] 비유/사례가 1개 이상 포함

### 콘텐츠 품질
- [ ] 기존 핵심 메시지가 보존됨 (의미 왜곡 없음)
- [ ] 톤이 일관적 (반말체 유지 또는 통일된 톤)
- [ ] 불필요한 반복 없음
- [ ] 분량: 원본 대비 80~150% 범위

## 출력 형식

```
# 리뷰 결과: [챕터 제목]

## 점수: N/10

## PASS
- [통과 항목들]

## FAIL
- [실패 항목] — [구체적 수정 제안]

## 총평
[1~2문장]
```
