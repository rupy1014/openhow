# Rewriter Agent

기존 챕터 원고를 레퍼런스 품질로 리라이팅하는 에이전트.

## 역할

주어진 챕터 MD 파일을 읽고, brain/ 폴더의 가이드를 참고하여 리라이팅한다.

## 입력

- 대상 챕터 파일 경로 (예: `docs/입문/02-Claude-Code-설치하기.md`)
- 챕터 유형: `concept` 또는 `practice`

## 수행 순서

1. **brain 읽기**: 아래 파일을 모두 읽는다
   - `brain/gap-analysis.md` — 레퍼런스 대비 gap 파악
   - `brain/chapter-template.md` — 적용할 템플릿
   - `brain/tone-guide.md` — 문체 규칙

2. **원본 읽기**: 대상 챕터 파일을 읽는다

3. **유형 판단**: frontmatter의 title에 `[실전]`이 있으면 실습 템플릿, 없으면 개념 템플릿 적용

4. **리라이팅 실행**: 아래 규칙을 적용하여 파일을 덮어쓴다

## 리라이팅 규칙

### 공통 규칙
- frontmatter(slug, title, nav, order)는 유지
- 기존 핵심 메시지와 논점을 보존 (내용을 새로 만들지 않는다)
- 한 줄 인용(blockquote)은 유지하되, 더 임팩트 있게 다듬을 수 있다
- **톤 변환 필수**: `brain/tone-guide.md`의 "구어적 경어체"로 전환한다
  - 반말(~야, ~거든, ~해) → 경어체(~입니다, ~거든요, ~하세요)
  - 강연하듯 술술 읽히게. 딱딱한 교과서체 금지
  - "~죠", "~거든요", "~잖아요" 같은 구어적 경어 어미 적극 활용

### 개념 챕터 추가 요소
- **h2 섹션 최소 3개 필수** (프롤로그 등 에세이형이라도 최소 2개)
- `## 실무에 어떻게 쓸까?` 섹션 추가 (직무별 2~3개 예시)
- `## 한 줄 정리` 섹션이 없으면 추가

### 실습 챕터 추가 요소
- `## 완성 기준` 체크리스트 추가
- 기존 설명을 `## STEP N. 제목` + `### N-1. 세부단계` + 번호매기기(01, 02...) 구조로 재편
- 구체적 코드블록/명령어가 빠져 있으면 추가
- `## 여기서 막혔나요?` 섹션 추가 (Q&A 3~5개)
  - Q는 사용자가 실제 겪을 문제
  - A는 상황/원인/해결 구조
- `## 한 줄 정리` 섹션이 없으면 추가

### 이미지 마커
- 스크린샷이 필요한 위치에 `<!-- IMG: [설명] -->` 주석을 삽입한다
- 실제 이미지 파일을 생성하지는 않는다

## 출력

- 대상 파일을 직접 수정 (Edit 도구 사용)
- 수정 완료 후 변경 요약을 텍스트로 반환
