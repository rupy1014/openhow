# AGENTS.md

이 워크스페이스는 Codex 입문 예제입니다.

## 목적

- 작은 정적 랜딩 페이지를 안전하게 수정하는 연습
- `AGENTS.md`, `config.toml`, `skills`가 어떻게 함께 작동하는지 체험
- 번들 built-in skill과 repo 전용 skill의 역할 차이 이해

## 작업 원칙

- 먼저 `site/` 폴더 구조를 읽고 설명합니다.
- 일반 계획은 번들 planning skill이나 일반 프롬프트로 처리하고, repo skill은 이 예제에 특화된 작업에만 씁니다.
- 한 번에 큰 변경을 하지 말고, 작은 단위로 제안합니다.
- UI 문구를 고칠 때는 과장된 마케팅 문장을 피합니다.
- 색상과 간격 수정은 기존 스타일 방향을 유지합니다.
- 작업이 끝나면 어떤 파일을 바꿨는지 요약합니다.

## 수정 범위

- 앱 코드는 `site/` 안에서만 수정합니다.
- 설정 예시는 `.codex/`와 `.agents/` 안에 있습니다.
- `site/.agents/skills`는 `site/` 폴더에서 Codex를 열었을 때만 같이 보이는 폴더 범위 skill 예시입니다.
- 이 예제에서는 외부 의존성을 새로 추가하지 않습니다.

## 완료 기준

- 요청한 변경이 실제 파일에 반영됩니다.
- 설명만 하지 않고, 바뀐 결과를 사용자가 확인할 수 있어야 합니다.
- 필요하면 다음에 어떤 작은 작업으로 이어가면 좋은지 한 줄 제안합니다.
