# 레퍼런스 vs 현재 원고 Gap 분석

## 레퍼런스 요약

### 안티그라비타 책 (01-1차_내지)
- 대상: 비개발자 (기획자/디자이너/마케터/대학생)
- 구조: DAY 0~6 (7일 여정), 각 DAY 안에 STEP 1/2/3
- 톤: 경어체 (~합니다)
- 핵심 장치:
  - **인물 사례 스토리**: 4명(김민준 기획자 34세, 이유진 디자이너 29세, 박준서 마케터 32세, 최민아 대학생 22세)의 좌절→극복 서사로 프롤로그 시작
  - **초세분화 STEP**: "01 웹 브라우저를 열고~", "02 버튼을 눌러~" 클릭 단위
  - **스크린샷 + 빨간박스**: 매 스텝마다 실제 화면, 클릭 위치 표시
  - **"여기서 막혔나요?"**: 매 챕터 끝 Q&A 트러블슈팅 4~6개
  - **로드맵 인포그래픽**: "7일간의 여정" 지도, DAY별 키워드(자신감/창작의즐거움/데이터관리능력/개인화/AI활용능력/서비스오너경험)
  - **TIP 박스**: 보충 설명용 아이콘 박스
  - **감성 풀페이지**: 전구 이미지 + "세상에 태어나지 못한 아이디어는, 처음부터 없었던 아이디어와 같다"

### OpenClaw 실전 가이드
- 대상: AI 도구를 실무에 붙이려는 직장인/개발자
- 구조: 31장 + 부록 5개 (300p+)
- 톤: 평어체 (~다)
- 핵심 장치:
  - **장면 기반 설명**: "아침 브리프 받는 장면", "회의 메모 장면" 등 실무 시나리오
  - **통일된 장 구조**: 들어가기 → 이해 → 흐름 → 전문가 팁 → 실무 적용
  - **개념 시각화**: "오픈클로 한 장 지도" 다이어그램
  - **비교 도식**: "브라우저 탭형 AI vs OpenClaw" 비교표
  - **실무 적용 섹션**: 직무별(사무직/영업/개발자/학생) 체감 포인트

## 현재 clauders_book 현황

- 42장 + 부록 3개, 6개 섹션
- 톤: 반말체 (~야, ~거든) — 친근하지만 출판용으로는 재검토 필요
- 글 품질: 개념 전달력은 좋음 (70% 기획론, 신입사원 비유 등)
- 프롤로그: 직접적, 솔직한 톤. 하지만 공감 스토리 약함

## 핵심 Gap 7가지

### Gap 1: 인물 사례 스토리 부재
- 현재: "마케터가 간단한 사내 도구를 만들고" (1줄 언급)
- 레퍼런스: 김민준(34세 기획자)의 2페이지짜리 서사 — 외주 견적 3000만원, 3개월 소요 → 좌절
- **필요**: 프롤로그 또는 1장에 2~3명의 구체적 인물 스토리 (이름/나이/직업/좌절/전환)

### Gap 2: 실습의 구체성
- 현재: "터미널을 열고, 설치 스크립트를 붙여넣고" (개념적)
- 레퍼런스: 실제 명령어 코드블록 + OS별 분기 + 매 단계 스크린샷
- **필요**: 코드블록, 실제 프롬프트 예시, 결과 화면 설명 (이미지는 별도)

### Gap 3: 트러블슈팅 코너 없음
- 현재: 없음
- 레퍼런스: "여기서 막혔나요?" — Q(파란)/A(회색) 형식, 원인/해결책 구조
- **필요**: 실습 챕터마다 `## 여기서 막혔나요?` 3~5개

### Gap 4: 로드맵/여정 시각화 없음
- 현재: index.md에 링크 목록만
- 레퍼런스: "7일간의 여정" 인포그래픽 + DAY별 감정 키워드
- **필요**: 전체 파트 흐름을 보여주는 여정 지도 (텍스트 기반이라도)

### Gap 5: 장 내부 구조 비통일
- 현재: 장마다 자유 구조 (h2 제목만 일관)
- 레퍼런스: 반복되는 템플릿 (들어가기/이해/흐름/팁/적용)
- **필요**: 공통 구조 정의 후 전 챕터 적용

### Gap 6: 비주얼 전략 부재
- 현재: 100% 텍스트
- 레퍼런스: 페이지당 1~2개 이미지 (스크린샷/도식/일러스트)
- **필요**: 이미지 삽입 위치 마커 (`![설치 화면](images/02-install-terminal.png)`) 선행 확보

### Gap 7: 문체 일관성
- 현재: 반말체, 온라인 콘텐츠 톤
- 안티그라비타: 경어체(~합니다), OpenClaw: 평어체(~다)
- **필요**: 출판 목적이면 톤 통일 결정 필요 (반말/평어/경어 중 택1)
