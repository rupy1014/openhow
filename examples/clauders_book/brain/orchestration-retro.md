# 오케스트레이션 회고 (프롤로그 1회차)

## 실행 결과

| Phase | 에이전트 | 소요 | 결과 |
|-------|--------|------|------|
| 1 | rewriter | ~84초 | 톤 변환 + 구조 정리 + IMG 마커 — 잘 됨 |
| 2 | story-injector | ~68초 | 3명 인물 사례 삽입 — 잘 됨 |
| 3 | troubleshoot-writer | skip | 개념 챕터라 해당 없음 |
| 4 | quality-reviewer | ~46초 | 9/10, FAIL 1건(h2 부족) |
| 수동 | h2 추가 | 즉시 | reviewer 피드백 반영 |

총 소요: ~200초 (3분 20초)

## 잘된 점

1. **톤 변환 품질 좋음**: "~야/~거든" → "~입니다/~거든요" 자연스럽게 전환. 딱딱하지 않음
2. **인물 사례 품질 좋음**: 이수현(기획자)/한소영(디자이너)/박재민(소상공인) — 직업/나이/금액/기간 디테일 구체적
3. **reviewer가 실제 문제를 잡음**: h2 부족 → 수동 수정으로 바로 해결
4. **원본 핵심 메시지 100% 보존**: 신입사원 비유, 70/30 원칙, 되는것/안되는것 모두 유지

## 발견된 개선점

### 1. rewriter가 h2를 안 넣음 → 템플릿 규칙 강화 필요
- 개념 챕터 템플릿에 "h2 최소 3개" 규칙이 있지만, 프롤로그를 에세이형으로 판단하고 skip함
- **대응**: rewriter 에이전트에 "프롤로그라도 h2 최소 2개는 필수" 규칙 추가

### 2. story-injector가 h3로 삽입 → h2와 불일치
- 인물 사례를 h3(###)로 넣었는데, reviewer가 h2 부족을 지적
- **대응**: story-injector에 "prologue 모드일 때 감싸는 h2 섹션 제목을 함께 생성" 규칙 추가

### 3. Phase 2 후 Phase 4로 바로 갔는데, 수동 수정이 필요했음
- reviewer FAIL → 수동 수정 → 재검토 없이 종료
- **대응**: FAIL 시 rewriter로 루프백하는 흐름을 실제로 실행해야 함 (현재는 수동)

### 4. 병렬화 가능 구간 미활용
- rewriter와 story-injector는 순차 실행했지만, story-injector가 원본 메시지를 보존한다면 병렬 가능한가? → 아님. story-injector는 rewriter 결과물 위에 작업해야 톤이 일치. 순차가 맞음.
- 단, 다른 챕터간 병렬은 가능: 프롤로그와 01장을 동시 처리 가능

## 오케스트레이션 개선안

### v2 워크플로우

```
Phase 1: rewriter (톤 + 구조 + h2 필수)
    ↓
Phase 2: story-injector (prologue/inline, h2 감싸기 포함)
    ↓
Phase 3: troubleshoot-writer (실습 챕터만)
    ↓
Phase 4: quality-reviewer
    ↓
    ├── PASS (8+/10) → 완료
    └── FAIL → Phase 1로 FAIL 항목만 수정 지시 → Phase 4 재실행
```

### 챕터 간 병렬화
- 같은 유형 챕터는 2~3개 동시 파이프라인 가능
- 예: 01장 + 02장 + 03장 병렬 → 각각 독립 파이프라인
