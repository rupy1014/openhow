## 기본 도구

### 터미널

명령어를 입력하는 창입니다.

검은 화면처럼 보여도 겁먹을 필요 없습니다.

컴퓨터와 글로 대화하는 창이라고 생각하시면 됩니다.

### VS Code

코드 편집기입니다.

메모장의 고급 버전이라고 보시면 됩니다.

파일을 보고, 수정하고, 터미널도 같이 열 수 있습니다.

### Codex

OpenAI의 코딩 에이전트 도구입니다.

터미널, IDE, 클라우드 작업 환경에서 쓸 수 있고,
`AGENTS.md`, 설정 파일, skills, 자동화 흐름과 함께 자주 언급됩니다.

### Claude Code

터미널에서 AI와 직접 일하는 도구입니다.

파일 읽기, 수정, 명령 실행까지 연결됩니다.

### Git

변경 기록 저장 도구입니다.

"저장"의 강화판이라고 보시면 됩니다.

언제 뭘 바꿨는지가 남습니다.

### GitHub

Git으로 관리하는 코드를 인터넷에 올려두는 곳입니다.

백업,
협업,
배포 연결에 자주 쓰입니다.

---

## 웹과 앱

### 프론트엔드

사용자에게 보이는 화면 쪽입니다.

버튼,
입력창,
레이아웃,
애니메이션이 여기에 들어갑니다.

### 백엔드

뒤에서 처리하는 서버 쪽입니다.

저장,
인증,
계산,
권한 확인이 여기에 들어갑니다.

### API

프로그램끼리 대화하는 통로입니다.

화면이 서버에게 "이 데이터 저장해줘"라고 말할 때 주로 씁니다.

### 엔드포인트

API 요청을 보내는 주소입니다.

어디로 요청할지 정해주는 문이라고 생각하시면 쉽습니다.

### 반응형

화면 크기에 따라 레이아웃이 바뀌는 방식입니다.

PC와 모바일에서 다르게 보이게 만드는 거죠.

## 저장과 배포

### 메모리

지금 실행 중일 때만 살아 있는 임시 상태입니다.

새로고침하면 사라집니다.

### localStorage

내 브라우저 안에 저장하는 방식입니다.

새로고침해도 남지만,
다른 기기에서는 안 보입니다.

### DB

데이터베이스입니다.

오래 남고,
다른 기기나 다른 사용자와 공유할 수 있는 저장소입니다.

### 환경변수

코드에 직접 쓰기 민감한 값을 따로 보관하는 방식입니다.

API 키를 여기에 넣습니다.

### 배포

내 컴퓨터에서만 돌던 결과물을 인터넷에 올리는 것입니다.

링크를 공유할 수 있게 되는 단계죠.

### Vercel

웹앱을 쉽게 배포하는 서비스입니다.

GitHub와 연결해서 많이 씁니다.

## AI와 자동화

### 프롬프트

AI에게 하는 요청입니다.

뭘 해달라고 말하는 문장 자체입니다.

### 컨텍스트

AI가 현재 참고하는 맥락입니다.

대화 내용,
파일 내용,
규칙 파일이 여기에 들어갑니다.

### 토큰

AI가 글을 세는 단위입니다.

대화가 길어질수록 토큰이 늘고,
비용과 품질에도 영향을 줍니다.

### Hallucination

AI가 자신 있게 틀리는 현상입니다.

없는 정보를 진짜처럼 말할 수 있어서 검증이 필요합니다.

### MCP

AI를 외부 도구와 연결하는 표준 방식입니다.

노션, 슬랙, 브라우저, DB 같은 바깥 도구를 붙일 때 씁니다.

### Webhook

"이 주소로 보내면 저 서비스가 받아준다"는 연결 방식입니다.

슬랙이나 디스코드 메시지 발송에 자주 씁니다.

### cron

정해진 시간에 프로그램을 실행하는 예약 장치입니다.

컴퓨터용 알람 시계라고 생각하시면 쉽습니다.

### 파이프라인

작업을 순서대로 연결한 흐름입니다.

수집 → 정리 → 생성 → 발송 같은 구조죠.

### 병렬 처리

여러 작업을 동시에 돌리는 것입니다.

뉴스 여러 개를 한꺼번에 요약할 때 같은 방식이 쓰입니다.

## 기획과 제품

### 가설

"이 문제를 이렇게 풀면 사람들이 쓸 것이다"라는 예상입니다.

정답이 아니라 검증해야 하는 믿음입니다.

### MVP

핵심 경험만 되는 최소 버전입니다.

처음부터 다 넣지 않고 가장 작은 버전으로 시작할 때 씁니다.

### PRD

제품 요구사항 문서입니다.

무엇을 만들고,
누구를 위한 것이고,
무엇을 안 만들지 적는 한 장짜리 기준표라고 보시면 됩니다.

### 레퍼런스

AI에게 보여주는 참고 자료입니다.

스크린샷,
비슷한 사이트,
문체 예시 같은 것들이죠.

### 린 스타트업

가장 작은 걸 먼저 만들고,
써보고,
고치는 방식입니다.

바이브 코딩과 궁합이 좋습니다.
