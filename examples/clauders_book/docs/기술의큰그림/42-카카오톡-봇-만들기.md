> 슬랙만 쓰는 사람보다 카카오톡 쓰는 사람이 훨씬 많습니다.

앞에서 만든 뉴스봇이나 자동화 결과를 한국에서 가장 익숙한 채널로 보내보는 장입니다. 바로 카카오톡이죠.

---

## 완성 기준

- [ ] 카카오 디벨로퍼스에서 앱을 등록하고 토큰을 발급받을 수 있다
- [ ] "나에게 보내기" API로 메시지를 전송할 수 있다
- [ ] 기존 뉴스봇의 발송 채널을 카카오톡으로 연결할 수 있다
- [ ] 환경 변수로 토큰을 관리할 수 있다

---

## STEP 1. 카카오톡 자동화의 두 방향 이해하기

### 1-1. 나에게 보내기

01 내 카카오톡으로 메시지를 보내는 방식입니다.

02 알림, 리마인드, 뉴스 브리핑에 잘 맞습니다.

03 이 책에서는 이 방식을 중심으로 진행합니다. 개인 자동화와 연결하기 좋고, 구조도 더 단순하기 때문입니다.

### 1-2. 채널 챗봇

01 고객과 대화하는 더 큰 구조입니다.

02 상담, 예약, 문의 응대 같은 데 쓰입니다.

03 이건 별도의 인프라와 심사가 필요하므로, 이 장에서는 다루지 않습니다.

> **확인**: "나에게 보내기"와 "채널 챗봇"의 차이를 설명할 수 있으면 됩니다.

---

## STEP 2. 카카오 디벨로퍼스 설정

### 2-1. 앱 등록하기

01 [카카오 디벨로퍼스](https://developers.kakao.com)에 접속해서 로그인합니다.

02 "내 애플리케이션" → "애플리케이션 추가하기"를 클릭합니다.

03 앱 이름과 사업자 정보를 입력합니다. 개인 용도라면 간단히 작성해도 됩니다.

<!-- IMG: [카카오 디벨로퍼스 애플리케이션 추가 화면] -->

### 2-2. 동의 항목 및 토큰 발급

01 "카카오 로그인" → "동의항목"에서 "카카오톡 메시지 전송"을 활성화합니다.

02 "REST API 키"를 복사해둡니다.

03 인가 코드를 받고, 이를 통해 액세스 토큰을 발급받습니다.

```bash
# 인가 코드 요청 URL (브라우저에서 열기)
https://kauth.kakao.com/oauth/authorize?client_id={REST_API_KEY}&redirect_uri={REDIRECT_URI}&response_type=code

# 토큰 발급
curl -X POST https://kauth.kakao.com/oauth/token \
  -d "grant_type=authorization_code" \
  -d "client_id={REST_API_KEY}" \
  -d "redirect_uri={REDIRECT_URI}" \
  -d "code={인가코드}"
```

04 발급받은 토큰은 환경 변수로 저장합니다.

```bash
# .env 파일
KAKAO_ACCESS_TOKEN=your_access_token_here
KAKAO_REFRESH_TOKEN=your_refresh_token_here
```

> **확인**: REST API 키와 액세스 토큰을 모두 확보했으면 성공입니다.

## STEP 3. 나에게 메시지 보내기

### 3-1. 메시지 전송 코드 작성

01 AI에게 이렇게 지시합니다.

> "카카오톡 나에게 보내기 API로 텍스트 메시지를 보내는 함수를 만들어줘. 환경 변수에서 토큰을 읽어오고, 실패하면 에러를 출력해줘."

02 핵심 API 엔드포인트는 이렇습니다.

```bash
curl -X POST https://kapi.kakao.com/v2/api/talk/memo/default/send \
  -H "Authorization: Bearer {ACCESS_TOKEN}" \
  -d 'template_object={
    "object_type": "text",
    "text": "뉴스봇 테스트 메시지입니다.",
    "link": { "web_url": "https://example.com" }
  }'
```

03 응답이 `{"result_code":0}`이면 성공입니다.

> **확인**: 내 카카오톡에 테스트 메시지가 도착하면 성공입니다.

### 3-2. 기존 뉴스봇과 연결

01 앞에서 만든 뉴스봇의 발송 함수를 찾습니다.

02 슬랙 웹훅 호출 부분을 카카오톡 API 호출로 교체합니다.

03 메시지 포맷만 카카오 템플릿 형식에 맞게 바꿔주면 됩니다.

구조는 슬랙과 거의 같습니다. 인증 정보가 있고, 특정 주소로, 형식 맞춘 데이터를 보내는 것이죠. 출구만 바뀐 겁니다.

> **확인**: 뉴스봇이 슬랙 대신 카카오톡으로 메시지를 보내면 성공입니다.

---

## STEP 4. 토큰 관리와 갱신

### 4-1. 환경 변수로 관리하기

01 토큰은 절대 코드에 직접 넣지 마세요. `.env` 파일에 넣고, `.gitignore`에 추가합니다.

02 AI에게 이렇게 지시하세요.

> ".env에서 KAKAO_ACCESS_TOKEN을 읽어서 사용하도록 바꿔줘."

### 4-2. 토큰 갱신 처리

01 카카오 액세스 토큰은 만료 기한이 있습니다.

02 리프레시 토큰으로 자동 갱신하는 로직을 추가하면, 매번 수동으로 발급받지 않아도 됩니다.

```bash
# 토큰 갱신
curl -X POST https://kauth.kakao.com/oauth/token \
  -d "grant_type=refresh_token" \
  -d "client_id={REST_API_KEY}" \
  -d "refresh_token={REFRESH_TOKEN}"
```

> **확인**: 토큰이 `.env`에만 있고, 코드에는 없으면 성공입니다.

## 여기서 막혔나요?

**Q. 토큰 발급 과정에서 redirect_uri 오류가 납니다.**

상황: 인가 코드를 받으려는데 "redirect URI mismatch" 에러가 나옵니다.

원인:
- 카카오 디벨로퍼스에 등록한 Redirect URI와 요청 URL의 redirect_uri가 다릅니다
- 프로토콜(http vs https)이나 포트 번호까지 정확히 일치해야 합니다

해결:
1. 카카오 디벨로퍼스 → "플랫폼" → "Web" → "사이트 도메인"과 "Redirect URI"를 확인하세요
2. 개발용이라면 `http://localhost:3000/callback` 같은 주소를 등록하세요
3. URL의 슬래시(`/`) 하나까지 정확히 일치해야 합니다

**Q. 메시지 전송이 -401 에러로 실패합니다.**

상황: API 호출했는데 인증 에러가 나옵니다.

원인:
- 액세스 토큰이 만료되었을 수 있습니다
- "카카오톡 메시지 전송" 동의항목이 비활성화되어 있을 수 있습니다

해결:
1. 리프레시 토큰으로 새 액세스 토큰을 발급받으세요
2. 카카오 디벨로퍼스에서 "동의항목" → "카카오톡 메시지 전송"이 활성화되어 있는지 확인하세요
3. 토큰 앞에 `Bearer ` (공백 포함)가 빠져 있진 않은지 확인하세요

**Q. 뉴스봇은 잘 되는데 카카오톡만 안 됩니다.**

상황: 슬랙으로는 잘 가는데 카카오톡으로 바꾸니 메시지가 안 옵니다.

원인:
- 메시지 포맷이 카카오 템플릿 형식에 맞지 않을 수 있습니다
- `template_object`가 JSON 문자열이 아니라 객체로 전달되고 있을 수 있습니다

해결:
1. `template_object`는 JSON 문자열로 변환해서 전달해야 합니다
2. AI에게 "카카오 메시지 API의 template_object 형식을 확인해줘"라고 물어보세요
3. 필수 필드(`object_type`, `text`, `link`)가 빠져 있진 않은지 확인하세요

**Q. 한 번 붙여보면 다른 채널도 할 수 있나요?**

상황: 카카오톡 외에 텔레그램이나 이메일도 붙이고 싶습니다.

원인:
- 사실 대부분의 메시지 채널은 구조가 비슷합니다

해결:
1. 핵심 구조는 동일합니다: 인증 → 엔드포인트 → 메시지 포맷 → 전송
2. 발송 함수만 채널별로 하나씩 만들어두면, 나중에 갈아끼우기가 쉽습니다
3. AI에게 "텔레그램 봇 API로 메시지 보내는 함수도 만들어줘"라고 하면 같은 패턴으로 만들어줍니다

## 한 줄 정리

카카오톡 봇은 자동화 시스템의 마지막 출구를 한국에서 가장 현실적인 메신저로 바꾸는 실전입니다. 한 채널을 붙여보면, 다른 채널도 같은 구조로 확장할 수 있습니다.
