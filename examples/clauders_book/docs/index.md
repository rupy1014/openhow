# 바로바로 자동화 with ChatGPT

> “이런 것도 되나요?”
>
> 된다. 그런데 그냥 되는 게 아니라,
> **어떤 도구를 왜 고르는지**부터 알면 훨씬 빨리 된다.

이 책은 AI를 검색처럼만 써보던 사람이,
**내 일을 조금씩 줄이는 자동화**부터 시작하게 만드는 입문서야.

초반에는 바로 Codex를 설치한다.
그다음에는 왜 Codex가 반복 작업과 자동화에 잘 맞는지,
Claude Code와 비교하면 어디서 결이 갈리는지 설명한다.

그리고 거기서 멈추지 않고,
**oh-my-codex(OMX)**까지 얹어서 왜 Codex를 더 오래, 더 안정적으로,
더 팀처럼 굴릴 수 있는지까지 안내한다.

즉,
이 책은
- Codex 설치와 첫 실행
- Codex의 강점 이해
- OMX까지 설치해서 운영 레일 깔기
- 첫 반복 작업 자동화 체감
- 바이브코딩과 자동화 이론 이해
- 실전 자동화 연결
순서로 독자를 데려가는 책이야.

:::success 이 책을 다 읽고 나면
- Codex를 단순한 코딩 도구가 아니라 **반복 작업 자동화 도구**로 보게 돼
- Claude Code와 Codex를 비교할 때 “무엇이 더 좋다”가 아니라 **언제 무엇을 써야 하는지** 설명할 수 있게 돼
- OMX를 왜 얹는지 이해하고, 설치 후 바로 써먹는 흐름을 잡게 돼
- 작은 서비스를 만들고 배포한 뒤 자동화 개념까지 연결하게 돼
- 작은 자동화 하나쯤은 직접 끝내볼 수 있게 돼
:::

## 이 책은 어떻게 흘러가?

| 파트 | 독자 변화 | 중심 질문 |
|---|---|---|
| Part 1. Codex로 자동화 시작하기 | 호기심 → 첫 체감 | 왜 Codex로 시작하고, 왜 OMX까지 얹어야 하지? |
| Part 2. 기초 다지기 — 바이브코딩 입문 | 체감 → 이해 | 서비스는 어떻게 만들고, 왜 여기서 자동화 이해가 생기지? |
| Part 3. 자동화 이론 입문 | 이해 → 구조 파악 | 자동화는 원래 어떤 구조로 굴러가지? |
| Part 4. 실전 자동화 | 구조 파악 → 실무 연결 | 내 일엔 어디부터 붙이면 좋지? |
| Part 5. 끝내보기 | 관찰 → 실행 | 나도 하나 끝낼 수 있을까? |
