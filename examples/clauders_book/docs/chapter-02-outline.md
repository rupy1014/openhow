# 2장 개요 — Codex는 어디에 강한가

## 부제

**Claude Code와 비교해서 보는 반복 작업 자동화**

---

## 이 장의 역할

이 장은 독자에게 **도구 선택의 이유**를 주는 장이야.

1장에서 설치와 첫 실행까지 갔다면,
독자는 이제 이런 상태야.

- 일단 켜보긴 했다
- 근데 왜 하필 Codex인지 아직은 잘 모르겠다
- Claude Code랑 뭐가 다른지도 궁금하다

그래서 2장은
**Codex가 특히 강한 장면이 무엇인지 설명해주는 장**이 돼야 해.

핵심은 “누가 더 세냐”가 아니야.

- Claude Code는 어디에 잘 맞는지
- Codex는 어디서 더 빨리 체감되는지
- 왜 이 책이 반복 작업 자동화 입문서에서 Codex를 먼저 잡는지

이걸 납득시키는 게 목적이야.

---

## 이 장을 읽고 남아야 하는 감정

1. **아, Codex는 대화형 도구라기보다 작업을 굴리는 느낌이구나**
2. **반복 작업, 배치 실행, 자동화 쪽에선 Codex가 먼저 체감되겠네**
3. **Claude Code와 경쟁이라기보다 역할이 다른 거구나**

---

## 이 장의 한 줄 목표

**Codex와 Claude Code의 결 차이를 설명하고, Codex가 반복 작업 자동화 입문에 적합한 이유를 이해하게 한다.**

---

## 추천 소제목 구조

## Codex와 Claude Code는 뭐가 다를까?

- 둘 다 터미널에서 도는 코딩 에이전트라는 공통점
- 하지만 대화 중심과 작업 실행 중심이라는 결 차이
- “더 좋다”가 아니라 “더 잘 맞는 장면이 다르다”는 프레임 제시

## 왜 Codex는 반복 작업에서 체감이 빠를까?

- `codex exec` 같은 비대화형 실행
- 같은 패턴을 여러 번 돌리는 작업에 강하다는 점
- 배치 실행과 자동화 흐름에 잘 들어간다는 점

## Claude Code와 비교하면 어떤 순간에 차이가 날까?

- 전략 짜기 / 구조 논의 / 긴 맥락 유지
- 짧고 반복적인 작업을 여러 번 실행하기
- 혼자 오래 붙들고 대화하느냐, 여러 작업을 레일에 태우느냐의 차이

## 이 책은 왜 Codex로 시작할까?

- 이 책의 출발점이 “코드 품질 논쟁”이 아니라 “내 시간 줄이기”이기 때문
- 초보 독자에게 반복 작업 자동화 체감을 먼저 주는 쪽이 더 중요하기 때문
- 이후 Part 2, Part 3에서 서비스와 구조 이해로 확장할 수 있기 때문

## 그러면 둘 중 하나만 써야 할까?

- 아니다. 같이 쓰는 흐름도 가능하다고 짚기
- 다만 입문 초반에는 도구를 하나의 관점으로 먼저 이해하는 게 중요하다고 안내
- 이 책에서는 Codex를 메인 레일로 잡고 나중에 확장한다고 정리

---

## 이 장에서 꼭 넣을 장면

### 장면 1. 비교표를 보는 독자의 질문

- “그래서 뭘 깔아야 해요?”
- “둘 다 비슷해 보이는데 왜 Codex부터예요?”

### 장면 2. 반복 작업 예시가 들어오는 순간

- 문서 초안 여러 개 만들기
- 폴더 안 파일 정리하기
- 같은 형식의 작업을 여러 번 돌리기

이때 독자가 “아, 여기서 Codex가 잘 맞겠네”를 느껴야 해.

### 장면 3. 경쟁 구도가 아니라 역할 구도로 이해하는 순간

- Claude Code는 생각을 오래 이어가는 쪽
- Codex는 작업을 반복 가능하게 굴리는 쪽

이 정리가 돼야 초보 독자가 도구 선택에서 덜 흔들린다.

---

## 이 장에서 피할 것

- 브랜드 팬덤처럼 비교하기
- 성능 비교만 길게 하기
- 독자의 실제 반복 업무와 연결하지 않기
- `codex exec` 같은 특징을 기능 소개로만 끝내기

이 장은 도구 리뷰가 아니라,
**자동화 관점의 도구 선택 장**이어야 해.
