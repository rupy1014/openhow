> 자동화가 자동화답게 되려면
>
> 내가 안 눌러도 돌아야 합니다.

여기서부터 진짜 자동화 감각이 들어옵니다.

지금까지는 반자동이었을 수 있습니다. 여러분이 직접 열고, 명령하고, 돌리는 방식이었죠.

그런데 정기 실행이 붙는 순간 달라집니다. 매일, 매주, 정해진 시간에 아무도 안 눌렀는데도 돌아가거든요.

이번 장에서는 macOS의 cron을 이용해 스크립트를 정기 실행하는 방법을 다룹니다.

---

## 완성 기준

- [ ] cron이 무엇인지 설명할 수 있다
- [ ] `crontab -e`로 작업을 등록할 수 있다
- [ ] 매일 아침 8시에 스크립트가 자동 실행된다
- [ ] 실행 로그를 확인할 수 있다

---

## STEP 1. cron 이해하기

### 1-1. cron은 컴퓨터용 알람 시계

아주 단순하게 말하면, cron은 **컴퓨터용 알람 시계**입니다.

알람 시간이 되면 소리를 내는 대신, 지정한 스크립트를 실행합니다.

예를 들면 이런 식이죠.

- 매일 아침 8시 → 뉴스봇 실행
- 매주 월요일 9시 → 리포트 생성
- 밤 11시 → 로그 정리

자동화에서 스케줄은 뼈대 역할을 합니다.

### 1-2. 왜 스케줄이 중요한가

아무리 좋은 자동화도 직접 눌러야만 돌면 아직 반쪽입니다.

스케줄이 붙으면 습관을 시스템으로 옮길 수 있습니다.

- 매일 챙기던 업계 뉴스
- 매주 하던 보고서
- 자꾸 잊던 리마인드

이런 걸 사람 기억이 아니라 기계 일정으로 옮기는 거예요. 사람은 자꾸 잊지만, 스케줄은 안 잊거든요.

> **확인**: "cron = 정해진 시간에 스크립트를 자동 실행하는 장치"라는 개념이 잡혔으면 OK입니다.

---

## STEP 2. cron 표현식 익히기

### 2-1. 기본 문법

cron 표현식은 다섯 칸으로 이루어져 있습니다.

```
분  시  일  월  요일  명령어
*   *   *   *   *     실행할 스크립트
```

| 위치 | 의미 | 범위 |
|------|------|------|
| 1번째 | 분 | 0-59 |
| 2번째 | 시 | 0-23 |
| 3번째 | 일 | 1-31 |
| 4번째 | 월 | 1-12 |
| 5번째 | 요일 | 0-7 (0과 7 모두 일요일) |

### 2-2. 자주 쓰는 예시

```bash
# 매일 아침 8시
0 8 * * * /path/to/script.sh

# 매주 월요일 오전 9시
0 9 * * 1 /path/to/script.sh

# 매일 밤 11시
0 23 * * * /path/to/script.sh

# 5분마다
*/5 * * * * /path/to/script.sh

# 평일(월~금) 아침 8시
0 8 * * 1-5 /path/to/script.sh
```

<!-- IMG: [cron 표현식 다섯 칸 구조 시각화 다이어그램] -->

> **확인**: `0 8 * * *`이 "매일 아침 8시"라는 걸 읽을 수 있으면 OK입니다.

---

## STEP 3. 실행할 스크립트 준비

### 3-1. 간단한 테스트 스크립트 만들기

먼저 잘 돌아가는지 확인할 간단한 스크립트를 만듭니다.

```bash
cat > ~/automation/projects/hello-cron.sh << 'EOF'
#!/bin/bash
echo "[$(date)] Hello from cron!" >> ~/automation/logs/hello-cron.log
EOF

chmod +x ~/automation/projects/hello-cron.sh
```

### 3-2. 수동 테스트

```bash
# 직접 실행해서 잘 되는지 확인
~/automation/projects/hello-cron.sh

# 로그 확인
cat ~/automation/logs/hello-cron.log
```

> **확인**: 로그 파일에 날짜와 "Hello from cron!" 메시지가 찍혀 있으면 성공입니다.

---

## STEP 4. crontab에 등록하기

### 4-1. crontab 편집

```bash
crontab -e
```

에디터가 열리면 아래 내용을 추가합니다.

```bash
# 매일 아침 8시 테스트 스크립트 실행
0 8 * * * ~/automation/projects/hello-cron.sh
```

저장하고 나옵니다.

### 4-2. 등록 확인

```bash
crontab -l
```

방금 등록한 내용이 보이면 성공입니다.

### 4-3. 빠른 테스트 (1분 뒤 실행)

8시까지 기다리기 어려우니, 테스트용으로 1분 뒤 실행을 등록해봅니다.

```bash
# 현재 시간 확인
date

# 예: 지금이 14:32이면 14:33에 돌리기
crontab -e
# 33 14 * * * ~/automation/projects/hello-cron.sh
```

1분 뒤 로그를 확인합니다.

```bash
cat ~/automation/logs/hello-cron.log
```

<!-- IMG: [crontab -l 실행 결과 화면] -->

> **확인**: 로그 파일에 새 줄이 추가되어 있으면, cron이 정상 동작하는 겁니다.

---

## STEP 5. 실전 스크립트 연결

### 5-1. 실제 자동화 스크립트 등록

테스트가 끝났으면, 실제로 돌리고 싶은 스크립트를 연결합니다.

```bash
crontab -e
```

```bash
# 매일 아침 8시 뉴스 수집
0 8 * * * ~/automation/projects/news-collect.sh >> ~/automation/logs/news.log 2>&1

# 매일 아침 8시 5분 요약
5 8 * * * ~/automation/projects/news-summary.sh >> ~/automation/logs/summary.log 2>&1

# 매일 아침 8시 10분 발송
10 8 * * * ~/automation/projects/news-send.sh >> ~/automation/logs/send.log 2>&1
```

### 5-2. 로그 리다이렉션 이해하기

`>> ~/automation/logs/news.log 2>&1` 부분이 중요합니다.

- `>>` — 기존 로그 뒤에 이어서 기록
- `2>&1` — 에러 메시지도 같은 파일에 기록

이게 없으면 "안 돌아갔는데 왜 안 돌아갔는지" 모르게 됩니다.

### 5-3. 주의사항: PATH 문제

cron은 여러분의 터미널과 다른 환경에서 실행됩니다. 스크립트 상단에 PATH를 명시해주세요.

```bash
#!/bin/bash
export PATH="/usr/local/bin:/opt/homebrew/bin:$PATH"

# 이하 스크립트 내용
```

> **확인**: 등록한 시간에 로그 파일이 갱신되면 완료입니다.

## 여기서 막혔나요?

**Q. crontab -e에서 에디터가 이상합니다.**

상황: vi 에디터가 열리는데 사용법을 모릅니다.

원인:
- macOS 기본 cron 에디터가 vi입니다

해결:
1. vi 기본 조작: `i`로 편집 모드 → 내용 입력 → `ESC` → `:wq` 입력 → 엔터
2. 에디터를 바꾸고 싶다면: `EDITOR=nano crontab -e`로 실행하세요

**Q. cron이 실행되지 않습니다.**

상황: 시간이 지났는데 로그 파일에 아무것도 안 찍힙니다.

원인:
- macOS에서 cron에 "전체 디스크 접근" 권한이 없을 수 있습니다
- 스크립트 경로가 틀렸을 수 있습니다

해결:
1. **시스템 설정 > 개인정보 보호 및 보안 > 전체 디스크 접근**에서 `/usr/sbin/cron`을 추가하세요
2. 스크립트 경로에 `~` 대신 절대 경로(예: `/Users/사용자이름/automation/...`)를 사용하세요
3. 스크립트에 실행 권한이 있는지 확인하세요: `chmod +x 스크립트경로`

**Q. 스크립트는 수동으로 돌리면 되는데 cron에서만 안 됩니다.**

상황: 터미널에서 직접 실행하면 잘 되는데, cron으로 돌리면 에러가 납니다.

원인:
- cron의 환경변수(PATH)가 터미널과 다릅니다
- 상대 경로를 썼을 수 있습니다

해결:
1. 스크립트 맨 위에 `export PATH="/usr/local/bin:/opt/homebrew/bin:$PATH"`를 추가하세요
2. 모든 파일 경로를 절대 경로로 바꾸세요
3. `.env` 파일을 사용한다면 스크립트에서 `source /절대경로/.env`로 로드하세요

**Q. cron 대신 launchd를 써야 하나요?**

상황: macOS에서는 cron보다 launchd가 권장된다는 글을 봤습니다.

원인:
- Apple은 공식적으로 launchd를 권장하지만, cron도 여전히 잘 동작합니다

해결:
1. 입문 단계에서는 cron이 훨씬 간단합니다. cron부터 시작하세요
2. 나중에 "로그인 시 자동 시작"이나 "실패 시 자동 재시도" 같은 기능이 필요하면 그때 launchd로 넘어가도 됩니다
3. 둘 다 배우는 것보다 하나를 확실히 쓰는 게 먼저입니다

---

## 한 줄 정리

정기 실행이 붙는 순간 도구는 시스템이 됩니다. cron은 그걸 가능하게 하는 가장 기본적인 알람 장치입니다.
