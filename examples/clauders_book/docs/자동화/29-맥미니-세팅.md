> 자동화는 결국 누군가 계속 켜져 있어야 합니다.
>
> 맥미니는 그 역할에 꽤 잘 맞습니다.

"왜 맥미니냐"보다 "작은 개인 서버를 갖는 감각"이 더 중요합니다.

항상 켜둘 수 있고, 조용하고, 전력 부담이 상대적으로 적고, 집이나 사무실 한쪽에서 계속 돌아가게 두기 좋죠. 그래서 개인 자동화 머신으로 자주 거론됩니다.

이번 장에서는 맥미니를 "개인 자동화 서버"로 세팅하는 과정을 다룹니다.

---

## 완성 기준

- [ ] 원격 접속(SSH 또는 화면 공유)으로 맥미니에 접근할 수 있다
- [ ] 잠자기 없이 24시간 켜져 있는 상태를 유지한다
- [ ] Node.js, Python 등 필요한 런타임이 설치되어 있다
- [ ] 프로젝트/로그 폴더 구조가 잡혀 있다

---

## STEP 1. 왜 별도 머신이 필요한지 이해하기

### 1-1. 노트북과의 차이

노트북은 여러분이 들고 다닙니다. 닫고, 이동하고, 배터리 신경 써야 하고, 항상 켜두기엔 불편하죠.

반면 자동화는 계속 켜져 있어야 할 때가 많습니다.

- 아침마다 돌고
- 밤마다 정리하고
- 여러분이 잘 때도 돌아갑니다

즉, 자동화 머신은 생활 머신과 분리되는 게 편합니다. 그 역할로 작은 데스크톱이 잘 맞는 거예요.

### 1-2. "운영용" 머신이라는 관점

이 기계는 "작업용"이 아니라 "운영용"입니다.

- 여러분이 자리를 비워도 계속 돌아갑니다
- 문제가 생기면 원격으로 확인합니다
- 모니터 없이도 원격으로 들어가고, 정해진 스크립트를 돌리고, 결과를 파일이나 메시지로 받습니다

이 감각이 생기면 맥미니가 그냥 컴퓨터가 아니라 **작은 운영 서버**처럼 느껴집니다.

> **확인**: "이 기계는 내가 안 써도 계속 돌아가는 머신이다"라는 관점이 잡혔으면 OK입니다.

## STEP 2. 원격 접속 설정

### 2-1. SSH 활성화

01 **시스템 설정** > **일반** > **공유**로 이동합니다.

02 **원격 로그인**을 켭니다.

03 터미널에서 접속을 테스트합니다.

```bash
ssh 사용자이름@맥미니IP주소
```

<!-- IMG: [macOS 시스템 설정 > 공유 > 원격 로그인 켜기 화면] -->

### 2-2. 화면 공유 활성화 (선택)

01 같은 **공유** 설정에서 **화면 공유**를 켭니다.

02 다른 맥에서 **Finder > 이동 > 서버에 연결** 또는 **VNC 클라이언트**로 접속합니다.

> **확인**: 다른 기기에서 맥미니에 원격으로 들어갈 수 있으면 성공입니다.

---

## STEP 3. 잠자기 방지 설정

### 3-1. 에너지 절약 설정

01 **시스템 설정** > **에너지 절약**(또는 **배터리** > **전원 어댑터**)으로 이동합니다.

02 **디스플레이가 꺼져 있을 때 자동으로 잠자기 방지**를 체크합니다.

03 **네트워크 접근 시 잠자기 해제**를 켭니다.

```bash
# 터미널에서 확인하는 방법
sudo pmset -g
# "sleep" 값이 0이면 잠자기 비활성화 상태
```

### 3-2. 터미널로 직접 설정 (확실한 방법)

```bash
# 잠자기 완전 비활성화
sudo pmset -a sleep 0
sudo pmset -a disablesleep 1

# 설정 확인
sudo pmset -g
```

> **확인**: `pmset -g`에서 sleep 값이 0으로 표시되면 성공입니다.

## STEP 4. 개발 환경 설치

### 4-1. Homebrew 설치

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

### 4-2. 필요한 런타임 설치

```bash
# Node.js (nvm 사용 권장)
brew install nvm
nvm install --lts

# Python
brew install python3

# Git
brew install git
```

### 4-3. 설치 확인

```bash
node --version
python3 --version
git --version
```

> **확인**: 세 명령어 모두 버전 번호가 출력되면 성공입니다.

## STEP 5. 폴더 구조 잡기

### 5-1. 표준 디렉토리 생성

자동화는 흩어지면 금방 관리가 어려워집니다. 초반에 구조를 잡아두세요.

```bash
mkdir -p ~/automation/projects    # 프로젝트 폴더
mkdir -p ~/automation/logs        # 로그 폴더
mkdir -p ~/automation/config      # 설정 파일
mkdir -p ~/automation/secrets     # 비밀 값 (API 키 등)
```

### 5-2. 비밀 값 관리

```bash
# .env 파일로 관리하는 예시
cat > ~/automation/secrets/.env << 'EOF'
OPENAI_API_KEY=sk-xxxxx
SLACK_WEBHOOK_URL=https://hooks.slack.com/services/xxxxx
EOF

# 권한 제한
chmod 600 ~/automation/secrets/.env
```

> **확인**: `ls ~/automation/`에서 projects, logs, config, secrets 폴더가 보이면 완료입니다.

## 여기서 막혔나요?

**Q. SSH 접속이 안 됩니다.**

상황: `ssh` 명령어를 쳤는데 "Connection refused"가 뜹니다.

원인:
- 원격 로그인이 꺼져 있을 수 있습니다
- 방화벽이 SSH 포트(22)를 막고 있을 수 있습니다
- IP 주소가 잘못됐을 수 있습니다

해결:
1. 맥미니에서 직접 **시스템 설정 > 공유 > 원격 로그인**이 켜져 있는지 확인하세요
2. `ifconfig | grep inet`으로 맥미니의 실제 IP를 확인하세요
3. 같은 네트워크에 있는지 확인하세요 (외부 접속은 포트포워딩 필요)

**Q. 잠자기 해제가 안 되는 것 같습니다.**

상황: 다음 날 아침에 보니 맥미니가 꺼져 있었습니다.

원인:
- GUI 설정만 바꾸고 `pmset` 값은 안 바꾼 경우가 많습니다
- macOS 업데이트 후 설정이 초기화되기도 합니다

해결:
1. `sudo pmset -a sleep 0 && sudo pmset -a disablesleep 1`을 다시 실행하세요
2. `sudo pmset -g`로 설정값을 확인하세요
3. macOS 업데이트 후에는 항상 다시 확인하세요

**Q. Homebrew 설치 중 에러가 납니다.**

상황: "Command Line Tools" 관련 에러가 뜹니다.

원인:
- Xcode Command Line Tools가 설치되지 않은 상태입니다

해결:
1. `xcode-select --install`을 먼저 실행하세요
2. 설치 완료 후 Homebrew 설치를 다시 시도하세요

**Q. 외부에서 맥미니에 접속하고 싶습니다.**

상황: 집 밖에서도 맥미니에 접근하고 싶습니다.

원인:
- 같은 네트워크가 아니면 직접 SSH 접속이 안 됩니다

해결:
1. 공유기에서 포트포워딩을 설정하세요 (외부 포트 → 맥미니 22번 포트)
2. 또는 Tailscale 같은 VPN 서비스를 사용하면 더 간편합니다
3. `brew install tailscale` 후 로그인하면 어디서든 접속할 수 있습니다
