> 자동화의 첫 타자는 뉴스봇이 좋습니다. 구조가 단순하고, 응용 범위가 넓거든요.

뉴스 요약 봇은 자동화 연습에 아주 좋은 프로젝트입니다. 이 장에서는 RSS로 뉴스를 수집하고, 필터링하고, 요약해서 발송하는 봇을 직접 만들어봅니다.

---

## 완성 기준

- [ ] 정해둔 RSS 소스에서 새 글을 가져올 수 있다
- [ ] 관심 키워드 기준으로 필터링하고 짧게 요약할 수 있다
- [ ] 요약 결과를 하나의 채널(슬랙, 이메일 등)로 발송할 수 있다

---

## STEP 1. 뉴스봇이 좋은 첫 자동화인 이유

### 1-1. 패턴이 단순합니다

뉴스봇의 전체 흐름은 네 단계가 거의 전부입니다.

1. 가져온다
2. 고른다
3. 요약한다
4. 보낸다

그리고 이 구조는 뉴스 말고도 엄청 많이 응용됩니다.

- 경쟁사 모니터링
- 채용공고 모니터링
- 업계 트렌드 요약
- 논문 요약
- 블로그 발행 알림

뉴스봇은 하나의 완성품이면서, **다른 자동화의 템플릿**이기도 합니다.

<!-- IMG: [뉴스봇 파이프라인 다이어그램 — 수집 → 필터링 → 요약 → 발송] -->

### 1-2. 이 프로젝트에서 진짜 배우는 것

겉으로는 "뉴스 요약" 같지만, 실제로 배우는 건 이겁니다.

- 데이터 수집
- 필터링
- 요약
- 발송
- 반복 실행

이 다섯 단계는 **자동화의 기본 골격**이죠. 이걸 한 번 몸으로 익히면 다음 프로젝트에서 훨씬 덜 헤맵니다.

> **확인**: "뉴스봇 = 자동화 기본 골격 연습"이라는 점을 이해했으면 다음으로 넘어가세요.

---

## STEP 2. RSS로 뉴스 수집하기

### 2-1. RSS를 쓰는 이유

많은 사이트가 새 글 목록을 기계가 읽기 좋은 형식으로 제공합니다. 크롤링보다 덜 무겁고, 구조도 단순하고, 자동화 시작점으로 좋습니다.

처음 자동화에선 이런 단순한 입력원이 좋습니다. 입력이 복잡하면 전체가 같이 어려워지거든요.

### 2-2. RSS 피드 가져오기

01 관심 분야의 RSS 피드 URL을 2~3개 준비합니다

예시:
```
https://news.ycombinator.com/rss
https://feeds.feedburner.com/TechCrunch
https://rss.nytimes.com/services/xml/rss/nyt/Technology.xml
```

02 Codex에게 RSS를 파싱하는 스크립트를 만들어달라고 합니다

```
이 RSS 피드 3개를 읽어서 최근 24시간 내 새 글 목록을
JSON 배열로 출력하는 Node.js 스크립트를 만들어줘.
각 항목은 title, link, pubDate, source를 포함해줘.
```

03 스크립트를 실행해서 데이터가 잘 들어오는지 확인합니다

```bash
node scripts/fetch-rss.js
```

<!-- IMG: [RSS 파싱 스크립트 실행 결과 — JSON 배열로 뉴스 목록이 출력된 터미널 화면] -->

> **확인**: 2~3개 소스에서 새 글 목록이 JSON으로 출력되면 성공입니다.

## STEP 3. 필터링과 요약하기

### 3-1. 필터 기준 정하기

모든 걸 다 요약하는 게 아닙니다. **나한테 중요한 것만 남기는 기준**이 중요하죠.

예를 들면 이렇습니다.

- AI 관련 키워드만
- 창업/제품 관련 소식만
- 개발 도구 업데이트만

필터가 없으면 정보가 너무 많아집니다. 자동화의 가치는 많이 가져오는 데 있지 않고, **내가 볼 만한 것만 남기는 데** 있습니다.

### 3-2. 요약 프롬프트 작성하기

01 요약 프롬프트는 짧지만 선명해야 합니다

```
아래 뉴스 목록에서 AI 코딩, 자동화, 제품 전략과
직접 관련 있는 내용만 남겨줘.
각 항목은 3줄 이내로 요약해줘.
과장된 마케팅 문구는 빼.
관련 없는 항목은 제거해.
```

02 Codex에게 필터링 + 요약을 하나의 스크립트로 묶어달라고 합니다

```
fetch-rss.js 결과를 받아서
위 프롬프트 기준으로 필터링하고 요약하는 스크립트를 만들어줘.
결과는 마크다운 형식으로 출력해줘.
```

03 실행해서 요약 결과를 확인합니다

> **확인**: 불필요한 기사가 걸러지고, 각 항목이 3줄 이내로 요약되어 나오면 성공입니다.

## STEP 4. 발송 채널 연결하기

### 4-1. 하나만 먼저 붙이세요

처음부터 카톡, 슬랙, 이메일 다 붙이지 마세요. **하나만 붙이세요.**

왜냐하면 채널은 바뀌어도 앞단 구조는 거의 안 바뀌거든요. 수집, 필터링, 요약이 먼저 안정돼야 합니다. 그래야 채널 추가도 쉽죠.

발송은 뒷단입니다. **핵심 골격이 먼저**입니다.

### 4-2. 슬랙 웹훅으로 발송하기 (예시)

01 슬랙 워크스페이스에서 Incoming Webhook URL을 생성합니다

02 Codex에게 요약 결과를 슬랙으로 보내는 기능을 추가해달라고 합니다

```
요약 결과를 슬랙 Incoming Webhook으로 보내는 기능을 추가해줘.
웹훅 URL은 환경변수 SLACK_WEBHOOK_URL에서 읽어줘.
메시지 포맷은 읽기 좋게 마크다운으로 해줘.
```

03 테스트 발송을 실행합니다

```bash
SLACK_WEBHOOK_URL="https://hooks.slack.com/services/..." node scripts/send-summary.js
```

<!-- IMG: [슬랙 채널에 뉴스 요약 메시지가 도착한 스크린샷] -->

> **확인**: 슬랙 채널에 요약 메시지가 도착하면 성공입니다. 가져오기 → 가공하기 → 보내기, 루프가 한 번 닫힌 것입니다.

---

## STEP 5. 반복 실행 설정하기

### 5-1. 전체를 하나의 명령으로 묶기

01 수집 → 필터링 → 요약 → 발송을 하나의 스크립트로 합칩니다

```bash
node scripts/news-bot.js
```

02 cron이나 GitHub Actions 같은 스케줄러로 매일 아침 자동 실행을 설정합니다

```bash
# crontab 예시: 매일 오전 8시 실행
0 8 * * * cd /path/to/project && node scripts/news-bot.js
```

> **확인**: 수동 실행 없이 정해진 시간에 뉴스 요약이 발송되면 완성입니다.

## 여기서 막혔나요?

**Q. RSS 피드 URL을 어디서 찾나요?**

상황: 관심 사이트의 RSS 주소를 모르는 경우입니다.

원인:
- 대부분의 블로그와 뉴스 사이트에 RSS가 있지만 눈에 안 띄는 경우가 많습니다

해결:
1. 사이트 URL 뒤에 `/rss`, `/feed`, `/atom.xml`을 붙여보세요
2. 브라우저에서 페이지 소스 보기로 `<link rel="alternate" type="application/rss+xml">`을 검색하세요
3. "사이트명 + RSS"로 구글링하면 대부분 나옵니다

**Q. 소스를 너무 많이 넣었더니 결과가 너무 길어요**

상황: 열 개 이상의 소스를 넣었더니 요약도 길어지고 읽기 싫어지는 경우입니다.

원인:
- 소스가 많으면 필터링 기준이 느슨해지고 전체 양이 폭증합니다

해결:
1. 소스를 2~3개로 줄이세요. 안정된 뒤에 하나씩 추가하면 됩니다
2. "최대 5개 항목만 남겨줘" 같은 상한선을 프롬프트에 넣으세요
3. 발송 메시지도 스크롤 없이 읽을 수 있는 길이를 목표로 하세요

**Q. 요약 품질이 들쭉날쭉합니다**

상황: 어떤 날은 요약이 좋고, 어떤 날은 핵심이 빠져 있는 경우입니다.

원인:
- 프롬프트의 필터 기준이 모호하면 결과도 불안정합니다

해결:
1. 프롬프트에 구체적인 키워드와 제외 조건을 명시하세요
2. "과장된 마케팅 문구 제거", "실질적 기술 변화만" 같은 가이드를 추가하세요
3. 좋았던 결과물을 예시로 프롬프트에 포함시키면 품질이 안정됩니다

**Q. 슬랙 웹훅 설정이 어렵습니다**

상황: 슬랙 워크스페이스 설정에서 웹훅을 어디서 만드는지 모르겠는 경우입니다.

원인:
- 슬랙 앱 설정 UI가 자주 바뀝니다

해결:
1. [api.slack.com/apps](https://api.slack.com/apps)에서 새 앱을 만드세요
2. "Incoming Webhooks" 기능을 활성화하세요
3. 원하는 채널을 선택하면 웹훅 URL이 발급됩니다
4. 슬랙이 복잡하면 이메일 발송부터 시작해도 괜찮습니다
