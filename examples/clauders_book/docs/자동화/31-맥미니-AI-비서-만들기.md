> 여기까지 오면 이제 보입니다.
>
> "작은 자동화들을 묶으면 AI 비서가 되겠구나."

앞에서 만든 조각들을 하나의 시스템으로 묶는 장입니다.

뉴스봇 하나만 있으면 아직 기능입니다. 뉴스봇 + 리마인드 + 요약 + 정기 실행 + 발송이 묶이면 그때부터 비서처럼 느껴지죠.

이번 장에서는 세 가지 자동화(아침 브리핑, 일정 리마인드, 일일 리포트)를 cron으로 묶어 하나의 AI 비서 시스템을 완성합니다.

---

## 완성 기준

- [ ] 아침 브리핑 스크립트가 매일 아침 정해진 시간에 실행된다
- [ ] 일정/할 일 리마인드가 동작한다
- [ ] 저녁 일일 리포트가 자동 발송된다
- [ ] 세 가지가 하나의 crontab에 등록되어 "비서 시스템"으로 동작한다
- [ ] 실패 시 로그에서 원인을 확인할 수 있다

---

## STEP 1. 전체 구조 설계하기

### 1-1. 비서가 하는 세 가지 일

사람 비서는 보통 이런 일을 합니다.

- 중요한 걸 먼저 가져오고
- 정리해서 보여주고
- 때가 되면 알려주고
- 반복 업무를 대신해줍니다

여러분의 AI 비서도 같은 구조입니다. 다만 사람 대신 스크립트, 스케줄, AI 요약, 메시지 발송이 그 역할을 나눠 맡는 거예요.

처음부터 만능 비서를 만들지 마세요. 세 가지 정도면 충분합니다.

| 기능 | 시간 | 역할 |
|------|------|------|
| 아침 브리핑 | 08:00 | 뉴스/업계 정보 요약 발송 |
| 일정 리마인드 | 08:30 | 오늘 일정 + 할 일 알림 |
| 일일 리포트 | 21:00 | 하루 결과 정리 발송 |

### 1-2. 하루 흐름으로 보기

**아침** — 뉴스와 일정이 한 메시지로 옵니다.

**낮** — 중요한 조건이 생기면 짧은 알림이 옵니다.

**저녁** — 하루 요약과 놓친 항목이 정리돼 옵니다.

이렇게 되면 "도구 세 개"가 아니라 "하루를 챙겨주는 흐름"처럼 느껴집니다. 비서라는 말은 이때부터 어울리죠.

> **확인**: 세 가지 기능의 역할과 실행 시점이 머릿속에 그려지면 OK입니다.

## STEP 2. 아침 브리핑 만들기

### 2-1. 스크립트 작성

```bash
cat > ~/automation/projects/morning-brief.sh << 'EOF'
#!/bin/bash
export PATH="/usr/local/bin:/opt/homebrew/bin:$PATH"
source ~/automation/secrets/.env

LOG=~/automation/logs/morning-brief.log
echo "[$(date)] === 아침 브리핑 시작 ===" >> $LOG

# 1. 뉴스 수집 (예: RSS 또는 API)
# 여기에 뉴스 수집 스크립트 호출
# node ~/automation/projects/collect-news.js >> $LOG 2>&1

# 2. AI 요약
# node ~/automation/projects/summarize.js >> $LOG 2>&1

# 3. 슬랙/카톡 발송
# node ~/automation/projects/send-slack.js >> $LOG 2>&1

echo "[$(date)] === 아침 브리핑 완료 ===" >> $LOG
EOF

chmod +x ~/automation/projects/morning-brief.sh
```

### 2-2. 발송 형식 설계

비서는 길게 말하는 사람보다 핵심만 정리하는 사람이 더 낫습니다.

메시지 형식을 고정하세요.

```
📰 오늘의 브리핑 (2026-03-27)

[주요 뉴스]
1. 제목 — 한 줄 요약 (링크)
2. 제목 — 한 줄 요약 (링크)
3. 제목 — 한 줄 요약 (링크)

[오늘 액션]
- 확인할 것 1개
```

> **확인**: 스크립트를 수동 실행해서 로그에 "시작/완료"가 찍히면 OK입니다.

## STEP 3. 일정 리마인드 만들기

### 3-1. 스크립트 작성

```bash
cat > ~/automation/projects/reminder.sh << 'EOF'
#!/bin/bash
export PATH="/usr/local/bin:/opt/homebrew/bin:$PATH"
source ~/automation/secrets/.env

LOG=~/automation/logs/reminder.log
echo "[$(date)] === 리마인드 시작 ===" >> $LOG

# 캘린더/할 일 데이터 읽기
# node ~/automation/projects/fetch-calendar.js >> $LOG 2>&1

# 알림 발송
# node ~/automation/projects/send-reminder.js >> $LOG 2>&1

echo "[$(date)] === 리마인드 완료 ===" >> $LOG
EOF

chmod +x ~/automation/projects/reminder.sh
```

### 3-2. 데이터 소스 선택

일정 데이터는 여러 곳에서 가져올 수 있습니다.

- Google Calendar API
- 노션 DB
- 로컬 JSON 파일 (가장 간단한 시작)

처음에는 로컬 JSON 파일로 시작하는 것을 추천합니다.

```json
{
  "reminders": [
    { "time": "09:00", "text": "팀 스탠드업 미팅" },
    { "time": "14:00", "text": "기획 문서 리뷰 마감" }
  ]
}
```

> **확인**: 수동 실행 후 로그에 정상 기록이 남으면 OK입니다.

## STEP 4. 일일 리포트 만들기

### 4-1. 스크립트 작성

```bash
cat > ~/automation/projects/daily-report.sh << 'EOF'
#!/bin/bash
export PATH="/usr/local/bin:/opt/homebrew/bin:$PATH"
source ~/automation/secrets/.env

LOG=~/automation/logs/daily-report.log
echo "[$(date)] === 일일 리포트 시작 ===" >> $LOG

# 오늘 수집 결과 모으기
# node ~/automation/projects/aggregate-today.js >> $LOG 2>&1

# 요약 생성
# node ~/automation/projects/daily-summary.js >> $LOG 2>&1

# 발송
# node ~/automation/projects/send-report.js >> $LOG 2>&1

echo "[$(date)] === 일일 리포트 완료 ===" >> $LOG
EOF

chmod +x ~/automation/projects/daily-report.sh
```

### 4-2. 리포트 형식

```
📋 일일 리포트 (2026-03-27)

[오늘 처리된 항목]
- 아침 브리핑: ✅ 정상 발송
- 리마인드: ✅ 2건 발송
- 수집 뉴스: 12건

[놓친 항목]
- 기획 문서 리뷰 (미확인)

[내일 체크]
- 주간 리포트 준비
```

> **확인**: 수동 실행 후 로그에 정상 기록이 남으면 OK입니다.

## STEP 5. crontab으로 통합하기

### 5-1. 세 스크립트를 한 번에 등록

```bash
crontab -e
```

아래 내용을 추가합니다.

```bash
# ===== AI 비서 시스템 =====
# 아침 브리핑 (매일 08:00)
0 8 * * * ~/automation/projects/morning-brief.sh

# 일정 리마인드 (매일 08:30)
30 8 * * * ~/automation/projects/reminder.sh

# 일일 리포트 (매일 21:00)
0 21 * * * ~/automation/projects/daily-report.sh
```

### 5-2. 등록 확인

```bash
crontab -l
```

세 줄이 모두 보이면 여러분의 AI 비서가 가동을 시작한 겁니다.

<!-- IMG: [crontab -l 결과 — 세 개의 자동화가 등록된 화면] -->

### 5-3. 실패 감지 구조

비서는 조용히 사라지면 안 됩니다. 자동화 시스템은 특히 더 그렇죠.

각 스크립트의 로그를 한눈에 확인하는 명령어를 만들어두세요.

```bash
# 오늘 로그 한눈에 보기
cat > ~/automation/projects/check-status.sh << 'EOF'
#!/bin/bash
echo "=== 아침 브리핑 ==="
tail -3 ~/automation/logs/morning-brief.log
echo ""
echo "=== 리마인드 ==="
tail -3 ~/automation/logs/reminder.log
echo ""
echo "=== 일일 리포트 ==="
tail -3 ~/automation/logs/daily-report.log
EOF

chmod +x ~/automation/projects/check-status.sh
```

> **확인**: `crontab -l`에 세 항목이 보이고, 등록한 시간에 각 로그가 갱신되면 완료입니다.

## 여기서 막혔나요?

**Q. 세 스크립트 중 하나만 안 돌아갑니다.**

상황: 아침 브리핑은 되는데 리마인드가 안 됩니다.

원인:
- 해당 스크립트의 실행 권한이 없을 수 있습니다
- 스크립트 내부에서 참조하는 파일 경로가 틀렸을 수 있습니다

해결:
1. `chmod +x ~/automation/projects/reminder.sh`로 권한을 확인하세요
2. 해당 스크립트를 수동으로 실행해서 에러 메시지를 확인하세요
3. 로그 파일(`~/automation/logs/reminder.log`)을 확인하세요

**Q. 스크립트 내에서 API 호출이 안 됩니다.**

상황: 수동 실행에서는 API가 잘 되는데 cron에서는 실패합니다.

원인:
- cron 환경에서 `.env` 파일이 로드되지 않았을 수 있습니다
- PATH에 `node`나 `python3` 경로가 없을 수 있습니다

해결:
1. 스크립트 상단에 `source ~/automation/secrets/.env`가 있는지 확인하세요
2. `export PATH="/usr/local/bin:/opt/homebrew/bin:$PATH"`가 있는지 확인하세요
3. `which node`로 확인한 경로를 스크립트에 절대 경로로 넣어보세요

**Q. 시간을 바꾸고 싶습니다.**

상황: 아침 8시가 아니라 7시에 돌리고 싶습니다.

원인:
- crontab 수정이 필요합니다

해결:
1. `crontab -e`로 편집하세요
2. `0 8`을 `0 7`로 바꾸면 됩니다
3. 저장 후 `crontab -l`로 확인하세요

**Q. 처음 버전에서 어디까지만 만들면 될까요?**

상황: 세 가지 다 만들기 부담스럽습니다.

원인:
- 전부 처음부터 넣지 않아도 됩니다

해결:
1. "매일 아침 요약이 제시간에 온다" — 이것 하나만 돼도 꽤 큰 체감입니다
2. 아침 브리핑부터 시작해서 안정적으로 돌아가는 걸 확인하세요
3. 그다음 리마인드, 그다음 리포트 순서로 하나씩 붙이면 됩니다

## 한 줄 정리

AI 비서는 거대한 한 기능이 아니라 작은 자동화 세 개쯤을 일관된 흐름으로 묶은 시스템입니다. 그렇게 만들면 진짜로 "나를 돕는 기계"가 됩니다.
