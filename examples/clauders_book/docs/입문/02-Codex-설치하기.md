> 설치는 어렵지 않습니다.
>
> 겁먹는 시간이 더 길거든요.

**10분 안에 Codex가 일할 준비를 끝내는 것.** 이 장의 목표는 딱 그겁니다.

설치에서 중요한 건 기술 지식이 아닙니다.

멈추지 않고 끝까지 해보는 겁니다.

---

## 완성 기준

- [ ] 터미널을 열 수 있다
- [ ] `codex --version`을 입력하면 버전이 나온다
- [ ] `codex`를 입력하면 AI 입력창이 뜬다
- [ ] "이 폴더에 뭐가 있어?" 같은 첫 질문을 던져봤다

---

## 왜 하필 터미널 도구부터 시작할까요?

많은 분이 처음엔 웹에서 시작합니다.

ChatGPT나 Claude 웹 화면에 들어가서 말을 걸죠.

입문으로는 좋습니다.

그런데 이 책 전체 흐름에는 조금 부족합니다.

왜냐하면 웹 AI는 주로 설명을 해주고,
우리가 지금 하려는 건 실행이기 때문입니다.

이 책에서 필요한 건 이런 겁니다.

- 파일을 직접 읽기
- 파일을 직접 만들기
- 코드를 직접 고치기
- 프로젝트 폴더 전체를 보고 일하기

이걸 하려면 브라우저 안이 아니라,
내 컴퓨터 안에서 움직이는 도구가 필요합니다.

그래서 Codex로 시작합니다.

---

## STEP 1. 필요한 도구 이해하기

### 1-1. 설치하면 뭐가 같이 필요할까요?

설치 한 번에 보통 세 가지를 이해하면 충분합니다.

| 도구 | 역할 | 비유 |
|------|------|------|
| `Node.js` | Codex CLI를 설치하고 실행하는 기반 | 엔진 |
| `Git` | 변경 이력 저장 | 블랙박스 |
| `Codex` | AI와 대화하며 파일과 명령을 다루는 도구 | 조수석의 AI |

하나씩 깊게 이해할 필요는 없습니다.

지금은 "돌아가게 만드는 부품들" 정도로 보면 충분합니다.

---

## STEP 2. Node.js 설치하기

### 2-1. 이미 설치되어 있는지 확인하기

01 터미널을 엽니다.

- **Mac**: `Cmd + Space` → "터미널" 검색 → Enter
- **Windows**: `Win + R` → `cmd` 입력 → Enter

02 아래 명령어를 입력합니다.

```bash
node -v
```

03 `v18` 이상의 버전이 나오면 이미 설치된 겁니다.

다음 STEP으로 넘어가세요.

> **확인**: 버전 번호가 출력되면 성공입니다. `command not found`가 나오면 설치가 필요합니다.

### 2-2. Node.js 설치하기

01 https://nodejs.org 에 접속합니다.

02 **LTS** 버전을 내려받습니다.

03 설치 파일을 실행하고 기본 설정으로 끝까지 진행합니다.

04 설치가 끝나면 터미널을 새로 열고 다시 확인합니다.

```bash
node -v
```

> **확인**: 버전 번호가 출력되면 준비 완료입니다.

## STEP 3. Codex 설치하기

### 3-1. 가장 쉬운 설치 방법

Mac이라면 둘 중 하나를 고르면 됩니다.

```bash
brew install --cask codex
```

또는

```bash
npm install -g @openai/codex
```

Windows라면 보통 npm 방식이 가장 간단합니다.

```bash
npm install -g @openai/codex
```

설치가 끝나면 아래 명령어로 확인합니다.

```bash
codex --version
```

> **확인**: 버전 번호가 출력되면 설치 완료입니다.

### 3-2. 처음 실행해보기

01 아무 폴더에서 아래 명령어를 실행합니다.

```bash
codex
```

02 처음 실행하면 로그인이나 인증 절차가 뜹니다.

안내에 따라 진행하세요.

03 입력 프롬프트가 뜨면 성공입니다.

> **확인**: 이제 Codex가 여러분의 말을 기다리고 있습니다.

---

## STEP 4. 첫 대화 해보기

### 4-1. 첫 질문은 작아야 합니다

처음부터 웹앱 전체를 만들게 하지 마세요.

오히려 작은 성공이 더 중요합니다.

01 아래 질문을 그대로 입력해보세요.

```text
지금 이 폴더에 뭐가 있는지 알려줘
```

02 이번에는 파일을 하나 만들어봅니다.

```text
README 초안 하나 만들어줘
```

03 마지막으로 그 파일을 수정해봅니다.

```text
방금 만든 README 첫 문장을 조금 더 친절하게 고쳐줘
```

> **확인**: 이 세 단계면 읽기, 만들기, 수정하기를 한 번에 경험한 겁니다.

### 4-2. 왜 작은 성공이 중요할까요?

설치 직후 바로 "앱 하나 만들어줘"를 던지면 피로가 빨리 옵니다.

설치 다음엔 무조건 작은 성공을 먼저 만드세요.

그래야 "이게 진짜 되는구나"가 몸에 들어옵니다.

## STEP 5. 승인과 샌드박스 이해하기

### 5-1. 왜 자꾸 허락을 묻나요?

좋은 질문입니다.

Codex는 파일을 읽고,
고치고,
명령을 실행할 수 있습니다.

그러니 처음엔 함부로 다 열어주지 않는 게 맞습니다.

### 5-2. 이 책 예제는 자동 진행 쪽으로 시작합니다

많은 분이 여기서 갈립니다.

"중요한 행동마다 물어보는 게 더 안전하지 않나요?"

맞는 말입니다.

그런데 책 예제는 학습 속도도 중요합니다.

그래서 이 책의 `codex-starter` 예제는 이렇게 잡았습니다.

```toml
approval_policy = "never"
sandbox_mode = "workspace-write"

[sandbox_workspace_write]
network_access = false
```

뜻은 이렇습니다.

- 승인 프롬프트는 띄우지 않습니다
- 대신 프로젝트 폴더 안에서만 움직입니다
- 네트워크는 막아둡니다

즉, **빠르게 일하게 하되, 울타리는 남겨두는 방식**입니다.

### 5-3. 더 조심해서 쓰고 싶다면?

보수적으로 시작하고 싶다면 예제에 같이 넣어둔 `cautious` 프로필을 쓰면 됩니다.

```bash
codex --profile cautious
```

이 프로필은 승인 확인을 다시 켜는 용도입니다.

그러니까 정리하면 이렇습니다.

- 빠르게 체감하고 싶다 → 기본 설정
- 낯선 저장소라 조금 더 조심하고 싶다 → `cautious`

## STEP 6. 이 책의 예제 프로젝트 열어보기

이제 진짜로 연습할 폴더를 하나 열어보면 좋습니다.

이 책 워크스페이스 안에는 Codex 연습용 예제가 같이 들어 있습니다.

`examples/clauders_book/examples/codex-starter`

이 폴더에는 이런 것들이 들어 있습니다.

- `AGENTS.md`
- `.codex/config.toml`
- `.agents/skills/`
- `site/.agents/skills/`
- `OFFICIAL-CODEX-SKILLS.md`
- 작은 정적 웹앱 파일

즉, Codex를 그냥 켜는 데서 끝나지 않고,
"어떻게 설정하면 더 잘 굴러가는지",
"repo 전체 skill과 폴더 전용 skill은 어떻게 나누는지",
"공식 built-in skill은 뭘 쓰면 되는지"까지 바로 볼 수 있게 해둔 겁니다.

## 여기서 막혔나요?

**Q. `codex` 치면 "command not found"가 나와요**

상황: 설치는 한 것 같은데 `codex` 명령어를 인식하지 못합니다.

원인:
- 터미널을 설치 전에 열어둔 상태일 수 있습니다
- npm 글로벌 설치 경로가 PATH에 안 잡혔을 수 있습니다

해결:
1. 터미널을 완전히 닫고 새로 열어보세요
2. 다시 `codex --version`을 실행해보세요
3. npm으로 설치했다면 아래 명령어로 글로벌 경로를 확인하세요

```bash
npm config get prefix
```

**Q. 설치는 됐는데 로그인에서 멈춰요**

상황: `codex`를 실행하면 인증 단계에서 진행이 안 됩니다.

원인:
- 브라우저 인증 과정이 막혔을 수 있습니다
- 네트워크나 계정 상태 문제가 있을 수 있습니다

해결:
1. 터미널 메시지를 끝까지 읽어보세요
2. 브라우저 인증 창이 떴는지 확인하세요
3. 필요하면 `codex login status`로 상태를 다시 확인하세요

**Q. 검은 화면이라 무서워요**

상황: 터미널을 열었는데 깜빡이는 커서만 보입니다.

원인:
- 정상입니다. 터미널은 원래 이렇게 생겼습니다

해결:
1. 메신저가 채팅창이듯, 터미널은 컴퓨터와 글로 대화하는 창이라고 생각해보세요
2. `pwd`를 입력해보세요. 지금 내가 어디 폴더에 있는지 보여줍니다
3. `ls`를 입력해보세요. 현재 폴더 파일 목록이 나옵니다

---

## 한 줄 정리

설치는 기술 시험이 아닙니다.

`codex --version`이 나오고,
`codex`를 쳤을 때 입력창이 뜨면,
여러분은 이미 시작할 준비가 끝난 겁니다.
